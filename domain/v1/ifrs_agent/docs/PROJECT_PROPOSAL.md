# IFRSseed AI 프로젝트 기획서

## 1. 프로젝트 개요

### 1.1 프로젝트명
**IFRSseed AI** - IFRS S1·S2 기준에 맞춘 지속가능경영보고서 자동 생성 AI 서비스

### 1.2 프로젝트 목적
IFRSseed AI는 TCFD 및 기존 ESG 데이터를 IFRS S1·S2 표준에 맞춘 보고서를 자동 생성하기 위해 설계된 AI 기반 솔루션입니다. 기준서(규정 문서)와 기업 ESG 보고서를 활용하여 정확하고 표준화된 지속가능성 보고서를 생성합니다.

## 2. 기술 아키텍처

### 2.1 핵심 기술 스택

#### 2.1.1 LoRA 학습 (Low-Rank Adaptation)
- **목적**: ESG 문체 및 용어 학습
- **기능**: 
  - ESG 보고서 작성에 특화된 문체 학습
  - IFRS S1·S2 표준 용어 및 표현 학습
  - 도메인 특화 언어 모델 최적화

#### 2.1.2 RAG 시스템 (Retrieval-Augmented Generation)
- **목적**: 데이터 검색 및 매핑
- **기능**:
  - 기준서(규정 문서) 검색 및 활용
  - 기업 ESG 보고서 데이터 검색
  - IFRS S1·S2 표준과의 매핑 및 매칭

#### 2.1.3 ERP/EMS/HRIS 시스템 연동 (System Integration)
- **목적**: 기업 내부 시스템과의 자동 데이터 수집
- **기능**:
  - ERP 시스템 연동 (SAP, Oracle, 더존 등)
  - EMS (환경관리 시스템) 연동
  - HRIS (인사정보 시스템) 연동
  - BI/DW (데이터웨어하우스) 연동
  - 자동 필드 매핑 (ERP 필드 → IFRS S2/GRI Data Point)
  - 배치/실시간 데이터 동기화
  - 데이터 검증 및 이상치 탐지

### 2.2 데이터 활용 전략

#### 2.2.1 데이터 기간
- **주요 데이터**: 2024년 데이터 중심 (최신성 확보)
- **보조 데이터**: 2022~2023년 데이터 제한적 포함 (트렌드 분석 지원)

#### 2.2.2 데이터 규모
- **기업 보고서**: 총 11개 기업 보고서
- **구현 가능성 극대화**: 제한된 데이터셋으로도 효과적인 구현 가능성 검증

#### 2.2.3 데이터 소스 다양화
- **외부 공시 데이터**: DART 전자공시, 기업 홈페이지
- **내부 시스템 데이터**: ERP, EMS, HRIS, BI/DW 시스템 연동
- **파일 기반 데이터**: Excel/CSV 자동 업로드, FTP/클라우드 연동
- **하이브리드 모드**: 일부 자동 연동, 일부 수동 입력

## 3. IFRS 공시기준의 중요성

### 3.1 통합된 글로벌 표준
IFRS 공시기준은 기존의 다양한 지속가능성 보고 프레임워크를 포괄합니다:
- **GRI** (Global Reporting Initiative)
- **CDP** (Carbon Disclosure Project)
- **SASB** (Sustainability Accounting Standards Board)
- **TCFD** (Task Force on Climate-related Financial Disclosures)

### 3.2 국제적 연계성
- **미국 SEC 기후 공시안**과 연계
- **유럽 ESRS** (European Sustainability Reporting Standards)와 연계
- 국제적으로 통합된 지속가능성 보고 표준으로 자리매김 예상

## 4. 시장 배경 및 필요성

### 4.1 기존 자발적 공시체계의 한계
- **단일화된 글로벌 기준 부재**
- 다양한 프레임워크 혼재:
  - GRI 권고안
  - SASB 표준
  - TCFD 권고안
  - 기타 지역별/산업별 프레임워크

### 4.2 의무 공시 체계로의 전환
- **2023년 이후**: 국가별 규제에 따른 의무 공시 중심 체계로 전환 추세
- **국내 전망**: IFRS 기반 지속가능성 보고기준이 **2025년부터 의무화** 예상

### 4.3 프로젝트의 시의성
- 의무 공시 전환 시점에 대비한 솔루션 제공
- 기업의 IFRS S1·S2 준수 비용 및 시간 절감
- 표준화된 보고서 품질 향상

## 5. 프로젝트 목표

### 5.1 단기 목표
- IFRS S1·S2 표준에 맞춘 보고서 자동 생성 기능 구현
- 11개 기업 보고서 기반 검증 완료
- LoRA 학습 및 RAG 시스템 최적화

### 5.2 장기 목표
- 다양한 산업 및 규모의 기업 지원 확대
- 실시간 데이터 업데이트 및 트렌드 분석 기능 강화
- 글로벌 표준 변화에 대한 지속적 업데이트
- **ERP/EMS/HRIS 시스템 자동 연동**: 기업 내부 시스템과의 통합을 통한 데이터 수집 자동화
- **ERP 데이터 검증 및 수정 요청 자동화**: 전처리, 검증, 이상 탐지, 수정 요청을 자동화하는 파이프라인 구축
- **4단계 역할 구조 및 Workflow 승인 프로세스**: KPMG 기반 조직 구조 반영 (최종 승인권자 → ESG팀 → 현업팀 → 일반 사용자)
- **HRIS 기반 자동 역할 할당**: HRIS 시스템 연동을 통한 사용자 역할 자동 할당 (사용자 선택 불가, 보안 강화)

## 6. 기대 효과

### 6.1 기업 측면
- 보고서 작성 시간 및 비용 절감
- IFRS S1·S2 준수 확보
- 보고서 품질 및 일관성 향상
- **데이터 수집 자동화**: ERP/EMS/HRIS 시스템 연동을 통한 수동 입력 시간 90% 감소
- **데이터 정확성 향상**: 기업 내부 시스템의 신뢰성 있는 데이터 활용으로 오류 80% 감소
- **실시간 ESG 모니터링**: 자동 동기화를 통한 실시간 ESG 지표 추적 가능
- **데이터 품질 관리 자동화**: 
  - 전처리 및 검증 자동화로 데이터 품질 보장
  - LLM 기반 이상 수치 탐지로 오류 조기 발견
  - 지능형 수정 요청으로 효율적인 데이터 수정 프로세스
  - SAP, Oracle 등 업계 표준 방식과 일치하는 검증 프로세스
- **역할 기반 Workflow 자동화**: ⭐ 신규
  - 4단계 역할 구조 (최종 승인권자, ESG팀, 현업팀, 일반 사용자)
  - HRIS 기반 자동 역할 할당 (사용자 선택 불가, 보안 강화)
  - 자동 승인 워크플로우 (현업팀 → ESG팀 → 최종 승인권자)
  - 부서별 책임 명확화 및 협업 효율성 향상
  - 실시간 상태 추적 및 알림 시스템

### 6.2 시장 측면
- 지속가능성 보고 표준화 촉진
- 투명성 및 비교 가능성 향상
- ESG 정보의 신뢰성 제고

## 7. 프로젝트 일정 및 마일스톤

### 7.1 개발 단계
1. **Phase 1**: 데이터 수집 및 전처리
   - 수동 입력 및 Excel/CSV 파일 업로드 (MVP)
   - **4단계 역할 구조 및 Workflow 승인 프로세스 구현** ⭐
   - **HRIS 기반 자동 역할 할당 시스템 구현** ⭐
2. **Phase 2**: LoRA 모델 학습
3. **Phase 3**: RAG 시스템 구축
4. **Phase 4**: 통합 및 테스트
   - 파일 자동 업로드 (클라우드/FTP/이메일)
5. **Phase 5**: 검증 및 최적화
6. **Phase 6**: ERP/EMS/HRIS 시스템 API 연동 (중기)
   - SAP, Oracle, 더존 ERP 연동
   - 환경관리 시스템 (EMS) 연동
   - 인사정보 시스템 (HRIS) 연동
   - 데이터 매핑 자동화
7. **Phase 7**: AI 기반 자동화 강화 (장기)
   - 실시간 데이터 스트리밍
   - 멀티소스 자동 통합
   - 이상 탐지 및 자동 검증

### 7.2 검증 계획
- 11개 기업 보고서 기반 검증
- IFRS S1·S2 표준 준수 여부 확인
- 보고서 품질 평가
