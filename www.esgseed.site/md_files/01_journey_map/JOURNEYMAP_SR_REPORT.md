# SR 보고서 작성 페이지 사용자 저니맵

**SR 작성** 메뉴(`/report`)에서 이루어지는 사용자 플로우를 **현재 프론트엔드 화면** 기준으로 정리합니다. 목차 선택 → 직접 작성/AI 문단 생성 → 저장 → 준수율 확인 등 **SR 보고서 작성 페이지 안에서의 행동**을 6열 구조로 기술합니다.

- **대상 화면**: SR 작성 페이지 (`/report`) — Table of Contents, 에디터(직접 작성 / AI 문단 생성 탭), 전체 준수율·관련 공시 기준·시각화 추천.
- **플로우 종료**: 문단 작성·**저장**을 마친 뒤 **대시보드**로 이동하여 진행 현황·최종 승인·보고서 출력을 확인합니다. (최종 완성도 점검·Excel/PowerPoint 다운로드는 대시보드에서 수행.)

---

## 1. 세부 시트 행(열) 구조

| 열 순서 | 열 이름 | 설명 |
|--------|----------|------|
| 1 | **단계 (Step)** | SR 작성 페이지에서 겪는 단계·순서 |
| 2 | **사용자 행동 (User Action)** | 해당 단계에서 사용자가 **직접 하는 행동** (클릭, 입력, 선택) |
| 3 | **화면 상태 (Screen State)** | 그때 화면에 **보이는 것** (목차, 에디터, 탭, 버튼, 준수율 등) |
| 4 | **감정/생각 (Emotion/Thought)** | 사용자가 느끼거나 생각할 수 있는 것 |
| 5 | **문제점 (Pain Point)** | 불편함, 혼란, 막힘 |
| 6 | **기회 요소 (Opportunities / Solutions)** | 개선·해결 방향 |

---

## 2. 작성 원칙

- **쓸 수 있는 것**: SR 작성 페이지에 있는 **버튼·라벨·카드 제목** — Table of Contents, **페이지 (NN)**, **직접 작성**·**AI 문단 생성** 탭, **내용**, **저장**, **이 문단은 아래 정보를 바탕으로 생성됩니다**, **AI 문단 생성**, **생성된 문단 미리보기**, **이 내용 사용**·**다시 생성**·**직접 수정**, **정량 데이터: 연동됨/미연동**, **전체 준수율**, **관련 공시 기준**, **시각화 추천**, **도표 생성으로 이동 →**, **대시보드** 등. 사용자 **행동** — "목차에서 온실가스 관리 선택", "직접 작성 탭에서 내용 입력 후 저장 클릭", "AI 문단 생성 후 이 내용 사용 클릭".
- **쓰지 말아야 할 것**: 컴포넌트·함수·변수명, 개발자 전용 용어.

## 4. 단계별 상세 저니맵 (6열)

| 단계 (Step) | 사용자 행동 (User Action) | 화면 상태 (Screen State) | 감정/생각 (Emotion/Thought) | 문제점 (Pain Point) | 기회 요소 (Opportunities / Solutions) |
|-------------|---------------------------|---------------------------|-----------------------------|---------------------|----------------------------------------|
| **1. 진입 및 목차 탐색** | 상단 메뉴에서 **SR 작성** 클릭 | SR 보고서 작성 페이지. 제목·부제. 좌측 **Table of Contents** 트리(페이지 번호·제목). 중앙 "목차 항목을 선택하여 페이지를 작성하세요" 또는 첫 목차 선택 시 에디터. 우측 **전체 준수율**·**관련 공시 기준** | "본격적으로 내용을 채워야 하는데, 어디서부터 손대지?" | 목차가 길어 스크롤 없이 필요한 항목을 찾기 어려울 수 있음 | 목차 상단 키워드 검색 또는 섹션 접기/펼치기 |
| **2. 작성 항목 선택 및 기준 확인** | 좌측 **Table of Contents**에서 항목 클릭(예: Environmental → 온실가스 관리) | 선택한 항목 제목·**페이지 (NN)** 배지. 중앙 **직접 작성**·**AI 문단 생성** 탭, **내용** 입력란·**저장**. 우측 해당 페이지의 **전체 준수율**·**관련 공시 기준**(● [필수] / ○ [권장], ✓/◐/○). **시각화 추천**·**도표 생성으로 이동 →** | "이 섹션에는 어떤 공시 기준이 연결되는지 확인해야겠다" | 선택한 목차와 공시 기준 매칭이 한눈에 들어오지 않을 수 있음 | 목차 또는 에디터 상단에 매칭 공시 기준 태그 강조 |
| **3. 직접 작성 — 본문 입력 및 저장** | **직접 작성** 탭에서 **내용** 텍스트 영역에 보고서 문장 입력 → **저장** 클릭 | 에디터·"30초마다 자동 임시저장 · 저장 시 준수율 재계산"·**저장** 버튼. 저장 시 "페이지 내용이 저장되었습니다." 토스트. 50자 미만 저장 시 "내용이 너무 짧습니다. 50자 이상 입력 후 저장해 주세요." 토스트. 우측 **전체 준수율**·공시 기준 목록 갱신 | "문장을 넣고 저장하면 준수율이 바뀌는구나" | 50자 미만일 때만 경고가 나와, 권장 분량에 대한 안내가 부족할 수 있음 | placeholder·도움말에 "50자 이상 권장" 등 상시 노출 |
| **4. AI 문단 생성 — 조건 확인·생성·삽입** | **AI 문단 생성** 탭에서 "이 문단은 아래 정보를 바탕으로 생성됩니다." 확인(회사정보·GHG·ERP ✓/✗) → **AI 문단 생성** 클릭 → **생성된 문단 미리보기** 확인 후 **이 내용 사용** 또는 **다시 생성** 또는 **직접 수정** 클릭 | 조건 요약(✓/✗). **AI 문단 생성** 버튼(클릭 시 "생성 중..."). 미리보기 영역·**이 내용 사용**·**다시 생성**·**직접 수정**. "이 내용 사용" 시 "에디터에 삽입되었습니다. 직접 작성 탭에서 수정 후 저장해 주세요." 토스트. 하단 **정량 데이터: 연동됨/미연동** 문구 | "AI로 초안 받고 수정해서 쓰면 빠르겠다" | ERP 미연동 시 "연동 시 더 정확한 문단 생성 가능"만 보여 구체적 연동 방법이 불명확할 수 있음 | ERP 연동 가이드 링크 또는 툴팁 |
| **5. 준수율 확인 및 보완** | 우측 **전체 준수율** 색상(녹/황/적) 및 **관련 공시 기준** 목록(✓/◐/○, ● [필수] / ○ [권장]) 확인. 미준수·부분 준수 항목이 있으면 **내용** 보완 후 다시 **저장** | 0~100% 진행 바, 준수(90%↑)·부분(60-89%)·미준수(59%↓) 개수. 공시 기준별 상태·제안 사항 | "아직 미준수인 기준이 있네. 내용을 보완해야겠다" | 미준수일 때 "어떤 문장을 추가하면 준수되는지" 구체적 가이드가 부족할 수 있음 | 미준수 기준 클릭 시 추천 문구·가이드 연동 |
| **6. 시각화 추천 및 도표 생성 이동** | 우측 **시각화 추천** 카드에서 현재 페이지에 맞는 추천(예: 온실가스 막대 그래프, 배출량 테이블) 확인. **도표 생성으로 이동 →** 클릭 | "이 섹션에 어울리는 자료" · 추천 제목·설명 · **도표 생성으로 이동 →** 링크(도표 및 그림 생성 페이지로 쿼리 파라미터와 함께 이동) | "텍스트만 있으면 딱딱하니까, 추천 차트 넣으면 좋겠다" | GHG 등 다른 메뉴 데이터를 차트에 다시 입력하기 번거로울 수 있음 | "GHG 데이터 불러오기" 등 산정 데이터 연동 버튼 |
| **7. 목차별 본문 작성 반복** | **Table of Contents**에서 다른 항목 선택 → 단계 2~5 반복. 필요한 섹션만 골라 **내용** 입력(직접 또는 AI 문단 사용 후 수정) 후 **저장** | 선택한 목차에 따라 중앙 에디터·우측 준수율·공시 기준·시각화 추천가 해당 페이지 기준으로 전환. 목차를 채워 나가며 보고서가 완성되는 느낌 | "챕터마다 채우니까 보고서가 만들어지는구나" | 목차가 길면 특정 섹션을 찾기 위해 스크롤이 필요할 수 있음 | 목차 상단 키워드 검색·섹션 접기/펼치기 |
| **8. 저장 후 대시보드로 이동** | 필요한 목차 항목 작성·**저장**을 마친 후 상단 메뉴에서 **대시보드** 클릭 | 대시보드 페이지. SR 작성 진행 현황(페이지별 상태·준수율), 보고서 출력(Excel/PowerPoint)·최종 승인 등 확인. SR 작성에서 저장한 내용이 대시보드·최종보고서 플로우에 반영됨 | "저장해 둔 문단이 반영됐구나. 대시보드에서 완성도 확인하고 다운로드하면 되겠다" | 수정이 필요할 때 다시 SR 작성으로 돌아가 수정 후 대시보드에서 다시 확인해야 한다는 점을 모를 수 있음 | 대시보드에 "내용 수정이 필요한가요? [SR 작성으로 가기]" 링크 제공 |

## 6. 요약

- **범위**: 이 저니맵은 **SR 작성 페이지**(`/report`)의 **현재 구현된 화면**을 기준으로 하며, 끝단계에서 **저장 후 대시보드로 이동**하여 진행 현황·최종 승인·보고서 출력을 확인하는 흐름까지 포함합니다.
- **단계 수**: 총 **8단계** — 1. 진입 및 목차 탐색 → 2. 작성 항목 선택 및 기준 확인 → 3. 직접 작성(본문 입력·저장) → 4. AI 문단 생성(조건 확인·생성·미리보기·이 내용 사용/다시 생성/직접 수정) → 5. 준수율 확인 및 보완 → 6. 시각화 추천 및 도표 생성 이동 → 7. 목차별 본문 작성 반복 → 8. 저장 후 대시보드로 이동.
- **현재 화면 기준**: **정량 데이터**는 수동 입력 필드 없이 "연동됨/미연동" 상태만 표시. **크롤링 문구 제안/생성**은 **AI 문단 생성** 탭(조건 요약·AI 문단 생성·생성된 문단 미리보기·이 내용 사용/다시 생성/직접 수정)으로 대체됨. **보고서 인덱스 기반 지표별 데이터 입력/조회**·**데이터 최종 제출** 버튼은 현재 SR 작성 페이지에 없으며, 대시보드 등 다른 화면에서 진행될 수 있음.
- **행 구조**: 단계 | 사용자 행동 | 화면 상태 | 감정/생각 | 문제점 | 기회 요소 (6열 고정).
- **언어**: Table of Contents, **페이지 (NN)**, **직접 작성**, **AI 문단 생성**, **내용**, **저장**, **이 문단은 아래 정보를 바탕으로 생성됩니다**, **AI 문단 생성**, **생성된 문단 미리보기**, **이 내용 사용**·**다시 생성**·**직접 수정**, **정량 데이터: 연동됨/미연동**, **전체 준수율**, **관련 공시 기준**, **● [필수]**·**○ [권장]**, **시각화 추천**, **도표 생성으로 이동 →**, **대시보드** 등 사용자가 보는 이름만 사용합니다.

이 문서는 **현재 프론트엔드** SR 보고서 작성 페이지를 참고하여 작성한 사용자 저니맵입니다.
