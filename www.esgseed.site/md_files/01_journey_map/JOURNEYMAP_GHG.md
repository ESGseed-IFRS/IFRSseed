# GHG 산정 Journey Map (상세)

본 문서는 **GHG 산정** 페이지의 사용자 저니맵을 정리합니다. **내부시스템 적재**와 **엑셀 업로드**를 데이터 입력 경로로 하며, **ESG 팀장(결재권자)** 과 **ESG 팀원** 두 사용자 유형에 맞춰 작성합니다. **감사·검증 대응** 섹션은 역할별로 구분합니다.

---

## 1. 개요

| 항목 | 내용 |
|------|------|
| **페이지** | GHG 산정 |
| **목적** | SR 보고서의 필수 정량 데이터인 온실가스(Scope 1, 2, 3) 배출량을 플랫폼 내에서 계산·관리하고, 결과를 SR 문단·차트·최종보고서·감사에서 재사용 |
| **산출물** | Scope 1/2/3 배출량(tCO₂e), 연도별·사업장별 집계, 증적 리포트(PDF·Excel), 검증의견서 부속 서류 |
| **데이터 입력 경로** | (1) 내부시스템 적재: 기설정 주기로 자동 적재 (2) **엑셀 업로드**: 표준 템플릿 일괄 입력 |
| **워크플로우** | **이미 적재된 데이터** 조회 또는 **엑셀 업로드**로 시작 → 검증·보완 → 산정 → **전체 결과 저장**·리포트 |

---

## 2. 페이지 구조 (화면에 보이는 이름 기준)

### 2-1. 페이지 헤더

| 구분 | 화면에 보이는 이름 |
|------|-------------------|
| 제목 | **온실가스 배출량 산정 (GHG Protocol)** |

### 2-2. 상단 탭 (3개)

| 탭 | 화면에 보이는 이름 | 내용 |
|----|-------------------|------|
| 1 | **GHG 산정·공시** | 산정 입력 6탭 + 활동자료 + 스텝 플로우(전력/연료) + 배출량 산정 결과 + Total Carbon Inventory |
| 2 | **감사·검증 대응** | 공시 프레임워크 선택 + 좌측 감사 메뉴 + 요건 체크리스트·데이터 계보 추적·수동 조정 이력·배출계수 적용 내역·데이터 품질 분포·산정 버전 히스토리·감사인 전용 뷰·증빙 패키지 내보내기 |
| 3 | **리포트 생성** | Evidence Report 생성 완료, PDF Report, Excel Details, JSON, 검증의견서 부속 서류 6항목 |

### 2-3. GHG 산정·공시 탭 — 레이아웃

| 영역 | 화면에 보이는 이름 / 동작 |
|------|---------------------------|
| **좌측 사이드바** (lg 이상) | **산정 입력** (상) — 6개 탭 버튼. **필터** (하) — 사업장·연도·에너지원·기간 등. K-ETS 선택 시 폐기물·물류·원료 탭 숨김 |
| **툴바** (상단 우측) | **히스토리 (N)** (**저장된 히스토리** 다이얼로그), **필터** (lg 미만 시 다이얼로그), **EMS 불러오기**, **엑셀 업로드**, **전체 결과 저장**, **CSV 내보내기**, **계수 기준 연도** (2024 / 2023 KR Emission Factors) |
| **메인 상단** | **공시 프레임워크** 셀렉트(ISSB, KSSB, K-ETS, GRI, ESRS), **선택 기준 대비 입력 완료율** N%, 6탭 요약 버튼(✓ 전력·열·스팀, 연료·차량 …) 클릭 시 해당 탭으로 전환 |
| **K-ETS/ESRS 안내** | K-ETS 선택 시: "월별 에너지 사용량 보고 의무. Scope 1 중심. 기존 입력 그리드 + ERP/엑셀 자동 채움 활용." / ESRS 선택 시: "Scope 2 location/market 기반 필수. 전환 계획·시나리오 분석 자료 입력 필드 (Phase 2 연동 예정)." |
| **메인 카드** | **2. [탭명] — 활동자료 입력 및 결과** (설명 + 프레임워크 안내). 내부: 선택한 탭에 따른 입력·결과 영역 |
| **우측 고정** (xl) | **Total Carbon Inventory** (총 tCO₂e, Scope 1/2/3 기여도 %·바), **저장된 히스토리** 카드 목록(날짜·총 tCO₂e, Scope 전체) |

### 2-4. 산정 입력 6탭

> "내가 어떤 데이터를 갖고 있는가" 기준. 결과는 Scope 1/2/3로 표시됩니다.

| 화면에 보이는 이름 | 소스 | 화면 동작 |
|-------------------|------|-----------|
| **전력·열·스팀** | EMS | 스텝 플로우(STEP 1→2→3). 1. 원시 데이터 수집 → 활동자료 → 배출량 산정 결과. **이 탭 결과 저장** |
| **연료·차량** | ERP | 동일 스텝 플로우. 고정/이동 연소 그리드 |
| **냉매** | EHS | 입력 테이블. "[EHS 불러오기] 또는 [엑셀 업로드]로 데이터를 가져오거나, [+ 행 추가]로 직접 입력하세요." 안내 |
| **폐기물** | EMS | 폐기물 입력 영역. 소각→Scope 1, 위탁·매립→Scope 3 |
| **물류·출장·통근** | SRM·HR | 물류·출장·통근 입력 영역 |
| **원료·제품** | SRM·PLM | 원료·제품 입력 영역 |

### 2-5. 전력·열·스팀 / 연료·차량 — 스텝 플로우

| 스텝 | 화면에 보이는 이름 | 화면 내용 |
|------|-------------------|-----------|
| STEP 1 | **데이터 확인** | **1. 원시 데이터 수집**: ERP·엑셀/수동 출처별 행 구분, **EMS 불러오기**·**엑셀 업로드** 버튼. 활동자료 그리드. **다음 단계 →** (데이터 있으면 활성화) |
| STEP 2 | **산정 실행** | "산정 실행 — 행별 배출계수 매칭" 안내. 동일 활동자료 그리드. **다음 단계 →** |
| STEP 3 | **결과 확인** | "[탭명] 탭 산정 결과". 전체 사업장 합계 tCO₂e, 최종 산정 배출량, 데이터 신뢰도, Scope 1/2/3. **이 탭 결과 저장** 버튼 |

### 2-6. 원시 데이터 수집·활동자료

| 구분 | 화면에 보이는 이름 |
|------|-------------------|
| 섹션 제목 | **1. 원시 데이터 수집** |
| 안내 | "ERP·엑셀에서 가져오거나 수동으로 입력한 활동자료를 먼저 확인합니다. 아래 테이블에 반영된 데이터가 Scope 1/2 산정에 사용됩니다." |
| 버튼 | **EMS 불러오기**, **엑셀 업로드** |
| 출처 표시 | ERP/엑셀(파랑), 수동(회색) 행 배경 |

### 2-7. 배출량 산정 결과

| 구분 | 화면에 보이는 이름 |
|------|-------------------|
| 전체 합계 | **전체 사업장 합계** — N tCO₂e. "모든 사업장의 Scope 1·2·3 합계입니다. 개별 사업장 데이터는 위 데이터 관리 영역에서 조회하세요." |
| 최종 산정 | **최종 산정 배출량** — 큰 숫자 tCO₂e |
| 신뢰도 | **데이터 신뢰도** — High (Level 1) / Medium (Level 2) |
| Scope별 | Scope 1: N tCO₂e, Scope 2: N tCO₂e (위치·시장 기반 표시 시), Scope 3: N tCO₂e |

### 2-8. 감사·검증 대응 탭 — 구조

| 구분 | 화면에 보이는 이름 |
|------|-------------------|
| 상단 | **공시 프레임워크 선택** — ISSB(IFRS S2), KSSB, K-ETS, GRI, ESRS 버튼. "선택 기준 \| 보고연도 \| 한국 \| 모든 서브 메뉴는 선택된 프레임워크에 따라 표시가 변경됩니다." |
| 좌측 메뉴 | **감사·검증 대응** (제목). **요건 체크리스트**, **데이터 계보 추적**, **수동 조정 이력**, **배출계수 적용 내역**, **데이터 품질 분포**, **산정 버전 히스토리**, **감사인 전용 뷰**, **증빙 패키지 내보내기** |

| 다운로드 | **PDF Report** (산정설정·Scope1·2·3·영수증 목록), **Excel Details** (산정설정·Scope1·2·3·영수증 목록 포함), **JSON**, **일괄 다운로드** (PDF·Excel) |

## 3. 데이터 소스 및 입력 경로 (화면 기준)

### 3-1. 데이터 소스별 역할 (화면에서 사용하는 소스명)

| 소스 | 역할 | GHG 범위 |
|------|------|-----------|
| **EMS** | 환경·에너지 데이터 | Scope 1·2 핵심 |
| **ERP** | 연료 구매·소비 데이터 | Scope 1 연료 |
| **EHS** | 냉매·폐기물 데이터 | Scope 1 냉매 |
| **SRM·HR** | 물류·출장·통근 데이터 | Scope 3 |
| **SRM·PLM** | 원료·제품 데이터 | Scope 3 |

### 3-2. 데이터 입력 경로

| 경로 | 설명 | 사용자 행동 |
|------|------|-------------|
| **내부시스템 적재** | 기설정 주기로 자동 적재 | 진입 시 **이미 적재된 데이터** 조회·검증 |
| **EMS 불러오기** | 필터 조건 기반 데이터 가져오기 | **EMS 불러오기** → **EMS 데이터 가져오기** |
| **엑셀 업로드** | 표준 컬럼(월, 사업장, 에너지원, 사용량, 단위) 엑셀 업로드 | **엑셀 업로드** → 파일 선택 → **확인 및 적용** |
| **수동 입력** | 각 탭 내 **연료 추가**·**전력 행 추가** 등 | 탭 내 버튼으로 행 추가·입력. **팀원이 수동 입력한 데이터는 팀장이 산정 입력 화면에서 조회 가능**하며, **수동 조정 이력**에서 입력자·일시 확인 가능 |

### 3-3. 진입 시 초기 화면

- **적재 데이터 있음**: 원시 데이터 영역에 테이블 표시. 출처(EMS/ERP/EHS 등) 배지. 사업장·연도·에너지원 요약.
- **적재 데이터 없음**: "적재된 데이터가 없습니다. **EMS 불러오기** 또는 **엑셀 업로드**로 데이터를 입력하세요." 또는 "**EMS 불러오기** 또는 **엑셀 업로드**로 데이터를 가져오거나, **+ 행 추가**로 직접 입력하세요." 안내.

## 4. 감사·검증 대응 — 상세 (역할별 구분 및 팀원 입력 가시성)

### 4-1. 개요 및 진입

| 항목 | 내용 |
|------|------|
| **목적** | 공시·감사 요건 충족 여부 확인, 산정 근거·계보 추적, 수동 조정·배출계수·품질 이력 조회, 증빙 패키지·리포트 제출 |
| **진입** | GHG 산정 페이지 상단 탭에서 **감사·검증 대응** 클릭 |
| **선택 사항** | **공시 프레임워크** (ISSB, KSSB, K-ETS, GRI, ESRS) 선택. 선택한 프레임워크에 따라 요건 체크리스트·표시 항목이 바뀜 |

### 4-2. 팀원 수동 입력 가시성 (팀장 조회)

| 구분 | 내용 |
|------|------|
| **팀원이 수동 입력한 데이터** | 팀원이 **연료 추가**, **전력 행 추가**, 엑셀 업로드, EMS 불러오기 등으로 입력·수정한 활동자료는 **팀장이 그대로 조회 가능**합니다. |
| **조회 위치** | **GHG 산정·공시** 탭 → **산정 입력** 6탭(전력·열·스팀, 연료·차량, 냉매 등)에서 전체 사업장·전체 Scope 기준으로 그리드와 결과를 볼 수 있습니다. 팀장은 필터로 사업장·연도 등을 좁혀서 특정 담당 구간만 볼 수도 있습니다. |
| **입력자·일시 확인** | **감사·검증 대응** 탭 → **수동 조정 이력** 메뉴에서 "누가(입력자), 언제, 어떤 항목을 수정·추가했는지" 이력으로 확인할 수 있습니다. 팀장은 전체 팀원 이력 조회, 팀원은 본인 입력분만 조회(권한에 따라 다름). |
| **정리** | 팀원이 수동 입력하면 팀장이 **산정 입력 화면**에서 바로 보고, **수동 조정 이력**에서 입력자·일시를 함께 확인할 수 있습니다. |

### 4-3. 메뉴별 상세 (화면·행동·역할)

#### 요건 체크리스트

| 항목 | 내용 |
|------|------|
| **화면** | 선택한 **공시 프레임워크**(ISSB, KSSB, K-ETS, GRI, ESRS)별로 "해당 프레임워크에서 요구하는 GHG·에너지 관련 요건" 목록. 항목별 충족 여부(충족/미충족/해당 없음 등) 표시. |
| **사용자 행동** | 요건 목록 훑어보기. 미충족 항목 클릭 시 **데이터 계보 추적**으로 이동하거나, **산정 입력** 탭(전력·연료 등)으로 가는 링크로 해당 입력 화면으로 이동. |
| **팀장** | ✓ 전체 요건 조회·확인. 결재 전 최종 점검. |
| **팀원** | ✓ 조회. 담당 영역 요건만 강조되면 담당 부분 확인에 유리. |

#### 데이터 계보 추적

| 항목 | 내용 |
|------|------|
| **화면** | 산정 결과(배출량 tCO₂e)까지 **어떤 활동자료·어떤 배출계수·어떤 산식**이 사용되었는지 단계별로 보여 주는 계보(Lineage) 뷰. 사업장·Scope·탭(전력/연료 등) 단위로 드릴다운. |
| **사용자 행동** | 계보 트리 또는 링크 따라가기. "이 숫자가 왜 나왔는지" 추적. 필요 시 **산정 입력** 탭(해당 전력·연료 등)으로 이동해 원본 활동자료·배출계수 확인. |
| **팀장** | ✓ 전체 계보 조회. 감사 질의 시 근거 제시용. |
| **팀원** | ✓ 조회. 담당 Scope·사업장 계보만 보이거나, 전체 조회(권한에 따라). |

#### 수동 조정 이력

| 항목 | 내용 |
|------|------|

## 9. 요약

- **라우트·구조**: GHG 산정 페이지. 상단 탭 3개 — **GHG 산정·공시** | **감사·검증 대응** | **리포트 생성**. 헤더 "온실가스 배출량 산정 (GHG Protocol)".
- **데이터 입력**: **내부시스템 적재** + **EMS 불러오기** + **엑셀 업로드** + 수동 입력(활동자료 그리드·각 Scope 탭 입력 영역). Scope 3은 EMS/엑셀 자동 입력 미지원(토스트 안내).
- **산정 입력 6탭**: 전력·열·스팀, 연료·차량, 냉매, 폐기물, 물류·출장·통근, 원료·제품. K-ETS 선택 시 폐기물·물류·원료 탭 숨김. **전력·연료** 탭은 **STEP 1 데이터 확인 → STEP 2 산정 실행 → STEP 3 결과 확인** 스텝 플로우.
- **툴바**: 히스토리(N), 필터, EMS 불러오기, 엑셀 업로드, 전체 결과 저장, CSV 내보내기, 계수 기준 연도(2024/2023).
- **감사·검증 탭**: 공시 프레임워크 선택. 좌측 메뉴 — 요건 체크리스트, 데이터 계보 추적, 수동 조정 이력, 배출계수 적용 내역, 데이터 품질 분포, 산정 버전 히스토리, 감사인 전용 뷰, 증빙 패키지 내보내기.
- **리포트 생성 탭**: Evidence Report 생성 완료, PDF Report, Excel Details, JSON, 일괄 다운로드, 검증의견서 부속 서류 6항목.
- **역할**: **ESG 팀장** — 필터·전체 수정·저장·감사인 뷰·증빙 패키지·리포트 다운로드. **ESG 팀원** — 담당 범위 필터·제한적 수정·저장(권한 시)·감사 메뉴 조회·리포트 제한 다운로드.
