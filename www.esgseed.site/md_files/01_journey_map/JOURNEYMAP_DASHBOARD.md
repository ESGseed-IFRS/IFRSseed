# 대시보드 페이지 저니맵

대시보드 페이지의 사용자 저니맵을 **ESG 팀장(결재권자)** 과 **ESG 팀원** 두 사용자 유형별로 상세 정리합니다. 구조(사용자 행동, 화면 상태, 감정/생각, 문제점, 기회 요소)에 맞추었으며, 현재 프론트엔드(`/dashboard`) 구현을 반영했습니다.

---

## 1. 세부 시트 행(열) 구조

| 열 순서 | 열 이름 | 설명 |
|--------|----------|------|
| 1 | **단계 (Step)** | 단계 번호 및 단계명 |
| 2 | **사용자 행동 (User Action)** | 해당 단계에서 사용자가 **직접 하는 행동** (클릭, 확인, 선택) |
| 3 | **화면 상태 (Screen State)** | 그때 화면에 **보이는 것** (카드, 버튼, 진행률, 알림, 다운로드) |
| 4 | **감정/생각 (Emotion/Thought)** | 사용자가 느끼거나 생각할 수 있는 것 |
| 5 | **문제점 (Pain Point)** | 불편함, 혼란, 막힘 |
| 6 | **기회 요소 (Opportunities / Solutions)** | 개선·해결 방향 |

---

## 2. 대시보드 페이지 — 사용자가 보는 요소 (현행 구현 기준)

### 2-1. 공통

| 구분 | 화면에 보이는 이름 |
|------|-------------------|
| 페이지 제목 | **대시보드** |
| 부제 | ESG 보고서 작성 현황을 한눈에 확인하고, 필요한 작업을 빠르게 진행하세요 |
| 역할 배지 | **팀장** (녹색) / **팀원** (파랑) |
| 역할 미리보기 | **팀장** · **팀원** 버튼 (역할 전환용) |
| **블록 1** | **전체 진행률** — 퍼센트 바, 완료/진행중/대기 개수, 마감일, D-day |
| **블록 2** | **페이지별 진행 상태** — 회사정보 / GHG 산정 / SR 작성 / 도표 생성 (각 카드: 상태 배지, 메시지, 진행률, "이동 →") |
| **블록 3** | **확인 필요 사항 · 다음 액션** — 미완료·미제출 항목 목록, "[페이지명]으로 이동" 링크 |
| **블록 6** | **보고서 출력** — 보고서 완성도 스코어카드, 섹션별 진행 상황, Excel(.xlsx)·PowerPoint(.pptx) 다운로드 |

### 2-2. 팀장 전용

| 구분 | 화면에 보이는 이름 |
|------|-------------------|
| 헤더 | **마감일** 날짜 입력 (date picker) |
| 블록 2 카드 | 각 카드 하단 **담당: 홍길동** 등 담당자 표시 |
| **블록 4** | **팀원 현황** — "가입 승인 대기 N명", 승인/반려 버튼, **+ 팀원 초대**, 팀원 목록(이름·부서·담당 페이지·진행률·활성/승인 대기) |
| **블록 6** | **최종 승인 — 완성도 충족 후 활성화** 버튼 (스코어카드 모두 충족 시 "최종 승인" 가능) |
| **블록 7** | **최근 활동 로그** — 팀 전체 활동 기록, "전체 로그 보기 →" |

### 2-3. 팀원 전용

| 구분 | 화면에 보이는 이름 |
|------|-------------------|
| **블록 5** | **팀장 피드백** — 수정 요청(rejected)·승인 완료(approved) 메시지, **재요청** 버튼 |

### 2-4. 보고서 완성도 스코어카드 (블록 6)

| 항목 | 설명 |
|------|------|
| 회사정보 제출 | ✓/✗ |
| GHG 산정 결과 저장 | ✓/✗ |
| SR 준수율 90% 이상 | 현재 N% |
| 차트·도표 저장 | ✓ (N건) / ✗ (0건) |

### 2-5. 섹션별 진행 상황 (블록 6)

회사정보, 지속가능경영 전략, 환경 성과, 사회적 책임, 지배구조, 성과 지표, 향후 계획 — 각 섹션별 진행률 바(0%).

### 2-6. 다운로드

- **Excel (.xlsx)** · **PowerPoint (.pptx)** — 팀장 **최종 승인** 후 활성화. 팀장·팀원 모두 다운로드 가능.

## 3. 저니맵 — ESG 팀장(결재권자)

### 3.1 세부 저니맵 표

| 단계 (Step) | 사용자 행동 (User Action) | 화면 상태 (Screen State) | 감정/생각 (Emotion/Thought) | 문제점 (Pain Point) | 기회 요소 (Opportunities / Solutions) |
|-------------|---------------------------|---------------------------|-----------------------------|---------------------|----------------------------------------|
| **1. 진입 및 역할 확인** | **대시보드** 메뉴 클릭 | 대시보드 페이지. 제목 "대시보드", 부제, **팀장** 배지(녹색). 역할 미리보기에서 **팀장** 선택 상태. 마감일 날짜 입력란 | "팀 전체 진행 상황을 여기서 보겠다." | — | — |
| **2. 전체 진행률 확인** | **전체 진행률** 블록에서 퍼센트·완료/진행중/대기 개수 확인. 마감일·D-day 확인 | 진행률 바(%), N% 숫자. 완료·진행중·대기 개수. "마감일 YYYY-MM-DD", "D-N" 배지 | "몇 %나 완료됐지. 마감일이 얼마나 남았지." | 마감일 미설정 시 "미설정"만 보여 다음 액션 불명확 | 마감일 설정 유도 문구, 권장 마감일 가이드 |
| **3. 페이지별 진행 상태 확인** | **페이지별 진행 상태** 카드 4개(회사정보, GHG 산정, SR 작성, 도표 생성) 확인. 각 카드의 상태(완료/진행중/대기)·메시지·담당자 확인 | 각 카드: ✓/◐/○ 배지, 상태 라벨, 메시지(예: "제출 완료", "결과 저장 필요"), 진행률 바(해당 시), "담당: 홍길동", "이동 →" | "회사정보는 제출됐고, GHG는 결과 저장이 필요하구나. 담당자별로 어디가 막혀 있는지 보자." | 담당자 클릭 시 연락·할당 변경 불가 | 담당자 클릭 시 담당 페이지로 이동 또는 연락처 툴팁 |
| **4. 확인 필요 사항 처리** | **확인 필요 사항 · 다음 액션** 목록 확인. "회사정보 제출 필요", "GHG 결과 저장 필요" 등 클릭하여 해당 페이지로 이동 | 각 항목: 우선순위별 점(빨강/주황/회색), 메시지, "[회사정보]로 이동" 등 링크 | "우선 회사정보 제출부터 시키자." | 항목이 많을 때 우선순위가 시각적으로만 구분됨 | 우선순위 숫자·정렬 안내, "오늘 할 일" 3개만 강조 |
| **5. 팀원 현황·승인 처리** | **팀원 현황** 블록에서 "가입 승인 대기 N명" 확인. **승인** 또는 **반려** 클릭. **+ 팀원 초대** 클릭 | 가입 승인 대기 영역(노란 배경). 팀원별 요청일, 승인/반려 버튼. 팀원 목록: 이름·부서·담당 페이지·진행률·활성/승인 대기 | "새로 가입 요청한 사람 승인해야겠다." | 승인 후 역할·담당 페이지 지정 플로우 없음 | 승인 시 역할·담당 페이지 지정 모달 |
| **6. 보고서 완성도 스코어카드 확인** | **보고서 출력** 블록에서 **보고서 완성도 스코어카드** 4항목(회사정보 제출, GHG 저장, SR 90%, 차트 저장) 확인 | 각 항목: ✓ 또는 ✗, 상세(현재 N% 등). 모두 충족 시 **최종 승인** 버튼 활성화 | "네 가지 다 충족되면 최종 승인할 수 있구나." | 스코어카드 미충족 시 "어디를 채워야 하는지" 한눈에 안 보임 | 미충족 항목 클릭 시 해당 페이지로 이동 |
| **7. 최종 승인** | 스코어카드 4항목 모두 충족 시 **최종 승인 — 완성도 충족 후 활성화** 버튼 클릭 | 버튼 클릭 후 "최종 승인 완료"로 전환, 비활성화. Excel·PowerPoint 다운로드 버튼 활성화 | "이제 팀원들도 보고서 다운로드할 수 있겠다." | 승인 후 취소·재검토 절차 없음 | "최종 승인 취소" (재검토) 옵션 (확장 시) |
| **8. 다운로드** | **Excel (.xlsx)** 또는 **PowerPoint (.pptx)** 버튼 클릭 (최종 승인 후) | 두 버튼 활성화(녹색/파랑). 클릭 시 파일 다운로드. "팀장 최종 승인 후 팀장·팀원 모두 다운로드 가능" 안내 | "공시·제출용 파일 받아두자." | PDF 옵션 없음 (현행 Excel·PPT만) | PDF 다운로드 추가 (확장 시) |
| **9. 최근 활동 로그 확인** | **최근 활동 로그** 블록에서 팀 전체 활동 기록 확인. **전체 로그 보기 →** 클릭 (확장 시) | 로그 목록: 사용자 이니셜·이름·행동 내용·"N분 전"/"N시간 전"/"N일 전" | "누가 최근에 뭘 했는지 확인하자." | 로그 필터(담당자·페이지별) 없음 | 담당자·페이지별 필터, 기간 선택 |
| **10. 페이지로 이동** | 페이지별 카드 또는 확인 필요 사항에서 **이동 →**, **[페이지명]로 이동** 클릭 | 해당 라우트(회사정보, GHG 산정, SR 작성, 도표 생성)로 이동 | "부족한 부분 채우러 가자." | — | 대시보드로 돌아오기 링크 상단 고정 |

## 4. 저니맵 — ESG 팀원

### 4.1 세부 저니맵 표

| 단계 (Step) | 사용자 행동 (User Action) | 화면 상태 (Screen State) | 감정/생각 (Emotion/Thought) | 문제점 (Pain Point) | 기회 요소 (Opportunities / Solutions) |
|-------------|---------------------------|---------------------------|-----------------------------|---------------------|----------------------------------------|
| **1. 진입 및 역할 확인** | **대시보드** 메뉴 클릭 | 대시보드 페이지. 제목 "대시보드", 부제, **팀원** 배지(파랑). 역할 미리보기에서 **팀원** 선택 상태. 마감일·팀원 현황·최근 활동 로그 블록 **비표시** | "내가 할 일과 전체 진행 상황을 확인하자." | — | — |
| **2. 전체 진행률 확인** | **전체 진행률** 블록에서 퍼센트·완료/진행중/대기 확인 | 진행률 바, N%, 완료·진행중·대기 개수. 마감일은 표시되나 팀원은 수정 불가 | "전체적으로 어디까지 됐는지 보자." | 마감일이 팀원에게 안 보이면 일정 파악 어려움 | 마감일 읽기 전용으로 팀원에게도 표시 |
| **3. 페이지별 진행 상태 확인** | **페이지별 진행 상태** 카드 4개 확인. 본인 담당 페이지 카드 클릭하여 해당 페이지로 이동 | 각 카드: 상태 배지, 메시지, 진행률, "이동 →". 담당자 표시 없음 | "내 담당인 GHG 산정이 진행중이네. 결과 저장하러 가야겠다." | 본인 담당 페이지가 불명확할 수 있음 | "내 담당" 뱃지 또는 담당 페이지 강조 (데이터 연동 시) |
| **4. 확인 필요 사항 처리** | **확인 필요 사항 · 다음 액션** 목록 확인. 본인과 관련된 항목 클릭하여 해당 페이지로 이동 | 팀장과 동일한 목록. "회사정보 제출 필요", "GHG 결과 저장 필요" 등, "[페이지명]으로 이동" | "GHG 결과 저장 필요하다고 나왔으니 GHG 페이지 가서 저장하자." | 팀원에게는 "본인 담당만" 필터 옵션 없음 | "내 담당만 보기" 토글 (확장 시) |
| **5. 팀장 피드백 확인** | **팀장 피드백** 블록에서 수정 요청·승인 결과 확인. 수정 요청 시 해당 페이지로 이동 후 수정, **재요청** 클릭 (필요 시) | 피드백 카드: ○(수정 요청) 또는 ✓(승인 완료). 섹션명·메시지·시간. 수정 요청 시 **재요청** 버튼 | "팀장이 GHG Scope 3 보완 요청했구나. 수정하고 재요청하자." | 피드백 클릭 시 해당 페이지 특정 위치로 이동 안 됨 | "해당 섹션으로 이동" 링크 |
| **6. 보고서 완성도·섹션별 진행 확인** | **보고서 출력** 블록에서 **보고서 완성도 스코어카드**·**섹션별 진행 상황** 확인. 최종 승인 버튼은 **비표시** | 스코어카드 4항목(✓/✗). 섹션별 진행률 바(0%). Excel·PowerPoint 버튼 — **최종 승인 전** 비활성화 | "아직 최종 승인이 안 됐구나. 내가 할 일만 하자." | 팀원은 "언제 승인될지" 알 수 없음 | "팀장 검토 중" 등 상태 문구 (확장 시) |
| **7. 다운로드 (최종 승인 후)** | 팀장이 최종 승인한 후 **Excel (.xlsx)** 또는 **PowerPoint (.pptx)** 버튼 클릭 | 두 버튼 활성화. 클릭 시 파일 다운로드. "팀장 최종 승인 후 팀장·팀원 모두 다운로드 가능" 안내 | "승인됐으니 보고서 받아두자." | — | 다운로드 이력 또는 알림 "최종 승인되었습니다" (확장 시) |
| **8. 페이지로 이동** | 페이지별 카드 또는 확인 필요 사항에서 **이동 →**, **[페이지명]으로 이동** 클릭 | 회사정보 / GHG 산정 / SR 작성 / 도표 생성으로 이동 | "담당 작업 하러 가자." | — | 대시보드로 돌아오기 링크 |
