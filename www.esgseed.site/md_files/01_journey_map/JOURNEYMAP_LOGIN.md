# 로그인·회원가입 저니맵

로그인 및 회원가입 과정의 사용자 저니맵을 정리합니다. **ESG 팀장(결재권자)** 과 **ESG 팀원** 두 사용자 유형에 따라 **회원가입 경로가 분리**되며, 로그인은 공통으로 동일한 화면을 사용합니다.

---

## 1. 사용자 유형 및 회원가입 경로 분리

### 1.1 두 가지 사용자 유형

| 사용자 유형 | 설명 | 계정 생성 경로 |
|-------------|------|----------------|
| **ESG 팀장(결재권자)** | 최종 승인권자, 회사 최초 관리자 | **회사 등록** → 초대 링크 수신 → ESG팀장 등록 → 비밀번호 설정 |
| **ESG 팀원** | 일반 사용자, ESG팀·환경안전팀·인사팀 등 | **회원가입** → 회사 정보 확인 → 이메일 인증 → 비밀번호 설정 → HRIS 조회 또는 승인 대기 |

### 1.2 회원가입 경로 요약

- **ESG 팀장**: 홈페이지 → **회사 등록** 버튼 → 회사 정보 입력 및 DART 자동 불러오기 → 최종 승인권자 이메일로 **초대 링크 발송** → 초대 링크에서 **ESG팀장(관리자) 등록** → 비밀번호 설정 → 관리자 대시보드로 이동
- **ESG 팀원**: 홈페이지 → **회원가입** 버튼 → 회사 정보 입력·확인(미등록 시 "회사 등록 먼저 필요" 안내) → 이메일 인증 → 비밀번호 설정 → **HRIS 조회**로 역할 자동 할당 또는 **승인 대기** → 회원가입 완료 후 로그인

## 2. 세부 시트 행(열) 구조

| 열 순서 | 열 이름 | 설명 |
|--------|----------|------|
| 1 | **단계 (Step)** | 단계 번호 및 단계명 |
| 2 | **사용자 행동 (User Action)** | 해당 단계에서 사용자가 **직접 하는 행동** (클릭, 입력, 선택, 확인) |
| 3 | **화면 상태 (Screen State)** | 그때 화면에 **보이는 것** (버튼, 입력 필드, 안내 문구, 배지) |
| 4 | **감정/생각 (Emotion/Thought)** | 사용자가 느끼거나 생각할 수 있는 것 |
| 5 | **문제점 (Pain Point)** | 불편함, 혼란, 막힘 |
| 6 | **기회 요소 (Opportunities / Solutions)** | 개선·해결 방향 |

## 3. 로그인 (공통 — ESG 팀장·ESG 팀원 동일)

두 사용자 유형 모두 계정 생성 완료 후 **동일한 로그인 화면**으로 접속합니다.

### 3.1 로그인 — 사용자가 보는 요소

| 구분 | 화면에 보이는 이름 |
|------|-------------------|
| **로그인 페이지** | 제목 **로그인**. 부제: "이메일과 비밀번호 또는 소셜 계정으로 IFRSseed에 접속하세요." |
| 입력 필드 | **이메일** (placeholder: name@company.com), **비밀번호** (placeholder: 비밀번호를 입력하세요) |
| 버튼 | **이메일로 로그인** · 구분선 "또는 소셜 계정으로 시작하기" · **Google로 시작하기** · **카카오로 시작하기** · **네이버로 시작하기** |
| 하단 링크 | **비밀번호를 잊으셨나요?** · **아직 계정이 없으신가요?** · **회사 등록** (ESG 팀장용) |
| **인증 콜백** | 구글/카카오/네이버 로그인 후 "로그인 처리 중..." → "로그인 성공! 홈으로 이동합니다..." 또는 실패 메시지 |
| **로그인 후** | 로그인 탭 숨김. 권한(ESG 팀장/ESG 팀원)에 따라 접근 가능한 탭만 표시. 사용자 정보·로그아웃 노출 |

### 3.2 로그인 저니맵 (공통)

| 단계 (Step) | 사용자 행동 (User Action) | 화면 상태 (Screen State) | 감정/생각 (Emotion/Thought) | 문제점 (Pain Point) | 기회 요소 (Opportunities / Solutions) |
|-------------|---------------------------|---------------------------|-----------------------------|---------------------|----------------------------------------|
| **1. 진입 및 인식** | 홈페이지 접속 후 플랫폼 첫 화면 확인 | **로그인** / **회원가입** / **회사 등록** 버튼 표시. 네비게이션에 **로그인** 탭, **아직 계정이 없으신가요?** 링크 | "이용하려면 로그인해야 하는 건가?" | — | 회사 최초 사용자와 팀원 구분 안내 (회사 등록 vs 회원가입) |
| **2. 로그인 입력** | 이메일과 비밀번호 입력 후 **이메일로 로그인** 클릭. 또는 **Google/카카오/네이버로 시작하기** 클릭 | **이메일**·**비밀번호** 필드, **이메일로 로그인** 버튼. 소셜 로그인 시 해당 서비스로 리다이렉트 | "내 계정으로 접속할 수 있겠다." | — | 자동 로그인(기억하기), 비밀번호 찾기 |
| **3. 로그인 처리** | 로그인 버튼 클릭 후 결과 대기 | "로그인 처리 중..." 표시. 완료 시 "로그인 성공! 홈으로 이동합니다...". 실패 시 오류 메시지 | "잘 로그인 되는지 확인하자." | 승인 대기 중 사용자는 로그인 불가 시 혼란 가능 | 승인 대기 시 "승인 대기 중입니다. ESG 팀장 승인 후 로그인 가능합니다" 안내 |
| **4. 서비스 시작** | 네비게이션 탭 확인. 회사정보·GHG·대시보드 등 메뉴 접근 | 로그인 탭 숨김. ESG 팀장은 관리자 대시보드·전체 메뉴, ESG 팀원은 권한별 메뉴만 표시 | "권한에 맞는 기능을 사용할 수 있겠다." | — | 권한별 기능 안내, 다음 단계 유도 |

## 4. 회원가입 — ESG 팀장(결재권자) 플로우

> 회사 최초 등록 시 **회사 등록** → 초대 링크 → ESG팀장 계정 생성 순서로 진행합니다.

| 단계 (Step) | 사용자 행동 (User Action) | 화면 상태 (Screen State) | 감정/생각 (Emotion/Thought) | 문제점 (Pain Point) | 기회 요소 (Opportunities / Solutions) |
|-------------|---------------------------|---------------------------|-----------------------------|---------------------|----------------------------------------|
| **1. 회사 등록 시작** | 홈페이지에서 **회사 등록** 버튼 클릭 | 회사 등록 페이지. 회사명·사업자등록번호 입력 필드, **회사정보 자동 불러오기** 버튼 | "우리 회사를 먼저 등록해야 하는구나." | — | "회사 등록 후 최종 승인권자에게 초대 링크가 발송됩니다" 안내 |
| **2. 회사 정보 입력** | 회사명·사업자등록번호 입력 후 **회사정보 자동 불러오기** 클릭 | DART API 호출. 회사명, 대표자명, 본사 주소, 업종, 최종 승인권자 이메일 자동 입력 | "자동으로 채워지니까 편하다." | DART 미조회 시 수동 입력 필요 | 로딩 상태 표시, 수동 입력 옵션 |
| **3. 회사 정보 확인** | 읽기 전용으로 표시된 정보 확인 후 **확인 완료** 체크 | 회사명, 대표자명, 본사 주소, 업종, 최종 승인권자 이메일. 확인 완료 체크박스 | "우리 회사가 맞는지 확인하자." | — | "자동 입력됨" 배지로 출처 명확히 |
| **4. 회사 등록 완료** | **회사 등록** 버튼 클릭 | 회사 ID 생성. 최종 승인권자 이메일로 초대 링크 발송 완료 메시지 | "이제 승인권자가 초대 링크로 가입할 수 있겠다." | — | 초대 이메일 재발송 옵션 |
| **5. 초대 링크 수신** | 최종 승인권자가 "[IFRSseed AI] 회사 등록 완료 - ESG팀장 초대" 이메일 수신 후 초대 링크 클릭 | 초대 링크 페이지. 초대 토큰 검증 후 회사명·대표자명 등 회사 정보 표시 | "우리 회사 ESG 팀장으로 등록하는구나." | 링크 만료 시 재발송 필요 | 링크 유효 기간 표시, 만료 시 안내 |
| **6. ESG팀장 등록** | **ESG팀장(관리자)으로 등록하기** 버튼 클릭 | 비밀번호 설정 화면. 특수문자·영문·숫자 8~14자, 비밀번호 확인 입력 | "비밀번호를 안전하게 설정하자." | — | 비밀번호 강도 표시 |
| **7. 비밀번호 생성 완료** | **비밀번호 생성 완료** 버튼 클릭 | 이메일·암호화된 비밀번호 저장. "최종 승인권자" 역할 자동 부여. is_first_admin=True, 회사 정보 연동 | "이제 관리자로 로그인할 수 있겠다." | — | 권한 안내, 관리자 대시보드 진입 유도 |
| **8. 등록 완료 및 대시보드** | 자동 로그인 또는 로그인 페이지로 이동 | "최초 ESG팀장(관리자) 등록이 완료되었습니다." + 최종 승인권자 역할 안내. 관리자 대시보드로 이동 | "회사 정보부터 설정해야겠다." | — | 온보딩 가이드, 회사정보·GHG 순서 안내 |

## 5. 회원가입 — ESG 팀원 플로우

> 이미 등록된 회사의 팀원이 **회원가입** 버튼으로 가입합니다. HRIS 연동 여부에 따라 역할 자동 할당 또는 승인 대기로 분기됩니다.

| 단계 (Step) | 사용자 행동 (User Action) | 화면 상태 (Screen State) | 감정/생각 (Emotion/Thought) | 문제점 (Pain Point) | 기회 요소 (Opportunities / Solutions) |
|-------------|---------------------------|---------------------------|-----------------------------|---------------------|----------------------------------------|
| **1. 회원가입 시작** | **회원가입** 또는 **아직 계정이 없으신가요?** 클릭 | 회원가입 페이지. 회사명·사업자등록번호 입력 필드, **회사정보 자동 불러오기** | "우리 회사에 이미 등록되어 있나?" | — | "이미 등록된 회사라면 회원가입을 진행하세요" 안내 |
| **2. 회사 정보 확인** | 회사명·사업자번호 입력 후 **회사정보 자동 불러오기** 클릭 | DART API 호출. 회사 등록 상태(등록됨/미등록) 확인 | "우리 회사가 등록되어 있는지 확인되네." | 미등록 시 "회사 등록 먼저 필요" 안내 | 회사 미등록 시 ESG 팀장에게 회사 등록 요청 문구 |
| **3. 이메일 입력 및 인증** | 사용자 이메일 입력 후 **이메일 인증** 클릭. 수신한 인증코드 입력 후 **인증 완료** 클릭 | 인증코드 포함 이메일 발송. 인증코드 입력란, **인증 완료** 버튼 | "본인 이메일이 맞는지 확인하자." | — | 인증코드 재발송, 유효 시간 표시 |
| **4. 비밀번호 설정** | 특수문자·영문·숫자 8~14자 비밀번호 설정 및 확인 입력. **비밀번호 생성 완료** 클릭 | 비밀번호·비밀번호 확인 필드. 비밀번호 강도 표시 | "안전한 비밀번호로 설정하자." | — | — |
| **5. HRIS 조회 (시스템)** | 회사 ID와 이메일로 HRIS API 호출. 회사 소속 여부·직급·부서·사번 조회 | "사용자 정보 확인 중..." 표시 | — | — | — |
| **6-a. HRIS 성공 시** | (자동) 직급·부서에 따라 역할 자동 할당 | ESG팀장→최종 승인권자, ESG팀→ESG팀, 환경안전팀/인사팀→현업팀, 기타→일반 사용자. 계정 생성 완료, 역할 표시 | "역할이 자동으로 지정됐구나." | — | 역할별 권한 안내 |
| **6-b. HRIS 실패 시** | (자동) 기본 역할 일반 사용자, **승인 대기** 상태로 계정 생성 | "ESG 팀장(관리자) 승인 대기 중입니다." 안내. 최초 ESG팀장에게 승인 요청 알림 발송 | "승인 후에 사용할 수 있겠다." | 승인 지연 시 불편 | 승인 현황 조회, ESG 팀장 연락처 안내 |
| **7. ESG팀장 승인 (HRIS 실패 시)** | ESG팀장이 역할 지정 후 승인. 사용자에게 승인 완료 알림 발송 | 사용자는 "승인 완료" 알림 수신. 로그인 가능 상태로 전환 | "이제 로그인할 수 있겠다." | — | — |
| **8. 회원가입 완료** | 할당된 역할 또는 승인 대기 안내 확인. 자동 로그인 또는 **로그인** 페이지로 이동 | 권한별 기능 안내. 로그인 후 대시보드 또는 권한별 메뉴 진입 | "이제 서비스를 이용할 수 있겠다." | — | 온보딩, 다음 단계 안내 |

## 8. 요약

- **사용자 유형**: **ESG 팀장(결재권자)** / **ESG 팀원** 두 가지. 회원가입 경로가 분리됨.
- **로그인**: 두 유형 공통. 이메일·비밀번호 또는 소셜 로그인. 승인 대기 시 로그인 불가 안내.
- **ESG 팀장**: **회사 등록** → DART 자동 불러오기 → 초대 링크 발송 → 초대 링크에서 ESG팀장 등록 → 비밀번호 설정 → 관리자 대시보드.
- **ESG 팀원**: **회원가입** → 회사 정보 확인(미등록 시 "회사 등록 먼저 필요") → 이메일 인증 → 비밀번호 설정 → **HRIS 조회**로 역할 자동 할당 또는 **승인 대기**.
- **DART API**: 기본(회사명·대표자·주소·업종·corp_code·ceo_email) + 확장(사업자번호·법인등록번호·설립일·재무정보·임직원 수·감사인 등) 가져오기 가능.
- **수기 입력**: ESG 목표 및 비전, 이해관계자 정보, 지배구조 데이터(이사회 구성 등)는 회사정보에서 별도 입력.
- **언어**: **로그인**, **회원가입**, **회사 등록**, **회사정보 자동 불러오기**, **ESG팀장(관리자)으로 등록하기**, **이메일 인증** 등 사용자 노출 문구 기준.
