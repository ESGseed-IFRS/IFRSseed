# JOURNEY MAP TOTAL — 전체 사용자 여정 (프론트엔드 기준)

IFRSseed AI 플랫폼의 **전체 저니맵**을 한 테이블로 정리한 Design/Build용 문서입니다.  
아래 **5개 행(카테고리)** — **단계 목적**, **사용자 주요 행동**, **핵심 기능**, **데이터/AI 개입**, **산출물** — 에 맞춰 각 단계를 채웠으며, **세부 저니맵 6개**의 내용을 반영했습니다.

- **JOURNEYMAP_LOGIN.md** — 로그인·회원가입
- **JOURNEYMAP_COMPANY_INFO.md** — 회사정보
- **JOURNEYMAP_GHG_v2.md** — GHG 산정
- **JOURNEYMAP_SR_REPORT.md** — SR 작성
- **JOURNEYMAP_ESG_DATA.md** — 도표·그래프(ESG 데이터)
- **JOURNEYMAP_FINAL.md** — 최종보고서

---

## 전체 Journey Map — 5개 행(카테고리) × 6단계

| 단계 | 세부 단계 | 단계 목적 | 사용자 주요 행동 | 핵심 기능 | 데이터/AI 개입 | 산출물 |
|------|-----------|-----------|------------------|-----------|---------------|--------|
| **1. 로그인** | 회원가입 | 보고서 작업을 회사 단위로 관리하고, 접근 권한을 부여하기 위함 | **회원가입** 또는 **아직 계정이 없으신가요?** 클릭 → 회사 이메일·비밀번호·회사명·사업자등록번호 입력 → **회사정보 자동 불러오기** 클릭(확장 시) → 직급·부서 선택·본인 인증(구글/카카오/네이버 또는 수동) → 제출 | 회원가입 폼(단계 안내), 사업자등록번호·도메인 검증, 직급·부서 선택, 본인 인증(이메일/소셜) | DART API로 사업자등록번호 기반 회사정보 자동 조회(확장 시). 직급·부서에 따른 권한(수정/조회) 자동 할당. 로그인 검증·세션 복원 | 가입 완료 계정, 권한 부여 |
| | 로그인 | 동일 회사·권한으로 재진입 | **이메일**·**비밀번호** 입력 → **이메일로 로그인** 클릭. 또는 **Google/카카오/네이버로 시작하기** 클릭 | 로그인 폼, 소셜 로그인 버튼, **비밀번호를 잊으셨나요?** 링크, 세션 유지 | 로그인 검증, 세션·권한 복원. 콜백 페이지에서 "로그인 처리 중..." → "로그인 성공! 홈으로 이동합니다..." | 로그인된 세션, 네비게이션(권한별 탭) 진입 |
| **2. 회사정보** | 진입·레이아웃 | 조직·기업 정보를 한곳에 정의하고, GHG·SR·최종보고서의 기준이 되게 하기 위함 | **회사정보** 탭 클릭 → 페이지 구조·섹션(기업 기본정보, 연락처, ESG 목표·이해관계자) 확인 | 회사정보 관리 페이지, 좌측·중앙 섹션 카드, 우측 **보고서 요약 미리보기** | (확장 시) DART API·시스템 연동으로 일부 필드 자동 채움 | — |
| | 기본정보 입력·제출 | 공시 기준에 필요한 기업 정보·ESG 데이터를 구조화해 입력하고 최종보고서에 반영 | 기업 기본정보·연락처·ESG 목표·이해관계자 필드 입력·수정 → **정보 확인 및 검증** 클릭 → **최종 보고서에 제출** 클릭 | **기업 기본정보**·**연락처 정보**·**ESG 목표 및 비전**·**이해관계자 정보** 카드, 검증·제출 버튼, 실시간 미리보기 | 입력값 검증·저장. 최종 제출 시 finalCompanyInfo로 저장 → SR 문단·차트·최종보고서 데이터 소스로 사용 | 검증된 회사정보, 최종 보고서 제출 가능 상태 |
| | (확장) 시스템 연동 | ERP/EMS/HRIS 등에서 ESG 데이터를 자동 수집·동기화하기 위함 | **시스템 연동** 카드 → **연동 설정 시작** → 시스템 종류 선택 → 연동 정보·인증 설정 → 엔드포인트·필드 매핑 → 동기화 주기·**테스트 연결** | 연동 설정 UI, API 엔드포인트·인증 방식·매핑·동기화 주기 설정, 테스트 연결·샘플 데이터 조회 | DART·외부 API 연동. 필드 매핑 검증, 동기화 스케줄 | 연동된 데이터 소스, 자동 입력 회사/ESG 데이터 |
| **3. GHG 산정** | 산정 설정 | 산정 경계·보고 연도·공시 기준을 정하여 일관된 배출량 산정 기준 확보 | 보고 목적(K-ETS/글로벌) 체크, 조직 경계·보고 연도·Scope 1/2 범위 선택, **계수 기준 연도**·가이드라인 입력 | 산정 설정 창(체크박스·선택·입력), 필수 항목 완료 시 준수 완료 표시 | 설정값 저장. 배출계수 연도·Scope 범위에 따라 계산 로직 적용 | 경계·연도·Scope 확정 |
| | 데이터 입력·관리 | 사업장·연도·에너지원별 활동자료를 수집해 배출량 계산의 입력값 확보 | **Filter**에서 **사업장**·**에너지원**·**연도**·**시기 단위** 선택 → **조회**·**초기화**. **EMS 불러오기**·**엑셀 업로드**로 일괄 입력. **Scope 1**(고정·이동)/**Scope 2**(전력·열) 그리드에 사용량·데이터 품질 입력 → **결과 저장** | 필터 창, EMS 불러오기·엑셀 업로드 창, Scope 1/2/3 탭·RAW/탄소 배출량 뷰, **연료 추가**·**결과 저장**·**전체 삭제** 등 버튼 | 필터 조건 기반 EMS 데이터 자동 채움. 엑셀 파싱·검증 후 그리드 반영. 활동자료·선택한 계수 연도로 배출량 자동 계산 | 사업장·에너지원·월별 활동자료, Scope 1/2/3 배출량(tCO₂e) |
| | 산정 결과·감사·리포트 | 산정 결과 확인, IFRS 감사 대응 근거 확인, 증적·보고서 생성 | **산정 결과** 탭에서 **전체 사업장 합계**·**배출량 산정 결과**·**적용 배출계수 및 산식** 확인. **IFRS 감사대응** 탭에서 **요건 반영 확인** 테이블·상세 요약 확인. **리포트 생성** 탭에서 **PDF Report**·**Excel Details**·**검증의견서 부속 서류** 다운로드 | 산정 결과 / IFRS 감사대응 / 리포트 생성 탭, **Total Carbon Inventory**·**저장된 히스토리** 카드, 다운로드 버튼 | 배출량 계산 결과·배출계수·산식 저장. SR 문단·차트·최종보고서에서 GHG 결과 참조 가능 | 최종 tCO₂e, 감사용 산출 근거·계보, 증적 파일(PDF/Excel) |
| **4. SR 작성** | 목차·공시 기준 확인 | 공시 기준에 맞는 문단 구조와 요구 데이터를 파악하기 위함 | **SR 작성** 또는 **회사정보** 진입. **Table of Contents** 목차 클릭, 연결된 공시 기준(GRI/IFRS/SASB/ESRS)·요구 데이터 포인트 확인 | 목차 기반 UI, **전체 준수율**·공시 기준 목록(준수/부분/미준수) | 공시 기준 규칙·문단 구조 로드 | 목차·공시 기준 매핑 인지 |
| | 문단 생성·수정·확정 | AI로 초안 생성 후 사용자가 수정·확정하여 보고서 문단 완성 | 목차 항목 선택 → **크롤링 문구 생성** 또는 스마트 문단생성(**텍스트 분석 시작**·**AI ESG Consultant** 채팅) → 생성 문단 검토·수정·**저장**·**최종보고서에 저장** | 문단 편집기, **정량 데이터**(GHG 배출량 등)·**크롤링 기반 문구 제안**·**저장** 버튼. Content 페이지: **IFRS S2 필수 정보 체크리스트**·**실시간 보고서 문단 프리뷰** | AI Agent: 회사정보·GHG 산정 결과·(선택) 과거 보고서·공시 기준 규칙 기반 문단 생성. 생성 문단 저장·최종보고서 반영 | 확정된 문단, 최종보고서용 저장 |
| **5. 그래프·도표** | 데이터 소스·차트 유형 선택 | SR 보고서용 시각화를 값 입력 또는 기존 데이터로 자동 생성하기 위함 | **도표 및 그림 생성** 메뉴 클릭 → **Environmental**·**Social**·**Governance** 탭 선택 → **데이터 소스**(데이터 유형) 선택 → **차트 유형**(막대·원형·선·혼합 등)·**차트 설정**·**데이터 포인트** 입력 | E/S/G 탭, **데이터 소스**·**차트 설정**·**차트 유형**·**차트 미리보기**, **ESG DATA 테이블**(테이블 선택·저장·다운로드) | 선택한 데이터 소스(GHG 산정 결과·회사정보 등)로 차트 데이터 자동 채움. **차트 생성하기** 시 미리보기 갱신 | 차트·도표·테이블 초안 |
| | 저장·보고서 반영 | 생성한 차트·도표를 보고서에 포함시키기 위함 | **차트 미리보기** 확인 → **저장** 클릭. **도표 갤러리**에서 테이블 저장. **최종보고서에 저장** 연동 | **저장**·**PNG**·**차트 갤러리**·**도표 갤러리** | 차트·도표를 최종보고서용 저장소에 저장, 문단과 연결 가능 | 보고서에 포함된 시각화, 최종보고서 반영 |
| **6. 최종 보고서** | 통합·미리보기 | 지금까지의 문단·차트·회사정보를 하나의 SR 보고서로 통합해 검토하기 위함 | **최종보고서** 메뉴 진입 → **보고서 완성도**·**섹션별 진행 상황**(완료/진행중/대기중) 확인 → **보고서 미리보기**·**목차**·**품질 체크리스트**로 구조·포함 데이터(회사정보 제출 여부·AI 문단 개수·차트 개수) 검토, 필요 시 해당 페이지로 이동해 수정 | 진행률·섹션별 상태, **보고서 미리보기** 카드, **목차**·**품질 체크리스트**, **미리보기** 버튼 | 저장된 문단·차트·도표·회사정보 통합, 공시 기준에 맞는 구조 정렬 | 통합된 보고서 초안 |
| | 출력·제출 | 공식 제출·배포용 파일 생성 | **PDF 다운로드** 또는 **Word 다운로드** 버튼 클릭. 회사정보 미제출 시 안내 후 비활성화. 생성 완료 시 "PDF가 성공적으로 다운로드되었습니다!" 알림 | **PDF 다운로드**·**Word 다운로드** 버튼, **PDF 생성 중...**·완료 메시지, 포함 데이터 요약 | 문서 렌더링(ESGReportDocument)·파일 생성. 다운로드 후(선택) 콘텐츠·차트 초기화 | PDF·Word 파일, 공시·제출 가능 |

## 행(카테고리) 정의 — 이미지 기준

| 행 이름 | 설명 |
|--------|------|
| **단계 목적** | 해당 단계의 목표·필요성(왜 이 단계가 있는지) |
| **사용자 주요 행동** | 해당 단계에서 사용자가 **직접 하는 행동**(클릭·입력·선택·확인). 세부 저니맵의 "사용자 행동"을 요약 |
| **핵심 기능** | 화면에 보이는 **버튼·탭·카드·입력 필드** 등 시스템이 제공하는 기능(사용자 관점) |
| **데이터/AI 개입** | 데이터 연동(DART·EMS·엑셀)·AI 문단 생성·배출량 계산·검증·세션 등이 어떻게 개입하는지 |
| **산출물** | 해당 단계 완료 시 나오는 **결과물**(계정·권한·검증된 회사정보·배출량·문단·차트·PDF 등) |
