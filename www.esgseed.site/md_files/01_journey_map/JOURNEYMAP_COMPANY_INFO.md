# 회사정보 페이지 저니맵

회사정보 관리 페이지의 사용자 저니맵을 **4단계**로 정리합니다. **ESG 팀장(결재권자)** 과 **ESG 팀원** 두 사용자 유형에 따라 접근·수정 권한이 다르며, 사용자 관점의 행동·화면·감정을 중심으로 기술합니다. 5~11단계(시스템 종류 선택·연동 정보 입력 등)는 제외하며, **시스템 연동**은 수기 설정이 아닌 **적재된 데이터 조회** 흐름으로 전제합니다.

---

## 1. 회사 기본 정보 출처 정의

### (*) DART에서 끌어올 회사 기본 정보

DART API(공시 데이터) 기반으로 자동 조회·입력되는 항목입니다.

| 카테고리 | 항목 | 비고 |
|----------|------|------|
| **기업 식별** | 회사명, 사업자등록번호 | 공시 기준 |
| **대표 정보** | 대표자명 | 정기보고서 등 |
| **소재지** | 본사 주소 | 사업보고서 등 |
| **업종** | 업종(코드·명칭) | 공시 분류 |
| **이해관계자 (공시)** | 주요 주주 | 주식등의대량보유상황보고서 등 |

### (**) 사용자가 수기로 업데이트해야 하는 회사 기본 정보

DART에 없거나, 보고서용으로 사용자가 직접 입력·보완하는 항목입니다.

| 카테고리 | 항목 | 구분 이유 |
|----------|------|-----------|
| **연락처** | 대표 전화번호, 대표 이메일, 웹사이트 | DART에 미제공 또는 실무 연락처와 상이 |
| **ESG 목표 및 비전** | 미션, 비전, ESG 핵심 목표 | 기업 자체 정의 항목, 공시 표준 없음 |
| **이해관계자 (수기)** | 임직원 수, 기타 이해관계자, 소통 채널 | 연간보고서·SR용 자체 산출, DART 미제공 |

### 출처별 요약

| 출처 | 항목 예시 |
|------|-----------|
| **DART 자동** | 회사명, 사업자등록번호, 대표자명, 본사 주소, 업종, 주요 주주 |
| **수기 입력** | 대표 전화·이메일·웹사이트, 미션·비전·ESG 목표, 임직원 수, 기타 이해관계자, 소통 채널 |

## 2. 세부 시트 행(열) 구조

| 열 순서 | 열 이름 | 설명 |
|--------|----------|------|
| 1 | **단계 (Step)** | 1~4 단계 번호 및 단계명 |
| 2 | **사용자 행동 (User Action)** | 해당 단계에서 사용자가 **직접 하는 행동** (클릭, 입력, 확인) |
| 3 | **화면 상태 (Screen State)** | 그때 화면에 **보이는 것** (탭, 카드, 버튼, 입력창, 메시지) |
| 4 | **감정/생각 (Emotion/Thought)** | 사용자가 느끼거나 생각할 수 있는 것 |
| 5 | **문제점 (Pain Point)** | 불편함, 혼란, 막힘 |
| 6 | **기회 요소 (Opportunities / Solutions)** | 개선·해결 방향 |

## 4. 저니맵 — ESG 팀장(결재권자)

### 4.1 4단계 세부 저니맵

| 단계 (Step) | 사용자 행동 (User Action) | 화면 상태 (Screen State) | 감정/생각 (Emotion/Thought) | 문제점 (Pain Point) | 기회 요소 (Opportunities / Solutions) |
|-------------|---------------------------|---------------------------|-----------------------------|---------------------|----------------------------------------|
| **1. 진입 및 인식** | **회사정보** 메뉴 클릭 | 회사정보 관리 페이지 표시. 권한: 수정·조회 가능. "최종 승인권자" 배지 또는 안내 | "이제 보고서에 들어갈 우리 회사 기본 정보를 정리해볼까?" | — | — |
| **2. 레이아웃 인식** | 기업 기본정보, 연락처, ESG 목표, 이해관계자, 보고서 요약 미리보기 등 전체 섹션 구조 확인 | 좌측·중앙: 섹션별 카드. 우측: **보고서 요약 미리보기**. DART 자동 입력 항목은 "DART" 배지 표시 | "입력할 게 많네. 어떤 순서로 채워야 하지?" | 정보량이 많아 한눈에 파악하기 다소 복잡함 | 섹션 바로가기(앵커), 진행률 표시바 제공 |
| **3. 기본정보 입력** | DART 자동 입력 항목(회사명·대표자명·주소·업종 등) 검토. 연락처·ESG 비전·임직원 수 등 **수기 입력** 항목 직접 입력 후 **정보 확인 및 검증** 클릭 | DART 항목: 읽기 전용 또는 수정 가능 + "DART" 배지. 수기 항목: 입력 필드. 실시간 **보고서 요약 미리보기** 반영. 빈 필드 시 "N개 필드가 비어있습니다" 알림 | "공시 데이터는 가져왔으니 나머지 우리 팀 자료만 채우면 되겠다." | 동일 기업의 다른 사용자가 가입 시 중복 입력 번거로움 | 기업 단위 데이터 공유·승계 기능 |
| **4. 시스템 연동 (적재 데이터 조회)** | **시스템 연동** 카드 클릭 | 기설정 I/F 주기에 따라 **이미 적재된 데이터** 조회 화면 표시. 수기 연동 설정 없음. ERP/EMS/HRIS 등에서 적재된 ESG 데이터 목록·요약 | "수동 입력 대신 이미 들어온 데이터를 확인하면 되겠다." | 적재 시점·갱신 주기 안내 부족 시 불명확 | 적재 시점·다음 동기화 예정일 표시, 데이터 출처·I/F 주기 안내 |

## 5. 저니맵 — ESG 팀원

### 5.1 4단계 세부 저니맵

| 단계 (Step) | 사용자 행동 (User Action) | 화면 상태 (Screen State) | 감정/생각 (Emotion/Thought) | 문제점 (Pain Point) | 기회 요소 (Opportunities / Solutions) |
|-------------|---------------------------|---------------------------|-----------------------------|---------------------|----------------------------------------|
| **1. 진입 및 인식** | **회사정보** 메뉴 클릭 | 회사정보 관리 페이지 표시. 권한: 조회 중심, 수정은 ESG 팀장 승인 또는 제한 | "회사 정보를 확인할 수 있겠다." | 수정 권한 없을 경우 보완 요청 절차가 불명확 | 수정 요청·보완 제안 기능, ESG 팀장 연락처 안내 |
| **2. 레이아웃 인식** | 기업 기본정보, 연락처, ESG 목표, 이해관계자, 보고서 요약 미리보기 등 전체 섹션 구조 확인 | 좌측·중앙: 섹션별 카드(일부 읽기 전용). 우측: **보고서 요약 미리보기** | "어떤 정보가 이미 입력되어 있지?" | DART vs 수기 구분이 없으면 출처 파악 어려움 | DART/수기 구분 배지, 최종 수정자·수정일 표시 |
| **3. 기본정보 확인 및 수정 요청** | DART·수기 항목 모두 확인. 수정이 필요하면 ESG 팀장에게 요청 또는 (권한 있는 경우) 일부 수기 항목 직접 입력 후 **정보 확인 및 검증** | 권한에 따라 입력 가능/읽기 전용 구분. 수정 권한 없을 시 "수정이 필요하면 ESG 팀장에게 요청하세요" 안내 | "내가 담당하는 연락처·ESG 목표 부분만 수정할 수 있나?" | 권한별 수정 범위가 불명확 | 담당 섹션별 권한 안내, 수정 요청 이력 표시 |
| **4. 시스템 연동 (적재 데이터 조회)** | **시스템 연동** 카드 클릭 | 기설정 I/F 주기에 따라 **이미 적재된 데이터** 조회 화면 표시. 수기 연동 설정 없음. 조회 전용 또는 일부 내역 다운로드 | "적재된 ESG 데이터가 어떤 상태인지 확인해보자." | 적재 실패·오류 시 담당자 안내 부족 | 오류 시 IT·ESG 팀장 연락처, 재적재 요청 안내 |

## 6. [시스템 연동 시작] — 전제 변경

### 6.1 기존 vs 변경 후

| 구분 | 기존 (삭제됨) | 변경 후 |
|------|---------------|---------|
| **연동 방식** | 사용자가 수기로 시스템(ERP/EMS/HRIS 등) 연동 설정 | **수기 연동 없음**. 기설정 I/F 주기에 따라 데이터가 이미 적재됨 |
| **사용자 행동** | 연동 설정 시작 → 시스템 종류 선택 → 연동 정보·인증 입력 → 매핑·동기화 설정 | **시스템 연동 카드 클릭 시 적재된 데이터 조회**부터 시작 |
| **화면 초기 상태** | 연동 설정 가이드, 시스템 선택 UI | 이미 적재된 데이터 목록·요약, 적재 시점·I/F 주기 안내 |

### 6.2 적재 데이터 조회 화면에서 제공할 정보 (권장)

- 적재된 데이터 목록(ERP/EMS/HRIS 등 출처별)
- 최종 적재 시점, 다음 동기화 예정일
- I/F 주기(일일/주간/월간 등) 안내
- 데이터 오류·실패 시 안내 및 담당자 연락처

## 7. 현재 구현과의 대응

| 단계 | 현재 화면 반영 | 비고 |
|------|----------------|------|
| 1~3 | ✓ 반영 | **회사정보** 메뉴, 기업 기본정보·연락처·ESG 목표·이해관계자, **정보 확인 및 검증**·**최종 보고서에 제출**, **보고서 요약 미리보기** |
| 4 | 변경 필요 | **시스템 연동** → "연동 설정 시작" 대신 **적재된 데이터 조회** 화면으로 진입 |

## 8. 요약

- **단계**: 4단계 (진입 및 인식 → 레이아웃 인식 → 기본정보 입력/확인 → 시스템 연동(적재 데이터 조회)). 5~11단계 삭제.
- **사용자 유형**: **ESG 팀장(결재권자)** (수정·조회) / **ESG 팀원** (조회 중심, 제한적 수정).
- **(*) DART**: 회사명, 사업자등록번호, 대표자명, 본사 주소, 업종, 주요 주주 등 공시 데이터.
- **(**) 수기 입력**: 대표 전화·이메일·웹사이트, 미션·비전·ESG 목표, 임직원 수, 기타 이해관계자, 소통 채널.
- **[시스템 연동]**: 수기 연동 설정 없음. 기설정 I/F 주기로 이미 적재된 데이터가 있으며, 카드 클릭 시 **적재된 데이터 조회**부터 시작.
