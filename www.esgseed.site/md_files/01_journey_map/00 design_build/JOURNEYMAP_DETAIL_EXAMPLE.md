# Journey Map 단계별 상세 (Design / Build용)

본 문서는 **설계·개발용 Design/Build Journey Map**으로, `JOURNEYMAP_PROJECT.md`의 6단계 고정 흐름과 형식 요구를 따르며, `JOURNEY_MAP_USER.md`의 상세 저니맵 형식(사용자 행동 / 화면 상태 / 감정·생각 / 개선 포인트)과 **실제 플랫폼 기능·라우트·용어**를 반영해 작성했습니다.

## 2. 단계별 상세 Journey Map

---

### 2-1. 1️⃣ 로그인 (`/login`)

**목적**: 보고서 작업을 회사 단위로 관리하고, 직급·부서 기반 권한으로 회사정보·SR·차트·최종보고서 접근을 제어한다.

#### 사용자 여정 단계별 상세

| 단계 | 사용자 행동 | 화면 상태 | 감정/생각 | 개선 포인트 |
|------|-------------|-----------|-----------|-------------|
| **1. 진입 및 인식** | 홈 또는 플랫폼 첫 화면 접속 | 로그인/회원가입 화면 표시 | "이용하려면 로그인해야 하는구나" | 플랫폼 가치 제안(IFRSseed AI, SR 작성) 명확 표시 |
| **2. 회원가입 선택** | "회원가입" 버튼 클릭 | 회원가입 페이지로 이동 | "어떤 정보를 입력해야 하지?" | 회원가입 단계 수(예: 3~5단계) 안내 |
| **3. 기본정보 입력** | 회사 이메일 주소·비밀번호 입력 | 이메일·비밀번호 필드 활성화 | "회사 이메일로 가입하는구나" | 회사 이메일 도메인 검증, 비밀번호 강도 표시 |
| **4. 회사정보 입력** | 회사명, 사업자등록번호 입력 | DART API 연동 안내 메시지 표시 | "사업자번호로 회사 정보를 자동 불러올 수 있나?" | 사업자등록번호 형식 검증(000-00-00000) |
| **5. DART API 연동** | "회사정보 자동 불러오기" 버튼 클릭 | DART API 호출, 회사 공시정보 자동 조회 | "자동으로 채워지니까 편하다" | 로딩 상태 표시, API 실패 시 수동 입력 옵션 |
| **6. 회사정보 확인** | DART로 불러온 회사명·대표자명·본사 주소·업종 확인 (조회 전용, 수정 불가) | 자동 입력 필드 읽기 전용 표시 | "우리 회사가 맞는지 확인하자" | "자동 입력됨" 배지, "확인 완료" 체크박스 |
| **7. 직급·부서 및 권한** | 직급 선택(대표이사, 사장, 부사장, 임원, 부서장, 팀장, 일반 임직원, 인턴), 부서 선택(경영지원팀, ESG팀, 재무팀 등) | 직급/부서 드롭다운, 선택에 따른 권한 레벨 안내(수정/조회) | "내 직급에 따라 권한이 달라지는구나" | 직급별 권한 안내 툴팁 |
| **8. 이메일 인증** | 인증 링크 클릭 또는 인증번호 입력 | 이메일 인증 대기/완료 화면 | "인증하면 가입이 완료되겠다" | 재발송 옵션, 유효시간 표시 |
| **9. 회원가입 완료** | 인증 완료 후 자동 로그인 또는 로그인 페이지 이동 | 완료 메시지, 환영 화면, 권한 레벨 표시 | "이제 권한에 맞는 기능을 쓸 수 있다" | 온보딩·권한별 기능 안내(선택) |
| **10. 로그인(기존 회원)** | 회사 이메일·비밀번호 입력 후 로그인 | 로그인 처리, 세션 생성, 권한 적용 | "로그인하면 내 권한으로 이용 가능하겠다" | 자동 로그인, 비밀번호 찾기, 권한 표시 |
| **11. 서비스 이용 시작** | 네비게이션에서 홈·회사정보·GHG 산정·도표 및 그림 생성·SR 작성·최종보고서 탭 확인 | 로그인 탭 숨김, 권한에 따른 탭 접근 가능 | "회사정보부터 확인하거나 수정할 수 있겠다" | 다음 단계 안내(권한별 맞춤) |

#### 주요 기능

- 회사 이메일 기반 회원가입, 비밀번호 강도·이메일 인증
- DART API: 사업자등록번호 기반 회사 공시정보 자동 조회(회사명, 대표자명, 본사 주소, 업종), 실패 시 수동 입력
- 회원가입 시 회사정보 확인은 조회 전용(본인 회사 확인 목적)
- 직급·부서 기반 권한: **수정 권한**(대표이사, 사장, 부사장, 임원, 부서장, 팀장), **조회 권한**(일반 임직원, 인턴), **부서별 입력 권한**(환경안전팀·인사팀·경영지원팀 등 담당 섹션만)
- 로그인 시 권한 레벨 확인·적용, 회사정보 탭에 DART 정보 자동 반영

#### 데이터 저장·다음 단계

- **저장**: 사용자 계정(이메일·암호화 비밀번호), 회사-사용자 매핑, 직급·부서·권한 레벨
- **연결**: 로그인 후 `/company-info` 접근, 권한에 따라 수정/조회/부서별 입력 가능

---

### 2-2. 2️⃣ 회사정보 (`/company-info`)

**목적**: GHG 산정·SR 문단의 기준이 되는 조직 경계, 사업장, ESG 데이터를 한 곳에서 관리하고, "최종 보고서에 제출"하여 AI·최종보고서에서 사용한다.

이 단계는 **모든 GHG 산정과 SR 문단의 기준값**이 되는 핵심 단계이다.

#### 권한별 접근 제어

| 권한 레벨 | 직급 예시 | 회사정보 탭 접근 |
|-----------|-----------|------------------|
| **수정 권한** | 대표이사, 사장, 부사장, 임원, 부서장, 팀장 | 모든 섹션 조회·수정 |
| **부서별 입력 권한** | 환경안전팀, 인사팀, 경영지원팀 | 담당 섹션만 입력(환경/사회/지배구조 데이터) |
| **조회 권한** | 일반 임직원, 인턴 | 조회만 가능, 수정 불가 |

#### 회사정보 8개 섹션

- **1~4 (DART 보완)**: 기업 기본정보, 연락처 정보, ESG 목표 및 비전, 이해관계자 정보  
- **5~7 (ESG 실적)**: 환경 데이터(온실가스, 에너지, 물/폐기물, 환경 인증), 사회 데이터(인력, 산업안전, 인권, 협력업체, 지역사회), 지배구조 데이터(이사회, 내부통제, 윤리경영, 리스크)  
- **8. 데이터 소스 관리**: 섹션별 입력 방식(수동/시스템 연동/파일 업로드), API·필드 매핑·동기화 주기

#### 사용자 여정 단계별 상세 (수정 권한)

| 단계 | 사용자 행동 | 화면 상태 | 감정/생각 | 개선 포인트 |
|------|-------------|-----------|-----------|-------------|
| **1. 진입** | 로그인 후 "회사정보" 탭 클릭 | 회사정보 페이지 표시, 권한 레벨 표시(예: 수정 권한) | "회사 정보를 확인하고 수정할 수 있겠다" | 권한 레벨 배지 |
| **2. 레이아웃 인식** | 페이지 구조 확인 | 왼쪽: 섹션 탭 메뉴(8개), 오른쪽: 실시간 미리보기 카드 | "8개 섹션에 ESG 데이터도 넣는구나" | 섹션별 완성도(0~100%) 표시 |
| **3. 기본정보 입력** | 섹션 1~4 순차 입력(DART 보완) | DART 자동 입력 필드는 "자동 입력됨" 배지, 수정 가능 | "DART 정보는 확인만 하고 부족한 것만 채우면 되겠다" | 자동/수동 입력 필드 구분 |
| **4a. 데이터 소스 설정** | 섹션 8에서 ESG 섹션(5~7)별 입력 방식 선택: 수동 / 시스템 연동(ERP·EMS·HRIS·BI) / 파일 자동 업로드 / 하이브리드 | 입력 방식 카드·설명·"방식 선택" 버튼 | "ERP 연동하면 자동 업데이트되겠다" | 방식별 소요 시간·자동화 수준 안내 |
| **4b. 시스템 연동**(선택 시) | 시스템 종류 선택 → API 정보 입력 → 필드 매핑 → 동기화 주기 → 테스트 연결·첫 동기화 | 단계별 설정 화면, 테스트 결과 표시 | "한 번 설정하면 자동으로 올라오겠네" | 주요 ERP/EMS 프리셋, 설정 검증 |
| **4c. 파일 연동**(선택 시) | FTP/클라우드/이메일 선택 → 경로·스케줄 설정 → Excel 템플릿 다운로드 → 테스트 업로드 | 업로드 방식별 설정, 템플릿·파싱 미리보기 | "매월 파일만 넣으면 되겠다" | 파일 형식 검증, 에러 알림 |
| **4d. ESG 데이터 자동 수집** | 연동 설정 후 자동 수집: 환경(Scope 1/2/3, 에너지, 물, 폐기물), 사회(인력, 안전, 인권 등), 지배구조(이사회, 감사, 윤리 등) | "자동 연동됨" 배지, 마지막 동기화 시간, "지금 동기화" 버튼 | "수동 입력 안 해도 되겠다" | 동기화 실패 알림, 수동 대체 가능 |
| **4e. ESG 데이터 수동 입력** | 섹션 5~7 웹 폼 직접 입력 | 필드별 단위, 전년 데이터 참조, Excel 템플릿 업로드 옵션 | "나중에 연동으로 바꿀 수 있을까?" | 나중에 연동 전환 가능 안내 |
| **5. 데이터 검증** | "정보 확인 및 검증" 버튼 클릭 | 필수 필드·IFRS S1·S2 항목·형식 검증, 이상치 경고 | "필수 항목과 범위를 확인해야겠다" | 업계 평균·전년 대비 변화율 표시 |
| **6. 완성도 확인** | 섹션별 완성도·전체 진행률 확인 | 완성도 그래프, IFRS 필수 항목 달성률(예: 90/100) | "어디가 부족한지 한눈에 보이네" | 부족 항목 하이라이트·권장 사항 |
| **7. 저장/제출** | "최종 보고서에 제출" 버튼 클릭 | finalCompanyInfo로 제출, 성공 메시지 | "이제 AI가 이 데이터로 보고서를 만들 수 있겠다" | 제출 후 다음 단계 안내(SR 작성·GHG 등) |
| **8. 확인/완료** | 제출 완료 메시지 확인 | 제출 데이터가 AI·최종보고서에 전달됨 | "회사정보 입력 끝. 다음은 GHG나 SR이겠지" | 문단생성·GHG 탭 이동 버튼, 데이터 요약 |

#### 사용자 여정 (조회 권한)

| **4. 데이터 검증** | 포함 데이터 확인: finalCompanyInfo, contentSections, charts, 필수 항목 체크 | 데이터 포함 여부·누락 경고 | "빠진 데이터는 없는지 확인해야겠다" | 필수 항목 체크리스트 |
| **5. 미리보기 확인** | 오른쪽 미리보기 패널에서 보고서 전체 구조·목차·섹션 레이아웃 확인 | 실시간 PDF 스타일 미리보기(스크롤) | "최종 모습을 미리 보자" | 목차 클릭 시 해당 위치 이동, 줌 |
| **6. 섹션별 수정** | 수정할 섹션 선택 → 텍스트 편집·차트 수정/교체·레이아웃 조정 | 해당 섹션 편집 모드, 미리보기 실시간 반영 | "이 부분만 고치면 되겠다" | WYSIWYG, 자동 저장, 변경 이력 |
| **7. 품질 검증** | 품질 체크리스트·IFRS 기준 준수 여부 확인 | 항목별 통과/경고/실패 표시 | "IFRS 기준을 잘 지켰는지 확인하자" | 자동 검증 결과·개선 권장 |
| **8. 최종 확인 및 승인** | 모든 수정 후 "최종 확인" 체크박스 클릭 | "최종 확인 완료" 상태, PDF/Word 다운로드 버튼 활성화 | "이제 다운로드할 수 있다" | 최종 확인 전 수정 불가 안내 |
| **9. 다운로드** | "PDF 다운로드" 또는 "Word 다운로드" 클릭 | PDF/Word 생성 진행 표시 | "보고서가 생성되고 있다" | 진행률 표시 |
| **10. 완료 및 공유** | 다운로드 완료 확인, 공유 옵션(이메일·링크 복사 등) 확인 | 완료 메시지, 공유 UI | "보고서 완성. 공시·제출할 수 있다" | 다운로드 이력, 공유 옵션 |

#### 주요 기능

- **완성도 관리**: 전체 진행률(%), 섹션별 상태(완료/진행중/대기/누락), IFRS S1·S2 필수 항목 체크
- **실시간 미리보기**: PDF 스타일 미리보기, 목차→위치 이동, 줌 인/아웃
- **편집**: WYSIWYG, 텍스트·차트·레이아웃 수정, 자동 저장, 변경 이력
- **품질**: IFRS 기준 준수 체크, 품질 점수·개선 제안
- **출력**: PDF, Word 다운로드; 고해상도/요약본 등 옵션
- **협업(설계)**: 댓글·승인 워크플로우·버전 관리

#### 데이터·연결

- **포함 요소**: finalCompanyInfo, contentSections, charts(·esgTables)
- **산출**: PDF/Word 보고서 파일
- **외부**: 공시·제출용 다운로드; CDP 등은 `/cdp-response` 등 별도 기능으로 확장
