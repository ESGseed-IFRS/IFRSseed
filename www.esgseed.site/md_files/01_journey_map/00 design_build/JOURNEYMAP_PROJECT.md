# SR 보고서 작성 플랫폼 – Design / Build Journey Map 작성 요청

## 1. 플랫폼 개요 (Context)

### 1-1. 플랫폼 정의

본 플랫폼(**IFRSseed AI**)은 **지속가능경영보고서(SR) 및 ESG 공시 보고서를 작성·관리하기 위한 전문 SaaS 플랫폼**이다.

- **이름**: IFRSseed AI
- **역할**: “보고서를 직접 쓰는 도구”가 아니라, **공시 기준을 구조화하고, 데이터 → 문단 → 시각화 → 최종 보고서까지 한 흐름으로 연결하는 플랫폼**
- **결과물**: 공시 기준에 맞는 SR 보고서, IFRS(ISSB) 기반 보고서, 필요 시 CDP 등 외부 제출용 자료

### 1-2. 타깃 사용자

| 구분 | 설명 |
|------|------|
| **주 사용자** | 기업 ESG 담당자, 지속가능경영팀, IR·재무팀 |
| **협업 사용자** | 환경안전팀, 인사팀, 경영지원팀 등 부서별 데이터 입력 담당 |
| **권한 모델** | 수정 권한(임원·부서장·팀장) / 부서별 입력 권한 / 조회 권한(일반 임직원·인턴) |

회원가입 시 **직급·부서**에 따라 권한이 자동 부여되며, 회사정보·문단·차트·최종보고서에 대한 **조회 전용 / 수정 / 부서별 입력** 접근이 구분된다.

### 1-3. 공시 기준과 플랫폼의 관계

SR 보고서는 아래 **공시 기준**에 따라 문단 구조, 요구 데이터 포인트, 정량·정성 정보가 정해진다. 플랫폼은 이 기준을 “구조화된 규칙”으로 갖고, 사용자 입력·GHG 산정 결과·AI 생성 문단을 해당 규칙에 맞춘다.

| 공시 기준 | 역할 | 플랫폼에서의 활용 |
|-----------|------|-------------------|
| **GRI** | 글로벌 지속가능성 보고 표준, 이슈별 공시 항목 정의 | 목차·문단 구조, 데이터 포인트 매핑, 체크리스트 |
| **IFRS (ISSB)** | IFRS S1·S2 등 기후·지속가능 관련 회계 공시 | 필수 공시 항목 검증, 문단 생성 규칙, 최종보고서 품질 체크 |
| **SASB** | 산업별 재무적 중요 ESG 이슈 | 산업·업종별 데이터 요구사항, KPI 매핑 |
| **ESRS** | EU 지속가능성 보고 기준 | 유럽 진출 기업용 문단 구조·데이터 포인트 참조 |

각 기준은 **정해진 문단 구조, 요구 데이터 포인트, 정량/정성 정보**를 요구하며, 플랫폼의 목적은 다음 세 가지이다.

1. **공시 기준에 맞는 문단을 AI Agent(멀티모달 에이전트)가 자동 생성**
2. **온실가스(Scope 1, 2, 3) 배출량을 플랫폼 내에서 계산·관리**
3. **입력·산정된 값 기반으로 그래프·도표·테이블 자동 생성 후 보고서에 반영**

---

### 1-4. 플랫폼 구조 및 화면 구성

실제 **라우트·메뉴**와 Journey Map 단계는 아래와 같이 대응된다.

| 순서 | Journey 단계 | 라우트(예시) | 주요 화면·기능 |
|------|--------------|--------------|----------------|
| 1 | 로그인 | `/login` | 회원가입, 회사 이메일·비밀번호, DART API 연동(사업자등록번호 기반 회사정보 자동 조회), 직급·부서 기반 권한 부여 |
| 2 | 회사정보 | `/company-info` | 조직 경계, 사업장, 8개 섹션(기본정보, 연락처, ESG 목표, 이해관계자, 환경/사회/지배구조 데이터, 데이터 소스 관리), 검증 후 “최종 보고서에 제출” |
| 3 | GHG 산정 | `/ghg-calculation` | Scope 1 / Scope 2 / Scope 3 각각 페이지, 데이터 업로드(EMS 연동·Excel), 검증, 배출량 계산, 사업장·에너지원·연도 필터 |
| 4 | SR 작성 | `/report` | 탭: “SR 보고서 작성” / “IFRS 보고서 작성”. 목차 기반 문단 생성, 공시 기준 연결, AI 문단 생성·수정·확정 |
| 5 | 그래프·도표 | `/charts` | Environmental / Social / Governance 탭, 테이블 프리셋·데이터 소스 선택, 차트 유형 선택, 값 입력, 차트·도표 저장 후 최종보고서에 포함 |
| 6 | 최종 보고서 | `/finalreport` | 진행률, 섹션별 상태, 실시간 미리보기, 편집, PDF/Word 다운로드 |

추가로 **CDP 응답**(`/cdp-response`), **콘텐츠**(`/content`) 등 보조 페이지가 있을 수 있으며, Journey Map에서는 위 6단계를 **고정 순서**로 둔다.

### 1-5. 데이터·AI 연동 개요

- **회사정보**: DART API로 기본정보 자동 조회; 8개 섹션 중 환경·사회·지배구조는 수동 입력 또는 (설계상) 시스템 연동·파일 업로드로 확장.
- **GHG 산정**: EMS 연동 RAW DATA, Excel 업로드 → 사업장별·에너지원별·월별·연도별 구조 → 배출량 계산 로직 → 결과는 재사용 가능 데이터로 저장.
- **SR 작성**: 회사정보 + GHG 산정 결과 + (선택) 과거 보고서·레퍼런스 + **공시 기준 규칙**을 입력으로 AI Agent가 문단 생성; 사용자는 수정·확정.
- **그래프·도표**: 사용자 “값 입력” 또는 “데이터 소스 선택” → 차트/테이블 자동 생성 → “저장” 시 최종보고서용 저장소에 포함.
- **최종 보고서**: 저장된 문단·차트·도표·회사정보를 하나의 문서로 통합, 구조 정렬, PDF/Word 등 출력.

## 2. Journey Map 목적

본 Journey Map은 **마케팅용 사용자 여정이 아니라**,  
👉 **설계·개발을 위한 Design / Build Journey Map**이다.

따라서 다음을 명확히 드러내야 한다.

- **왜 이 단계 순서가 필요한지**
- 각 단계가 **SR 보고서 작성 흐름**에서 어떤 역할을 하는지
- **사용자 행동(User Action)**과 **시스템 기능(System / AI / Data)**이 어떻게 연결되는지
- **AI Agent, 데이터, 공시 기준**이 어디서 개입되는지

## 3. 전체 사용자 흐름 (고정 순서)

아래 순서는 고정이며, Journey Map은 이 흐름을 기준으로 작성한다.

2️⃣ **회사정보 입력** (조직 경계, 사업장)

## 5. 단계별 상세 Journey Map 작성 지침

전체 Journey Map 이후, **각 단계를 개별 섹션으로 나누어 상세 작성**한다.

### 5-1. 1️⃣ 로그인

**목적**

- 보고서 단위의 작업을 **회사 단위**로 관리하기 위함
- 직급·부서 기반 **권한 부여**로 회사정보·SR·차트·최종보고서에 대한 접근 제어

**실제 구현 참고**

- 회사 이메일·비밀번호, 회원가입 시 사업자등록번호 + DART API로 회사정보 자동 조회(확인용)
- 직급(대표이사, 사장, 부사장, 임원, 부서장, 팀장, 일반 임직원, 인턴) 및 부서 입력 → 수정 권한 / 조회 권한 / 부서별 입력 권한 자동 할당

**상세 Journey Map 테이블 열 구조**

| 항목               | 내용 |
| ------------------ | ---- |
| 사용자 역할        |      |
| 사용자 행동        |      |
| 시스템 기능        |      |
| 데이터 저장        |      |
| 다음 단계와의 연결 |      |

---

### 5-2. 2️⃣ 회사정보 (조직 경계, 사업장)

이 단계는 **모든 GHG 산정과 SR 문단의 기준값**이 되는 핵심 단계이다.

**반드시 포함할 요소**

- 조직 경계 설정 (법인 / 연결 / 운영 기준 등)
- 사업장 관리 (복수 사업장 가능)
- 사업장별 속성 (국가, 산업, 운영 여부 등)
- 8개 섹션: 기업 기본정보, 연락처, ESG 목표·비전, 이해관계자, **환경 데이터**, **사회 데이터**, **지배구조 데이터**, **데이터 소스 관리**(입력 방식: 수동/시스템 연동/파일 업로드 등)

**상세 테이블 열 구조**

| --------------------------- | ---- |
| 입력 정보                   |      |

| 시스템 처리                 |      |
| 이후 단계에서 사용되는 방식 |      |

### 5-3. 3️⃣ GHG 산정 (핵심 계산 단계)

본 단계는 SR 보고서의 **필수 정량 데이터 생성 단계**이다.

#### 기능 요구사항

- **Scope 1, Scope 2, Scope 3** 대상 (각각 전용 페이지 또는 탭)
- **데이터 수집 방식**

  - Excel 업로드
- **데이터 구조**

  - 에너지원별

- 데이터 업로드
- 데이터 검증
- 배출량 계산 결과

## 6. 작성 시 유의사항

- 기능 나열이 아니라 **“왜 이 기능이 필요한지”** 중심으로 작성할 것
- 모든 단계는 **SR 보고서 작성이라는 하나의 목적**으로 연결될 것
- **AI / 데이터 / 공시 기준**이 **어디서 어떻게 작동하는지** 명확히 표현할 것
- 실제 라우트·메뉴·권한 모델은 **1-4, 1-5**를 참고하여, 설계·개발 시 일관되게 반영할 것
