# GHG 산정 페이지 저니맵 작성 전략 (v2)

변경된 GHG 산정 프론트엔드에 맞춰 **세부 시트**에서 사용할 저니맵 작성 전략을 정리합니다. **사용자 관점**과 **화면에 보이는 버튼·단계**만 사용하며, 시스템/코드 용어(GHGFilterSidebar, isBoundaryComplete() 등)는 사용하지 않습니다.

---

## 1. 세부 시트 행(열) 구조

세부 저니맵 시트의 **행(열)** 은 아래 6개로 고정합니다.

| 열 순서 | 열 이름 | 설명 |
|--------|----------|------|
| 1 | **단계 (Step)** | 사용자가 겪는 단계·순서 (예: 1. 진입, 2. 조건 선택) |
| 2 | **사용자 행동 (User Action)** | 해당 단계에서 사용자가 **직접 하는 행동** (클릭, 입력, 선택) |
| 3 | **화면 상태 (Screen State)** | 그때 화면에 **보이는 것** (탭, 블록, 버튼, 테이블, 메시지) |
| 4 | **감정/생각 (Emotion/Thought)** | 사용자가 느끼거나 생각할 수 있는 것 |
| 5 | **문제점 (Pain Point)** | 불편함, 혼란, 막힘 |
| 6 | **기회 요소 (Opportunities / Solutions)** | 개선·해결 방향 |

---

## 2. 작성 원칙 (프론트엔드 언어만)

- **쓸 수 있는 것**
  - 사용자가 **눈으로 보는 요소**: 탭 이름, 버튼 문구, 입력란 라벨, 테이블·카드 제목, 안내 문구
  - 사용자 **행동**: "연도 선택", "조회 클릭", "Scope 1 탭 클릭", "결과 저장 클릭"
  - **화면 상태**: "산정 결과 탭이 열림", "좌측에 사업장·연도 선택 영역", "배출량 산정 결과 블록이 보임"
- **쓰지 말아야 할 것**
  - 컴포넌트·함수·변수명: `GHGFilterSidebar`, `GHGDataManagementCenter`, `isBoundaryComplete()`, `applyFilters()` 등
  - 개발자 전용 용어: "스토어", "payload", "엔드포인트" 등

## 3. 변경된 GHG 산정 페이지 — 사용자가 보는 요소 정리

아래는 **실제 화면에 노출되는 이름**만 정리한 목록입니다. 세부 시트의 "단계", "사용자 행동", "화면 상태" 작성 시 이 이름을 그대로 사용합니다.

### 3-1. 상단 (페이지 제목·탭·툴바)

| 구분 | 화면에 보이는 이름 |
|------|-------------------|
| 페이지 제목 | 온실가스 배출량 산정 (GHG Protocol) |
| 메인 탭 1 | **산정 결과** |
| 메인 탭 2 | **IFRS 감사대응** |
| 메인 탭 3 | **리포트 생성** |
| 툴바 버튼 | **히스토리 (N)** · **필터** · **EMS 불러오기** · **엑셀 업로드** · **전체 결과 저장** · **CSV 내보내기** |
| 툴바 선택 | **계수 기준 연도** (2024 / 2023 KR Emission Factors 등) |

### 3-2. 산정 결과 탭 — 좌측 (조건 선택·조회)

| 구분 | 화면에 보이는 이름 |
|------|-------------------|
| 영역 제목 | Filter (또는 필터) |
| 선택 항목 | **사업장** (드롭다운: 전체 / 본사 / 공장A …) |
| | **에너지원** (드롭다운: 전체 / 도시가스(LNG) / 전력 …) |
| | **연도** (드롭다운) |
| | **시기 단위** (월별 / 분기별 / 연간) |
| 버튼 | **조회** · **초기화** |
| 안내 문구 | "선택한 한 사업장 데이터만 테이블에 반영됩니다. 전체 합계는 하단 [전체 사업장 합계]에서 확인하세요." |

### 3-3. 산정 결과 탭 — 중앙 (데이터 입력·결과)

| 구분 | 화면에 보이는 이름 |
|------|-------------------|
| Scope 탭 | **Scope 1** · **Scope 2** · **Scope 3** |
| Scope 1 서브탭 | **고정 연소** · **이동 연소** |
| Scope 2 서브탭 | **전력** · **열/스팀/온수** |
| 뷰 전환 | **RAW 데이터 관리** · **탄소 배출량 관리** |
| 버튼(Scope 1) | **연료 추가** · **항목 추가** (이동연소) · **결과 저장** · **전체 삭제** |
| 버튼(Scope 2) | **전력 행 추가** · **표준열(증기/온수) 추가** · **KDHC(지역난방) 추가** · **결과 저장** |
| 입력 필드 | 월, 사업장, 에너지원, 사용량, 단위, **데이터 품질**(실측/추정 등), 추정 방법, 가정 사항 |
| 블록 제목 | **전체 사업장 합계** |
| 블록 제목 | **배출량 산정 결과** (최종 tCO₂e, 데이터 신뢰도, Scope 1·2·3) |
| 블록 제목 | **본 산정에 적용된 배출계수 및 산식** (또는 적용 배출계수·산식) |

### 3-4. 산정 결과 탭 — 오른쪽 패널

| 구분 | 화면에 보이는 이름 |
|------|-------------------|
| 카드 제목 | **Total Carbon Inventory** (총 탄소 인벤토리) |
| 카드 제목 | **저장된 히스토리** |
| 안내 | "저장된 히스토리가 없습니다. 각 Scope에서 결과 저장을 눌러 기록을 남겨보세요." |

### 3-5. IFRS 감사대응 탭

| 구분 | 화면에 보이는 이름 |
|------|-------------------|
| 탭 구성 | 서브 탭 없음. **요건 반영 확인**만 단일 화면으로 표시 |
| 안내(선택) | "검증의견서 부속 서류를 다운로드하려면 **리포트 생성** 탭을 이용하세요." · **리포트 생성 탭으로 이동** 버튼 |
| 요건 반영 확인 | 테이블: **요건 분류** · **요건 항목** · **상태**(충족/일부/미충족/해당 없음), 행 클릭 시 **상세 요약** |

### 3-6. 리포트 생성 탭

| 구분 | 화면에 보이는 이름 |
|------|-------------------|
| 완료 메시지 | **Evidence Report 생성 완료** (모든 산정 근거와 메타데이터가 포함된 디지털 증적 자료가 준비되었습니다) |
| 카드/버튼 | **PDF Report** · **Excel Details** (산정설정·Scope1·2·3·영수증 목록) |
| 검증의견서 부속 서류 | 안내: "아래 서류는 GHG Protocol·KMR 검증의견서 제출 시 참고용으로 활용할 수 있습니다. 검증기관별 추가 요청이 있을 수 있으니 확인 후 제출하세요." |
| | 서류 카드 6종: **검증 범위 요약**, **배출량 집계표**, **데이터 품질 통계**, **적용 배출계수·산식 요약**, **활동자료 요약**, **증빙 목록** ("PDF·Excel 일괄 다운로드 시 아래 항목이 포함됩니다") |
| | **PDF 다운로드** · **Excel 다운로드** · **검증의견서 부속 서류 일괄 다운로드** |

### 3-7. 다이얼로그(팝업)

| 구분 | 화면에 보이는 이름 |
|------|-------------------|
| 히스토리 창 | **저장된 히스토리** (제목), 각 항목 날짜·총 tCO₂e |
| 필터 창 | 연도·월, **사업장 관리**(추가/삭제/저장/되돌리기), 사업장·에너지원 체크, **적용** · **초기화** |
| EMS 불러오기 창 | **EMS 데이터 가져오기** 버튼, 필터 조건 요약 |
| 엑셀 업로드 창 | 파일 선택/드래그, **확인 및 적용** · **취소** |

## 4. 세부 시트 작성 예시 (행 1행)

아래는 **한 행**만 위 6열 구조에 맞춰 채운 예시입니다. 나머지 단계도 같은 원칙으로 "사용자 행동·화면에 보이는 이름"만 사용해 채웁니다.

| 단계 (Step) | 사용자 행동 (User Action) | 화면 상태 (Screen State) | 감정/생각 (Emotion/Thought) | 문제점 (Pain Point) | 기회 요소 (Opportunities / Solutions) |
|-------------|---------------------------|---------------------------|-----------------------------|---------------------|----------------------------------------|
| 1. 진입 | 네비에서 "GHG 산정" 또는 해당 메뉴 클릭 | 페이지 제목 "온실가스 배출량 산정 (GHG Protocol)", 상단 탭 **산정 결과**·**IFRS 감사대응**·**리포트 생성**, 툴바에 히스토리·필터·EMS 불러오기·엑셀 업로드·전체 결과 저장·CSV 내보내기·계수 기준 연도. **산정 결과** 탭 활성 시 좌측 사업장·에너지원·연도·시기 단위 선택 영역와 **조회**·**초기화** 버튼, 가운데 Scope 1/2/3 탭과 데이터 영역, 오른쪽 Total Carbon Inventory·저장된 히스토리 | "온실가스 배출량을 여기서 산정하는구나" | 첫 진입 시 데이터가 비어 있으면 그리드·결과가 비어 목적이 불명확할 수 있음 | 첫 진입 시 "사업장·연도 선택 후 **조회**를 눌러보세요" 안내 또는 툴팁 |

## 5. 단계(Step) 나누기 가이드 — 변경된 플로우 기준

세부 시트에서 **단계**를 나눌 때 참고할 순서입니다. 각 단계마다 위 6열을 채웁니다.

| 순서 | 단계 개요 | 사용자 관점에서 쓸 이름 예시 |
|------|-----------|------------------------------|
| 1 | 페이지 진입 | 진입 |
| 2 | 좌측에서 사업장·에너지원·연도 선택 | 조건 선택 (사업장·연도·에너지원) |
| 3 | **조회** 버튼 클릭 | 조회 클릭 |
| 4 | Scope 1 / 2 / 3 탭 선택, 고정연소/이동연소 또는 전력/열 선택 | Scope·입력 유형 선택 |
| 5a | 연료 추가·전력 행 추가 등으로 수동 입력 | 활동자료 수동 입력 |
| 5b | **EMS 불러오기** → **EMS 데이터 가져오기** | EMS로 데이터 불러오기 |
| 5c | **엑셀 업로드** → 파일 선택 → **확인 및 적용** | 엑셀으로 데이터 불러오기 |
| 6 | 스크롤하여 배출량 산정 결과·전체 사업장 합계·적용 배출계수·산식 확인 | 배출량 산정 결과 확인 |
| 7 | Scope 1 또는 2 화면에서 **결과 저장** 클릭 | Scope별 결과 저장 |
| 8 | 상단 탭 **IFRS 감사대응** 클릭 → **요건 반영 확인** 테이블 확인 (서브 탭 없음). 필요 시 **리포트 생성 탭으로 이동** 버튼 클릭 | IFRS 감사대응(요건 반영 확인) |
| 9 | 상단 탭 **리포트 생성** 클릭 → **PDF Report**·**Excel Details** 또는 검증의견서 부속 서류 **PDF 다운로드**·**Excel 다운로드**·**검증의견서 부속 서류 일괄 다운로드** 클릭 | Evidence Report 및 검증의견서 부속 서류 다운로드 |
| 10 | 툴바 **전체 결과 저장** 클릭 | 전체 결과 저장 |
| 11 | 툴바 **필터** 클릭 → 사업장 관리·연도·월 등 변경 → **적용** | 필터 창에서 조건 변경 |
| 12 | **계수 기준 연도** 드롭다운에서 2024/2023 선택 | 계수 기준 연도 변경 |
| 13 | **히스토리 (N)** 클릭하여 저장된 히스토리 목록 확인 | 저장된 히스토리 확인 |
| 14 | **CSV 내보내기** 클릭 | CSV 내보내기 |
| 15 | 산정·증적·IFRS 확인 후 다른 메뉴로 이동 | 완료 및 다음 |

## 6. 요약

- **세부 시트 행 구조**: 단계 | 사용자 행동 | 화면 상태 | 감정/생각 | 문제점 | 기회 요소 (6열 고정).
- **언어**: 사용자가 보는 **버튼·탭·라벨·블록 제목**만 사용 (예: **조회**, **산정 결과**, **요건 반영 확인**, **리포트 생성 탭으로 이동**, **검증의견서 부속 서류 일괄 다운로드**). `GHGFilterSidebar`, `isBoundaryComplete()` 등 시스템/코드 용어는 사용하지 않음.
- **반영된 변경 (IFRS_RESULT_REPORT.md)**: IFRS 감사대응 탭은 **서브 탭 없이** 요건 반영 확인만 표시. 검증의견서·서류 작성(6종 서류 + PDF/Excel/일괄 다운로드)은 **리포트 생성** 탭으로 통합됨.
- **내용**: §3의 화면 요소 목록과 §5의 단계 순서를 기준으로, 각 단계별로 6열을 채워 넣으면 됩니다.

이 문서와 기존 `JOURNEYMAP_GHG.md` §14 세부 시트 표를 함께 참고하여, 변경된 GHG 산정 페이지에 맞춘 저니맵 시트를 완성하면 됩니다.
