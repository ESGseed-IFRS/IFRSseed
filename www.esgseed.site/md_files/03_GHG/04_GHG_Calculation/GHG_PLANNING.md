# GHG 산정 탭 재구현 전략 (GHG_PLANNING)

현재 GHG 산정·공시 탭의 구조와 흐름이 직관적이지 않아, **Scope 1/2/3**과 **공시·보고에서 요구하는 항목**(월별 에너지 사용량, 월별 폐기물 반출량, Scope별 배출량 등)을 명확히 구분하고, 하나의 일관된 페이지에서 온실가스 산정을 할 수 있도록 **재구현하기 위한 전략·기획 문서**입니다.

---

## 1. 목표·원칙

| 항목 | 내용 |
|------|------|
| **목표** | 실무자가 "무엇을 채워야 하는지" 바로 이해하고, Scope별·공시 항목별로 데이터를 입력·확인할 수 있는 **단일·명확한 GHG 산정 페이지** 구성 |
| **구분 축** | **(1) Scope 1 / 2 / 3** — 산정 로직·저장·감사 기준 **(2) 공시·보고 요구 항목** — 월별 에너지 사용량, 월별 폐기물 반출량, Scope 1/2/3 배출량(tCO₂e) 등 |
| **원칙** | 계산·저장은 Scope 1/2/3 단위 유지. UI는 **공시 항목**과 **Scope**를 모두 드러내어, 항목 선택 시 해당 입력·결과 뷰로 직행할 수 있게 함. |

---

## 2. 공시·보고 요구 항목 정의

다음 항목을 **공시·보고 요구 항목**으로 명시하고, 각 항목별로 **어떤 Scope/데이터와 연결되는지** 매핑합니다.

| 구분 | 공시·보고 요구 항목 | 단위/형식 | 데이터 소스(Scope/유형) |
|------|---------------------|-----------|--------------------------|
| 에너지 | **월별 에너지 사용량** | 월별 kWh/GJ/Nm³ 등 | Scope 1 고정·이동 연소 + Scope 2 전력·열 (활동자료) |
| 에너지 | 연료별 에너지 사용량 | 연료별·월별 | Scope 1/2 입력 그리드(에너지원·월별 사용량) |
| 온실가스 | **Scope 1 배출량** (tCO₂e) | 월별/연도별 tCO₂e | Scope 1 산정 결과 |
| 온실가스 | **Scope 2 배출량** (tCO₂e) | 월별/연도별 tCO₂e (위치/시장 기반) | Scope 2 산정 결과 |
| 온실가스 | **Scope 3 배출량** (tCO₂e) | 카테고리별 tCO₂e | Scope 3 산정 결과 |
| 폐기물 | **월별 폐기물 반출량** | 톤/㎥ 등, 유형별 | 환경 데이터(폐기물) — Scope 3 또는 별도 환경 지표 |
| 폐기물 | 재활용률, 매립·소각 비율 | % | 동일 데이터 기반 집계 |
| 용수 | 월별 용수 사용량(취수·방류) | ㎥ 등 | 환경 데이터(용수/폐수) — 별도 섹션 |
| 기타 | 재생에너지 사용량·비율, REC/PPA | kWh, % | Scope 2 시장 기반·재생에너지 실적 |

- **우선 구현**: 월별 에너지 사용량, Scope 1/2/3 배출량. **추가 검토**: 월별 폐기물 반출량, 용수, 재생에너지(필요 시 별도 블록 또는 공시 항목 메뉴 확장).

## 3. 페이지 구조 제안 (재구현 시 적용)

### 3-1. 전체 레이아웃

```
┌─────────────────────────────────────────────────────────────────────────────┐
│  [GHG 산정·공시]  [IFRS 감사대응]  [리포트 생성]   (상단 탭 — 유지)            │
├──────────────┬──────────────────────────────────────────────────────────────┤
│              │  공시 프레임워크: [IFRS S2 ▼]   입력 완료율: 33%               │
│  공시·보고    │  ─────────────────────────────────────────────────────────────  │
│  항목        │                                                              │
│  (좌측       │  [선택된 공시 항목 또는 Scope에 해당하는 메인 영역]              │
│   목록)      │  - 조건 선택(사업장·연도·에너지원 등)                            │
│              │  - 활동자료 입력(월별 RAW 데이터 그리드 등)                      │
│  · 월별      │  - 결과 미리보기(해당 항목/Scope 배출량·사용량)                   │
│    에너지    │                                                              │
│  · Scope 1   │  ─────────────────────────────────────────────────────────────  │
│  · Scope 2   │  [배출량 산정 결과 요약]  Total / Scope1 / 2 / 3               │
│  · Scope 3   │  [적용 배출계수·산식]                                          │
│  · 월별      │                                                              │
│    폐기물    │                                                              │
│  (선택)      │                                                              │
├──────────────┼──────────────────────────────────────────────────────────────┤
│  필터        │  (필터는 좌측 사이드바에 공시 항목과 통합 — 기본 제안)   │
│  (사업장·    │                                                              │
│   연도·      │                                                              │
│   에너지원)  │                                                              │
└──────────────┴──────────────────────────────────────────────────────────────┘
```

- **좌측**: **(1) 공시·보고 항목** 목록(월별 에너지 사용량, Scope 1/2/3 배출량, 월별 폐기물 반출량 등) + **(2) 필터**(사업장, 연도, 에너지원)를 **한 사이드바**에 통합. **필터 위치**: 기본 제안 = 좌측 사이드바 통합. (대안: 메인 상단 고정.)
- **메인**: 선택한 **공시 항목**(또는 Scope)에 따라 **해당하는 입력 폼·그리드·결과 요약**만 노출. 한 화면에 “조건 선택 → 활동자료 입력 → 결과 확인”이 한 흐름으로 보이도록 구성.

### 3-2. 공시 항목별 진입·표시

| 공시 항목 | 클릭 시 동작 | 메인 영역 표시 |
|-----------|--------------|----------------|
| 월별 에너지 사용량 | Scope 1+2 활동자료(사용량) 뷰로 전환 | 사업장·연도·에너지원 필터 + **에너지원/월별 RAW 데이터** 그리드 + 사용량 합계 |
| Scope 1 배출량 | Scope 1 입력·결과 뷰로 전환 | 고정연소/이동연소 선택 + 해당 월별 그리드 + Scope 1 배출량(tCO₂e) |
| Scope 2 배출량 | Scope 2 입력·결과 뷰로 전환 | 전력/열 선택 + 월별 그리드 + Scope 2 배출량(위치/시장 기반) |
| Scope 3 배출량 | Scope 3 입력·결과 뷰로 전환 | 카테고리별 입력 + Scope 3 배출량 |
| 월별 폐기물 반출량 | (구현 시) 폐기물 데이터 뷰로 전환 | 폐기물 유형·월별 반출량 입력/표시 |

- 각 공시 항목 행에 **상태(입력됨/미입력)** 와 **이동** 버튼을 두어, 클릭 시 해당 입력 화면으로 이동하는 방식 유지·강화.

### 3-3. Scope와 공시 항목의 관계

- **저장·계산**: 기존처럼 **Scope 1/2/3** 단위(고정연소, 이동연소, 전력, 열, Scope3 카테고리)로 유지.
- **UI 진입**: **공시 항목**을 1차로 두고, 항목 선택 시 필요한 **Scope**가 자동으로 선택되거나, **Scope 1 / 2 / 3** 탭을 두고 각 탭 상단에 “이 데이터가 채우는 공시 항목: 월별 에너지 사용량, Scope N 배출량” 등을 배지로 표시.
- **매핑**: 기존 `disclosure.ts`·`disclosureReportMapping.ts`의 **공시 항목 마스터·매핑 테이블**을 그대로 활용. (월별 에너지 → Scope 1+2 사용량, Scope N 배출량 → Scope N 산정 결과.)

## 4. 워크플로 단순화

| 기존 문제 | 개선 방향 |
|-----------|-----------|
| 단계(1→2→3)와 서브탭(고정연소/이동연소/RAW/탄소배출량)이 혼재 | **한 공시 항목(또는 한 Scope)** 안에서 “필터 → 입력 그리드 → 결과”만 보이게 하고, 단계는 **최대 2~3개**로 통일(예: 조건 선택 | 활동자료 입력 | 결과 확인). |
| 필터가 팝업/접이식 등으로 위치가 불명확 | 필터는 **좌측 사이드바 통합**(기본 제안)으로 두고, 사업장·연도·에너지원을 항상 같은 위치에서 변경. (대안: 메인 상단 고정.) |
| “공시 항목”과 “Scope”가 동시에 강조되어 정신이 분산 | **1차 진입**: 공시 항목 목록(월별 에너지, Scope 1/2/3 배출량, 월별 폐기물 등). 선택한 항목에 따라 **메인에 하나의 입력·결과 흐름**만 표시. |

## 5. 공시 프레임워크·필수 항목

- **공시 프레임워크** 선택(IFRS S2, GRI, CDP, K-ETS, K-ESG, ESRS)은 **메인 상단**에 유지.
- 선택한 프레임워크에 따라 **필수 요구 항목**을 공시 항목 목록에서 **필수/선택** 배지로 표시.
- **선택 기준 대비 입력 완료율(%)** 은 Scope 1/2/3 입력 여부 기반으로 유지.

## 6. 빈 상태·로딩·에러

| 항목 | 적용 |
|------|------|
| 빈 상태 | 월별 RAW 데이터 그리드 등 데이터가 없을 때 **동일한 톤의 문구** 사용. (예: “데이터가 없습니다. 연료 추가 또는 EMS/엑셀 업로드를 이용하세요.”) — `emptyState.ts` 상수 활용. |
| 로딩 | EMS·데이터 조회 시 **스켈레톤** 또는 로딩 스피너를 **한 패턴**으로 통일. (`Skeleton` 컴포넌트·`LOADING_MESSAGE` 등) |
| 에러 | 검증 실패·API 오류 시 토스트·인라인 메시지를 플랫폼 공통 패턴으로 표시. |

## 7. 데이터·리포트 연동

- **공시 항목 마스터·매핑**: `constants/disclosureReportMapping.ts`의 **DISCLOSURE_ITEM_MASTER** 및 리포트 시트 매핑 유지.
- **Excel 리포트**: 공시 항목별 시트(공시_월별에너지사용량, 공시_Scope1/2/3_배출량 등)는 기존 `evidenceReport.ts` 로직 유지.
- **월별 폐기물 반출량** 등 새 항목 추가 시: 마스터에 항목 추가 → 매핑(데이터 소스) 정의 → 해당 입력/결과 뷰 추가.

## 8. 재구현 적용 순서 제안

| 순서 | 내용 |
|------|------|
| 1 | **레이아웃 확정**: 좌측 “공시·보고 항목” 목록 + **필터(좌측 사이드바 통합, 기본 제안)** + 메인(입력·결과) 구조로 단일 페이지 재구성. |
| 2 | **공시 항목 목록 UI**: 월별 에너지 사용량, Scope 1/2/3 배출량, (선택) 월별 폐기물 반출량. 항목 클릭 시 메인 영역이 해당 Scope/데이터 뷰로 전환되도록 연결. |
| 3 | **메인 영역 단순화**: 선택된 항목(또는 Scope)에 따라 **하나의 입력 흐름**만 표시(조건 선택 → 활동자료 입력 → 결과 확인). 서브탭·단계 수 최소화. |
| 4 | **필터 일원화**: 사업장·연도·에너지원 필터를 **좌측 사이드바**(공시 항목과 통합)에 고정하고, 모든 그리드에 동일 필터 적용. |
| 5 | **배출량 산정 결과·적용 계수**: 기존처럼 Total / Scope 1/2/3 요약과 적용 배출계수·산식 블록을 메인 하단 또는 측면에 고정 표시. |
| 6 | **빈 상태·로딩·스켈레톤** 통일 및 **접근성**(포커스 링, ARIA) 점검. |

## 11. 요약

- **Scope 1/2/3**으로 산정 로직을 구분하되, **공시·보고 요구 항목**(월별 에너지 사용량, 월별 폐기물 반출량, Scope 1/2/3 배출량 등)으로 **진입 경로와 화면 구성을 명확히** 한다.
- **한 페이지**에서 “공시 항목 선택 → 해당 입력·결과만 보기”가 되도록 레이아웃과 워크플로를 단순화하고, 필터·빈 상태·로딩을 일관되게 적용한다.
- 기존 **공시 항목 마스터·매핑·리포트 시트**는 유지하고, 재구현 시 이 문서와 상기 참조 문서를 기준으로 단계별 적용한다.

### 참고 및 팁 (시스템 재구현 시)
- **공시 출력 커스터마이징**: 프레임워크 선택 시 자동으로 "연간 총합 + 시장 기반 Scope 2 토글 + Scope 3 중대 카테고리 필터" 적용.
- **월별 데이터 활용**: 입력은 월별로 편하게 → 공시용으로는 연간 합계 + 그래프(추이)로 변환 출력.
- **업데이트 필요**: 배출계수(DB)는 환경부/K-ETS/IPCC 최신 버전 연동. Scope 3 카테고리 15개 중대성 판단 로직 추가.
- **출처**: 금융위원회·KSSB(2025~2026 공시 로드맵), ISSB IFRS S2 교육자료, CDP 2025 데이터 공개, K-ETS 제4차 기본계획, POSCO/LG화학 2024 지속가능경영보고서.
