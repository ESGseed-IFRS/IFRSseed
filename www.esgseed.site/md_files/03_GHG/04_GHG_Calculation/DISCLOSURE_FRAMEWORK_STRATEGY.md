# 공시 프레임워크 연동 전략 (DISCLOSURE_FRAMEWORK_STRATEGY)

현재 GHG 산정 페이지의 **공시 프레임워크** 선택(IFRS S2, GRI, CDP, K-ETS, K-ESG, ESRS)은 **입력 완료율 계산과 필수/선택 배지**에만 반영되고, **메인 입력·결과 영역은 프레임워크에 따라 달라지지 않습니다**.  
이 문서는 **프레임워크 선택 시 실제로 무엇이 바뀌어야 하는지**를 정리하고, 2025~2026년 한국 기업 공시 기준을 반영한 **구체적인 수정 전략**을 담습니다.

---

## 1. 현재 반영 현황 vs 의도

| 구분 | 현재 반영 | 의도(원래 기획) |
|------|-----------|-----------------|
| 공시 프레임워크 선택 | 드롭다운으로 선택 가능 | ✅ |
| 입력 완료율(%) | Scope 1/2/3 입력 여부 기반 0~100% | ✅ |
| 필수/선택 배지 | FRAMEWORK_REQUIRED_ITEMS로 공시 항목에 표시 | ✅ |
| **메인 입력·결과 UI** | 프레임워크 변경해도 **동일** | ❌ 프레임워크별로 **달라져야 함** |
| **Scope 2 위치/시장 기반** | 일부만 지원, 프레임워크와 연동 없음 | IFRS S2 선택 시 강조·토글 |
| **Scope 3 카테고리** | 15개 카테고리 공통 표시 | IFRS S2/CDP 선택 시 중대성·필수 강조 |
| **월별 vs 연간 공시 강조** | 월별 RAW 입력만 강조 | K-ETS: 월별 의무, IFRS S2: 연간 총합 중심 |
| **리포트 출력 형태** | 공통 Excel 시트 | 프레임워크별 연간/월별·시트 구성 차이 |

**결론**: 공시 프레임워크 선택이 “어떤 기준으로 보고할지”를 정하는 핵심인데, **화면·리포트에 거의 반영되지 않음**. 사용자는 “IFRS S2 선택했는데 왜 K-ETS 스타일 화면이지?”라고 느낄 수 있음.

## 2. GHG 공시 프레임워크 비교 (2026년 2월 기준)

한국 기업의 GHG 공시 기준을 주요 프레임워크별로 정리. **프레임워크 선택 시 UI·리포트에 반영해야 할 차이**를 이 테이블 기준으로 정의.

| 공시 프레임워크 | 주요 적용 대상 | GHG 공시 요구사항 핵심 | Scope 1/2/3 필수 | 월별 에너지/폐기물 공시 | 한국 적용 현황 |
|-----------------|----------------|------------------------|------------------|--------------------------|----------------|
| **GHG Protocol** | 거의 모든 기업 (CDP, K-ETS, ESG 기본) | Scope 1/2/3 배출량 tCO₂e, 배출계수·GWP | Scope 1/2: 필수, Scope 3: 선택적→필수화 | 연간 총합 위주 | 현재 시스템 주 기준 |
| **IFRS S2** | **2026년 공시(FY2025 데이터)부터 단계적 의무화** (자산 2조 원 이상 상장사 중심, 2030년 전 상장사 확대 유력) | 절대 총배출량 tCO₂e, Scope 2 **위치+시장 기반**(REC/PPA), Scope 3 **카테고리별 중대성** | Scope 1/2: 필수, Scope 3: 필수(중대 카테고리, 적용 첫해 면제·유예 가능성 있음) | 연간 총합 + 측정 방법·가정 | KSSB 국내 기준 초안·금융위 로드맵 2026년 이후. 시장 기반 Scope 2·Scope 3 세부 공시 미지원 시 기준 미달 |
| **CDP** | 자발적 (국내 800+ 기업, 2025년 기준 약 865개 수준) | Scope 1/2/3 상세 + **검증 여부** | Scope 1/2: 대부분, Scope 3: 점점 증가 | 연간 중심, 추이 그래프 | CDP 질문지·Scope 3 카테고리 강화 필요 |
| **K-ETS** | 대상 기업만 (**772개**, 2026~2030 제4차 계획기간 기준, 의무 764+자발적 8) | **Scope 1 중심** + 에너지 사용량 보고 | Scope 1/2 일부 | **월별 에너지 사용량 보고 의무** | K-ETS 배출계수 적용, 공시용 한정 |
| **K-ESG / GRI** | 자발적 ESG 보고서 (상장사) | Scope 1/2/3 + 에너지·폐기물·재활용률 | Scope 1/2: 대부분, Scope 3: 선택적 | **연간 합계 + 재활용률·추이** (POSCO·LG화학 등) | 월별 에너지/폐기물 K-ESG 스타일 |

### 참고 (시스템 재구현 시)
- **공시 출력**: 프레임워크별 “연간 총합 + 시장 기반 Scope 2 토글 + Scope 3 중대 카테고리 필터” 자동 적용.
- **입력**: 월별 입력 유지 → 공시용은 연간 합계·그래프(추이)로 변환 출력.
- **출처**: 금융위원회·KSSB(국내 기준 초안), 금융위 로드맵 2026년 이후, ISSB IFRS S2, CDP 2025, K-ETS 제4차, POSCO·LG화학 보고서.

## 3. 프레임워크 선택 시 달라져야 할 내용 (구체)

### 3-1. 공시 항목·필수/선택 표시

| 프레임워크 | 월별 에너지 | Scope 1 | Scope 2 | Scope 3 | 월별 폐기물 |
|------------|-------------|---------|---------|---------|-------------|
| GHG Protocol | 선택 | 필수 | 필수 | 선택 | 선택 |
| IFRS S2 | 선택 | 필수 | 필수 | 필수(중대) | 선택 |
| CDP | 선택 | 필수 | 필수 | 선택→필수 | 선택 |
| K-ETS | **필수** | **필수** | 선택 | - | 선택 |
| K-ESG/GRI | 선택 | 필수 | 필수 | 선택 | 선택 |

→ **FRAMEWORK_REQUIRED_ITEMS**를 위 표에 맞게 조정. K-ETS는 `monthly_energy`를 필수로.

### 3-2. 메인 입력·결과 영역

| 프레임워크 | 달라져야 할 UI·동작 |
|------------|---------------------|
| **IFRS S2** | Scope 2: **위치 기반 / 시장 기반** 토글 또는 구분 표시. Scope 3: **중대 카테고리** 강조 또는 필터. 공시용 안내: “연간 총합 + 측정 방법·가정” 중심. |
| **CDP** | Scope 3 카테고리 입력 강화. **검증 여부** 입력/표시. |
| **K-ETS** | Scope 1·월별 에너지 강조. “월별 에너지 사용량 보고 의무” 문구. Scope 2·3는 보조. |
| **K-ESG/GRI** | 에너지·폐기물·재활용률 관련 공시 항목 노출. 연간 합계·추이 시각화 안내. |
| **GHG Protocol** | 현재 스타일 유지 (기준). |

### 3-3. 입력 완료율 계산

- 현재: Scope 1/2/3 “데이터 있음” 여부만으로 0~100% 계산.
- 개선: 선택된 프레임워크의 **필수 항목**만 기준으로 완료율 계산.  
  예: K-ETS 선택 시 → Scope 1 + 월별 에너지 필수이면, 이 두 항목 입력 여부로 완료율 계산.
- **세분화 UX**: 필수 항목 100% 입력 시 100%, 부분 입력 시 비율 표시 (예: 3개 필수 중 2개 입력 → 67%). 사용자가 무엇을 더 입력해야 하는지 직관적으로 파악 가능.

### 3-4. 리포트(Excel/PDF) 출력

| 프레임워크 | 출력 시 강조 |
|------------|--------------|
| IFRS S2 | 연간 총합 시트, Scope 2 위치/시장 구분, Scope 3 카테고리별. |
| K-ETS | 월별 에너지 사용량 시트, Scope 1 상세. |
| K-ESG/GRI | 연간 에너지·폐기물·재활용률 시트. |
| CDP | Scope 1/2/3 상세 + 검증 여부. |

### 3-5. 각 Scope 블록 상단 배지 (IFRS S2, K-ESG 등)

**현황**: Scope 1/2/3 각 섹션 상단에 "월별 에너지 사용량", "연료별 에너지 사용량", "Scope 1 배출량 (tCO₂e)", **"IFRS S2"**, **"K-ESG"** 배지가 표시됨.

**문제점**:
- 상단 공시 프레임워크 드롭다운에서 이미 선택 가능. 같은 IFRS S2·K-ESG를 Scope별로 반복 표시하면 **선택된 프레임워크와 연동되지 않고**, "이 Scope가 어느 기준에 적용되는지"라는 정보성 가치도 낮음.
- 사용자: "IFRS S2·K-ESG가 왜 여기 붙어 있지? 선택한 기준과 무슨 관계지?" → **맥락이 불명확**.

**제안 (택 1)**:

| 옵션 | 내용 | 장단 |
|------|------|------|
| **A. 삭제** | IFRS S2, K-ESG 배지 제거. "월별 에너지 사용량", "연료별 에너지 사용량", "Scope N 배출량 (tCO₂e)"만 유지. | 단순. 공시 기준은 상단 선택으로 충분. |
| **B. 동적 표시** | **선택된 프레임워크**만 1개 표시. 예: IFRS S2 선택 시 → "선택 기준: IFRS S2 (필수)" 배지 1개. 툴팁: "이 Scope 데이터는 IFRS S2 필수 항목을 채웁니다." | 프레임워크 선택과 명확히 연동. |
| **C. 안내 문구로 대체** | 배지 대신, "이 입력은 상단에서 선택한 공시 기준(IFRS S2 등)의 Scope N 필수 항목을 채웁니다." 한 줄 텍스트. | 간결하고 설명적. |

**권장**: **A (삭제)** 또는 **B (동적 표시)**.  
- A: 화면 정리 우선. **→ 적용 완료 (2026-02)**  
- B: 선택 기준과 Scope 연결을 시각적으로 보여주고 싶을 때.

## 4. 수정 전략 (우선순위)

### Phase 1: 즉시 반영 가능 (데이터·상수)

| 순서 | 내용 | 구현 |
|------|------|------|
| 1 | FRAMEWORK_REQUIRED_ITEMS를 프레임워크별로 정확히 정의 (K-ETS: monthly_energy 필수 등) | `constants/disclosure.ts` |
| 2 | 입력 완료율을 “선택 프레임워크의 필수 항목” 기준으로 계산 | `index.tsx` |
| 3 | 프레임워크 선택 시 **안내 문구** 변경: “IFRS S2 기준: 연간 총합 중심”, “K-ETS: 월별 에너지 의무” 등 | 메인 카드 설명 |

### Phase 2: UI 반응 (가이드형)

| 순서 | 내용 | 구현 |
|------|------|------|
| 4 | IFRS S2 선택 시 Scope 2 영역에 “위치 기반 / 시장 기반” 토글 표시. **시장 기반 선택 시 REC/PPA 입력 필드 자동 노출** (체크박스 또는 전용 입력란) | Scope 2 폼/결과 |
| 5 | IFRS S2/CDP 선택 시 Scope 3 “중대 카테고리” 강조 또는 필터 | Scope 3 폼 |
| 6 | K-ETS 선택 시 “월별 에너지 사용량 보고 의무” 배너 또는 강조 | 월별 에너지·Scope 1 영역 |
| 7 | Scope 블록 상단 IFRS S2·K-ESG 배지: **삭제(A)** 또는 **동적 표시(B)** (선택 기준 1개만 표시) | `Step2ActivityData.tsx` |

### Phase 3: 리포트 연동

| 순서 | 내용 | 구현 |
|------|------|------|
| 8 | 프레임워크별 Excel 시트 구성 분기 (IFRS S2: 연간+위치/시장, K-ETS: 월별 에너지 등) | `evidenceReport.ts`, `disclosureReportMapping.ts` |
| 9 | PDF/리포트에도 “공시 기준: IFRS S2” 등 표기 | 리포트 헤더 |

## 5. 요약

1. **현재**: 공시 프레임워크 선택이 완료율·필수 배지에만 쓰이고, **메인 UI·리포트는 거의 동일**.
2. **목표**: 프레임워크별 **필수 항목·강조 영역·리포트 형태**가 달라지도록 연동.
3. **우선**: Phase 1 (상수·완료율·안내 문구) → Phase 2 (Scope 2/3, K-ETS 강조, Scope 배지 처리) → Phase 3 (리포트 분기).

이 전략을 반영하면 “공시 프레임워크를 바꿨을 때 화면·결과가 그에 맞게 변한다”는 인상을 줄 수 있습니다.
