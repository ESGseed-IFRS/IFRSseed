# 필터 통합 전략 — 좌측 사이드바 단일 필터 (FILTER_MERGE_STRATEGY)

현재 GHG 산정·공시 탭에는 **필터가 두 군데** 존재하여 사용자 혼선과 중복 입력이 발생합니다.  
**좌측 사이드바에 하나의 통합 필터**를 두고, **메인 영역의 필터(오른쪽 필터) 디자인**을 기준으로 통일하는 전략을 담은 문서입니다.

---

## 1. 현재 상태

| 위치 | 컴포넌트 | 포함 항목 | 스타일 |
|------|-----------|-----------|--------|
| **좌측 사이드바** | `GHGFilterPanel` | 년도(단일/범위), 월(단일/범위), 사업장 관리(체크박스 목록 + 새 사업장/지점 추가), 되돌리기/저장 | 카드·라이트 배경, 복잡한 년/월 모드 전환 |
| **메인 영역** (데이터 관리 센터 내) | `GHGFilterSidebar` | 사업장(드롭다운 전체/개별), 에너지원(드롭다운), 연도, 시기 단위(월별/분기별/연간), 조회·초기화 버튼 | 다크 슬레이트(`slate-800`), 드롭다운 기반, 깔끔한 레이아웃 |

**문제점**
- 필터 기준(사업장·연도·에너지원 등)이 **두 곳**에 분산
- 어떤 필터가 실제 조회에 적용되는지 직관적이지 않음
- GHG_PLANNING에서 제안한 “필터 좌측 사이드바 통합”과 일치하지 않음

---

## 2. 목표

- **단일 필터**: 좌측 사이드바에 **하나의 통합 필터**만 존재
- **디자인 기준**: 메인 영역의 필터(`GHGFilterSidebar`) 스타일을 기준으로 통일  
  - 어두운 회색 배경(`slate-800`), 필터 아이콘·섹션 라벨, 드롭다운 UI, 조회·초기화 버튼
- **기능 통합**: 좌측 패널의 사업장 관리(목록·추가) + 메인 필터의 사업장·에너지원·연도·시기 단위를 **한 인터페이스**로 병합

---

## 3. 통합 필터 항목 정의

단일 필터에 포함할 항목은 다음과 같습니다.

| 항목 | 타입 | 설명 | 참조 |
|------|------|------|------|
| **사업장** | Select (전체 \| 개별) | 선택한 사업장 데이터만 테이블에 반영. “전체” 선택 시 모든 사업장 | GHGFilterSidebar |
| **에너지원** | Select (전체 \| 개별) | 해당 에너지원만 테이블에 표시 | GHGFilterSidebar |
| **연도** | Select | 산정·조회 기준 연도 (최근 5~6년) | GHGFilterPanel, GHGFilterSidebar |
| **월** | Select (선택) | 특정 월만 조회 시 사용. “전체” 또는 1~12월 | GHGFilterPanel |
| **시기 단위** | Select | 월별 / 분기별 / 연간 | GHGFilterSidebar |
| **사업장 관리** | 링크 또는 버튼 | 새 사업장/지점 추가 → 모달 또는 인라인 폼 | GHGFilterPanel |

- **우선**: 사업장, 에너지원, 연도, 시기 단위, 조회·초기화  
- **추가 검토**: 월(단일 선택), 사업장 관리(추가) — 필요 시 통합 필터 하단에 “사업장 관리” 섹션으로 포함

---

## 4. 디자인 스펙 (오른쪽 필터 스타일 기준)

통합 필터는 **GHGFilterSidebar** 디자인을 따릅니다.

| 요소 | 스타일 |
|------|--------|
| **컨테이너** | `bg-slate-800`, `border-slate-700`, 좌측 사이드바 내 고정 |
| **섹션 헤더** | Filter 아이콘 + "FILTER" 대문자 라벨, `text-slate-400` |
| **항목별 라벨** | `text-[10px] font-bold uppercase tracking-wider text-slate-400` |
| **설명 문구** | `text-xs text-slate-400` (선택 시) |
| **Select 트리거** | `h-9`, `bg-slate-700`, `border-slate-600`, `text-slate-100` |
| **조회 버튼** | `bg-[#669900]` (primary green), `w-full h-10` |
| **초기화 버튼** | `variant="outline"`, `border-slate-600`, `RotateCcw` 아이콘 |

- 레이아웃: 상단부터 **사업장 → 에너지원 → 연도 → 시기 단위 → [조회] → [초기화]** 순서
- 사업장 관리(추가)가 필요한 경우, 하단에 “사업장 관리” 영역 또는 “새 사업장 추가” 링크 배치

---

## 5. 적용 방식 제안

| 단계 | 내용 |
|------|------|
| 1 | **GHGFilterSidebar** 확장: 사업장 관리(목록·추가) 기능을 GHGFilterPanel 로직에서 이전 또는 연동. 필요 시 `onFacilitiesUpdate` prop 유지 |
| 2 | **index.tsx** 좌측 사이드바: `GHGFilterPanel` 제거 후, 확장된 **GHGFilterSidebar**를 단일 필터로 사용. `GHGFilterPanel`과 동일한 props(사업장 목록, 업데이트 콜백 등) 전달 |
| 3 | **GHGDataManagementCenter**: 내부 **GHGFilterSidebar** 제거. 메인 영역은 Scope 1/2/3 탭 + 활동자료 입력 그리드만 표시 (필터는 좌측에서만 조작) |
| 4 | **FilterState** 단일 소스: `filtersByScope[activeScope]`가 좌측 통합 필터 결과만 반영하도록 정리. EMS/엑셀 업로드 시에도 동일 필터 상태 사용 |

---

## 6. 레이아웃 개략

```
┌────────────────────┬─────────────────────────────────────────────────────────┐
│ 공시·보고 항목      │  공시 프레임워크  |  입력 완료율                          │
│  · 월별 에너지      │  ─────────────────────────────────────────────────────  │
│  · Scope 1         │  [선택 공시 항목] — 활동자료 입력 및 결과                  │
│  · Scope 2         │  (필터 없음 — 조건 선택은 좌측에서)                         │
│  · Scope 3         │  [Scope 1/2/3 탭] [활동자료 그리드]                        │
├────────────────────┤                                                         │
│ FILTER (통합)      │                                                         │
│  사업장    [전체▼] │                                                         │
│  에너지원  [전체▼] │                                                         │
│  연도      [2026년▼]│                                                         │
│  시기 단위 [월별▼] │                                                         │
│  [조회]            │                                                         │
│  [초기화]          │                                                         │
│  (사업장 관리)     │                                                         │
└────────────────────┴─────────────────────────────────────────────────────────┘
```

- 좌측: **공시·보고 항목** + **통합 필터** (오른쪽 필터 스타일)
- 메인: 공시 항목 선택에 따른 입력·결과 흐름만 표시, 필터 UI 없음

---

## 8. 요약

- **좌측 사이드바**에 **공시·보고 항목**과 **통합 필터**만 두고, 메인 영역의 필터는 제거한다.
- 통합 필터의 **디자인**은 **GHGFilterSidebar**(오른쪽/메인 필터)를 기준으로 한다.
- **사업장, 에너지원, 연도, 시기 단위, 조회·초기화**를 한 인터페이스로 모으고, **사업장 관리(추가)**는 필요 시 동일 스타일로 통합한다.
