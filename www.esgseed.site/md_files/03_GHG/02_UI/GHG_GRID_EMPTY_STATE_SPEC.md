# GHG 산정 입력 탭 — 그리드 빈 상태(Empty State) 스펙

> **핵심 요구사항**: 더미데이터 또는 실데이터가 없어도 그리드 틀(컬럼 헤더 + 빈 행)은 항상 표시된다.  
> 데이터 유무와 관계없이 사용자가 "어떤 데이터를 입력해야 하는지" 컬럼을 보고 파악할 수 있어야 한다.

---

## 현재 문제

```
현재 동작:
데이터 없음 → 그리드 전체 숨김 → "입력된 원시 데이터가 없습니다" 텍스트만 표시

원하는 동작:
데이터 없음 → 그리드 컬럼 헤더는 항상 표시 → 빈 행 영역 + 안내 문구 병행
```

---

## 요구 동작 상세

### 상태 1 — 데이터 있을 때 (정상)

```
┌────────┬────┬──────────┬──────────┬───────┬───────┬────────┬────┬────┬────┐
│ 사업장 │ 월 │ 에너지종류│  공급원  │ 사용량 │ 단위  │재생E%  │품질│출처│이력│
├────────┼────┼──────────┼──────────┼───────┼───────┼────────┼────┼────┼────┤
│서울본사│ 1월│    전력   │ 한국전력 │125,000│  kWh  │ 12.5%  │ M1 │EMS │ ✅ │
│서울본사│ 1월│    열     │ 지역난방 │  3,200│  Gcal │   0%   │ M1 │EMS │ ✅ │
│수원공장│ 1월│    전력   │ 한국전력 │280,000│  kWh  │  8.2%  │ M1 │EMS │ ⚠️ │
└────────┴────┴──────────┴──────────┴───────┴───────┴────────┴────┴────┴────┘
```

### 상태 2 — 데이터 없을 때 (빈 상태) ← 현재 누락된 부분

```
┌────────┬────┬──────────┬──────────┬───────┬───────┬────────┬────┬────┬────┐
│ 사업장 │ 월 │ 에너지종류│  공급원  │ 사용량 │ 단위  │재생E%  │품질│출처│이력│
├────────┴────┴──────────┴──────────┴───────┴───────┴────────┴────┴────┴────┤
│                                                                              │
│          EMS 불러오기 또는 엑셀 업로드로 데이터를 가져오세요.                │
│          직접 입력하려면 [+ 행 추가] 버튼을 사용하세요.                      │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘
```

**핵심**: 컬럼 헤더는 항상 렌더링. 데이터 행이 0건일 때 빈 영역 안에 안내 문구 표시.

---

## 탭별 빈 상태 컬럼 헤더 정의

데이터 없을 때도 아래 헤더가 항상 보여야 한다.

### 탭 1 — 전력·열·스팀
```
사업장 | 월 | 에너지 종류 | 공급원 | 사용량 | 단위 | 재생에너지% | 품질 | 출처 | 수정이력
```

### 탭 2 — 연료·차량
```
사업장 | 월 | 연소 구분 | 연료 종류 | 소비량 | 단위 | 구매량 | 품질 | 출처 | 수정이력
```

### 탭 3 — 냉매
```
사업장 | 설비 ID | 설비 유형 | 냉매 종류 | 충전량(kg) | 누출량(kg) | GWP | 점검일 | 품질 | 출처 | 수정이력
```

### 탭 4 — 폐기물
```
사업장 | 월 | 폐기물 종류 | 폐기물명 | 발생량(톤) | 처리 방법 | 소각량 | 재활용량 | Scope | 품질 | 출처 | 수정이력
```

### 탭 5 — 물류·출장·통근
```
구분 | 월 | 이동 수단 | 출발 | 도착 | 거리(km) | 중량/인원 | Scope Cat. | 품질 | 출처 | 수정이력
```

### 탭 6 — 원료·제품
```
구분 | 공급업체/제품명 | 배출량(tCO₂e) | 보고 방식 | Scope Cat. | 품질 | 출처 | 수정이력
```

## 구현 가이드

### 조건부 렌더링 제거

```javascript
// ❌ 현재 (잘못된 방식) — 데이터 없으면 그리드 자체를 숨김
{data.length > 0 ? <DataGrid rows={data} /> : <EmptyMessage />}

// ✅ 원하는 방식 — 그리드는 항상 렌더링, 빈 상태는 그리드 내부에서 처리
<DataGrid
  columns={columns}
  rows={data}          // 빈 배열([])이어도 그리드는 렌더링
  emptyRowsRenderer={<EmptyState />}   // 행이 0건일 때 그리드 내부에 표시
/>
```

### 빈 상태 컴포넌트

```javascript
const EmptyState = ({ tabName, systemName }) => (
  <div className="empty-state">
    <p className="empty-state-message">
      입력된 데이터가 없습니다.
    </p>
    <p className="empty-state-hint">
      [{systemName} 불러오기] 또는 [엑셀 업로드]로 데이터를 가져오거나,
      [+ 행 추가]로 직접 입력하세요.
    </p>
  </div>
)
```

### 상단 적재 현황 — 빈 상태에서도 표시

```
데이터 없을 때:
● 0건   출처별: 데이터 없음   수정이력: —

데이터 있을 때:
● 240건   출처별: EMS 240건 / 수동 0건   수정이력 있음: 1건 ⚠️
```

## 요약

| 항목 | 현재 | 변경 후 |
|------|------|---------|
| 데이터 없을 때 그리드 | 숨김 | 항상 표시 |
| 컬럼 헤더 | 데이터 있을 때만 표시 | 항상 표시 |
| 빈 상태 안내 | 그리드 대신 텍스트만 | 그리드 내부에 안내 문구 |
| 적재 현황 카운터 | 데이터 없으면 미표시 | 0건으로 항상 표시 |
