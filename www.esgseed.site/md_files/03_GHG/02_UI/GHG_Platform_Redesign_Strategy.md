# 온실가스 배출량 산정 플랫폼 재설계 전략

## 개요
GHG Protocol 기반 플랫폼의 현재 UI 문제를 진단하고, 규제 준수·감사 대응 중심으로 재설계합니다.  
핵심은 Raw Data 변환 과정에 **‘산정 판단’과 ‘근거’ 레이어**를 추가하여, 단순 계산기가 아닌 **감사·검증·IFRS S2·CDP 대응이 가능한 신뢰성 있는 플랫폼**으로 업그레이드하는 것입니다.

## ① 현재 화면 구조 진단 (왜 불안한가)
지금 화면은 **전형적인 ‘계산기 UI’**입니다.  

**현재 기능**  
- 활동자료 입력 (연료, 사용량 등)  
- 자동 계산  
- Scope별 합계 출력  
- CSV 다운로드  

**치명적인 문제점**  
❌ GHG Protocol 준수 여부·방법이 전혀 드러나지 않음  
❌ 사용된 배출계수(EF)의 출처·버전 설명 불가  
❌ 입력 데이터가 실측인지 추정인지 구분 없음  
❌ 조직경계(Operational Control / Equity Share) 및 운영경계 설정 기록 미표시  
❌ 검증기관·감사 질문에 즉시 답변할 수 있는 근거(Evidence) 부족  

👉 결과: 숫자는 나오지만, **기업 지속가능성 보고서·외부 검증·IFRS S2 대응에는 바로 사용할 수 없는 상태**

## ② 핵심 문제 요약 (한 문장)
현재 UI는 **Raw Data → 계산 → 결과**만 지원하며, **Raw Data → ‘산정 판단’ → 계산 → 근거** 레이어가 없어 블랙박스 상태입니다.

## ③ 반드시 추가돼야 할 개념 레이어 (UI에 반드시 드러나야 함)

### 1) Boundary & Policy Layer (GHG Protocol 준수 증명)
**문제**  
- Scope는 표시되지만 Boundary 선택·기록이 없음  

**추가 항목**  
- 조직경계: 운영통제법 / 지분비율법  
- 운영경계 설명: Scope 1 포함 기준 등  
- 보고연도 및 기준 가이드라인 명시  

**UI 제안**  
[산정 설정] 탭 (최초 1회 설정, 이후 템플릿 저장 가능)  
조직경계: 운영통제법
운영경계: Scope 1 (직접 연료 연소 포함), Scope 2 (위치 기반)
기준 가이드라인: GHG Protocol Corporate Standard (2004 + Scope 3 Guidance 보완)
보고연도: 2025

👉 이 설정 없이는 GHG Protocol 준수 주장 자체가 불가능

### 2) Methodology Layer (산정 논리의 핵심)
**문제**  
- 연료 선택 → 자동 계산이지만 “왜 이 방식인지” 기록 없음  

**추가 항목**  
각 산정 결과마다 Methodology ID 부여  

Methodology ID: S1-FUEL-KR-2025

Guideline: GHG Protocol Scope 1
Method: Activity Data × Emission Factor
EF Source: 환경부 국가 온실가스 배출계수 (2025 버전)
GWP: IPCC AR6
Version: 1.2 (2025-01 업데이트)


**UI 제안**  
- 활동자료 입력 폼 하단에 **[산정 방식 상세 보기]** 버튼  
- 클릭 시 팝업/사이드 패널: 가이드라인, 산식, EF 출처, 버전, 적용 날짜 표시  

👉 이 블록 하나가 감사 대응의 80%를 해결

### 3) Data Quality & Assumption Layer
**문제**  
- 입력값의 신뢰도(실측/추정) 구분 없음  
- 결측치·추정 처리 로직 미존재  

**추가 항목** (입력 시 필수)  
- 데이터 유형: ⭕ 실측 / ⭕ 추정 / ⭕ 공급자 제공 / ⭕ 기타  
- 추정 방법: 평균값 / 이전연도 보정 / 산업평균 등  
- 가정 사항 (자유 텍스트)  

**UI 제안**  
데이터 품질: 추정
추정 방법: 월 평균 사용량 적용 (1~2월 실측 기반)
가정 사항: 계량기 점검 지연으로 3월 데이터 추정


👉 ISO 14064-1, IFRS S2 검증 시 필수 요구사항

## ④ 재설계된 화면 흐름
**BEFORE (현재)**  
연료 선택 → 사용량 입력 → 결과  

**AFTER (권장)**  
[1] 산정 설정
└ 조직경계 / 운영경계 / 보고연도 / 기준 가이드라인
[2] 활동자료 입력
└ 연료 / 사용량 / 데이터 유형 / 추정 방법 / 가정 사항
[3] 산정 방식 자동 연결
└ Methodology ID / 산식 / EF 출처 / GWP
[4] 계산 결과
└ 수치 + "왜 이렇게 계산됐는지" 설명 링크
[5] Evidence 자동 저장 & 보고서 생성
└ 모든 메타데이터 포함 (JSON / PDF / Excel)


## ⑤ 경쟁력 분석
| 구분              | 지금 화면     | 개선 후       |
|-------------------|---------------|---------------|
| 기본 계산         | ⭕            | ⭕            |
| GHG Protocol 설명 | ❌            | ⭕            |
| 감사·검증 대응    | ❌            | ⭕            |
| IFRS S2 문장 생성 | ❌            | ⭕ (AI 지원)  |
| CDP / TCFD 연계   | 어려움        | 쉬움          |
| AI 문단·보고서 생성 | 한계         | 최적화        |

**핵심 경쟁력**  
숫자 계산 플랫폼 → **규제 대응 & 감사 준비 엔진**으로 전환  
2026년 ESG 공시 의무화(한국 대기업 대상 확대) 시대에 결정적 차별화 요소

## 구현 팁 & 우선순위
- **백엔드**: 모든 메타데이터를 불변 로그 형태로 DB 저장 (예: MongoDB + 타임스탬프 + 해시)
- **프론트엔드**: 위자드 형태 UI + 실시간 유효성 검사 (경계 미설정 시 입력 차단)
- **AI 통합**: LLM 기반 가정 사항 자동 추천 + IFRS S2 문장 초안 생성
- **추가 기능**: EF DB 자동 업데이트 (환경부·IPCC API 연동), 버전 비교 기능
- **테스트**: 실제 감사 시나리오 모의 테스트 + GHG 전문가 리뷰
