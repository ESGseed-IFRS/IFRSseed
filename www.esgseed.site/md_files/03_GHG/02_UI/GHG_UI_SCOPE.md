# GHG 산정 페이지 구조·UI 통합 전략

GHG 산정 페이지를 **(1) Scope 1/2/3 + 공시·보고 요구 항목**으로 재구성하고, **(2) 조화롭고 세련된 UI**로 정리하기 위한 전략을 담은 문서입니다.

---

## Part 1. Scope 1/2/3 + 공시·보고 요구 항목으로 구분하는 전략

### 1-1. 배경·목표

| 현재 | 목표 |
|------|------|
| Scope 1(고정연소/이동연소), Scope 2(전력/열), Scope 3 위주로만 구분 | **Scope는 유지**하되, **공시/보고에서 요구하는 항목**(월별 에너지 사용량, 월별 폐기물 반출량 등)으로 **진입 경로·뷰**를 추가해 실무자가 "무엇을 채워야 하는지"를 바로 찾을 수 있게 함 |
| "Scope 2 전력"처럼 산정 관점 위주 | K-ESG·GRI·IFRS S2·CDP·K-ETS 등 **공시 항목명**과 매핑되어 "이 화면이 어떤 공시 항목에 대응하는지" 명시 |

### 1-2. 공시·보고 요구 항목 후보 (참조: K-ESG, GRI, IFRS S2, CDP, K-ETS)

아래는 **공시/보고서에서 자주 요구되는 항목** 예시입니다. 플랫폼에서 지원할 항목을 선정한 뒤, 각 항목을 **어떤 Scope/데이터와 연결할지** 매핑합니다.

| 구분 | 공시·보고 요구 항목 예시 | 단위/형식 | GHG 산정·데이터와의 매핑 |
|------|---------------------------|-----------|---------------------------|
| **에너지** | 월별 에너지 사용량, 연도별 에너지 사용량 | 월별 kWh/GJ/Nm³ 등 | Scope 1 고정·이동 연소 사용량 + Scope 2 전력·열 사용량 (활동자료) |
| **에너지** | 연료별 에너지 사용량 (도시가스, 경유, 전력 등) | 연료별·월별 | Scope 1/2 입력 그리드(에너지원·월별 사용량) |
| **온실가스** | Scope 1 배출량 (tCO₂e), 월별/연도별 | tCO₂e | Scope 1 산정 결과 |
| **온실가스** | Scope 2 배출량 (위치 기반/시장 기반), 월별/연도별 | tCO₂e | Scope 2 산정 결과 |
| **온실가스** | Scope 3 배출량 (카테고리별) | tCO₂e | Scope 3 산정 결과 |
| **폐기물** | 월별 폐기물 반출량(또는 발생량), 유형별 | 톤/㎥ 등 | 환경 데이터(폐기물) — Scope 3 또는 별도 환경 지표 |
| **폐기물** | 재활용률, 매립·소각 비율 | % | 동일 데이터 기반 집계 |
| **용수** | 월별 용수 사용량(취수량), 방류량 | ㎥ 등 | 환경 데이터(용수/폐수) — 별도 섹션 |
| **기타** | 재생에너지 사용량·비율, REC/PPA 실적 | kWh, % | Scope 2 시장 기반·재생에너지 이행 실적 |

- **월별 에너지 사용량**, **월별 폐기물 반출량** 등은 **공시 항목으로 노출**하고, 화면에서는 해당 항목을 선택하면 **필요한 입력 필드·결과 뷰**로 바로 연결되도록 합니다.

### 1-3. 구조 전략: 이중 축 (Scope 축 + 공시 항목 축)

- **계산·저장은 기존처럼 Scope 1/2/3 단위 유지** (배출계수·산식·감사 추적과 호환).
- **사용자 진입·뷰**는 두 가지 경로를 모두 제공합니다.

| 축 | 용도 | UI에서의 표현 |
|----|------|----------------|
| **Scope 축** | 산정 로직·저장·감사 (Scope 1 고정/이동, Scope 2 전력/열, Scope 3) | **GHG 산정·공시** 탭 내 **Scope 1 / Scope 2 / Scope 3** 탭, 필터·그리드·결과 저장 |
| **공시 항목 축** | "무엇을 채워야 하는지" 기준 네비게이션·리포트 | **공시·보고 항목** 메뉴/카드: "월별 에너지 사용량", "월별 폐기물 반출량", "Scope 1/2/3 배출량" 등 → 클릭 시 해당 데이터 입력/결과 화면으로 이동 |

- **공시 항목 → Scope/데이터 매핑**을 한 번 정의해 두면, "공시 항목별 보기"에서 같은 저장소(Scope 1/2/3 활동자료·결과)를 **항목별로 필터/그룹**해서 보여 주면 됩니다.

### 1-4. UI 구성 제안

#### 옵션 A: Scope 탭 유지 + 공시 항목 레이블·링크

- **현재** Scope 1 / Scope 2 / Scope 3 탭 구조는 유지.
- 각 Scope 블록(또는 서브탭) 상단에 **"이 데이터가 채우는 공시 항목"**을 배지/문구로 표시.
  - 예: Scope 2 전력 블록 → "K-ESG 월별 에너지 사용량(전력)", "IFRS S2 Scope 2 배출량" 등.
- **공시 항목별 요약 페이지** 하나 추가: "이번 연도 공시 항목별 입력·산출 현황" 테이블에서 항목 클릭 시 해당 Scope/입력 화면으로 이동.

#### 옵션 B: 공시 항목을 1차 네비게이션으로 (권장 검토)

- **1차 메뉴**: 공시·보고 요구 항목 — **월별 에너지 사용량** | **월별 폐기물 반출량** | **Scope 1 배출량** | **Scope 2 배출량** | **Scope 3 배출량** | (필요 시 용수·재생에너지 등).
- **2차**: 항목 선택 시 해당 항목에 필요한 **입력 화면**(에너지원·월별 그리드 등)과 **결과 뷰**(tCO₂e, 사용량 요약) 표시.
- 백엔드/스토어는 기존 Scope·에너지원·월별 구조 유지하고, **공시 항목 ID ↔ Scope/데이터 소스 매핑**으로 뷰만 항목별로 꾸미는 방식.

#### 옵션 C: 하이브리드 (탭 + 공시 항목 사이드/상단)

- **좌측 또는 상단**: "공시·보고 항목" 목록 (월별 에너지 사용량, 월별 폐기물 반출량, Scope 1/2/3 배출량 …).
- **메인**: 선택한 항목에 따라 **Scope 1/2/3 중 해당하는 탭·그리드**가 자동 선택되고, 필요 시 "이 항목에 사용되는 데이터"만 강조 표시 (필터 고정 또는 하이라이트).

### 1-5. 데이터·매핑 정리

| 작업 | 내용 |
|------|------|
| **공시 항목 마스터** | 공시 항목 ID, 한글명(예: 월별 에너지 사용량, 월별 폐기물 반출량), 적용 공시 프레임(K-ESG, GRI, IFRS S2 등), 단위 정의. |
| **매핑 테이블** | 공시 항목 ID → (Scope, 에너지원/카테고리, 기간 단위). 예: "월별 에너지 사용량" → Scope 1+2 활동자료(사용량), 월별. "월별 폐기물 반출량" → 폐기물 데이터 소스, 월별. |
| **저장 구조** | 기존처럼 Scope 1/2/3·에너지원·월별로 저장. 리포트/엑셀 생성 시 **공시 항목별 시트 또는 컬럼**을 매핑 테이블로 생성. |

### 1-6. 적용 순서 제안

| 단계 | 내용 |
|------|------|
| **1단계** | 공시·보고 요구 항목 목록 확정 (월별 에너지 사용량, 월별 폐기물 반출량, Scope 1/2/3 배출량 등) 및 각 항목 ↔ Scope/데이터 매핑 정의. |
| **2단계** | **옵션 A** 적용: Scope 탭 유지, 각 블록에 "이 데이터가 채우는 공시 항목" 배지/문구 추가. |
| **3단계** | "공시 항목별 요약" 페이지 추가 — 항목별 입력·산출 현황 테이블, 클릭 시 해당 Scope/화면으로 이동. |
| **4단계** | (선택) **옵션 B 또는 C** 도입 — 공시 항목을 1차 메뉴 또는 사이드 메뉴로 두고, 항목 선택 시 해당 입력/결과만 노출. |
| **5단계** | (선택) **1-7 가이드형 UI** — 공시 프레임워크(GRI, IFRS S2, CDP, K-ETS 등) 선택 시 필수 요구 항목 배지·완료율(%) 표시. |


### 1-7. 공시 프레임워크 선택 기반 ‘가이드형 UI’ (보고서 중심 역설계)

본 GHG 산정 페이지는 **보고서 작성 관점에서 역설계된 구조**를 지향한다.  
사용자는 산정 시작 전 또는 도중에 적용할 **공시 프레임워크**를 선택할 수 있으며,  
선택된 기준에 따라 해당 기준에서 **필수로 요구되는 공시·보고 항목**이 화면에 가이드 형태로 표시된다.

#### 공시 프레임워크 선택

| 지원 프레임워크 예시 | 비고 |
|----------------------|------|
| **GRI** (Global Reporting Initiative) | 지속가능경영 보고 표준 |
| **IFRS S2** (Climate-related Disclosures) | 기후 관련 공시 |
| **CDP** (Carbon Disclosure Project) | 설문·배출량 공개 |
| **ESRS** (European Sustainability Reporting Standards) | EU 공시 기준 |
| **K-ETS** | 국내 배출권거래제 |
| **K-ESG** | 국내 ESG 공시·가이드 |

- UI: 드롭다운 또는 라디오로 **하나 또는 복수** 선택. 선택 시 해당 프레임워크의 필수·권장 항목이 1-2의 공시 항목 목록과 매핑되어 강조 표시.

#### 가이드형 UI 동작 방식

| 구분 | 동작 |
|------|------|
| **필수 요구 항목** | 공시 항목 카드·행에 **필수** 배지 표시. 미입력 시 강조(색상, 아이콘, 미완료 개수). |
| **선택/미해당 항목** | 기본 상태 유지 또는 시각적 강조 제거. |
| **완료율** | 상단 또는 사이드에 **“선택 기준 대비 입력 완료율(%)”** 표시. |

#### 사용자 경험 효과
- 실무자는 "내가 무엇을 더 입력해야 하는지"를 즉시 인지 가능.
- 공시 기준별 누락 리스크 감소.
- 보고서 작성 및 외부 감사 대응 시 **체크리스트 역할** 수행.

## Part 2. UI 조화·세련화 전략

### 2-1. 목표

- **조화**: 페이지·컴포넌트 간 시각적 일관성, 플랫폼 전역과의 통일감.
- **세련됨**: 정보 계층이 분명하고, 불필요한 시각 소음 감소, 여백·타이포·컬러가 정돈된 인상.

### 2-2. 진단 (현재 UI에서 개선할 부분)

- **일관성**: GHG 페이지만의 스타일이 플랫폼(회사정보, SR 작성, 최종보고서)과 다르게 느껴질 수 있음.
- **계층**: 탭·필터·그리드·결과 블록이 한 화면에 많아 **무엇이 1순위인지** 흐릿할 수 있음.
- **밀도**: 버튼·라벨·테이블이 빽빽하면 "세련됨"보다 "업무용 폼" 느낌만 강해짐.
- **상태 표현**: 로딩·빈 데이터·에러 시 메시지/스켈레톤이 없거나 제각각일 수 있음.

### 2-3. 디자인 원칙 (조화·세련)

| 원칙 | 적용 방향 |
|------|-----------|
| **단일 디자인 토큰** | 색상·간격·타이포·모서리·그림자를 플랫폼 공통 토큰(css variables 또는 tailwind 설정)으로 통일. GHG 페이지도 동일 토큰 사용. |
| **명확한 시각 계층** | 1) 페이지 제목·부제 2) 단계/탭(Scope 또는 공시 항목) 3) 필터·조건 4) 입력 그리드·결과 블록 순으로 위계 유지. 제목·섹션 제목은 크기·굵기로 구분. |
| **카드·패널 통일** | 모든 블록(필터, 그리드, 산정 결과, Total Carbon Inventory 등)을 동일한 카드 스타일(배경, 테두리, 패딩, 모서리)로 통일. |
| **여백 정리** | 섹션 간 간격은 1~2단계만 사용(예: 16px / 24px). 블록 내부 패딩도 동일 규칙 적용. |
| **버튼·입력 필드** | Primary / Secondary / Ghost 등 역할별 스타일을 플랫폼 공통으로 두고, GHG에서도 동일 컴포넌트 사용. 아이콘+텍스트 버튼은 정렬·간격 통일. |
| **타이포** | 제목·본문·캡션용 폰트 크기·줄간격을 플랫폼 규격에 맞춤. 표·숫자는 고정폭 또는 tabular-nums로 정렬감 유지. |

### 2-4. 레이아웃·구성

| 항목 | 권장 |
|------|------|
| **사이드바(필터)** | GHG_UI_Strategy_v2처럼 **좌측 고정 사이드바** 사용 시, 플랫폼 다른 페이지(예: SR 작성)의 사이드/패널과 **너비·배경·텍스트 스타일** 통일. |
| **메인 콘텐츠** | Scope 탭(또는 공시 항목) + 단계(조건 선택 → 활동자료 입력 → 결과)가 **한 흐름**으로 보이도록, 탭·스텝 인디케이터를 상단에 일관된 형태로 배치. |
| **그리드·테이블** | 헤더·행 높이·셀 패딩 통일. 짝/홀 행 배경 차이는 미미하게. 중요한 수치(총 tCO₂e 등)는 강조(크기·색상). |
| **빈 상태·로딩** | 데이터 없을 때: "데이터가 없습니다. [연료 추가] 또는 EMS/엑셀 업로드를 이용하세요" 등 **한 가지 톤**의 문구 + 일관된 빈 상태 일러스트/아이콘. 로딩 시 스켈레톤 또는 프로그레스를 플랫폼 공통 컴포넌트로. |

### 2-5. 색상·강조

| 용도 | 권장 |
|------|------|
| **배경** | 페이지 배경은 플랫폼 배경색과 동일. 카드/패널은 약한 대비(예: 배경보다 한 단계 진한 neutral). |
| **강조** | Primary(CTA, 진행): 브랜드 컬러. 성공/완료: 녹색 계열. 경고/미입력: amber 또는 red 한 가지 톤으로 통일. |
| **텍스트** | 제목: foreground, 본문: muted-foreground, 캡션/보조: 더 연한 muted. 링크는 primary. |
| **Scope/공시 항목** | Scope 1/2/3 또는 공시 항목별로 색을 나누고 싶다면, **연한 배지·좌측 보더** 정도로만 사용해 과하지 않게. |

### 2-6. 적용 순서 제안 (UI 세련화)

| 단계 | 내용 |
|------|------|
