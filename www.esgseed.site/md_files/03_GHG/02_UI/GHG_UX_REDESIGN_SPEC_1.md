# GHG 산정 입력 탭 — UX 구조 재설계 스펙

> **현재 문제**: 단계들이 세로로 나열된 구조, 조회 버튼 의존, 더미데이터 미연결, 산정 결과 미추적  
> **목표**: 스텝 기반 흐름 + 탭 진입 시 더미데이터 즉시 로드 + 행별 산정 결과 추적

---

## 1. 전체 구조 변경 — 세로 나열 → 스텝 플로우

### 현재 (❌)
```
[전력·열·스팀 탭 진입]
  ↓ 세로 스크롤
  1. 원시 데이터 수집 섹션
  2. 활동자료 입력 및 결과 섹션
  3. 배출량 산정 결과 섹션
  (전부 한 화면에 세로로 나열)
```

### 목표 (✅)
```
[전력·열·스팀 탭 진입]
  → STEP 1: 데이터 확인   [다음 단계 →]
  → STEP 2: 산정 실행     [다음 단계 →]
  → STEP 3: 결과 확인     [저장]

한 번에 하나의 STEP만 화면에 표시.
다음 단계 버튼으로 진행.
상단에 스텝 인디케이터 표시.
```

### 스텝 인디케이터 UI
```
● STEP 1 데이터 확인  ──  ○ STEP 2 산정 실행  ──  ○ STEP 3 결과 확인
```

## 2. STEP 1 — 데이터 확인

### 핵심 요구사항
- 탭 진입 시 **해당 탭의 더미데이터가 그리드에 즉시 로드**되어야 한다
- 조회 버튼 없이 초기 진입부터 전체 데이터 표시
- 필터(사업장·연도·월)는 **이미 로드된 데이터를 필터링**하는 용도

### 탭별 초기 로드 더미데이터

| 탭 | 자동 로드 파일 | 예상 행 수 |
|----|--------------|-----------|
| 전력·열·스팀 | `EMS_ENERGY_USAGE.csv` | 240행 |
| 연료·차량 | `ERP_FUEL_PURCHASE.csv` | 180행 |
| 냉매 | `EHS_REFRIGERANT.csv` | 50행 |
| 폐기물 | `EMS_WASTE.csv` | 120행 |
| 물류·출장·통근 | `SRM_LOGISTICS.csv` + `HR_COMMUTE_BUSINESS_TRAVEL.csv` | 240행 |
| 원료·제품 | `SRM_SUPPLIER_ESG.csv` + `PLM_PRODUCT_CARBON.csv` | 80행 |

### 로드 동작 순서
```
1. 탭 클릭
2. 해당 탭 더미 CSV 자동 로드 (로딩 스피너 표시)
3. 그리드에 전체 행 즉시 표시
4. 상단 카운터 업데이트: "● 240건  EMS 240건 / 수동 0건"
5. 필터는 초기값 "전체"로 세팅
```

### STEP 1 화면 구성
```
┌─────────────────────────────────────────────────────────────────────┐
│ ● STEP 1 데이터 확인  ── ○ STEP 2 산정 실행  ── ○ STEP 3 결과 확인  │

│ [EMS 불러오기]  [엑셀 업로드]  [+ 행 추가]                           │

│ 필터: 사업장▼  연도▼  월▼  에너지종류▼                               │
├──────┬────┬──────────┬──────────┬───────┬──────┬───────┬────┬────┬──┤

│  └─ 동기화: 2024-02-05  수정자: user_002  수정전: 275,000 kWh        │
```

### STEP 1 컬럼 — 탭별 정의

**전력·열·스팀**
```
사업장 | 월 | 에너지 종류 | 공급원 | 사용량 | 단위 | 재생에너지% | 품질 | 출처 | 수정이력
```
※ Scope 컬럼 없음. 사용자는 Scope 몰라도 됨.

**연료·차량**
```
사업장 | 월 | 연소구분(고정/이동) | 연료종류 | 소비량 | 단위 | 구매량 | 품질 | 출처 | 수정이력
```

**냉매**
```
사업장 | 설비ID | 설비유형 | 냉매종류 | 충전량(kg) | 누출량(kg) | GWP | 점검일 | 품질 | 출처 | 수정이력
```

**폐기물**
```
사업장 | 월 | 폐기물종류 | 폐기물명 | 발생량(톤) | 처리방법 | 소각량 | 재활용량 | Scope자동분류 | 품질 | 출처 | 수정이력
```

**물류·출장·통근**
```
구분(물류/출장/통근) | 월 | 이동수단 | 출발 | 도착 | 거리(km) | 중량/인원 | Cat. | 품질 | 출처 | 수정이력
```

**원료·제품**
```
구분(구매물품/제품사용/제품폐기) | 공급업체/제품명 | 배출량(tCO₂e) | 보고방식 | Cat. | 품질 | 출처 | 수정이력
```

## 3. STEP 2 — 산정 실행

### 핵심 요구사항
- STEP 1 그리드의 각 행에 **배출계수가 1:1로 매칭**되어 보여야 함
- "어떤 배출계수가 이 행에 적용됐는가"가 행 단위로 추적 가능해야 함
- 산정 실행 버튼 클릭 시 전체 행 일괄 산정

### STEP 2 화면 구성
```
┌─────────────────────────────────────────────────────────────────────┐
│ ✅ STEP 1 완료  ── ● STEP 2 산정 실행  ── ○ STEP 3 결과 확인        │
├─────────────────────────────────────────────────────────────────────┤
│ 적용 배출계수: 환경부 2024 KR Emission Factors  [변경]               │
│ GWP 기준: IPCC AR6                                                   │
├──────┬────┬──────┬───────┬──────┬────────────┬──────────┬──────────┤
│사업장│ 월 │에너지│ 사용량│ 단위 │   배출계수  │   산식   │배출량    │
│      │    │종류  │       │      │            │          │(tCO₂e)  │
├──────┼────┼──────┼───────┼──────┼────────────┼──────────┼──────────┤
│서울  │ 1월│ 전력 │125,000│ kWh  │0.4781      │125,000   │ 59.8    │
│      │    │      │       │      │kgCO₂e/kWh  │× 0.4781  │         │
│      │    │      │       │      │[EF-전력-24↗]│          │         │
├──────┼────┼──────┼───────┼──────┼────────────┼──────────┼──────────┤
│서울  │ 1월│ 열   │  3,200│ Gcal │0.2500      │3,200     │  0.8    │
│      │    │      │       │      │tCO₂e/Gcal  │× 0.2500  │         │
│      │    │      │       │      │[EF-열-24↗] │          │         │
├──────┼────┼──────┼───────┼──────┼────────────┼──────────┼──────────┤
│...                                                                   │
├─────────────────────────────────────────────────────────────────────┤
│ 소계: 전력 열·스팀 탭 합계  567.3 tCO₂e                              │
│                                          [← 이전]  [다음 단계 →]    │
└─────────────────────────────────────────────────────────────────────┘
```

- **[EF-전력-24↗]** 클릭 시 MDG 배출계수 상세 팝업
- 배출계수는 `MDG_EMISSION_FACTOR.csv`에서 `energy_type` 기준으로 자동 매칭
- 산정 완료 시 각 행에 `applied_factor_id`, `applied_factor_value`, `calculated_at` 저장

## 4. STEP 3 — 결과 확인

### 핵심 요구사항
- 이 탭의 산정 결과만 표시 (전체 Scope 합산은 우측 패널에)
- 각 행의 배출량과 원천 데이터가 연결된 상태로 보여야 함
- "저장" 시 히스토리에 버전으로 기록 (v1, v2...)

## 5. 우측 패널 — TOTAL CARBON INVENTORY

우측 패널은 현재 구조(실시간 합산)는 맞는데, **저장된 탭만 반영**되어야 함.

```
TOTAL CARBON INVENTORY
  ← 저장된 탭 결과의 합산값

  Scope 1: 1,234.0 tCO₂e  [연료·차량↗] [냉매↗] [폐기물↗]
  Scope 2:   567.3 tCO₂e  [전력·열·스팀↗]
  Scope 3: 3,890.0 tCO₂e  [물류·출장·통근↗] [원료·제품↗]

  저장된 히스토리:
  v2  2024-02-15  user_001  5,691.3 tCO₂e  [현재]
  v1  2024-02-05  user_001  5,632.1 tCO₂e
```

## 6. 산정 결과 화면 문제 — 현재 vs 목표

| 항목 | 현재 (❌) | 목표 (✅) |
|------|----------|----------|
| 결과 표시 단위 | 전체 합산 숫자만 | 행별 배출량 + 합계 |
| 배출계수 표시 | 하단에 텍스트 나열 | 각 행에 1:1 인라인 표시 |
| 원천 추적 | 불가 | 행 클릭 시 STEP 1 원천 행으로 드릴다운 |
| 수동 조정 표시 | 없음 | ⚠️ 배지 + 상세 보기 |
| 저장 | 없음 | 탭별 저장 + 버전 히스토리 |

## 7. 더미데이터 연결 — 탭별 CSV 매핑 (개발자 체크리스트)

```
전력·열·스팀 탭  ← EMS_ENERGY_USAGE.csv
  - energy_type: '전력' → 전력 서브탭
  - energy_type: '열' | '스팀' → 열/스팀 서브탭
  - 배출계수 매칭: MDG_EMISSION_FACTOR.csv (factor_category = '전력')

연료·차량 탭  ← ERP_FUEL_PURCHASE.csv
  - fuel_category: '고정연소' → 고정연소 행
  - fuel_category: '이동연소' → 이동연소 행
  - 배출계수 매칭: MDG_EMISSION_FACTOR.csv (factor_category = '고정연소' | '이동연소')

냉매 탭  ← EHS_REFRIGERANT.csv
  - 배출량 = leak_amount_kg × gwp_factor / 1000 (tCO₂e)
  - gwp_factor: MDG_EMISSION_FACTOR.csv에서 refrigerant_type 기준 매칭

폐기물 탭  ← EMS_WASTE.csv
  - disposal_method = '소각' → Scope 1
  - disposal_method ≠ '소각' → Scope 3 Cat.12
  - 배출계수 매칭: MDG_EMISSION_FACTOR.csv (factor_category = '폐기물')

물류·출장·통근 탭  ← SRM_LOGISTICS.csv + HR_COMMUTE_BUSINESS_TRAVEL.csv
  - SRM: logistics_type으로 Cat.4 / Cat.9 구분
  - HR: category로 '출장'(Cat.6) / '통근'(Cat.7) 구분
  - 배출계수 매칭: MDG_EMISSION_FACTOR.csv (transport_mode 기준)

원료·제품 탭  ← SRM_SUPPLIER_ESG.csv + PLM_PRODUCT_CARBON.csv
  - SRM: supplier_emission_tco2e → Cat.1 (값이 이미 tCO₂e, 배출계수 불필요)
  - PLM: use_phase_emission → Cat.11 / eol_emission → Cat.12
```

## 8. 핵심 변경 사항 요약 (개발자 전달용)

| # | 변경 내용 | 우선순위 |
|---|-----------|---------|
| 1 | 세로 나열 구조 → STEP 1/2/3 스텝 플로우로 변경 | P0 |
| 2 | 탭 진입 시 더미 CSV 자동 로드 (조회 버튼 없이) | P0 |
| 3 | 탭별 컬럼 구성 변경 (Scope 컬럼 제거, 품질·수정이력 추가) | P0 |
| 4 | STEP 2: 행별 배출계수 1:1 매칭 표시 | P1 |
| 5 | STEP 3: 행별 배출량 + 합계 구조로 변경 | P1 |
| 6 | 더미 CSV ↔ MDG 배출계수 자동 매칭 로직 | P1 |
| 7 | 탭별 저장 + 버전 히스토리 | P1 |
| 8 | 수동 조정 이력 ⚠️ 배지 표시 | P2 |
