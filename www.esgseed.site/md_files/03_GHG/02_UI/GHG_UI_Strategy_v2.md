# GHG 산정 UI 전략 v2 (멀티필터·테이블 구조)

## 1. 개요

### 1-1. 핵심 변경 사항

| 항목 | 기존 | 변경 |
|------|------|------|
| **산정 설정** | GHGBoundarySettings 탭 유지 | **완전 제거** — 별도 페이지/설정에서 관리하거나 최소화 |
| **멀티 필터** | 상단 인라인 필터 바 | **사이드바 형태** — RE:SEED 사이드바와 동일한 스타일로 왼쪽 수직 배치 |
| **사업장 선택** | 필터·테이블 등 여러 위치 | **사이드바에서만 선택** — 테이블에는 사업장 열 없음 |
| **결과 테이블** | 행=월(1~12), 열=사업장 | **행=에너지원, 열=월(1~12)** — 행·열 교체 |
| **활동자료 입력** | 스크롤 내리면 한 폼에 모두 노출 | **단계별 설정** — 고정연소/이동연소, 전기/재생에너지/열을 단계로 분리하여 설정 |

### 1-2. 적용 범위

- GHG 산정 메인 화면 (`/ghg-calculation`)
- Scope 1·2 결과 테이블 (행=에너지원, 열=월)
- 멀티 필터 UI 전면 개편

## 2. 산정 설정 제거

### 2-1. 제거 대상

- **GHGBoundarySettings** 탭/섹션 전체 제거
- 경계 설정(Boundary & Policy)은 다음 중 하나로 처리:
  - 회사정보(`/company-info`) 페이지에서 사전 설정
  - 별도 설정 메뉴로 이동
  - 또는 모달/최소 노출로 축소 (필요 시만 진입)

### 2-2. 탭 구조 변경

| 기존 탭 | 변경 후 탭 |
|---------|------------|
| 산정 설정 | **제거** |
| 데이터 관리 센터 | **메인 진입** (첫 화면) |
| 결과 | 결과 |
| 리포트 | 리포트 |

- **진입 시**: 데이터 관리 센터가 기본 탭
- 결과·리포트 탭만 유지

## 3. 멀티 필터: 사이드바 스타일 (첫 번째 이미지 기준)

### 3-1. 디자인 원칙

- **RE:SEED 사이드바와 동일한 스타일**
  - 어두운 회색 수직 사이드바
  - 깔끔한 다크 모드 느낌
  - 왼쪽 고정, 콘텐츠 영역과 분리

### 3-2. 레이아웃

```
┌──────────────────────────────────────────────────────────────┐
│  [멀티 필터 사이드바]  │         메인 콘텐츠 영역              │
│  (다크 회색, 수직)     │  (Scope 1/2/3, 데이터 입력·테이블)    │
│                        │                                     │
│  필터                   │  에너지원별·월별 결과 테이블          │
│  ─────────             │  (행=에너지원, 열=1~12월)             │
│  사업장 (선택만 여기서) │  ※ 사업장은 사이드바에서만 선택       │
│  에너지원               │                                     │
│  연도                   │                                     │
│  시기 단위              │                                     │
│                        │                                     │
└──────────────────────────────────────────────────────────────┘
```

### 3-3. 사이드바 구성

| 섹션 | 항목 | UI 타입 | 비고 |
|------|------|---------|------|
| **필터** | **사업장** | 다중 선택 (Checkbox) | **사이드바에서만 선택** — 선택한 사업장 데이터만 테이블에 반영 (테이블 열에 사업장 표시 안 함) |
| | 에너지원 | 다중 선택 (Checkbox) | 선택한 에너지원만 테이블 행으로 표시 |
| | 연도 | Select | 단일 연도 |
| | 시기 단위 | Select | 월별 / 분기별 / 연간 |
| | 초기화 | Button | 필터 전체 해제 |

### 3-4. 스타일 가이드 (첫 번째 이미지 반영)

- 배경: `slate-800` ~ `slate-900` (어두운 회색)
- 텍스트: 흰색 또는 `slate-100`
- 아이콘: 윤곽선 스타일, 흰색
- 섹션 제목: `DASHBOARD` 스타일 대문자, 연한 회색 (`slate-400`)
- 구분선: 얇은 가로 회색 선
- 너비: 약 240~280px (고정)

## 4. 활동자료 입력: 스크롤 → 단계별 설정

### 4-1. 문제점 (기존)

- **한 화면에 모든 입력 폼**이 세로로 쌓여 있어 스크롤이 길어짐
- 고정연소·이동연소·전기·재생에너지 이행·열/스팀/온수 등이 한 페이지에 혼재
- 사용자 관점에서 **정보 과부하·초점 분산**, 필요한 항목을 찾기 어려움

### 4-2. 변경 방향: 단계별 설정 (Step-by-Step)

스크롤 기반의 긴 폼을 **단계(Step)**로 분리하여, 한 번에 한 설정만 집중적으로 입력할 수 있게 한다.

### 4-3. Scope 1 (연료 연소) — 단계 구성

| 단계 | 단계명 | 포함 내용 | 비고 |
|------|--------|-----------|------|
| **Step 1** | 고정 연소 | 월·사업장·에너지원·사용량·계수/배출량·데이터 품질 | 필수 |
| **Step 2** | 이동 연소 | 월·사업장·에너지원·사용량·계수/배출량·데이터 품질 | 회사 차량 |
| **완료** | 입력 요약 | 추가된 고정/이동 연소 행 목록 | 다음 단계로 진행 |

- 각 단계 전환: "다음 단계" / "이전 단계" 버튼
- 단계 인디케이터: ① 고정 연소 → ② 이동 연소

### 4-4. Scope 2 (전력·열) — 단계 구성

| 단계 | 단계명 | 포함 내용 | 비고 |
|------|--------|-----------|------|
| **Step 1** | 구매 전기 | 월·사업장·전력 사용량·계수/배출량·데이터 품질 | 필수 |
| **Step 2** | 재생에너지 이행 실적 | 녹색프리미엄·REC·PPA·자가발전·EAC 인증서 등 | 시장 기반 산정·RE100 대응 |
| **Step 3** | 구매 열/스팀/온수 | 표준 추가 / KDHC 추가 | 선택 |
| **완료** | 입력 요약 | 전력·재생에너지·열 입력 목록 | 다음 단계로 진행 |

- 각 단계 전환: "다음 단계" / "이전 단계" 버튼
- 단계 인디케이터: ① 구매 전기 → ② 재생에너지 이행 → ③ 열/스팀/온수

### 4-5. Scope 3 — 현행 유지 또는 단계화 검토

- Scope 3은 카테고리별 구조가 상이하므로, 현행 탭/카드 구조 유지 후 필요 시 단계화 검토

### 4-6. UI 패턴

- **메인 콘텐츠 영역** 내에 단계 인디케이터(가로 또는 세로 스텝) + 현재 단계 폼만 표시
- 스크롤 최소화 — 한 단계당 한 화면 내에 수납
- 단계 간 전환 시 입력값 유지 (스토어에 저장)

## 5. 결과 테이블: 행·열 구조 변경

### 5-1. 기존 vs 변경

| 구분 | 기존 | 변경 |
|------|------|------|
| **행(Rows)** | 월 (1월 ~ 12월) | **에너지원** (도시가스, 경유, 전력 등) |
| **열(Columns)** | 사업장 (본사, 공장A, 공장B, 지점1, 지점2) | **월 (1월 ~ 12월)** |
| **셀 값** | tCO2e (월·사업장별 합산) | tCO2e (에너지원·월별 합산, 사업장 필터 적용) |

### 5-2. 테이블 구조

**제목**: `에너지원/월별 결과 테이블`

**행**: 에너지원
- Scope 1: 도시가스(LNG), 경유, 휘발유, LPG, 벙커유, 무연탄 등
- Scope 2: 전력, 열/스팀/온수 등
- 필터에서 에너지원 미선택 시: 전체 에너지원 표시

**열**: 월 (1월 ~ 12월)
- 시기 단위가 월별일 때: 1월 ~ 12월 열
- 분기별: 1~4분기 열
- 연간: 연도 합산 1열

**셀**: 해당 에너지원·월 조합의 tCO2e 합산값  
※ **사업장은 사이드바에서만 선택** — 선택한 사업장들의 데이터만 합산하여 셀에 반영 (테이블 열에는 사업장 미표시)

### 5-3. 필터 연동 규칙

| 필터 선택 | 테이블 반영 |
|-----------|-------------|
| **사업장** (사이드바에서만) | 선택한 사업장들의 데이터만 합산 — 테이블 구조(행=에너지원, 열=월) 유지 |
| 에너지원 3개 선택 | 행 = 해당 3개 에너지원만 |
| 에너지원 전체 선택 / 미선택 | 행 = 전체 에너지원 |
| 연도 2024 | 2024년 데이터만 집계 |
| 시기 단위: 월별 | 열 = 1월 ~ 12월 |
| 시기 단위: 분기별 | 열 = 1~4분기 |
| 시기 단위: 연간 | 열 = 연도 합산 1열 |

### 5-4. 하단 설명 문구

> 단위: tCO2e - 입력한 데이터를 에너지원별·월별로 합산해 표시합니다. **사업장은 사이드바에서 선택**하며, 선택한 사업장의 데이터만 반영됩니다.

## 7. 결과·리포트 섹션 — 사용자 관점 필요성 분석

### 7-1. 결과 탭 (Step4Results) 현황

| 구성 요소 | 설명 | 사용자 필요성 | 권장 |
|-----------|------|---------------|------|
| **총 배출량 (tCO2e)** | Scope 1+2+3 합계 | ✅ K-ETS/MRV 제출, 내부 보고 핵심 | **유지** |
| **Scope 1/2/3 각각 배출량** | Scope별 breakdown | ✅ 원인 분석·감사 대응 | **유지** |
| **데이터 신뢰도** | High/Medium (실측 vs 추정) | ⚠️ K-ETS·감사 시 중요하나 현재 추정 플래그 활용도 낮음 | **유지** (감사 대비) |
| **검증 필요성** | "제3자 검증 권고" 고정 문구 | ⚠️ 상시 동일 메시지, 정보량 적음 | **제거** 또는 요약 영역으로 통합 |

**결과 탭 권장 최소 구성**: 총 배출량 + Scope 1/2/3 breakdown + (선택) 데이터 신뢰도

---

### 7-2. 리포트 탭 (Step5Report) 현황

| 구성 요소 | 설명 | 사용자 필요성 | 권장 |
|-----------|------|---------------|------|
| **PDF 다운로드** | 산정설정·Scope1·2·3·영수증 목록 | ✅ K-ETS/MRV 증적, 감사 제출 | **유지** |
| **Excel 다운로드** | 요약·산정설정·Scope1·2·3·영수증 시트 | ✅ 데이터 분석, K-ETS 온라인 제출 | **유지** |
| **JSON 다운로드** | 전체 메타데이터·영수증 | ⚠️ 시스템 연동·개발자용, 일반 사용자 사용 빈도 낮음 | **삭제** 또는 고급 메뉴로 이동 |

**리포트 탭 권장 구성**: PDF + Excel (2개만 노출), JSON은 설정·고급 메뉴에서 제공

---

### 7-3. 방법론 패널 (Step3Methodology) 현황

| 구성 요소 | 설명 | 사용자 필요성 | 권장 |
|-----------|------|---------------|------|
| 배출계수·GWP·산식 | 환경부 계수 버전, 산정 공식 | ✅ 감사·검증 대응 | **유지** |
| 결과 탭 내 배치 | 결과 하단 접이식 영역 | ⚠️ 필요 시 펼쳐서 확인 | **유지** (접기/펼치기 가능) |

---

### 7-4. 삭제/축소 대상 요약

| 항목 | 조치 |
|------|------|
| 결과 탭 "검증 필요성" | **삭제** — 정보가 고정적이라 유용성 낮음 |
| 리포트 탭 JSON 다운로드 | **삭제** 또는 고급 메뉴로 이동 — 일반 사용자 노출 최소화 |
