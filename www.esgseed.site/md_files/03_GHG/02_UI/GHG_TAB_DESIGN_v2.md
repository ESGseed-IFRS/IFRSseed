# GHG 산정 — 데이터 입력 탭 구조 설계

> **핵심 원칙**: 사용자는 Scope를 몰라도 된다. "내가 어떤 데이터를 갖고 있는가" 기준으로 탭을 찾을 수 있어야 한다.  
> 시스템은 내부적으로 Scope를 알고 있고, 산정 결과 화면에서 Scope 1/2/3로 보여준다.  
> **입력 화면 형태**: 그리드(테이블) — 행별 데이터 나열 + 계보 정보 인라인 표시

---

## 탭 구조 한눈에 보기

```
GHG 산정 입력
├── 1. 전력·열·스팀      [Scope 2]   EMS
├── 2. 연료·차량         [Scope 1]   ERP
├── 3. 냉매              [Scope 1]   EHS
├── 4. 폐기물            [Scope 1·3] EMS
├── 5. 물류·출장·통근    [Scope 3]   SRM·HR
└── 6. 원료·제품         [Scope 3]   SRM·PLM
```

---

## 그리드 공통 설계 원칙

### 컬럼 구성 규칙

모든 탭의 그리드는 **3개 영역**으로 구성된다.

```
[데이터 영역]              [계보 영역]                  [상태 영역]
사업장·월·활동자료 값 등   출처·동기화·수정이력 등      품질등급·산정여부
```

### 계보 인라인 표시 방식

행마다 계보 정보를 접을 수 있는 형태로 표시한다.

```
▶ 행 (기본 접힘 상태)
  사업장  월  에너지종류  사용량  단위  품질  출처  수정이력

▼ 행 (펼침 상태 — 계보 상세)
  사업장  월  에너지종류  사용량  단위  품질  출처  수정이력
  └─ 출처: EMS  |  동기화: 2024-02-05 08:55  |  적재: 2024-02-05 09:00
     수정이력: 없음 ✅   (또는 수정자: user_002 | 2024-02-10 14:32 | 수정전: 275,000 ⚠️)
```

### 수동 조정 플래그 표시

| 상태 | 아이콘 | 조건 |
|------|--------|------|
| 정상 | ✅ | `updated_at` = `synced_at` (수정 없음) |
| 수동 조정 | ⚠️ | `updated_at` - `synced_at` > 24시간 |
| 수동 입력 | 🖊 | `source_system` = 수동 |

### 상단 공통 UI 요소

모든 탭에 공통으로 제공되는 상단 컨트롤:

```
[시스템 불러오기 버튼]  [엑셀 업로드]  [행 추가]  [전체 삭제]

필터: 사업장 ▼   연도 ▼   월 ▼   [조회]  [초기화]

적재 현황: ● N건   출처별: EMS N건 / 수동 N건   수정이력 있음: N건 ⚠️
```

## 탭별 상세

---

### 탭 1 — 전력·열·스팀

**왜 이렇게 나눴나**  
우리 회사가 외부에서 **사서 쓰는 에너지**다. 직접 태우는 게 아니라 한전·지역난방에서 구매해서 쓰는 것. 담당자가 월별 전기요금 고지서, 열 사용 내역서를 갖고 오는 경우가 대부분이라 하나로 묶었다.

| 항목 | 내용 |
|------|------|
| **소스 시스템** | EMS |
| **소스 파일** | `EMS_ENERGY_USAGE.csv` |
| **내부 분류** | Scope 2 |
| **포함 데이터** | 전력 사용량(kWh), 열 사용량(Gcal), 스팀 사용량(GJ), 재생에너지 비율 |
| **자동 분류 기준** | `energy_type` 컬럼값: `전력` / `열` / `스팀` |

**그리드 컬럼 구성**:

| 컬럼 | 소스 필드 | 설명 |
|------|-----------|------|
| 사업장 | `site_name` | MDG 기준 |
| 월 | `month` | 1~12 |
| 에너지 종류 | `energy_type` | 전력 / 열 / 스팀 |
| 공급원 | `energy_source` | 한국전력 / 지역난방 등 |
| 사용량 | `usage_amount` | 숫자 |
| 단위 | `usage_unit` | kWh / Gcal / GJ |
| 재생에너지 비율 | `renewable_ratio` | % |
| 품질 | `data_quality` | M1 / M2 / E1 / E2 배지 |
| 출처 | `source_system` | EMS / 수동 |
| 수정이력 | `updated_at` vs `synced_at` | ✅ / ⚠️ |

**화면 예시**:

```
사업장    월   에너지종류  공급원    사용량     단위   재생E%  품질  출처  이력
서울본사  1월  전력        한국전력  125,000    kWh    12.5%   M1   EMS   ✅
서울본사  1월  열          지역난방  3,200      Gcal   0%      M1   EMS   ✅
수원공장  1월  전력        한국전력  280,000    kWh    8.2%    M1   EMS   ⚠️
  └─ 동기화: 2024-02-05 08:55  |  수정자: user_002  |  2024-02-10 14:32  |  수정전: 275,000 kWh
수원공장  1월  스팀        자가발전  1,800      GJ     100%    M1   EMS   ✅
```

### 탭 2 — 연료·차량

**왜 이렇게 나눴나**  
우리 회사가 **직접 태우는 에너지**다. 보일러·발전기에 넣는 가스·기름(고정연소), 트럭·지게차·업무차량에 넣는 기름(이동연소). 둘 다 ERP의 구매·소비 기록에서 오고, 담당자가 "연료비 얼마 썼냐"로 묶어서 생각하는 데이터라 같은 탭에 넣었다.

| 항목 | 내용 |
|------|------|
| **소스 시스템** | ERP |
| **소스 파일** | `ERP_FUEL_PURCHASE.csv` |
| **내부 분류** | Scope 1 (고정연소·이동연소) |
| **포함 데이터** | LNG·LPG·경유·휘발유·B-C유 구매량·소비량 |
| **자동 분류 기준** | `fuel_category` 컬럼값: `고정연소` / `이동연소` |

**그리드 컬럼 구성**:

| 컬럼 | 소스 필드 | 설명 |
|------|-----------|------|
| 사업장 | `site_name` | MDG 기준 |
| 월 | `month` | 1~12 |
| 연소 구분 | `fuel_category` | 고정연소 / 이동연소 배지 |
| 연료 종류 | `fuel_type` | LNG / 경유 / 휘발유 등 |
| 소비량 | `consumption_amount` | 숫자 |
| 단위 | `fuel_unit` | Nm³ / L / kg |
| 구매량 | `purchase_amount` | 재고 검증용 |
| 품질 | `data_quality` | M1 / M2 / E1 / E2 배지 |
| 출처 | `source_system` | ERP / 수동 |
| 수정이력 | `updated_at` vs `synced_at` | ✅ / ⚠️ |

**화면 예시**:

```
사업장    월   구분     연료종류  소비량   단위  구매량   품질  출처  이력
수원공장  1월  고정연소  LNG      85,000  Nm³   85,500  M2   ERP   ✅
수원공장  1월  이동연소  경유     12,500  L     13,000  M2   ERP   ✅
구미공장  1월  고정연소  B-C유   42,000  L     42,000  M2   ERP   ⚠️
  └─ 동기화: 2024-02-05 07:30  |  수정자: user_003  |  2024-02-12 09:15  |  수정전: 41,500 L
```

### 탭 3 — 냉매

**왜 따로 분리했나**  
냉매는 에너지도 아니고 연료도 아니다. 에어컨·냉동기에 충전된 가스가 **누출**되면서 배출되는 것으로, 데이터 출처도 EHS(환경안전보건) 시스템이고 담당자도 다른 경우가 많다. 다른 탭에 섞이면 묻혀버리는데 GWP(지구온난화지수)가 높아서 배출량 비중이 크기 때문에 별도 탭으로 분리했다.

| 항목 | 내용 |
|------|------|
| **소스 시스템** | EHS |
| **소스 파일** | `EHS_REFRIGERANT.csv` |
| **내부 분류** | Scope 1 (탈루 배출) |
| **포함 데이터** | 냉매 종류(HFC-134a 등), 충전량, 누출량, GWP 계수 |
| **자동 분류 기준** | 파일 자체가 냉매 전용 |

**그리드 컬럼 구성**:

| 컬럼 | 소스 필드 | 설명 |
|------|-----------|------|
| 사업장 | `site_name` | MDG 기준 |
| 설비 ID | `equipment_id` | EQ-AC-001 등 |
| 설비 유형 | `equipment_type` | 에어컨 / 냉동기 / 칠러 |
| 냉매 종류 | `refrigerant_type` | HFC-134a / HFC-410A 등 |
| 충전량(kg) | `charge_amount_kg` | 숫자 |
| 누출량(kg) | `leak_amount_kg` | 숫자 — 배출량 계산 기준 |
| GWP | `gwp_factor` | MDG 배출계수 연동값 |
| 점검일 | `inspection_date` | DATE |
| 품질 | `data_quality` | M1 / M2 배지 |
| 출처 | `source_system` | EHS / 수동 |
| 수정이력 | `updated_at` vs `synced_at` | ✅ / ⚠️ |

**화면 예시**:

```
사업장     설비ID      설비유형  냉매종류    충전(kg)  누출(kg)  GWP    점검일      품질  출처  이력
서울본사   EQ-AC-001   에어컨    HFC-134a    5.2       0.8       1,430  2024-06-15  M1   EHS   ✅
판교연구소 EQ-CH-003   칠러      HFC-410A    12.0      1.2       2,088  2024-06-20  M1   EHS   ✅
수원공장   EQ-FR-007   냉동기    HFC-134a    8.5       0.0       1,430  2024-07-01  M1   EHS   ✅
```

### 탭 4 — 폐기물

**왜 이렇게 나눴나**  
폐기물은 "우리가 버리는 것"으로 직원들이 직관적으로 알아듣는 단어다. 소각하면 Scope 1, 외부 위탁 처리하면 Scope 3로 내부 분류가 갈리지만 사용자는 그걸 몰라도 된다. 데이터 출처도 EMS지만, 담당자가 "에너지 담당"과 "폐기물 담당"으로 나뉘는 경우가 많아서 탭을 분리했다.

| 항목 | 내용 |
|------|------|
| **소스 시스템** | EMS |
| **소스 파일** | `EMS_WASTE.csv` |
| **내부 분류** | Scope 1 (소각분) / Scope 3 Cat.12 (위탁 처리분) |
| **포함 데이터** | 폐기물 종류·발생량·처리 방법(소각/매립/재활용/위탁) |
| **자동 분류 기준** | `disposal_method` = `소각` → Scope 1 / 그 외 → Scope 3 |

**그리드 컬럼 구성**:

| 컬럼 | 소스 필드 | 설명 |
|------|-----------|------|
| 사업장 | `site_name` | MDG 기준 |
| 월 | `month` | 1~12 |
| 폐기물 종류 | `waste_type` | 일반 / 지정 / 건설 |
| 폐기물명 | `waste_name` | 폐합성수지 / 폐유 등 |
| 발생량(톤) | `generation_amount` | 숫자 |
| 처리 방법 | `disposal_method` | 소각 / 매립 / 재활용 / 위탁 |
| 소각량(톤) | `incineration_amount` | Scope 1 산정 기준 컬럼 |
| 재활용량(톤) | `recycling_amount` | 숫자 |
| Scope 자동 분류 | 내부 로직 | Scope 1 / Scope 3 배지 |
| 품질 | `data_quality` | M1 / M2 배지 |
| 출처 | `source_system` | EMS / 수동 |
| 수정이력 | `updated_at` vs `synced_at` | ✅ / ⚠️ |

**화면 예시**:

```
사업장    월   폐기물종류  폐기물명   발생량(톤)  처리방법  소각량  재활용량  Scope   품질  출처  이력
수원공장  3월  지정폐기물  폐합성수지  15.3       재활용    0       10.2     Scope3  M1   EMS   ✅
수원공장  3월  지정폐기물  폐유        2.1        위탁처리  0       0        Scope3  M1   EMS   ✅
구미공장  3월  지정폐기물  폐산        0.8        소각      0.8     0        Scope1  M1   EMS   ✅
```

### 탭 5 — 물류·출장·통근

**왜 이렇게 나눴나**  
"우리 회사 사람과 물건이 이동하는 것"을 모두 담았다. 화물 운송(SRM), 임직원 출장·통근(HR)은 데이터 출처가 다르지만 사용자 관점에서는 "이동하는 것"으로 같은 카테고리로 인식한다. 탄소 발자국에서 물류·출장이 생각보다 비중이 크기 때문에 한 탭에서 한눈에 보이게 했다.

| 항목 | 내용 |
|------|------|
| **소스 시스템** | SRM + HR |
| **소스 파일** | `SRM_LOGISTICS.csv`, `HR_COMMUTE_BUSINESS_TRAVEL.csv` |
| **내부 분류** | Scope 3 Cat.4(인바운드 물류)·Cat.6(출장)·Cat.7(통근)·Cat.9(아웃바운드 물류) |
| **포함 데이터** | 운송 수단·거리·중량(물류), 이동 수단·거리·인원(출장·통근) |
| **자동 분류 기준** | 파일 출처 + `category` / `logistics_type` 컬럼으로 구분 |

**그리드 컬럼 구성**:

| 컬럼 | 소스 필드 | 설명 |
|------|-----------|------|
| 구분 | `category` / `logistics_type` | 물류(인바운드/아웃바운드) / 출장 / 통근 |
| 월 | `month` | 1~12 |
| 이동 수단 | `transport_mode` | 항공 / 해상 / 도로 / 철도 / 자가용 등 |
| 출발 | `origin_country` | 국가 코드 |
| 도착 | `destination_country` | 국가 코드 |
| 거리(km) | `distance_km` | 숫자 |
| 중량/인원 | `weight_ton` / `person_trips` | 물류는 톤, 출장·통근은 명 |
| Scope Cat. | 내부 로직 | Cat.4 / Cat.6 / Cat.7 / Cat.9 배지 |
| 품질 | `data_quality` | M1 / M2 / E2 배지 |
| 출처 | `source_system` | SRM / HR / 수동 |
| 수정이력 | `updated_at` vs `synced_at` | ✅ / ⚠️ |

**화면 예시**:

```
구분             월   이동수단  출발  도착  거리(km)  중량/인원  Cat.   품질  출처  이력
물류(인바운드)   1월  해상      CN    인천  2,100     85.5톤    Cat.4  E2   SRM   ✅
물류(아웃바운드) 1월  항공      인천  LA    9,500     12.3톤    Cat.9  E2   SRM   ✅
출장             1월  항공      서울  도쿄   1,160     12명      Cat.6  M2   HR    ✅
통근             1월  자가용     —     —      —        245명     Cat.7  E2   HR    ⚠️
  └─ 동기화: 2024-02-05 07:00  |  수정자: user_006  |  2024-02-08 11:00  |  수정전: 240명
```

### 탭 6 — 원료·제품

**왜 이렇게 나눴나**  
"우리가 만드는 것의 앞뒤"다. 원료를 사올 때(업스트림)와 우리 제품이 고객에게 쓰이고 폐기될 때(다운스트림)의 배출량. 데이터 출처가 SRM(공급망)과 PLM(제품 수명주기)으로 나뉘지만, 사용자 관점에서는 "제품과 원료에 관한 것"으로 묶인다. Scope 3에서 가장 비중이 크고 복잡한 영역이라 별도 탭으로 분리했다.

| 항목 | 내용 |
|------|------|
| **소스 시스템** | SRM + PLM |
| **소스 파일** | `SRM_SUPPLIER_ESG.csv`, `PLM_PRODUCT_CARBON.csv` |
| **내부 분류** | Scope 3 Cat.1(구매 물품)·Cat.11(제품 사용)·Cat.12(제품 폐기) |
| **포함 데이터** | 공급업체별 배출량(SRM), 제품 생애주기 단계별 배출량(PLM) |
| **자동 분류 기준** | 파일 출처로 구분 — SRM → Cat.1, PLM → Cat.11·12 |

**그리드 컬럼 구성**:

| 컬럼 | 소스 필드 | 설명 |
|------|-----------|------|
| 구분 | 파일 출처 | 구매물품(SRM) / 제품사용(PLM) / 제품폐기(PLM) |
| 공급업체 / 제품명 | `supplier_name` / `product_name` | 텍스트 |
| 배출량(tCO₂e) | `supplier_emission_tco2e` / `use_phase_emission` / `eol_emission` | 숫자 |
| 보고 방식 | `ghg_reported_yn` | 직접보고 / 추정 |
| Scope Cat. | 내부 로직 | Cat.1 / Cat.11 / Cat.12 배지 |
| 품질 | `data_quality` | M2 / E2 배지 |
| 출처 | `source_system` | SRM / PLM / 수동 |
| 수정이력 | `updated_at` vs `synced_at` | ✅ / ⚠️ |

**화면 예시**:

```
구분      공급업체/제품명       배출량(tCO₂e)  보고방식      Cat.    품질  출처  이력
구매물품  (주)한국소재          12,500         직접보고      Cat.1   M2   SRM   ✅
구매물품  (주)글로벌부품         8,200         추정(spend)   Cat.1   E2   SRM   ✅
제품사용  스마트디스플레이A1     8,500          산정값        Cat.11  E1   PLM   ✅
제품폐기  스마트디스플레이A1     420            산정값        Cat.12  E1   PLM   ✅
```

## EMS 이중 소속 처리

EMS는 **전력·열·스팀**과 **폐기물** 두 탭에 데이터를 공급한다.  
"EMS 전체 불러오기" 버튼 하나로 시스템이 파일 내 컬럼을 보고 자동 분배한다.

```
"EMS 전체 불러오기" 클릭
    ↓
EMS_ENERGY_USAGE.csv → energy_type 컬럼 확인 → 전력·열·스팀 탭 (240건)
EMS_WASTE.csv        → waste_type 컬럼 확인  → 폐기물 탭 (120건)
    ↓
토스트: "전력·열·스팀 240건, 폐기물 120건 적재 완료"
```

## 데이터 입력 → 산정 결과 흐름

```
[산정 입력 탭]              [내부 Scope 매핑]           [산정 결과]

전력·열·스팀         →      Scope 2               →   Scope 2: 567 tCO₂e
연료·차량            →      Scope 1 고정·이동      ↗
냉매                 →      Scope 1 탈루           →   Scope 1: 1,234 tCO₂e
폐기물 (소각)        →      Scope 1 소각           ↗
폐기물 (위탁)        →      Scope 3 Cat.12         ↘
물류·출장·통근       →      Scope 3 Cat.4,6,7,9    →   Scope 3: 3,890 tCO₂e
원료·제품            →      Scope 3 Cat.1,11,12    ↗
```

산정 결과 탭에서 각 수치 옆에 **(탭명)** 태그로 역추적 가능:

```
Scope 1: 1,234 tCO₂e
  └─ 고정연소 852.4 tCO₂e  [연료·차량 탭↗]
  └─ 이동연소 298.6 tCO₂e  [연료·차량 탭↗]
  └─ 탈루     61.2 tCO₂e   [냉매 탭↗]
  └─ 소각     22.0 tCO₂e   [폐기물 탭↗]
```
