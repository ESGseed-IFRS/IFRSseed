# GHG 감사 대응 탭 전략

> **목적**: GHG 산정 화면의 **감사 대응 탭**이 IFRS S2 요건별 데이터 계보를 어떻게 추적·표시·증빙하는지 설계한다.  
> 산정 입력·결과 탭은 감사 대응 탭에 데이터를 공급하는 upstream으로 다루며, 감사 대응 탭이 이 문서의 핵심이다.  
> **관련 문서**: `DUMMY_DATA_PLANNING.md`, `GHG_DUMMY_DATA_STRATEGY.md`, `GHG_TAB_DESIGN_v2.md`, `JOURNEYMAP_GHG_v2.md`

---

## 1. 감사 대응 탭이 필요한 이유

### 1.1 IFRS S2 감사 요건

IFRS S2는 단순히 "배출량이 얼마냐"가 아니라 **"그 숫자가 어디서 왔고, 어떻게 산정됐으며, 누가 검증했는가"** 를 요구한다.  
감사 대응 탭은 이 요건들을 **한 화면에서 자동으로 체크하고 근거를 제시**하는 역할을 한다.

| IFRS S2 요구 항목 | 감사 대응 탭 충족 방법 |
|------------------|----------------------|
| 데이터 출처 명시 | `source_system` — 어느 시스템에서 왔는가 |
| 데이터 수집 시점 | `synced_at` — ON-PREM에서 언제 넘어왔는가 |
| 수동 조정 여부 | `synced_at` vs `updated_at` 시간 차이 자동 감지 |
| 수정 주체 | `updated_by` — 누가 수정했는가 |
| 배출계수 근거 | `applied_factor_id`, `applied_factor_value` 스냅샷 |
| 산정 방법론 | `calculation_method`, `calculation_formula` 저장·표시 |
| 검증 의견 | 요건별 충족 여부 자동 체크 + 근거 드릴다운 |

### 1.2 GHG 3탭과 감사 대응 탭의 관계

감사 대응 탭은 산정 입력·결과 탭이 쌓은 데이터를 집계·검증·표시한다.

```
[산정 입력 탭 — 6개]                        [감사 대응 탭에 공급]
전력·열·스팀 / 연료·차량 / 냉매              → source_system, synced_at, data_quality
폐기물 / 물류·출장·통근 / 원료·제품          → updated_at, updated_by, 수정전후값

[산정 결과 탭]                               [감사 대응 탭에 공급]
Scope 1/2/3 배출량 계산                      → applied_factor_id, applied_factor_value
버전 히스토리 저장                            → calculated_at, calculated_by, tCO₂e 버전별

                    ↓ 전체 집계

[감사 대응 탭 — 이 문서의 핵심]
  IFRS S2 요건 자동 체크
  수동 조정 이력 플래그
  배출계수 버전 불일치 경고
  데이터 품질 분포 차트
  계보 드릴다운 (숫자 → 원천 레코드 추적)
  증빙 패키지 5종 다운로드
```

## 2. 데이터 계보 컬럼 체계

### 2.1 산정 입력 단계 — 전체 파일 공통 컬럼

> `DUMMY_DATA_PLANNING.md` §6-1에 정의된 컬럼. 감사 대응 탭이 이 컬럼들을 읽어 요건 충족 여부를 판단한다.

| 컬럼명 | 타입 | 설명 | 감사 대응 탭 활용 |
|--------|------|------|-----------------|
| `source_system` | STRING | 원천 시스템 (EMS/ERP/EHS/SRM/HR/PLM/수동) | 데이터 출처 명시 요건 |
| `synced_at` | DATETIME | ON-PREM → 플랫폼 동기화 일시 | 수집 시점 증빙 |
| `created_at` | DATETIME | 플랫폼 최초 적재 일시 | 적재 기록 |
| `updated_at` | DATETIME | 마지막 수정 일시 | 수동 조정 감지 |
| `updated_by` | STRING | 수정한 사용자 ID | 수정 주체 추적 |

### 2.2 수동 조정 감지 로직

감사 대응 탭이 자동으로 수행하는 판단 로직.

```
if (updated_at - synced_at) > 24시간:
    → manual_adjustment_flag = true
    → 감사 대응 탭: "수동 조정 이력 있음 ⚠️" 표시
    → updated_by, updated_at, 수정 전후 값 로그 보존
else:
    → 감사 대응 탭: "정상 적재 ✅" 표시
```

> **임계값 근거**: 동기화 직후 24시간 이내 수정은 데이터 정제(정상),  
> 24시간 초과 수정은 의도적 수동 조정으로 간주하여 감사 플래그 처리.

### 2.3 산정 결과 단계 — 추가 저장되는 계보 컬럼

산정 완료 시 활동자료 레코드에 아래 컬럼이 추가 저장되며, 감사 대응 탭의 배출계수 근거·산정 방법론 요건에 활용된다.

| 컬럼명 | 타입 | 설명 | 예시값 |
|--------|------|------|--------|
| `applied_factor_id` | STRING | 적용 배출계수 ID (MDG 스냅샷) | EF-LNG-2023 |
| `applied_factor_value` | FLOAT | 적용 배출계수 값 (산정 시점 고정) | 2.15900 |
| `applied_factor_version` | STRING | 배출계수 버전·기준연도·출처 | 2023-환경부 |
| `applied_gwp_basis` | STRING | 적용 GWP 기준 | AR5 / AR6 |
| `calculation_method` | STRING | 산정 방법론 | 연료연소법 / spend-based / distance-based |
| `calculation_formula` | STRING | 산정 산식 (텍스트) | 82500 × 2.159 × 1.0 |
| `ghg_emission_tco2e` | FLOAT | 산정된 배출량 (tCO₂e) | 178.1 |
| `calculated_at` | DATETIME | 산정 실행 일시 | 2024-02-15 10:30:00 |
| `calculated_by` | STRING | 산정 실행 사용자 ID | user_001 |

> **핵심 원칙**: `applied_factor_value`는 산정 시점 값을 **스냅샷으로 고정** 저장.  
> MDG 배출계수가 이후 갱신되더라도 당시 산정 근거가 보존된다.

## 3. Upstream 탭 — 감사 대응 탭에 공급하는 데이터

### 3.1 산정 입력 탭 (6개) — 계보 확인 포인트

감사 대응 탭은 아래 포인트를 기준으로 각 입력 탭의 데이터를 자동 점검한다.

| 입력 탭 | 소스 파일 | 감사 대응 탭 점검 포인트 |
|---------|-----------|------------------------|
| 전력·열·스팀 | `EMS_ENERGY_USAGE.csv` | `source_system = EMS` 여부, `synced_at` 존재, `data_quality` M1/M2 비율 |
| 연료·차량 | `ERP_FUEL_PURCHASE.csv` | `source_system = ERP` 여부, `consumption_amount` vs `purchase_amount` 재고 정합성 |
| 냉매 | `EHS_REFRIGERANT.csv` | `source_system = EHS` 여부, `gwp_factor` MDG 현행값 일치 여부 |
| 폐기물 | `EMS_WASTE.csv` | `disposal_method` 기반 Scope 자동 분류 정확도 |
| 물류·출장·통근 | `SRM_LOGISTICS.csv`, `HR_COMMUTE_BUSINESS_TRAVEL.csv` | 파일별 `source_system` 분리 (SRM / HR) |
| 원료·제품 | `SRM_SUPPLIER_ESG.csv`, `PLM_PRODUCT_CARBON.csv` | `supplier_emission_tco2e` 직접 보고 vs 추정 비율, `data_quality` E2 비중 |

**그리드 행별 계보 표시 (입력 탭 화면)**:

```
EMS-E-2024-001 | 서울본사 | 1월 | 전력 | 125,000 kWh
  └─ 출처: EMS  |  동기화: 2024-02-05 08:55  |  품질: M1  |  수정이력: 없음 ✅

EMS-E-2024-002 | 수원공장 | 1월 | 전력 | 280,000 kWh
  └─ 출처: EMS  |  동기화: 2024-02-05 08:55  |  품질: M1  |  수정이력: 있음 ⚠️
     → 수정자: user_002  |  수정일시: 2024-02-10 14:32  |  수정전: 275,000 kWh
```

---

### 3.2 산정 결과 탭 — 감사 대응 탭에 공급하는 정보

**산정 결과 표시 구조 (결과 탭 화면)**:

```
Scope 1  합계: 1,234 tCO₂e

활동자료            배출계수                     산식                    배출량
──────────────────────────────────────────────────────────────────────────────
LNG 82,500 Nm³     2.159 tCO₂e/TJ             82,500 × 2.159 × 1.0    178.1 tCO₂e
[연료·차량 탭↗]    EF-LNG-2023 [환경부·AR5↗]

경유 12,500 L      2.581 tCO₂e/TJ             12,500 × 2.581 × 1.0     32.3 tCO₂e
[연료·차량 탭↗]    EF-경유-2023 [환경부·AR5↗]
```

- **[연료·차량 탭↗]**: 클릭 시 해당 입력 탭 원천 레코드로 드릴다운
- **[환경부·AR5↗]**: 클릭 시 MDG 배출계수 상세 표시
- 배출계수 값은 산정 시점 스냅샷(`applied_factor_value`) 기준 — MDG 현행값과 다를 수 있음

**산정 버전 히스토리 (결과 탭 화면)**:

```
v3  2024-02-15 10:30  user_001  Scope1: 1,234  Scope2: 567  Scope3: 3,890  [현재]
v2  2024-02-10 09:15  user_002  Scope1: 1,198  Scope2: 567  Scope3: 3,890  ← 수원공장 수동 수정 반영
v1  2024-02-05 11:00  user_001  Scope1: 1,198  Scope2: 545  Scope3: 3,890  [최초 산정]
```

> 버전별 산정값·변경 사유가 모두 감사 대응 탭 "산정 버전 이력" 항목에 연동된다.

## 5. 감사 대응 탭이 감지하는 시나리오

더미데이터에 **의도적으로 포함된 시나리오**와 감사 대응 탭이 이를 표시하는 방식.

### 5.1 정상 케이스 (전체의 약 90%)

```
synced_at:  2024-02-05 08:55
updated_at: 2024-02-05 09:00  ← 5분 차이 = 정상 적재
updated_by: system
→ 감사 대응 탭: ✅ 정상 적재, 수동 조정 없음
```

### 5.2 수동 조정 케이스 (3~5건 의도적 포함)

```
레코드: ERP-F-2024-013 (수원공장 1월 LNG)
synced_at:  2024-02-05 07:30
updated_at: 2024-02-10 14:32  ← 5일 후 수동 수정 (24시간 초과)
updated_by: user_002
수정전값:   275,000 Nm³
수정후값:   280,000 Nm³
수정사유:   "ERP 데이터 오류 — 계량기 오독, 현장 확인 후 정정"
→ 감사 대응 탭: ⚠️ 수동 조정 이력 1건 — 사유·수정자·전후값 드릴다운 제공
```

### 5.3 데이터 품질 혼재 케이스

```
Scope 2 전력:   data_quality = M1 → ✅ 고신뢰
Scope 3 통근:   data_quality = E2 → ⚠️ 추정 근거 명시 권장
Scope 3 Cat.1:  data_quality = E2 → ⚠️ 공급업체 직접 보고 권장
→ 감사 대응 탭: 품질 분포 차트 + 등급별 개선 권장사항 자동 생성
```

### 5.4 배출계수 버전 불일치 케이스 (1~2건 의도적 포함)

```
레코드: EMS-E-2024-구버전 (일부 사업장)
applied_factor_id:    EF-전력-2022  ← MDG 현행(EF-전력-2023)과 불일치
applied_factor_value: 0.4560        ← 현행 0.4781과 차이
→ 감사 대응 탭: ⚠️ 구버전 배출계수 적용 감지 — 재산정 권장 알림
```
