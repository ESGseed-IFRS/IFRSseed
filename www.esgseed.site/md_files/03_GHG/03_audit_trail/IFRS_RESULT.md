# 📄 [화면 B] 산출 근거 및 증적 상세 — IFRS 감사 대응 전략

> **설계 목적**: 감사인이 시스템 외부의 물리적 서류를 확인하지 않고도 시스템 내에서 데이터의 **완전성(Completeness), 정확성(Accuracy), 유효성(Validity)**을 합리적으로 확신(Reasonable Assurance)할 수 있도록 설계함.

---

## 0. 상단 탭 3개 (산정 결과 | IFRS 감사대응 | 리포트 생성)

**순서 고정**: 좌→우 **산정 결과 | IFRS 감사대응 | 리포트 생성**. 탭은 정확히 3개.

### 0-1. 탭 구성 (고정)

| 순서 | 탭 라벨 | 표시 내용 |
|------|---------|-----------|
| **1** | **산정 결과** | 배출량 산정 결과(최종 tCO₂e, 신뢰도, Scope 1/2/3) + 적용 배출계수·산식 요약. **접기/펼치기 없이** 한 화면 고정 표시. |
| **2** | **IFRS 감사대응** | 감사·검증용 전용 뷰. §2~§6 내용을 **섹션으로 한 화면에 고정** 배치. **접기/펼치기 없음**. |
| **3** | **리포트 생성** | 증적·보고서 생성(기존 Step5Report) |

### 0-2. UI 요구

- **버튼 수**: 상단 탭은 **정확히 3개**. 2개로 두지 않음.
- **버튼 형태**: SIDBAR_CONNECT 0-2 준수 — **직사각형, 날카로운 모서리(rounded 없음)**. 둥근 모서리 사용 금지.
- **활성/비활성**: 활성 탭 = primary 녹색 배경+흰 글씨, 비활성 = 흰/연회색 배경+진한 글씨.

## 0B. 화면 구조 전략 (배출량 산정 결과 위치)

**현재 이슈**: 배출량 산정 결과가 페이지 **밑**에 있어서 사용자·감사인이 최종 수치를 먼저 인지하기 어렵다.

### 0B-1. 화면 배치 원칙

| 원칙 | 설명 |
|------|------|
| **결과 우선** | "배출량 산정 결과"는 해당 뷰의 **최상단** 또는 **스크롤 시 처음 보이는 영역**에 배치한다. |
| **한 눈에 보이기** | 최종 산정 배출량(tCO₂e), 데이터 신뢰도, Scope 1/2/3, 적용 계수·산식 요약을 **접기/펼치기 없이** 한 화면에 고정 노출. |

### 0B-2. 권장 섹션 순서 (위 → 아래)

| 순서 | 섹션 | 내용 | 비고 |
|------|------|------|------|
| **1** | **배출량 산정 결과** | 제목 "배출량 산정 결과" + 최종 산정 배출량(큰 숫자) + 데이터 신뢰도 + Scope 1/2/3 | **밑이 아닌 맨 위** |
| **2** | **적용 배출계수·산식 안내** | §0C 참고. 결과 블록 바로 아래 또는 같은 폴드에 고정 표시. **접기/펼치기 없음**. | §0C |

**구현 시 필수**: "배출량 산정 결과" 제목과 최종 tCO₂e·신뢰도·Scope 요약이 **페이지 상단(또는 결과 탭 진입 시 첫 폴드)** 에 오도록 레이아웃·DOM 순서를 조정한다. 결과가 스크롤 밑에 가라앉지 않도록 한다.

## 0C. 두 번째 영역 (방법론·적용 계수 블록) 재구성 및 UI 수정

**대상**: "지능형 방법론 자동 매핑 완료" 제목과, 그 아래 배출계수(EF)·GWP 카드·어두운 알고리즘 박스·푸터 문구가 들어 있는 **카드 전체**(두 번째 이미지 부분).

### 0C-1. 배치: 밑에 뜨지 않게

| 요구 | 설명 |
|------|------|
| **금지** | 이 블록을 **페이지 맨 밑**에 두지 않는다. 스크롤 끝에서만 보이게 하지 않는다. |
| **권장** | 결과 블록 **바로 아래** 또는 같은 폴드에 적용 배출계수 출처·GWP·산식 요약을 한 블록으로 고정 표시. 상세 내용은 **IFRS 감사대응** 탭에서 섹션으로 표시. |
| **결과** | 방법론·계수 정보가 **화면 하단에만** 떠 있는 구조를 제거한다. |

### 0C-2. 메시지 명확화: 뭘 말하는지 알 수 있게

| 현재 문제 | 수정 방향 |
|-----------|-----------|
| "지능형 방법론 자동 매핑 완료" | **사용 금지**. 목적이 불명확하고 마케팅·AI 톤으로 들림. |
| 대체 문구 예시 | **"본 산정에 적용된 배출계수 및 산식"** 또는 **"적용 배출계수·GWP (감사·검증 참고)"**. 한 문장으로 **이 블록이 무엇을 보여주는지**(감사·검증용 적용 기준 안내)를 명시. |
| "입력된 활동자료를 바탕으로 최적의..." | **사용 금지**. 모호하고 GPT 느낌. |
| 대체 문구 예시 | **"아래 계수·산식은 현재 산정에 적용된 기준입니다. 감사·검증 시 참고용으로 활용하세요."** 등 **역할을 명시**하는 중립 문장. |

### 0C-3. GPT 느낌 제거 — UI·문체 수정

| 항목 | 현재 문제 | 수정 요구 |
|------|-----------|-----------|
| **제목·설명** | "지능형", "자동 매핑 완료", "최적의 ... 연결했습니다" 등 과장·AI 스타일 문구 | **전부 제거**. 중립적·전문 용어만 사용. 예: "적용 배출계수", "적용 산식", "감사 참고용". |
| **어두운 코드 박스** | `// 산정 알고리즘 (Scope 1)` 형태의 코드 스타일 박스(흑/남색 배경+모노스페이스) | **제거 또는 대체**. 코드 블록 느낌이 나지 않도록, **일반 본문·표**로 산식 표시. 예: "Scope 1: CO₂ 배출량(tCO₂e) = 활동자료(사용량) × 배출계수(EF)" 를 카드 행 또는 표 한 행으로 표기. |
| **아이콘·장식** | 녹색 원형 아이콘(연결/완료 느낌) 등 과한 비주얼 | **축소 또는 제거**. 정보가 주인공이 되도록, 불필요한 장식은 줄인다. |
| **푸터 문구** | "활동자료 입력 단계에서 추가한 연료/전력에 따라 위 계수가 자동 적용됩니다" | **단순화**. 예: "적용 연도: 2024 KR Emission Factors. 연료·전력 종류에 따라 위 계수가 적용됩니다." — "자동" 등 AI 연상 표현 최소화. |

### 0C-4. 구현 체크리스트 (두 번째 영역)

- [ ] 방법론·적용 계수 블록이 **페이지 맨 밑**에만 노출되지 않도록 배치(결과 블록 아래 또는 같은 폴드).
- [ ] "지능형 방법론 자동 매핑 완료" 문구 **삭제** 후, "적용 배출계수·산식" 등 **목적이 드러나는 제목**으로 교체.
- [ ] 설명 문장을 **역할 명시형·중립 문체**로 변경 (GPT/마케팅 톤 제거).
- [ ] 어두운 **코드 스타일 박스** 제거 또는 **표·일반 본문** 형태로 산식 표시.
- [ ] 아이콘·장식 최소화, **정보 위주** 레이아웃으로 정리.

## 1. IFRS 감사대응 탭 — 표시 내용 (접기/펼치기 없음)

**산정 결과** 탭에는 버튼·펼침 패널 없음. **IFRS 감사대응** 탭에서 §2~§6 내용을 **섹션으로 한 화면에 고정** 표시(접기/펼치기 없음).

| 위치 | 설명 |
|------|------|
| **결과 블록 직하단** | "배출량 산정 결과" 섹션(최종 tCO₂e · 데이터 신뢰도 · Scope 1/2/3) **바로 아래** 한 줄에 3개 버튼 가로 배치 |
| **금지** | 버튼 2개만 두기, 버튼을 결과/방법론 영역 밖(상단 탭 등)에 두기, 결과 블록을 버튼보다 아래로 두기 |

### 1-2. 버튼 3개 (고정)

| 순서 | 버튼 라벨 | 클릭 시 표시 내용 | 본 문서 섹션 매핑 |
|------|-----------|------------------|-------------------|
| **1** | **산출 근거 상세** | 산출 공식, 적용된 배출계수·GWP·순발열량, 불확실성 등급 등 방법론·투명성 내용 | §1 산출 공식 및 방법론 공시 |
| **2** | **데이터 계보·감사 추적** | 수집 경로, Raw Snapshot, 변경 이력, 수정 사유, 승인 로그 등 계보·거버넌스 | §2 데이터 계보 및 거버넌스 로그 |
| **3** | **IFRS 대응·증빙** | Side-by-Side 증빙 비교, OCR 검증, 데이터 품질 등급, IFRS S2 공시 특화(기준 연도 재산정, Location/Market Based, 중요성) | §3 디지털 증빙 및 품질 매칭, §4 IFRS S2 공시 특화 |

### 1-3. UI 동작

- **표시 방식**: 버튼 클릭 시 해당 항목의 **상세 패널을 같은 페이지에서 펼치거나** 모달/드로어로 열어 §2~§6 내용을 텍스트/카드 형태로 표시.
- **토글**: 같은 버튼 재클릭 시 패널 닫기. 다른 버튼 클릭 시 기존 패널 닫고 해당 패널만 열기.
- **버튼 스타일**: SIDBAR_CONNECT 0-2 준수 — 직사각형, **날카로운 모서리(rounded 없음)**. 활성(선택된 버튼)은 primary 녹색, 비활성은 흰색/연회색 배경.

### 1-4. 구현 체크리스트

- [ ] **배출량 산정 결과**가 화면 상단(또는 결과 뷰 첫 폴드)에 노출되며, **밑**에 있지 않음
- [ ] 결과 블록 직하단에 **정확히 3개** 버튼만 노출
- [ ] 버튼 1 클릭 → 산출 공식, 계수 세부, 불확실성(§2) 패널 표시
- [ ] 버튼 2 클릭 → 데이터 계보, 감사 추적, 승인 로그(§3) 패널 표시
- [ ] 버튼 3 클릭 → 디지털 증빙, IFRS S2 공시, 수정 절차(§4, §5, §6) 패널 표시

## 2. 산출 공식 및 방법론 공시 (Methodology & Transparency)

**IFRS 감사대응 탭 — 산출 근거 섹션.** IFRS S2는 산정 방법론의 구체적 설명을 요구하며, 감사인은 사용된 계수와 산식의 적절성을 검증한다.

### 2-1. 산출 공식의 가시화

| 항목 | 상세 | 화면 표시 요구 |
|------|------|----------------|
| **공식** | Total tCO₂e = Σ (Activity Data × NCV × Emission Factor × GWP) | 공식을 코드 블록 또는 수식 형태로 명확히 표시 |
| **Scope별 산식** | Scope 1: 사용량×배출계수, Scope 2 전력: 전력 사용량×그리드 배출계수 등 | Scope 1 / Scope 2 전력·열 구분하여 각각 산식 한 줄씩 표시 |
| **계수 세부** | 순발열량(NCV): 에너지원별 국가 표준값(Tier 2) 적용 여부 명시 | EF·GWP와 별도 행 또는 카드로 "NCV 적용 기준" 표시 |
| **배출계수(EF)** | 국가 고유 배출계수(National EF) vs IPCC 기본 계수 구분, 버전·출처 | 카드 1: "배출계수 (Emission Factor)" — 출처, 버전, 적용 연도 |
| **GWP** | IPCC AR5 / AR6 중 적용 버전, 100-year Horizon 명시 | 카드 2: "지구온난화지수 (GWP)" — 버전, Horizon |

### 2-2. 불확실성(Uncertainty) 관리

| 항목 | 상세 | 화면 표시 요구 |
|------|------|----------------|
| **불확실성 등급** | 활동자료 수집 장치(계량기) 오차 범위 또는 데이터 품질 등급(Tier) | "데이터 신뢰도"와 연계해 High(Level 1)/Medium(Level 2)/Low 등 텍스트·색상으로 표시 |
| **산정 제외(Exclusion)** | 특정 설비·지점 데이터 누락 시 사유 및 영향도(중요성 평가) | 별도 문단 또는 리스트: "산정 제외 항목", "사유", "영향도" |

### 2-3. 구현 시 데이터 소스

- 배출계수·GWP·산식 문구: 스토어/상수(MethodologyDetailPanel 등)에서 EF Source, Version, GWP, Formula 필드 사용.
- 데이터 신뢰도: 활동자료의 dataQuality(실측/추정 등) 집계 결과와 매핑.

## 3. 데이터 계보 및 거버넌스 로그 (Lineage & Governance)

**IFRS 감사대응 탭 — 데이터 계보·감사 추적 섹션.** 내부 통제(Internal Control) 체계 입증을 위한 '탄생부터 현재까지' 기록.

### 3-1. 최초 수집 정보 (Origin & Lineage)

| 항목 | 상세 | 화면 표시 요구 |
|------|------|----------------|
| **수집 경로** | EMS API 연동(Connector ID), 엑셀 업로드, 수동 입력 등 + 적재 시각 | 리스트 또는 테이블: "경로", "Connector ID", "일시" |
| **Raw Snapshot** | 시스템에 처음 적재된 수정 전 원본값 보존 (감사 시 대조용) | "원본값 보존 여부", "최초 적재 시점" 등 안내 문구 또는 링크 |

### 3-2. 감사 추적(Audit Trail) 및 승인 로그

| 항목 | 상세 | 화면 표시 요구 |
|------|------|----------------|
| **변경 이력** | 누가(User ID), 언제(Timestamp), 무엇을(From → To) 변경 | 이력 테이블: 사용자, 일시, 필드명, 변경 전·후 값 |
| **수정 사유** | 사유 코드 선택 필수 (계량기 오동작, 고지서 오기재 등) | 이력 행에 "사유" 컬럼 또는 모달에서 사유 표시 |
| **승인 체계** | 담당자 입력 → 검토자 확인 → 책임자 승인, 타임스탬프·디지털 서명 로그 | 단계별 타임스탬프 리스트 또는 승인 상태 뱃지 |

### 3-3. 구현 시 유의

- 백엔드/스토어에 lineage·audit 로그가 있으면 API로 조회 후 테이블로 렌더.
- 없으면 "수집 경로·변경 이력·승인 로그는 시스템 연동 후 표시됩니다" 플레이스홀더 문구와 필드 정의만 명시.

## 4. 디지털 증빙 및 품질 매칭 (Digital Evidence & QA)

**IFRS 감사대응 탭 — 증빙·품질 섹션.** 감사인에게 제공할 실질적 '증거물' 기능.

### 4-1. Side-by-Side 비교 뷰 (Digital Evidence)

| 항목 | 상세 | 화면 표시 요구 |
|------|------|----------------|
| **구성** | 좌측: 업로드된 고지서/영수증(PDF, JPG), 우측: 시스템 입력 데이터 | 2단 레이아웃 또는 탭으로 "증빙 파일" \| "시스템 입력" |
| **증빙 연결** | 특정 월·셀과 증빙 파일 1:1 매핑, 클릭 시 해당 파일 표시 | 월별 또는 셀별 "증빙 보기" 링크/버튼 → 미리보기 또는 다운로드 |

### 4-2. OCR 검증 및 품질 지표

| 항목 | 상세 | 화면 표시 요구 |
|------|------|----------------|
| **OCR 결과** | 이미지 내 텍스트 인식값과 시스템 입력값 일치 여부, 매칭율(%) | "OCR 매칭율: 98%" 등 지표 한 줄 또는 카드 |
| **데이터 품질 등급** | Primary Data (High) / Secondary Data (Low) 구분 | 셀/행 단위 또는 요약 영역에 DQ 뱃지·문구 |

### 4-3. 구현 시 유의

- 증빙 파일·OCR은 첨부/연동 기능이 있으면 해당 API와 연결. 없으면 "증빙 업로드 및 OCR 연동 후 표시" 플레이스홀더와 요구 필드 정의.

## 5. IFRS S2 공시 특화 기능 (Compliance Layer)

**IFRS 감사대응 탭** 내에서 함께 표시할 공시 특화 항목.

| 항목 | 상세 내용 | 시스템 구현 로직 |
|------|-----------|------------------|
| **기준 연도 재산정** | 조직 경계 변경(합병, 분할 등) 시 기준 연도 배출량 수정 | 과거 시점 데이터 Versioning 및 소급 적용(Recalculation) 엔진 |
| **Location/Market Based** | Scope 2 전력 배출량 이중 공시 요구 충족 | 동일 활동자료에 대해 Location/Market 배출계수 각각 적용, 별도 저장·표시 |
| **중요성(Materiality)** | 공시 대상 포함/제외 항목의 판단 근거 | 산정 범위 설정 단계에서 입력한 판단 근거(Text)를 상세 화면에 연결 표시 |

## 6. [핵심 워크플로우] 수정 발생 시 강제 절차

**IFRS 감사대응 탭** 내 "수정 시 절차" 섹션. IFRS 대응을 위해 수정 시 다음 워크플로우를 강제한다.

| 단계 | 내용 | 구현 요구 |
|------|------|-----------|
| **1** | 수정 모드 진입 — 기존 데이터는 읽기 전용 보존 | 수정 진입 시 스냅샷 또는 버전 복사, 기존 값은 감사용으로 조회 가능 |
| **2** | 수정 사유 선택 — [오류 정정 / 방법론 변경 / 조직 경계 변경 / 기타] 중 선택 필수 | 셀/행 수정 저장 전 사유 선택 드롭다운 또는 라디오 필수 |
| **3** | 증빙 재업로드 — 수정된 값의 근거(고지서·내부 품의서) 필수 첨부 | 첨부 없이 저장 불가 또는 경고 + "나중에 제출" 플로우 정의 |
| **4** | 영향도 기록 — 전체 배출량 변화량(tCO₂e) 자동 계산·기록 | 수정 전·후 총 tCO₂e 차이 계산 후 audit 로그에 저장 |

## 7. 문서·화면 매핑 요약

| 화면 요소 | 문서 섹션 | 비고 |
|-----------|-----------|------|
| **상단 탭 3개** | §0 | **산정 결과 \| IFRS 감사대응 \| 리포트 생성** (순서 고정). 직사각형·날카로운 모서리. |
| 배출량 산정 결과(상단) | §0B | 결과가 **밑**이 아닌 **위**에 배치. 접기/펼치기 없이 고정 표시. |
| **방법론·적용 계수 블록** | §0C | **밑에 뜨지 않게** 배치. GPT 느낌 제거, 목적 명확한 제목·표/본문 위주. |
| **IFRS 감사대응 탭** | §1, §2~§6 | §2~§6 내용을 **섹션으로 한 화면 고정** 표시. **접기/펼치기 없음**. |

이 문서에 맞춰 **탭 순서(산정 결과 \| IFRS 감사대응 \| 리포트 생성)** 를 적용하고, **접기/펼치기 없이** 산정 결과 탭에는 결과+적용 계수만, IFRS 감사대응 탭에는 §2~§6를 섹션으로 고정 표시하면 된다.
