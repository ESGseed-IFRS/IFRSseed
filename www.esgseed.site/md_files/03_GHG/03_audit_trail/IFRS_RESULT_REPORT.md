# 리포트 생성 vs 검증의견서·서류 작성 — 동일 여부 및 변경 전략

## 0. 채택 전략 (결정 사항)

**다운로드 내용이 같다면**, 아래 두 가지로 변경한다.

1. **검증의견서·서류 작성** → **리포트 생성** 탭으로 옮긴다.  
   (6개 서류 설명 + PDF/Excel/일괄 다운로드는 모두 리포트 생성 탭에서만 제공)
2. **IFRS 감사대응** 탭에서 **서브 탭을 제거**한다.  
   (“요건 반영 확인” | “검증의견서·서류 작성” 구분 없이, “요건 반영 확인” 콘텐츠만 단일 화면으로 노출)

이 문서는 “같은지 확인 → 같으면 이렇게 변경”하는 전략을 정리한 것이다.

---

## 1. 다운로드 파일 동일 여부 (전략 전제)

### 결론: **동일한 내용** → 위 전략대로 진행한다.

| 구분 | IFRS 감사대응 → 검증의견서·서류 작성 | 리포트 생성 탭 |
|------|--------------------------------------|----------------|
| **데이터 소스** | `buildEvidencePayload(...)` | 동일 |
| **PDF** | `EvidencePdfDocument` | 동일 |
| **Excel** | `downloadEvidenceExcel(payload, ...)` | 동일 |
| **파일명 접두어** | `ghg-verification-support-*` | `ghg-evidence-*` |
| **추가 옵션** | PDF+Excel 일괄 다운로드 | JSON 다운로드 |

- PDF·Excel **내용**은 동일(산정설정, Scope1·2·3, 데이터 품질, 영수증 목록 등).
- 따라서 **같은 증적 자료**를 두 곳에서 주는 것이므로, **한 곳(리포트 생성)** 으로 모으고 **서브 탭은 제거**하는 방향이 맞다.

---

## 2. 변경 전략 요약

| 구분 | 변경 내용 |
|------|-----------|
| **리포트 생성 탭** | 검증의견서·서류 작성 **전체**를 이전한다. (6개 서류 설명 카드 + 안내 문구 + PDF/Excel/일괄 다운로드). 기존 Evidence Report 완료 메시지·PDF/Excel/JSON 유지. |
| **IFRS 감사대응 탭** | **서브 탭 제거.** “요건 반영 확인”만 남기고, “검증의견서·서류 작성” 서브 탭·UI·다운로드 로직 전부 제거. |

---

## 3. 구체 변경 사항 (이 전략에 따른 작업)

### 3-1. 리포트 생성 탭(Step5Report)에 넣을 것

- **검증의견서 부속 서류 설명**
  - 안내 문구: “아래 서류는 GHG Protocol·KMR 검증의견서 제출 시 참고용으로 활용할 수 있습니다. 검증기관별 추가 요청이 있을 수 있으니 확인 후 제출하세요.”
  - “PDF·Excel 일괄 다운로드 시 아래 항목이 포함됩니다” + **6개 카드**: 검증 범위 요약, 배출량 집계표, 데이터 품질 통계, 적용 배출계수·산식 요약, 활동자료 요약, 증빙 목록.
- **다운로드**
  - 기존: PDF Report, Excel Details, (선택) JSON.
  - 추가: **검증의견서 부속 서류 일괄 다운로드** (PDF + Excel 동시).  
  - 6개 서류 설명은 현재 IFRSAuditView의 `VERIFICATION_DOCS`와 동일 내용을 Step5Report 또는 공용 상수로 이전.

### 3-2. IFRS 감사대응(IFRSAuditView)에서 제거·변경할 것

- **서브 탭 제거**
  - “요건 반영 확인” | “검증의견서·서류 작성” 탭 UI 삭제.
  - `subTab === 'documents'` 블록 전체 삭제 (6개 카드, PDF/Excel/일괄 다운로드 버튼, 관련 설명).
- **다운로드 관련 제거**
  - `handleDownload`, `handleDownloadAll`, `downloadLoading`, `VERIFICATION_DOCS`, `FILE_PREFIX_VERIFICATION` 등 검증의견서·서류 작성 전용 코드 제거.
- **결과**
  - IFRS 감사대응 탭에는 **요건 반영 확인 테이블만** 단일 화면으로 표시.

### 3-3. (선택) IFRS 감사대응에 안내만 추가

- “검증의견서 부속 서류를 다운로드하려면 **리포트 생성** 탭을 이용하세요.” 문구 + 리포트 생성 탭으로 이동하는 버튼/링크.

### 3-4. 파일별 정리

| 파일 | 작업 |
|------|------|
| `Step5Report.tsx` | 6개 카드·안내 문구·일괄 다운로드 버튼 추가. 기존 PDF/Excel/JSON 유지. |
| `IFRSAuditView.tsx` | 서브 탭 제거, `subTab === 'documents'` 및 검증의견서 다운로드 로직·상수 제거. 요건 반영 확인만 렌더. |
| `evidenceReport.ts`, `EvidencePdfDocument.tsx` | 변경 없음. |

### 3-5. 작업 순서

1. Step5Report에 6개 카드·안내·일괄 다운로드 추가.
2. IFRSAuditView에서 서브 탭 + 검증의견서·서류 작성 UI·로직 제거.
3. (선택) IFRSAuditView에 “리포트 생성 탭에서 다운로드” 안내 추가.
4. 리포트 생성 탭에서 PDF/Excel/일괄/JSON 동작, IFRS 감사대응에 요건 반영 확인만 보이는지 테스트.

---

## 4. 최종 확인

- **같다** → **검증의견서·서류 작성은 리포트 생성으로 옮기고, IFRS 감사대응에서는 서브 탭 제거**하는 전략으로 진행한다.
- 이 md는 그에 따른 **변경 전략**을 정리한 문서이다.
