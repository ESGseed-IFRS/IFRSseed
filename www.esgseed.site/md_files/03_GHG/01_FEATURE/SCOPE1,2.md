# 온실가스 산정 플랫폼 고도화 전략 (기존 UI 유지 및 로직 확장)

## 1. 프로젝트 개요 및 원칙
- **목적:** 국내 **배출권거래제(K-ETS)**와 글로벌 **CDP/RE100** 공시를 단일 플랫폼에서 동시 대응 (Dual Compliance).
- **UI 유지 원칙:** 현재 구현된 카드형 레이아웃, 드롭다운 메뉴, 그린/화이트 톤의 디자인 시스템을 최대한 유지한다.
- **사용자 경험:** 유저가 데이터를 한 번만 입력하면, 시스템이 내부적으로 두 가지 기준에 맞춰 결과를 산출한다.

## 2. [산정 설정] UI 및 로직 상세 가이드

### 2.1 조직 경계 (Organizational Boundary)
- **기존 UI:** 드롭다운 선택 방식.
- **변경 로직:** - 상단에 '보고 목적(K-ETS / 글로벌)' 체크박스 추가(UI 최소 변경).
    - **K-ETS 활성화 시:** 드롭다운을 '운영통제법'으로 강제 고정하고 하단에 "국내 법령에 따라 운영통제법이 적용됩니다" 안내 문구 노출.
    - **글로벌 전용 시:** 기존처럼 재무통제, 지분비율 등 선택 가능.

### 2.2 운영 경계 (Operational Boundary)
- **Scope 1 (직접 배출):**
    - **UI 변경:** 드롭다운 대신 **'멀티 셀렉트(Multi-select)'** 또는 **'체크박스'** 형태로 변경.
    - **이유:** 제조업 실무상 '연료 연소'와 '공정 배출'이 동시에 발생하는 경우가 많아 중복 선택이 필수적임.
- **Scope 2 (간접 배출):**
    - **UI 유지:** 드롭다운 선택지는 유지하되, **'위치 기반 & 시장 기반 동시 산정'** 옵션을 기본값(Default)으로 설정.
    - **로직 확장:** 시장 기반(Market-based) 산정을 위해, 전기 사용량 입력 단계에서 '재생에너지 이행 실적(REC, PPA 등)'을 입력할 수 있는 필드가 조건부로 나타나도록 설계.

## 3. 데이터 및 백엔드 설계

### 3.1 배출계수(EF) 관리
- **연도별 매칭:** '보고 연도' 선택 시, 해당 연도의 환경부 국가 배출계수 DB가 자동으로 백엔드에 매칭되도록 구성.
- **DB 이원화:**
    - **K-ETS용:** 환경부 공시 계수 적용.
    - **글로벌용:** IEA(국제에너지기구) 전력 배출계수 라이브러리 연동.

### 3.2 Dual-Calculation 엔진
- **Input:** 사용자가 입력한 활동자료(전기 사용량, 연료 사용량 등)는 하나로 관리.
- **Output:** - **K-ETS 탭:** 국내 지침에 따른 산정 결과 명세서 출력.
    - **Global 탭:** 시장 기반/위치 기반 결과가 포함된 CDP 대응 대시보드 출력.

## 4. 제외 사항
- **Scope 3 (공급망 배출):** 이번 고도화 범위에서 완전히 제외함.


## 3. 데이터 입력 및 DB 설계 전략 (상세)

### 3.1 배출계수(EF) 라이브러리 고도화
- **연도별 자동 매칭 및 API 연동:**
    - 유저가 선택한 '보고 연도(Reporting Year)'에 맞춰 해당 연도의 계수를 자동 로드.
    - **국내용:** 환경부 NGMS(국가온실가스종합관리시스템) 공시 계수 DB 구축. 가능하다면 NGMS 공개 API와 연동하여 최신 계수(2025-12 업데이트본 등) 상시 동기화.
    - **해외용:** IEA(국제에너지기구) 전력 배출계수 및 주요 국가별(US eGRID 등) 최신 라이브러리 구축.
- **DB 스키마 구성:** `[연도] - [국가/지역] - [에너지원] - [배출계수 값] - [출처(NGMS/IEA)]` 형태의 테이블 구조 설계.

### 3.2 재생에너지 이행 실적 및 로직 (Scope 2 시장 기반)
- **이행 수단별 분리 입력:** 시장 기반 산정을 위해 다음 항목을 각각 독립적인 필드로 구성.
    1. **녹색프리미엄:** K-ETS 명세서에는 반영되지 않으나(감축 불인정), RE100/시장 기반 산정 시에는 0kg 배출로 처리.
    2. **REC(재생에너지 공급인증서) 구매:** K-ETS 및 RE100 모두 감축 실적으로 반영 가능하도록 설계.
    3. **PPA (제3자/직접):** 장기 계약 데이터 입력 및 연도별 발전량 매칭.
    4. **자가발전 (On-site):** 소비량 중 자가 소비분에 대한 배출량 차감 로직.
- **EAC(에너지 속성 인증서) 규제 대응:** 2027년 강화되는 RE100 가이드라인(EAC 취소 의무 등)에 대비하여, 각 이행 수단별 '인증서 번호' 및 '유효기간' 관리 필드를 선제적으로 추가.

## 4. Dual-Calculation 산정 로직
- **K-ETS 로직:** 국내 법령에 따라 [위치 기반 전기 사용량 - 허용된 REC 실적]으로 최종 배출량 계산.
- **Global 로직:** [위치 기반 결과]와 [시장 기반 결과(모든 이행 수단 차감)]를 동시에 산출하여 나란히 비교 출력.

## 5. 제외 항목
- Scope 3(공급망 배출)은 이번 고도화 범위에서 제외함.
