# Scope 1·2 상세 전략 (사업장·에너지원·시장/위치·수동입력)

실무자 관점에서 따라갈 수 있는 GHG 산정 플랫폼을 위한 **단계별 위저드**, **사업장 선택 방식**, **에너지원·결과 반영**, **Scope 2 시장/위치 기반 복원**, **수동입력 직관성** 개선 전략을 정리합니다.

---

## 0. 단계별 위저드(1 → 2 → 3): 한 번에 한 단계만 표시

### 0-1. 현재 문제

- **모든 단계를 하나의 페이지에** 보여 주고 있음: 조건 선택·조회 안내, 저장된 산정값, 활동자료 입력, 결과 요약이 **한 화면에 스크롤로** 모두 노출됨.
- 사용자가 “지금 뭘 해야 하는지”, “다음에 뭘 하면 되는지”를 **순서대로** 따라가기 어렵고, 수동입력 칸도 여러 블록 사이에 묻혀 직관이 떨어짐.

### 0-2. 목표 UX

- **1 → 2 → 3 순서**를 정하고, **다음 단계** 버튼을 누르면 **그다음 단계로만** 화면이 전환되게 한다.
- **한 번에 한 단계만** 보이도록 하여, “지금은 1단계만 하면 된다”가 명확해지게 한다.

### 0-3. 단계 정의 (Scope 1·Scope 2 공통 개념)

| 단계 | 제목(화면 표시) | 화면에 보이는 내용 | 버튼 |
|------|-----------------|-------------------|------|
| **1** | 조건 선택·조회 | 연도·사업장(드롭다운)·에너지원 선택. **조회** 버튼. “선택한 조건으로 데이터를 조회합니다.” 안내. (EMS·엑셀으로 불러올 수도 있다는 링크/버튼 가능.) | **[다음 단계]** |
| **2** | 활동자료 입력 | 해당 Scope에 맞는 **입력 그리드만** 표시(고정연소/이동연소 또는 전력/열). 월·사업장·에너지원·**사용량** 입력, 행 추가·삭제. “여기에 사용량을 입력하세요.” 안내. | **[이전 단계]** **[다음 단계]** |
| **3** | 산정 결과·저장 | **배출량 산정 결과**(고정연소·이동연소 또는 전력·열 tCO₂e, 총계). **결과 저장** 버튼. **저장된 산정값** 목록. | **[이전 단계]** (결과 저장 / 전체 삭제 등) |

- **진입 시**: 항상 **1단계**부터 시작.
- **다음 단계** 클릭: 1 → 2, 2 → 3. **이전 단계** 클릭: 3 → 2, 2 → 1.
- 2단계에서 “다음 단계”를 누르면 **자동으로 계산**을 수행한 뒤 3단계에 결과를 표시하는 흐름으로 설계.

### 0-4. 구현 방향

| 항목 | 내용 |
|------|------|
| **상태** | 현재 단계 번호(1 / 2 / 3)를 스토어 또는 컴포넌트 state로 보관. Scope·서브타입(고정연소/이동연소, 전력/열)별로 step 1~3 유지 가능. |
| **렌더링** | **현재 단계에 해당하는 블록만** 렌더링. 1단계일 때는 “조건 선택·조회” 블록만, 2단계일 때는 “활동자료 입력” 블록만, 3단계일 때는 “산정 결과·저장” 블록만 표시. |
| **버튼** | 각 단계 하단에 **다음 단계** 버튼(1·2단계), **이전 단계** 버튼(2·3단계). 버튼 라벨: “다음 단계” / “이전 단계”. (필요 시 “1. 조건 선택” “2. 활동자료 입력” “3. 결과 확인” 같은 단계 인디케이터를 상단에 표시.) |
| **진행 조건** | 1→2: 조회 버튼을 눌렀을 때만 “다음 단계” 활성화하거나, 연도·사업장 선택만 되어 있어도 다음 단계 허용(정책 결정). 2→3: 입력값이 있으면 다음 단계 클릭 시 계산 후 3단계로. |
| **Scope/서브타입** | Scope 1 고정연소, Scope 1 이동연소, Scope 2 전력, Scope 2 열 각각에 위 1→2→3 위저드를 적용. 탭으로 Scope·서브타입을 바꿀 때 해당 흐름의 **현재 단계**가 유지되도록. |

### 0-5. 적용 범위

- **Scope 1** (고정 연소, 이동 연소): 각각 1→2→3 위저드.
- **Scope 2** (전력, 열/스팀/온수): 각각 1→2→3 위저드.
- **금지**: “모든 단계를 한 페이지에 스크롤로 모두 노출”하는 기존 형태로 되돌리지 않음. **한 번에 한 단계만** 보이도록 유지.

## 1. 사업장 선택 방식 전환: “각각 선택 + 전체 합계”

### 1-1. 현재 문제

- **현재**: 사업장을 **멀티 체크**로 선택하면, 선택한 사업장들의 값이 **한꺼번에 합쳐져** 그리드에 표시됨.
- **사용자 요구**: “저렇게 선택해서 값 합치지 말고, **연도 선택하듯 각각 선택**하게 하고, **전체 사업장에 대한 합계 하나**를 추가해 달라.”

### 1-2. 목표 UX

| 구분 | 현재 | 목표 |
|------|------|------|
| **사업장** | 멀티 체크 → 선택 사업장 합산만 표시 | **단일 선택(드롭다운)** — 선택한 **한 사업장** 데이터만 그리드에 표시 |
| **연도** | 단일 선택(드롭다운, 예: 2026년) | 유지 |
| **전체 합계** | 없음 또는 불명확 | **“전체 사업장 합계”** 한 블록/한 행을 추가해, **모든 사업장** 합계를 항상 한 곳에서 표시 |

즉, **연도**처럼 **사업장도 드롭다운으로 한 개만 선택** → 그 사업장만 그리드에 나오고, 별도로 **“전체 사업장 합계”**는 항상 보이게 한다.

### 1-3. 구현 방향

| 항목 | 내용 |
|------|------|
| **사업장 컨트롤** | 필터 사이드바의 “사업장”을 **멀티 체크** → **단일 선택 드롭다운**으로 변경. 라벨 예: “사업장 (조회 대상)” / “사업장”. 옵션: “전체”, “본사”, “공장A”, “공장B”, “지점1”, “지점2” … |
| **그리드 데이터** | 선택값이 **“전체”**가 아니면: `facility === selectedFacility` 인 row만 표시. **“전체”**면: 모든 사업장 row 표시(기존과 유사). |
| **“전체 사업장 합계” 블록** | 산정 결과 탭(또는 같은 페이지)에 **고정 블록** 추가. 제목: “전체 사업장 합계”. 내용: **모든 사업장**에 대한 Scope 1(또는 2) **총 tCO₂e**, 필요 시 Scope·에너지원별 소계. 사업장 드롭다운과 무관하게 **항상** 전체 합계만 표시. |
| **연도** | 기존처럼 단일 선택 유지. “연도” + “사업장” 조합으로 “해당 연도·해당 사업장(또는 전체)” 데이터만 조회·표시. |

### 1-4. 적용 범위

- **Scope 1** (고정 연소, 이동 연소): 사업장 단일 선택 + 전체 사업장 합계 블록.
- **Scope 2** (전력, 열/스팀/온수): 동일하게 사업장 단일 선택 + 전체 사업장 합계 블록.
- **필터 사이드바** 문구: “선택한 사업장 데이터만 테이블에 반영됩니다.” → “선택한 **한** 사업장 데이터만 테이블에 반영됩니다. 전체 합계는 하단 [전체 사업장 합계]에서 확인하세요.” 등으로 정리.

## 1-5. 에너지원도 단일 선택 드롭다운 + 전체

### 목표

- **에너지원**도 사업장과 동일하게 **단일 선택 드롭다운**으로 변경. 멀티 체크(해제/전체) 대신 **한 개만 선택** 또는 **전체** 선택.
- “전체” 선택 시 해당 Scope의 **모든 에너지원**이 그리드·계산에 포함되고, 한 개 선택 시 해당 에너지원만 필터링.

### 구현 방향

| 항목 | 내용 |
|------|------|
| **에너지원 컨트롤** | 필터 사이드바의 “에너지원”을 **멀티 체크** → **단일 선택 드롭다운**으로 변경. 옵션: **“전체”** + Scope별 에너지원 목록(고정연소: 도시가스(LNG), 경유, 휘발유, LPG, 벙커유, 무연탄 / 이동연소: 경유(차량), 휘발유(차량) / Scope2 전력·열: 해당 목록). |
| **저장/표시** | 선택값이 “전체”면 `energySources` 빈 배열 또는 특수값으로 간주해 전체 에너지원 표시. 한 개 선택 시 `facility + year + energySource` 조건으로 그리드·결과 필터링. |
| **문구** | “에너지원을 선택하면 해당 에너지원만 테이블에 표시됩니다. 전체를 보려면 ‘전체’를 선택하세요.” |

## 2. 에너지원 선택과 결과 반영 (Scope 1 이동연소, Scope 2 전력·열)

### 2-1. 현재 문제

- **Scope 1 이동연소**: 사이드바에서 에너지원(예: 경유(차량), 휘발유(차량))을 선택해도 **선택이 결과(그리드·배출량)에 반영되지 않음**.
- **Scope 2 전력**: 에너지원(전력) 선택이 **결과 반영 안 됨**.
- **Scope 2 열/스팀/온수**: 동일하게 **에너지원 선택과 결과 반영이 안 됨**.

### 2-2. 원칙

- **Single source of truth**: 필터(사이드바)에서 선택한 **연도·사업장·에너지원**이 **그리드·배출량 산정 결과**에 반드시 반영되어야 함.
- **AND 조건**: `year === selectedYear` AND `facility === selectedFacility`(또는 전체) AND `energySource IN selectedEnergySources` 인 데이터만 집계·표시.

### 2-3. 구현 방향

| 구분 | 조치 |
|------|------|
| **Scope 1 이동연소** | 이동연소용 **에너지원 목록**을 사이드바/필터에 노출(경유(차량), 휘발유(차량) 등). `appliedFilters.energySources`(또는 동일 개념)를 이동연소 그리드·계산 입력에 전달. 그리드·배출량 결과가 **선택한 에너지원만** 필터링되도록 수정. |
| **Scope 2 전력** | 전력 데이터에 대해 `selectedEnergySources`(예: [전력]) 적용. 그리드 및 “배출량 산정 결과”의 Scope 2 전력 tCO₂e가 **선택한 에너지원·필터 조건**과 일치하도록 연동. |
| **Scope 2 열/스팀/온수** | 열(표준·KDHC 등) 에너지원을 필터 옵션에 포함. 선택한 열 에너지원만 그리드·계산에 반영. |
| **에너지원 라벨 일치** | 사이드바 옵션(예: “경유(차량)”)과 스토어/그리드의 `energySource` 키(예: `diesel_vehicle`)가 다를 수 있음. **매핑 테이블**로 표시명↔키 일치시키고, 필터 적용 시 동일 기준으로 비교. (SIDBAR_CONNECT §0-6 참고) |
| **조회 타이밍** | “조회” 클릭 시에만 `appliedFilters` 갱신. 조회 후 그리드·결과가 **갱신된 필터**로 다시 계산·표시되도록 보장. |

### 2-4. 검증 체크리스트 (항상 확인)

- [ ] **Scope 1 고정연소**: 에너지원 단일 선택/전체 → 그리드·tCO₂e 반영.
- [ ] **Scope 1 이동연소**: 에너지원(경유(차량), 휘발유(차량)) 선택·전체 → 그리드·배출량 결과에 **반드시 반영**되는지 매 배포/수정 시 확인.
- [ ] **Scope 2 전력**: 전력 선택 → 전력 사용량·배출량만 반영.
- [ ] **Scope 2 열/스팀/온수**: 열 관련 에너지원 선택 → 열 배출량만 반영.
- [ ] 에너지원 “전체” 선택 시 해당 Scope 전체 에너지원 표시.

## 2-5. 테이블 표시: 전체 에너지원 + 합계 기본, 필터로 축소

### 목표

- **선택·입력된 에너지원만** 테이블에 보이지 말고, **기본적으로 해당 Scope의 전체 에너지원 행 + 합계 행**이 테이블에 표시되게 함.
- **필터에서 에너지원을 (단일) 선택**했을 때만, 해당 에너지원만 보이거나 선택한 것만 보이게 함.

### 구현 방향

| 항목 | 내용 |
|------|------|
| **기본 표시** | 에너지원/월별 테이블(RAW 또는 탄소 배출량)에 **Scope별 전체 에너지원**을 행으로 나열. 입력값이 없으면 0 또는 ‘-’ 표시. **합계 행**을 마지막에 두어 월별·연간 합계 표시. |
| **필터 적용 시** | 사이드바에서 에너지원을 **단일 선택**(한 개 선택) 시 → 해당 에너지원 행만 테이블에 표시. “전체” 선택 시 → 위 전체 에너지원 + 합계 표시. |
| **데이터 소스** | 수동 입력·EMS·엑셀에서 들어온 데이터는 동일 테이블에 반영. 행은 “전체 에너지원 기준”으로 고정하고, 셀 값만 입력/연동 데이터로 채움. |

## 3. Scope 2 시장 기반·위치 기반 복원 및 반영

### 3-1. 현재 상태

- **산정 설정(GHGBoundarySettings)** 에는 Scope 2 포함 기준이 있음:  
  `위치 기반 & 시장 기반 동시 산정` / `위치 기반 (Location-based)` / `시장 기반 (Market-based)`.
- **Scope 2 전력·열 입력/결과 화면**에서는 이 선택이 **명시적으로 보이지 않거나**, 위치 기반/시장 기반 **각각의 배출량**이 따로 표시되지 않는 상태로 보임. (“있었는데 왜 없어진 거냐”는 인상에 대응.)

### 3-2. 목표

- **선택 노출**: Scope 2 화면(전력·열 입력 또는 결과 블록)에서 **현재 산정 기준**이 “위치 기반” / “시장 기반” / “동시 산정” 중 무엇인지 **항상 표시**.
- **결과 분리**:  
  - **동시 산정** 선택 시: **위치 기반 배출량**과 **시장 기반 배출량**을 **둘 다** 계산·표시.  
  - **위치만** 또는 **시장만** 선택 시: 해당 하나만 계산·표시.
- **재생에너지 실적**: 시장 기반(또는 동시 산정) 선택 시에만 재생에너지 이행 실적(REC, PPA, 녹색프리미엄 등) 입력란 노출·반영. (현재 조건부 노출 로직 유지·보강.)

### 3-3. 구현 방향

| 항목 | 내용 |
|------|------|
| **Scope 2 상단/옆 안내** | “현재 산정: 위치 기반 & 시장 기반 동시 산정” 등 **한 줄 요약** 표시. 산정 설정에서 바꾸면 여기만 바뀌어도 됨. |
| **결과 블록** | “배출량 산정 결과” 내 Scope 2 영역에 **위치 기반 tCO₂e** / **시장 기반 tCO₂e** (동시 산정 시 둘 다, 단일 선택 시 하나만) 표시. |
| **계산 로직** | 위치 기반: 그리드 전력·열 사용량 × 위치 기반 배출계수. 시장 기반: 사용량 × 시장 기반 배출계수 − 재생에너지 실적 반영. 기존 `calcScope2Electricity`, `calcHeatStandard` 등에서 **scope2Included**에 따라 분기. |
| **타입·스토어** | `Scope2Step` / `scope2Included` 등 기존 타입 유지. 결과 저장 시 `locationBasedEmissions`, `marketBasedEmissions` (또는 동일 의미 필드) 저장·표시 가능하도록 확장. |

### 3-4. 문서·UI 정리

- 산정 설정에서 “Scope 2” 선택지 라벨을 **그대로 유지** (위치 기반 & 시장 기반 동시 산정 / 위치 기반 / 시장 기반).
- Scope 2 전력·열 화면에 **시장 기반·위치 기반이 없어진 것처럼 보이지 않도록** 반드시 복원: 한 화면 안에서 **선택 요약**(현재 산정: 위치 기반 / 시장 기반 / 동시 산정) + **결과**(위치 기반 tCO₂e, 시장 기반 tCO₂e)를 함께 표시.

### 3-5. 복원 체크리스트

- [ ] Scope 2 전력·열 상단 또는 옆에 “현재 산정: 위치 기반 & 시장 기반 동시 산정”(또는 선택한 값) **항상 노출**.
- [ ] 배출량 산정 결과 블록의 Scope 2 영역에 **위치 기반 tCO₂e** / **시장 기반 tCO₂e** 각각 표시(동시 산정 시 둘 다).
- [ ] 시장 기반(또는 동시) 선택 시 **재생에너지 이행 실적** 입력란 노출·반영.

## 4. 수동입력 칸 직관성: “추가 버튼” 안내 + EMS·엑셀 연동 안내

### 4-1. 현재 문제

- **수동입력 칸**을 누군가 알려주지 않으면, 사용자가 “저 부분에 값을 입력해야겠다”고 **직관적으로 느끼지 못함**.
- **실제로 입력하려면 “추가” 버튼(연료 추가/전력 행 추가 등)을 눌러야 한다**는 안내가 없어, 빈 화면만 보이고 입력 방법을 모름.
- EMS·엑셀 연동이 **어떤 순서로, 어디서** 가능한지 화면·문서에 자세한 안내가 부족함.

### 4-2. 원칙 (§0 위저드와 연동)

- **2단계(활동자료 입력)** 에서만 수동 입력 그리드가 노출됨. “1단계 → 다음 단계 → **2단계 = 여기가 입력 단계**”가 자연스럽게 인지되도록 함.
- **“입력하려면 먼저 [연료 추가] / [전력 행 추가] 등을 눌러 주세요”** 문구를 2단계 화면에 **반드시** 표시.

### 4-3. 구현 방향

| 항목 | 내용 |
|------|------|
| **추가 버튼 안내** | 2단계 상단 또는 빈 그리드 위에 **안내 문구** 표시: “**실제로 데이터를 입력하려면 오른쪽 [연료 추가]** (고정연소) **또는 [항목 추가]** (이동연소) **/ [전력 행 추가]** (Scope2) **버튼을 눌러** 행을 추가한 뒤, 월·사업장·에너지원·사용량을 입력하세요.” (버튼 라벨은 실제 UI와 동일하게.) |
| **2단계 제목** | “2. 활동자료 입력” 제목 + 서브 문구: “월·사업장·에너지원·**사용량**을 입력하세요. **입력 행이 없으면 [연료 추가] 등 추가 버튼을 먼저 눌러 주세요.**” |
| **입력 영역** | 2단계에서만 노출되는 그리드(카드)를 테두리·배경으로 구분. “연료 추가” / “전력 행 추가” 버튼을 **눈에 띄게** 배치. |
| **컬럼 라벨·플레이스홀더** | “월”, “사업장”, “에너지원”, “**사용량**”, “단위” 명확 표시. 사용량 셀 플레이스홀더: “0” 또는 “숫자 입력”. |
| **첫 행 예시(선택)** | 예시 1행(월 1, 본사, 도시가스, 1200, Nm³)을 비활성/회색으로 두어 “이렇게 채운다”는 것을 보여줄 수 있음. |

### 4-4. EMS·엑셀 연동 안내 (화면·문서에 자세히)

| 구분 | 안내 내용 |
|------|-----------|
| **EMS 연동** | 1) 상단 툴바 **EMS 불러오기** 클릭 → 2) 열리는 창에서 현재 필터(연도·사업장·에너지원) 확인 → 3) **EMS 데이터 가져오기** 버튼 클릭 → 4) 연동 성공 시 **해당 Scope의 에너지원/월별 테이블(2단계 화면)에 자동 반영**. 실패 시 “엑셀 업로드 또는 수동 입력을 이용하세요” 안내. |
| **엑셀 업로드** | 1) 상단 툴바 **엑셀 업로드** 클릭 → 2) 필수 컬럼(월, 사업장, 에너지원, 사용량, 단위)이 있는 엑셀 파일 선택/드롭 → 3) 미리보기에서 검증 후 **확인 및 적용** 클릭 → 4) **같은 Scope의 테이블에 데이터 반영**. 템플릿 다운로드 링크 제공. |
| **표시 위치** | EMS·엑셀으로 불러온 데이터는 **2단계 활동자료 입력 화면의 에너지원/월별 테이블**에 그대로 반영됨. 1단계에서 “EMS·엑셀으로도 불러올 수 있습니다” 한 줄 링크를 두고, 툴바 버튼(EMS 불러오기, 엑셀 업로드)과 연결. |

### 4-5. 기본 화면 옵션: 상세 입력 폼을 기본으로

- **옵션 A(권장)**  
  - 2단계 진입 시 **세 번째 이미지처럼** “에너지원/월별 RAW 테이블 + 월·사업장·에너지원·사용량 상세 입력 폼(연료 추가 버튼)”이 **기본으로** 보이게 함.  
  - EMS·엑셀 연동 또는 업로드를 하면 **같은 테이블·같은 폼 영역에** 데이터가 반영되게 함.  
- **옵션 B**  
  - 2단계에서 “입력된 데이터가 없습니다”만 보이다가, **연료 추가** 등 추가 버튼을 누르면 상세 입력 폼이 나타나는 방식. 이 경우에도 **반드시** “입력하려면 [연료 추가] 버튼을 눌러 주세요” 안내 문구를 빈 화면에 크게 표시.

### 4-6. 실무자 플로우 정리

- **목표**: “**1단계** 조건 선택·조회 → **다음 단계** → **2단계**에서 **추가 버튼으로 행 추가 후** 월·사업장·에너지원·사용량 입력(또는 EMS·엑셀 반영) → **다음 단계** → **3단계** 결과 확인·저장.”
- **접근성**: 2단계 진입 시 “데이터 입력은 이 화면에서 하시면 됩니다. **먼저 [연료 추가] 등 추가 버튼을 눌러 주세요.**” 수준의 안내를 2단계 상단에 표시.
