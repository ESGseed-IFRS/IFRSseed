# CarbonOps v1.0 상세 설계 사양서 (GHG 산정 화면)

> **핵심**: 데이터의 위계(Hierarchy)를 명확히 하고, 사용자의 인지 부하를 줄이는 **Depth 기반의 UX 설계**. 모든 정보를 한 화면에 때려 넣는 방식이 아니라, 필터링(GNB/검색바)을 통해 내가 보고 싶은 정보만 **그리드** 형태로 집중 관리하되, 그 결과가 어떻게 도출되었는지 **증적(Evidence)** 을 증명하는 구조.

---

## 1. UX/UI 설계 원칙: **"Focus & Trace"**

### 1-1. 정보의 위계화

- 상위 레벨(Scope)에서 하위 레벨(월별·에너지원)로 이어지는 **전이 과정을 명시적으로 분리**
- 한 화면에 모든 정보를 담지 않고, **탭·서브탭·필터**로 단계적 드릴다운

### 1-2. 검색 기반 대시보드

- 고정된 전체 화면이 아닌, **사용자가 선택한 조건**(사업장, 기간, 에너지원)에 맞는 데이터만 그리드에 노출
- 조회(확인) 버튼 클릭 시에만 조건 적용

### 1-3. 증적 투명성 (Traceability)

- IFRS 및 국제 표준 대응을 위해 **모든 배출량 결과값의 산출 근거(Raw Data)를 추적 가능(Traceable)** 하게 설계
- "이 결과값이 어떻게 나왔는가?"에 대한 답을 화면에서 제공

### 1-4. 핵심 원칙: **한 화면 = 한 집중 단위**

현재 GHG 산정 화면은 한 탭 안에 **Scope 1·2, 고정·이동 연소, 전력·열·LNG·경유·휘발유 등** 모든 에너지원, 모든 사업장, 월별·연별이 한꺼번에 쌓여 있어 정보 과부하가 발생한다.

**변경 방향**: 한 화면(탭)에 보여주는 단위를 **명시적으로 쪼개어** 사용자가 집중할 수 있게 한다.

| 예시 | 한 화면에 보여줄 단위 |
|------|------------------------|
| 전력 사용량 월별 | **전력** 사용량, **A사업장**, 1월~12월 |
| 전력 전체 | **전력** 사용량, **전체 사업장** |
| LNG 고정연소 | **Scope 1 고정연소**, **LNG**, 사업장·월별 |

---

### 1-5. 계층 구조 (Drill-down)

```
Scope 1  ─┬─ 고정 연소 ─┬─ LNG, 도시가스, 경유, 휘발유, LPG, 벙커유, 무연탄
          │             └─ 에너지원 선택 → 사업장 선택 → 월별/연별
          │
          └─ 이동 연소 ─┬─ 경유(차량), 휘발유(차량)
                        └─ 에너지원 선택 → 사업장 선택 → 월별/연별

Scope 2  ─┬─ 전력 ─── 사업장 선택 → 월별/연별
          │
          └─ 열/스팀/온수 ─ 사업장 선택 → 월별/연별
```

- **레벨 1**: Scope 1 vs Scope 2
- **레벨 2**: 고정 연소 vs 이동 연소 (Scope 1) / 전력 vs 열 (Scope 2)
- **레벨 3**: 에너지원 선택 (LNG, 경유, 전력 등)
- **레벨 4**: 사업장 선택 (A사업장 / 전체 사업장)
- **레벨 5**: 시기 단위 (1월~12월 / 연별)

### 1-6. 데이터 위계 구조 (Data Hierarchy)

| Level | 구분 | 상세 항목 (예시) |
|-------|------|------------------|
| **Level 1** | Scope | Scope 1 (직접 배출), Scope 2 (간접 배출) |
| **Level 2** | Category | 고정연소, 이동연소, 공정배출 / 전력 사용, 스팀 사용 |
| **Level 3** | Source | LNG, 휘발유, 디젤, 무연탄, 전력 등 에너지원 |
| **Level 4** | Site | 사업장 A, 사업장 B, 본사 등 |
| **Level 5** | Time/Month | 2025년 01월 ~ 12월 (데이터 입력 최소 단위) |

## 2. 탭·서브탭 구조

### 2-1. 상위 탭

| 탭 | 설명 |
|----|------|
| **Scope 1** | 고정·이동 연소 |
| **Scope 2** | 전력·열/스팀/온수 |
| **Scope 3** | 공급망 등 (별도 구조) |

### 2-2. Scope 1 서브탭

| 서브탭 | 에너지원 예시 |
|--------|----------------|
| **고정 연소** | LNG, 도시가스, 경유, 휘발유, LPG, 벙커유, 무연탄 |
| **이동 연소** | 경유(차량), 휘발유(차량) |

### 2-3. Scope 2 서브탭

| 서브탭 | 에너지원 예시 |
|--------|----------------|
| **전력** | 전력 사용량 |
| **열/스팀/온수** | 열(표준), KDHC |

- 서브탭 선택 후 → **에너지원 선택** → **사업장 선택** → **기간(년·월)** 선택 → **확인** 클릭
- 이 조건이 적용된 화면이 **한 화면** 단위가 됨

## 3. 화면 구성 (레이아웃)

### 3-1. 상단 GNB

- **내비게이션 바**: 홈, 회사정보, GHG 산정, 도표·그림, SR 작성, 최종보고서, 로그인
- **GHG 산정** 진입 시: 사용자 정보·설정 등 상단 고정

### 3-2. 상단 검색 영역 (Global Navigation & Filter)

사용자가 데이터를 관리하기 전 반드시 거쳐야 하는 **입구** 역할.

| 항목 | 타입 | 설명 |
|------|------|------|
| **사업장 선택** | Single/Multi-select | 단일 사업장 또는 전체 사업장 선택 |
| **에너지원 분류** | Select | Scope 1(고정연소, 이동연소), Scope 2(전력, 스팀) 등 **계층별** 선택 |
| **타임 피어리드** | Select/Range | 연도별, 월별 기간 설정 |
| **제어 버튼** | Button | **조회(확인)**, **초기화** |

### 3-3. 데이터 관리 그리드 (Activity Data Grid)

조회 조건이 적용된 후 나타나는 **데이터 편집·조회 영역**.

**구조**:
- **옵션 A (행 기반)**: 1월~12월이 **수직(Row)** 으로 배열 → 세로 스크롤로 12월까지 확인
- **옵션 B (열 기반)**: 1월~12월이 **가로(Column)** 으로 배열 → 가로 스크롤로 12월까지 확인

**데이터 상태 표시**:
| 상태 | 표시 | 비고 |
|------|------|------|
| **EMS 연동** | Green | 인터페이스를 통해 자동 수집된 데이터 (수정 불가 또는 표기) |
| **수동 입력** | Yellow | 엑셀 업로드 또는 직접 입력 데이터 |
| **Warning System** | Red Box | 로직에 어긋나거나 전월 대비 오차가 큰 이상치 데이터 |

**수정 로직**:
- 특정 **셀(Cell)** 클릭 → **팝업(Modal)** 표시
- **수정 사유 입력** + 데이터 변경 후 저장

## 4. 기능 레이어: Raw Data에서 결과까지

### 4-1. RAW 데이터 관리 (Raw Data Management)

**역할**: EMS 인터페이스 데이터 및 사용자 입력값의 **원본**을 관리하는 페이지.

**핵심 기능**:
| 기능 | 설명 |
|------|------|
| **단위 변환** | kWh↔MWh, Nm³↔m³ 등 Unit Conversion |
| **누락 데이터 체크** | 해당 (연도·월·사업장·에너지원)에 데이터 없는 셀 표시 |
| **이상치 탐지** | 전월 대비 급변, 0/음수, 과대값 등 Red Box로 경고 |

**페르소나**:
- EMS 연동 시: "LNG 사용량은 EMS에서 인터페이스로 자동 수집된 데이터가 여기 표시됩니다."
- 화면 진입 시 **데이터가 이미 있음** → 확인·검증 위주
- **이상치(Red Box)**: 셀 클릭 → **모달**에서 수정 사유 입력 + 데이터 변경

### 4-2. 탄소 배출량 산출 및 증적 (Calculation & Evidence)

**IFRS 대응 핵심**: "이 결과값이 어떻게 나왔는가?"에 대한 답을 제공.

**화면 구성**:
| 항목 | 내용 |
|------|------|
| **산출 공식 노출** | 사용된 배출계수(EF), 발열량, GWP 값을 **수식과 함께** 노출 |
| **증빙 서류 매칭** | 해당 월의 고지서 스캔본, 계량기 사진, **EMS 통신 로그 데이터**와 연결 |
| **버전 이력** | 데이터 수정 시 **누가, 언제, 왜** 수정했는지 히스토리 관리 |

- RAW 데이터 → 배출계수 적용 → **tCO₂e** 결과
- 감사·검증 대응용

## 5. SIDEBAR_Strategy 연동

### 5-1. 멀티 필터 바 (사이드바)

- `[사업장 선택] | [에너지원] | [연도] | [월별/분기별/연간]`
- **사이드바에서만** 사업장·에너지원·연도·시기 선택
- **확인** 클릭 시 조건 적용

### 5-2. 데이터 현황 (dataType별 색상)

| 상태 | 색상 | 조건 |
|------|------|------|
| EMS 연동 | Green | dataType === 'ems' |
| 엑셀 업로드 | Blue | dataType === 'excel' |
| 수동 입력 | Yellow | dataType === 'manual' |
| 미입력 | Red | 해당 (연도·월·사업장·에너지원)에 데이터 없음 |

- 미입력(Red) 셀 클릭 → 수동 입력·엑셀 업로드·EMS 불러오기 안내

### 5-3. Collection before Calculation

- **Collection Layer**: EMS·엑셀·수동 → RAW DATA 적재
- **Calculation Layer**: RAW + 배출계수 → emissions 계산
- RAW 관리 화면 / 탄소 배출량 화면을 위 구조에 맞게 분리

## 6. 사용자 플로우 요약

```
1. GHG 산정 진입
   ↓
2. Scope 선택 (Scope 1 / Scope 2)
   ↓
3. 서브탭 선택 (고정 연소 / 이동 연소 또는 전력 / 열)
   ↓
4. 필터 조건 선택: 사업장, 에너지원, 연도, 기간
   ↓
5. [확인] 클릭
   ↓
6. "2024년 1월~12월, A사업장, 전력 사용량 관리" 그리드 표시
   - RAW 데이터: 사용량 입력·수정·이상치 확인
   - 탄소 배출량: tCO2e + 산출 근거
   ↓
7. 셀 클릭 → 팝업 수정 (이상치 등)
```

## 7. 향후 구현 시나리오 (Persona-based)

### 7-1. 관리자 시나리오

| 단계 | 행동 | 화면 상태 |
|------|------|-----------|
| **1. 로그인** | 관리자 로그인 | GNB 노출 |
| **2. 조건 선택** | GNB·필터에서 **사업장 A**, **Scope 2 - 전력** 선택 후 **조회** | — |
| **3. 데이터 확인** | 1월~12월 그리드 출력 | 4월 데이터에 **EMS 통신 오류**로 인한 **Red Box** 발견 |
| **4. 수정 및 증빙** | 4월 데이터 **셀 클릭** | 모달: 고지서 기반 **수동 입력**으로 전환, **증빙 파일(PDF)** 업로드 |
| **5. 결과 검증** | 산출 페이지 이동 | 최종 **tCO₂e** 값 확인, IFRS 공시용 **산출 근거 보고서** 생성 |

---

## 8. 구현 시 유의사항

1. **한 화면 = 한 집중 단위** 유지: 한 번에 Scope·에너지원·사업장·기간을 과도하게 섞지 않음
2. **탭·서브탭**으로 계층 명시: Scope → 고정/이동 or 전력/열 → 에너지원
3. **조회(확인)** 버튼으로 조건 확정 후 그리드 갱신
4. **RAW 관리**와 **탄소 배출량 관리** 화면 분리
5. **IFRS 산출 근거**를 탄소 배출량 화면에서 반드시 제공
6. **수정 시 사유 입력** 및 **버전 이력** 관리
