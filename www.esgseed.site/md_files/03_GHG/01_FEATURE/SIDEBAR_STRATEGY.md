# Scope 1·2 EMS·엑셀 RAW DATA 연동 전략

## 1. 개요

### 1-1. 목표

Scope 1·2 온실가스 배출량 산정 시, **활동자료(RAW DATA)**를 EMS 연동·Excel 업로드로 수집하고, **데이터 저장소**를 먼저 구축한 뒤 산정 엔진이 이를 불러와 계산하는 **"Collection before Calculation"** 구조로 전환한다.

| 경로                   | 설명                                                | 대상             |
| ---------------------- | --------------------------------------------------- | ---------------- |
| **EMS 연동**     | 내부 EMS(환경관리시스템)에서 RAW DATA 자동 가져오기 | Scope 1, Scope 2 |
| **Excel 업로드** | 엑셀 파일 업로드로 RAW DATA 일괄 입력               | Scope 1, Scope 2 |
| **수동 입력**    | 폼에서 직접 입력                                    | Scope 1, Scope 2 |

**핵심 변경**:

- **단계별 위저드 제거**: "이전" / "다음 단계" 버튼 삭제 → 각 섹션(산정 설정·데이터 관리·결과·리포트)으로 **직접 이동**
- **멀티 필터 바 추가**: `[사업장 선택] | [에너지원] | [연도] | [월별/분기별/연간]` 상단 인라인 배치 (핵심 추가 사항)
- **데이터 관리 센터(Smart Dashboard)**: EMS/엑셀 입수, 데이터 현황(EMS Green / 엑셀 Blue / 미입력 Red), 전월 대비 증감율로 이상치 파악

### 1-2. Scope 구분

| Scope             | EMS 연동 | Excel 업로드 | 비고                                    |
| ----------------- | -------- | ------------ | --------------------------------------- |
| **Scope 1** | ✅       | ✅           | 고정연소(도시가스·경유 등)·이동연소   |
| **Scope 2** | ✅       | ✅           | 전력·열(증기·지역난방 등)             |
| **Scope 3** | ❌       | ❌           | 공급망·출장 등, 구조 상 별도 설계 필요 |

## 2. 데이터 아키텍처 전략: "Collection before Calculation"

### 2-1. 핵심 원칙

사용자가 매번 수동 입력하는 대신, **데이터 저장소(Collection Layer)**를 먼저 구축하고, **산정 엔진(Calculation Engine)**이 이를 불러와 배출량을 계산하는 구조로 전환한다.

- **Collection Layer**: EMS·엑셀·수동 입력 → RAW DATA 적재
- **Calculation Layer**: 저장된 RAW DATA + 배출계수 → emissions 계산

### 2-2. 스키마 설계 (Schema)

| 요소                         | 설명                                | 예시                                   |
| ---------------------------- | ----------------------------------- | -------------------------------------- |
| **사업장 (Site)**      | 본사, A공장, B지사 등               | 본사, 공장1, 지점A                     |
| **에너지원 (Source)**  | LNG, 전력, 디젤 등 (고유 코드 부여) | lng, electricity, diesel               |
| **시기 (Period)**      | 연도(YYYY) 및 월(MM)                | 2024, 1~12                             |
| **데이터 유형 (Type)** | 입수 채널 구분                      | 실측(EMS), 입력(Manual), 업로드(Excel) |

- **Type** 필드로 각 row의 출처를 추적 → 데이터 현황 테이블에서 색상(EMS: Green, Excel: Blue, 미입력: Red) 표시 및 감사 증빙에 활용.

### 2-3. UI 구조 변경: 단계별 위저드 제거

| 기존                                                     | 변경                                                                                         |
| -------------------------------------------------------- | -------------------------------------------------------------------------------------------- |
| Step 1 → Step 2 → Step 3 → Step 4 → Step 5 순차 진행 | **플랫 화면**: 산정 설정(경계)·데이터 관리 센터·결과·리포트가 탭 또는 섹션으로 분리 |
| "이전" / "다음 단계" 버튼                                | **삭제**. 각 섹션으로 직접 이동                                                        |
| Step 2 = 활동자료 입력만                                 | **데이터 관리 센터**로 확장: 멀티 필터 바, 데이터 현황, EMS/엑셀 입수                  |

## 3. RAW DATA 구조 (에너지원별·월별)

### 3-1. 공통 필드

모든 RAW DATA(EMS·엑셀·수동 공통)는 아래 필드를 갖는다.

| 필드                   | 타입                        | 필수 | 설명                                           |
| ---------------------- | --------------------------- | ---- | ---------------------------------------------- |
| **year**         | number                      | ✅   | 보고 연도 (예: 2024)                           |
| **month**        | number                      | ✅   | 월 (1~12)                                      |
| **facility**     | string                      | ✅   | 사업장/지점 (예: 본사, 공장1, 지점A)           |
| **energySource** | string                      | ✅   | 에너지원/연료 종류                             |
| **amount**       | number                      | ✅   | 사용량                                         |
| **unit**         | string                      | ✅   | 단위 (kWh, MWh, Nm³, L, GJ 등)                |
| **dataType**     | 'ems'\| 'manual' \| 'excel' | ✅   | 입수 채널 (EMS 연동 / 수동 입력 / 엑셀 업로드) |

### 3-2. Scope 1 에너지원(energySource) 매핑

| EMS/엑셀 값   | 플랫폼 내부 키 | 단위 | 비고       |
| ------------- | -------------- | ---- | ---------- |
| 도시가스, LNG | lng            | Nm³ | 고정연소   |
| 경유          | diesel         | L    | 고정·이동 |
| 휘발유        | gasoline       | L    | 고정·이동 |
| LPG           | lpg            | kg   | 고정연소   |
| 벙커유, 중유  | bunkerC        | ton  | 고정연소   |
| 무연탄        | anthracite     | ton  | 고정연소   |

- **고정연소 vs 이동연소**: 키워드 기반 자동 분류 (예: 'lng', 'lpg', 'bunker' 포함 → 고정연소)

### 3-3. Scope 2 에너지원 매핑

| EMS/엑셀 값          | 플랫폼 내부 키  | 단위      | 비고         |
| -------------------- | --------------- | --------- | ------------ |
| 전력, electricity    | 전력            | kWh / MWh | Scope 2 전력 |
| 열, 증기, 스팀, 온수 | 별도 열 입력 폼 | GJ / TJ   | Scope 2 열   |

- 전력은 `electricity` 배열에 적재, 열은 `heat` 배열로 별도 관리.

### 3-4. 데이터 적재 형태 (EmissionData)

```
{
  id: string;           // 유일 ID (ems-{timestamp}-{idx} 등)
  dataType: 'ems' | 'manual' | 'excel';  // 입수 채널
  year: number;
  month: number;
  facility: string;
  energySource: string;
  amount: number;
  unit: string;
  emissions: number;    // 계산 후 채움 (초기 0)
  createdAt: Date;
  dataQuality?: DataQuality;  // 실측/추정 등 (선택)
}
```

## 4. EMS RAW DATA 연동 전략

### 4-1. 데이터 소스 설정 UI

**위치**: GHG 산정 화면 내 "데이터 소스 설정" 메뉴 (또는 별도 설정 페이지)

| 항목                    | 타입             | 설명                                                    |
| ----------------------- | ---------------- | ------------------------------------------------------- |
| **EMS 서버 주소** | Input (URL)      | EMS API 엔드포인트 (예:`https://ems.company.com/api`) |
| **API Key**       | Input (Password) | 인증용 API 키                                           |
| **연동 테스트**   | Button           | 연결 성공 시 "연동됨" 배지 표시, 실패 시 에러 메시지    |

**연동 성공 시**:

- "연동됨" 배지 표시 (Green)
- EMS 불러오기 버튼 활성화

**연동 실패 시**:

- "연동 해제" 또는 "연동 실패" 배지
- 수동 입력·엑셀 업로드로 대체 안내

### 4-2. 연동 방식

| 방식                | 설명                                                 | 구현 우선순위 |
| ------------------- | ---------------------------------------------------- | ------------- |
| **API Pull**  | 플랫폼이 EMS API를 주기적으로 호출해 데이터 수집     | Phase 1       |
| **스케줄링**  | Cron Job으로 매달 초 자동 Pull                       | Phase 2       |
| **API Push**  | EMS가 플랫폼 API로 데이터 전송                       | Phase 2       |
| **파일 연동** | EMS가 FTP/클라우드에 파일 생성 → 플랫폼이 정기 수집 | Phase 2       |

### 4-3. API Pull 상세 (Phase 1)

**요청 파라미터 (EMS → 플랫폼 요구사항)**

플랫폼은 EMS API 호출 시 아래 조건을 전달한다.

| 파라미터      | 타입                | 설명                       |
| ------------- | ------------------- | -------------------------- |
| year          | number              | 보고 연도 (단일 또는 범위) |
| yearRange     | { start, end }      | 연도 범위 (선택)           |
| month         | number              | 월 (단일)                  |
| monthRange    | { start, end }      | 월 범위 (선택)             |
| facilities    | string[]            | 사업장 ID/코드 목록        |
| energySources | string[]            | 에너지원 코드 목록         |
| scope         | 'scope1'\| 'scope2' | 대상 Scope                 |

**응답 형식 (EMS → 플랫폼)**

```json
{
  "rows": [
    {
      "year": 2024,
      "month": 1,
      "facility": "본사",
      "energySource": "전력",
      "amount": 12500,
      "unit": "kWh"
    },
    {
      "year": 2024,
      "month": 1,
      "facility": "공장1",
      "energySource": "도시가스",
      "amount": 3500,
      "unit": "Nm³"
    }
  ]
}
```

### 4-4. EMS 연동 UI 흐름

1. 사용자가 **필터**에서 연도·월·사업장·에너지원 선택
2. "**EMS 불러오기**" 버튼 클릭
3. 필터 조건을 EMS API 요청 파라미터로 변환
4. EMS API 호출 → 응답 rows 수신
5. 각 row를 `EmissionData` 형태로 변환 (id, emissions: 0 생성)
6. **현재 활성 Scope**에 맞춰 `scope1` 또는 `scope2.electricity`에 추가
7. 배출계수 적용으로 `emissions` 계산 (실시간)

### 4-5. 필드 매핑 (EMS → 플랫폼)

EMS 시스템별 컬럼명이 다를 수 있으므로 **필드 매핑 설정**을 지원한다.

| EMS 필드 (예시)                    | 플랫폼 필드  | 기본 매핑    |
| ---------------------------------- | ------------ | ------------ |
| site_cd, plant_id, location        | facility     | facility     |
| energy_type, fuel_type, meter_type | energySource | energySource |
| usage, consumption, value          | amount       | amount       |
| unit_cd, uom                       | unit         | unit         |
| yyyy, report_year                  | year         | year         |
| mm, report_month                   | month        | month        |

- 초기에는 고정 매핑, 이후 **매핑 설정 UI**로 확장 가능.

## 5. Excel 업로드 전략

### 5-1. 필수 컬럼

| 컬럼명 (한글/영문)      | 필수      | 설명                    |
| ----------------------- | --------- | ----------------------- |
| 월 / month              | ✅        | 1~12                    |
| 사업장 / facility       | ✅        | 사업장명                |
| 에너지원 / energySource | ✅        | 도시가스, 전력 등       |
| 사용량 / amount         | ✅        | 숫자                    |
| 단위 / unit             | ✅        | kWh, Nm³, L 등         |
| 연도 / year             | ⚠️ 선택 | 없으면 필터의 연도 적용 |

### 5-2. 엑셀 템플릿 예시

**Scope 1용**

| 월 | 사업장 | 에너지원      | 사용량 | 단위 |
| -- | ------ | ------------- | ------ | ---- |
| 1  | 본사   | 도시가스(LNG) | 1200   | Nm³ |
| 1  | 공장1  | 경유          | 500    | L    |
| 2  | 본사   | 도시가스(LNG) | 1150   | Nm³ |

**Scope 2용 (전력)**

| 월 | 사업장 | 에너지원 | 사용량 | 단위 |
| -- | ------ | -------- | ------ | ---- |
| 1  | 본사   | 전력     | 12500  | kWh  |
| 1  | 공장1  | 전력     | 82000  | kWh  |

### 5-3. 유효성 검사 체크리스트 (Validation)

| 항목                  | 검증 내용                                       | 실패 시 처리                                       |
| --------------------- | ----------------------------------------------- | -------------------------------------------------- |
| **필수 컬럼**   | 월·사업장·에너지원·사용량·단위 존재         | 업로드 중단, 에러 목록 표시                        |
| **단위 일치**   | Nm³ vs m³, kWh vs MWh 등 표준 단위로 정규화   | 경고 + 자동 변환 또는 사용자 확인 요청             |
| **중복 데이터** | (year, month, facility, energySource) 동일 조합 | **덮어쓰기** vs **건너뛰기** 옵션 제공 |
| **이상치**      | 사용량 0·음수·과대값                          | 경고 표시, 사용자 확인 후 적용                     |

### 5-4. 업로드 흐름

1. 사용자가 엑셀 파일 **드래그앤드롭** 또는 **파일 선택**
2. 첫 번째 시트의 **첫 행을 헤더**로 인식
3. **필수 컬럼 검증**: 월, 사업장, 에너지원, 사용량, 단위 존재 여부
4. **단위 검증**: Nm³/m³, kWh/MWh 등 일치·정규화
5. **에너지원 정규화**: "도시가스", "LNG" → lng 등 매핑
6. **중복 검출**: 동일 (year, month, facility, energySource) 존재 시 덮어쓰기 vs 건너뛰기 선택 UI
7. **검증 통과 시** 미리보기 테이블 표시
8. "**확인 및 적용**" 클릭 → `dataType: 'excel'`로 저장

### 5-5. 단위·에너지원 매핑

| 엑셀 입력값                    | 플랫폼 매핑                             |
| ------------------------------ | --------------------------------------- |
| 도시가스, LNG, lng             | energySource: 도시가스(LNG), unit: Nm³ |
| 전력, electricity, electricity | energySource: 전력, unit: kWh           |
| kWh, MWh                       | unit 유지                               |
| Nm3, Nm³                      | unit: Nm³                              |

## 6. UI/UX: Smart Dashboard & 멀티 필터 바

### 6-1. 데이터 관리 센터 (Smart Dashboard)

기존 Step 2(Activity Data)를 **데이터 관리 센터** 형태로 확장한다. 단계별 위저드가 아니라 **플랫 화면**으로, 멀티 필터 바·데이터 현황·시각화를 한 화면에 배치한다.

### 6-2. 멀티 필터 바 (핵심 추가 사항)

**위치**: Scope 1/2 세부 탭 **상단**에 인라인 배치 (항상 노출)

```
┌───────────────────────────────────────────────────────────────────────────────────┐
│ [사업장 선택 ▼]  |  [에너지원 ▼]  |  [연도 ▼]  |  [월별 / 분기별 / 연간 ▼]  [적용] [초기화] │
└───────────────────────────────────────────────────────────────────────────────────┘
```

| 필터                  | 타입         | 옵션                    | 설명                          |
| --------------------- | ------------ | ----------------------- | ----------------------------- |
| **사업장 선택** | Multi-select | 본사, 공장1, 지점A 등   | facilities 목록에서 다중 선택 |
| **에너지원**    | Multi-select | 도시가스, 전력, 경유 등 | Scope별 energySources         |
| **연도**        | Single/Range | 2020~2030               | 단일 또는 범위(시작~종료)     |
| **시기 단위**   | Select       | 월별 / 분기별 / 연간    | 조회·집계 단위               |

- **월별**: 1~12월 개별 선택 또는 범위
- **분기별**: 1~4분기 선택
- **연간**: 연도 전체

### 6-3. 데이터 현황 테이블 (상태값 표시)

| 상태                    | 색상      | 표시 조건                                            |
| ----------------------- | --------- | ---------------------------------------------------- |
| **EMS 연동 완료** | 🟢 Green  | dataType === 'ems'                                   |
| **엑셀 업로드됨** | 🔵 Blue   | dataType === 'excel'                                 |
| **수동 입력**     | 🟡 Yellow | dataType === 'manual'                                |
| **미입력**        | 🔴 Red    | 해당 (연도·월·사업장·에너지원) 조합에 데이터 없음 |

- 테이블 각 행 또는 셀에 **상태 배지** 표시 → 입수 채널 및 결측 여부를 한눈에 파악.
- **미입력(Red)** 셀: 클릭 시 수동 입력·엑셀 업로드·EMS 불러오기 안내.

### 6-4. 데이터 시각화 (전월 대비 증감율)

| 항목                           | 설명                                                          |
| ------------------------------ | ------------------------------------------------------------- |
| **전월 대비 증감율(%)**  | 테이블 옆 또는 셀 내에 전월 대비 증감률 표시                  |
| **이상치(Outlier) 파악** | 급격한 증감(예: ±30% 초과) 시 경고 표시 또는 하이라이트      |
| **트렌드 표시**          | 월별·사업장별·에너지원별 사용량 추이를 차트로 시각화 (선택) |

- 이상치 감지 로직: 전월 대비 ±N% 초과 시 경고 색상 또는 아이콘 표시.

### 6-5. 기존 필터 vs 멀티 필터 바

| 구분           | 기존                           | 변경                                           |
| -------------- | ------------------------------ | ---------------------------------------------- |
| **위치** | "필터" 버튼 클릭 시 다이얼로그 | **상단 인라인 멀티 필터 바** (항상 노출) |
| **항목** | 연도·월·사업장·에너지원     | +**시기 단위(월별/분기별/연간)**         |
| **UX**   | 숨겨진 패널                    | 한 화면에서 즉시 조회·필터링                  |

## 7. 세부 탭 필터 전략 (사업장·에너지원·연도)

### 7-1. 필터 적용 범위

| 대상                             | 적용 내용                          |
| -------------------------------- | ---------------------------------- |
| **데이터 현황 테이블**     | 멀티 필터 바 조건에 맞는 행만 표시 |
| **EMS 불러오기**           | 필터 조건을 API 파라미터로 전달    |
| **엑셀 적용**              | year 미포함 시 필터의 연도로 보정  |
| **월×사업장 결과 테이블** | 필터된 데이터만 집계               |

### 7-2. 사업장 관리

- 멀티 필터 바 또는 **사업장 관리** 섹션에서 사업장 추가·수정·삭제
- 저장 시 `ghg.store.facilities` 업데이트 → 사업장 Select, 월×사업장 테이블 헤더에 즉시 반영
- 회사정보(company-info)의 사업장과 **동기화** 옵션 (향후 확장)

### 7-3. 에너지원 목록

- **Scope 1**: 고정연소 + 이동연소 목록에서 추출
- **Scope 2**: 전력 + 열(표준/KDHC) 구분
- 사용자가 **입력·업로드한 energySource**가 기존 목록에 없으면 목록에 동적 추가 (선택)

## 8. 데이터 흐름 종합

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                         RAW DATA 수집 경로                                    │
├─────────────────────────────┬───────────────────────────────────────────────┤
│  EMS API                    │  Excel Upload                                  │
│  - 필터(연도·월·사업장·     │  - 드래그앤드롭 / 파일 선택                     │
│    에너지원) 기반 요청       │  - 필수 컬럼 검증 (월·사업장·에너지원·        │
│  - 응답 rows → EmissionData │    사용량·단위)                                 │
│  - Scope 1/2 자동 분류      │  - rows → EmissionData 변환                    │
└─────────────────────────────┴───────────────────────────────────────────────┘
                                        │
                                        ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│  ghg.store                                                                   │
│  - scope1: { stationary, mobile }                                            │
│  - scope2: { electricity, heat }                                             │
│  - facilities, filtersByScope, factorYear                                    │
└─────────────────────────────────────────────────────────────────────────────┘
                                        │
                    ┌───────────────────┼───────────────────┐
                    ▼                   ▼                   ▼
            ┌───────────────┐   ┌───────────────┐   ┌───────────────┐
            │ Scope 1 폼    │   │ Scope 2 폼    │   │ 필터 적용     │
            │ - 고정/이동   │   │ - 전력/열     │   │ - 연도·월     │
            │ - 월×사업장   │   │ - 월×사업장   │   │ - 사업장      │
            │   결과 테이블 │   │   결과 테이블 │   │ - 에너지원    │
            └───────────────┘   └───────────────┘   └───────────────┘
                                        │
                                        ▼
                              배출계수 적용 → emissions 계산
                                        │
                                        ▼
                              배출량 결과 · 증적 리포트

```

- **단계별 위저드 제거**: "이전" / "다음 단계" 버튼 없음. 산정 설정·데이터 관리 센터·결과·리포트는 **탭 또는 섹션**으로 직접 이동.

## 10. 유의사항

1. **데이터 정합성**: EMS·엑셀로 들어온 데이터는 **배출계수 적용 전** 원본 보존 권장 (audit trail).
2. **중복 방지**: 동일 (year, month, facility, energySource) 조합이 이미 있으면 **합산** 또는 **경고 후 덮어쓰기** 정책 필요.
3. **단위 통일**: kWh↔MWh, Nm³ 등 단위별 변환 로직 필수.
4. **Scope 3**: 본 전략은 Scope 1·2 한정. Scope 3은 카테고리 구조가 달라 별도 설계 필요.
5. 지금 UI느낌과 너무 달라지지 않게 작업할 것.
6. 지금 나눠져 있는 산정 설정 단계들은 삭제할 것
