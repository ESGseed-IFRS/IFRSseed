## ESG 공시 전 과정 TOTAL 플로우 (기능·업무 기준)

이 문서는 시스템 상단 메뉴를 아래 4개 영역으로 나누고, 각 영역 안에서 **실무자가 수행할 업무(등록·조회·신청·요청·회신·리마인드 등)** 를 기준으로 기능 요구사항을 정리한 것입니다.

- **1. 로그인 / 회원가입**
- **2. GHG 배출량 산정**
- **3. SR 보고서 작성 (공시데이터 + ESG 데이터 + 리포트 생성)**
- **4. 대시보드 (공시 진행 모니터링)**

각 영역은 “누가 / 언제 / 무엇을 / 어떻게 / 어떤 알림과 승인으로” 처리하는지까지 포함해 설계합니다.

---

## 1. 로그인 / 계열사별 그룹 계정

> 기본 원칙: **각 자회사·물류 법인·국내 인프라(데이터센터 등)는 “그룹당 1개의 공용 계정”을 사용**하고, 최종적으로 **지주사(삼성SDS) 담당자가 모든 데이터를 취합한 뒤 FREEZING(데이터 동결)** 한다.

- **지주사(삼성SDS)**
  - `SDS_ADMIN`: 전체 통합 관리·조회 + FREEZING 권한
  - 필요 시 `SDS_VIEW`(읽기 전용) 정도만 추가
- **계열사·법인 그룹 계정 (각 1개)**
  - 미라콤아이앤씨
  - 시큐아이
  - 에스코어
  - 멀티캠퍼스
  - 오픈핸즈
  - 엠로
  - 국내 인프라(데이터센터 등): 상암, 수원, 춘천, 동탄 등
  - 해외 사업장 및 물류 법인들
  - → 각 그룹별로 **동일 ID/PW를 공유**하여, 팀장 개인 계정 없이 실무 담당자들이 공용 계정으로 데이터 입력

### 1-1. 계열사 그룹 계정 생성·배포 (초기 1회 설정)

- **지주사에서 직접 계정 생성**
  - 법인/사업장 정보 등록: 회사명, 법인코드, 구분(국내/해외/물류/인프라 등)
  - 그룹 계정 ID/PW 자동 생성 (예: `MIRACOM_GHG`, `S2_SUWON_DC` 등)
  - 역할: `CORP_INPUT` (해당 법인의 데이터 입력·수정 권한)
- **계정 정보 배포**
  - 계열사 ESG 담당자에게 공용 ID/PW 전달
  - 추후 PW 변경은 **SDS_ADMIN만 가능** (팀장 승인 플로우 없음)

### 1-2. 로그인 / 조직 선택

- **계열사·법인 사용자**
  - 공용 ID/PW로 로그인 → 자동으로 해당 법인/데이터센터 스코프에만 접근
  - 자신 그룹 외의 법인 데이터는 조회 불가
- **지주사 사용자**
  - `SDS_ADMIN` ID로 로그인 시 전체 법인·사업장 선택 가능한 뷰 활성화
  - 통합 뷰 / 법인별 뷰 전환 가능

### 1-3. FREEZING 권한 모델

- **FREEZING(데이터 동결)**
  - 권한 주체: **지주사 `SDS_ADMIN`만 수행 가능**
  - 단위:
    - 연도 단위(예: 2024년)
    - 공시 프레임워크 단위(IFRS S2, K-ETS 등)
  - 동결 후:
    - 계열사 그룹 계정에서는 해당 연도·프레임워크의 데이터가 **read-only** 로 전환
    - 수정 필요 시 SDS 담당자가 “동결 해제 요청” 기록 후, 다시 삭제/수정

### 1-4. 비밀번호·접근 관리 (간소화)

- **PW 관리**
  - 공용 계정 PW 변경은 SDS에서 일괄 관리(계열사 개별 요청만 접수)
  - 변경 이력(언제, 누가, 어느 계정을 변경했는지)만 Audit trail에 남김
- **접근 제어**
  - IP/국가 기반 접근 제한(필요 시)
  - 계열사별 계정 잠금/해제 기능 (장기간 미사용, 보안 이슈 발생 시)

## 2. GHG 배출량 산정

> 목표: GHG 데이터 수집(I/F + 수기), 검증, 이력관리, 산정 결과 출력까지 **연간 산정 사이클**을 모두 시스템 안에서 처리.

### 2-1. GHG 산정 (등록·조회·검증)

- **(1) 산정 프로젝트 생성**
  - 보고 연도, 조직 경계, 공시 프레임워크 선택
  - Scope 1/2/3 포함 범위, 기준 가이드라인 선택
  - **등록 액션**: `산정 프로젝트 생성` → 이후 모든 데이터가 이 프로젝트에 귀속

- **(2) GHG 데이터 취합 (I/F + 수기 입력)**
  - **시스템 I/F 연동 데이터 조회**
    - ERP: 연료 구매, 원료, 제품
    - EMS/BMS: 전력·에너지 사용량
    - HR/출장 시스템: 출장, 통근
    - SRM/물류: 물류·운송
  - **수기 업로드/입력**
    - 표준 템플릿(엑셀) 업로드: 전력, 연료, 냉매, 폐기물, 물류, 출장, 통근, 원료·제품
    - 개별 행 직접 입력/수정(웹 폼)
  - **작업 유형**
    - 데이터 `등록`: 신규 행 추가, 엑셀 업로드
    - 데이터 `수정`: 값 변경, 배출계수 선택 변경
    - 데이터 `삭제/비활성`: 잘못된 행 비활성 처리 (논리 삭제)

- **(3) 데이터 이상치 검증**
  - 전년 대비 증감률, 단위당 사용량, 특정 기간 급증 등 자동 검증
  - 이상치 리스트 조회(테이블 + 필터)
  - **조치 액션**
    - `재검토 요청`: 현업/원 데이터 담당자에게 확인 요청
    - `확인 완료`: 정상으로 승인
    - `보정 입력`: 추정치/보정값 재입력
  - 검증 결과에 따른 **상태**: 미검토 / 검토중 / 검토완료

- **(4) 배출계수 매핑·산정 실행**
  - 에너지원·활동유형별 배출계수 라이브러리 조회
  - 기준연도/국가/출처(IPCC, 환경부 등) 선택
  - 자동 매핑 + 수동 오버라이드(특수 계약/자체계수)
  - `배출량 재계산` 버튼 → Scope별 배출량 산정
  - 산정 결과 로그(언제, 누가, 어떤 조건으로 재계산했는지) 이력 저장

### 2-2. Audit trail (계열사 입력 + 지주사 취합·동결)

- **계열사 입력 이력관리**
  - 각 활동자료 행에 대해:
    - 입력 주체: 어느 그룹 계정(미라콤, 시큐아이, 상암 DC 등)에서 입력했는지
    - 입력/수정 일시
    - 수정 전/후 값, 수정 사유
  - 계열사 계정에서는 **본인 그룹 데이터만 조회** 가능
- **지주사 취합·검토**
  - SDS 담당자는 전체 법인 데이터를 통합 조회(법인별 필터, 사업장별 필터)
  - 이상치/누락 데이터에 대해 **계열사별 코멘트 요청**:
    - 예: “시큐아이 2024년 3월 전력 사용량 급증 사유 회신 요청”
    - 요청 알림은 해당 법인 그룹 계정에 표시
  - 계열사는 공용 계정으로 로그인해 코멘트에 **회신**(메모 입력·수정 등)
- **FREEZING 전 사전 체크**
  - SDS 담당자는 다음 상태를 확인 후 동결:
    - 모든 계열사의 데이터 제출 여부(제출/미제출)
    - 주요 이상치에 대한 회신 여부(미회신 항목 여부)
  - 체크리스트 충족 시 `연도별 FREEZING 실행` 버튼 활성화
- **외부 검증사 접근**
  - 검증사는 통합 데이터만 조회 (계열사 개별 계정 사용 안 함)
  - 검증 질의는 SDS 계정에 남기고, SDS가 필요한 경우 계열사에 재요청

### 2-3. GHG 보고서 출력 (리포트 생성·다운로드)

- **내부용 산정 결과 리포트 조회**
  - Scope 1/2/3별 배출량 테이블, 사업장별/Scope별 그래프
  - 기간 필터(연간, 반기, 분기, 월)
- **보고서 다운로드**
  - PDF 요약본, Excel 상세본, JSON 원본
  - 공시 프레임워크 선택에 따른 포맷 자동 구성(예: K-ETS용, IFRS S2용 등)
- **보고서 승인 요청**
  - 초안 → 검토 → 확정 상태
  - 확정된 버전만 3번 SR 보고서 작성 단계에서 사용 가능

## 3. SR 보고서 작성

> GHG 산정 결과와 ESG 데이터(정량/정성)를 결합하여 **공시/지속가능경영보고서(SR)** 를 작성·관리하는 단계.

### 3-1. 공시데이터 관리 (등록·조회·승인·리마인드)

- **공시 프레임워크별 항목 정의 조회**
  - IFRS S2, KSSB, K-ETS, GRI, ESRS E1 별 필수·선택 공시 항목 리스트 조회
  - 항목별 담당 부서·담당자 매핑
- **공시 항목별 데이터 요청**
  - 각 항목에 대해 “요청 양식” 템플릿 제공
    - 예: `S2-1 GHG 배출량`, `S2-2 기후 리스크`, `E1-5 감축목표` 등
  - 담당자에게 데이터 입력/업로드 **요청 발송**
    - 이메일/인앱 알림 + 요청 기한
- **담당자 입력·등록**
  - 정량 데이터: 수치, 단위, 기준, 주석
  - 정성 데이터: 서술형 텍스트, 첨부파일(근거자료)
  - ESG팀이 초안 작성 후, 각 부서 확인·수정 요청 가능
- **승인·버전 관리**
  - 항목별 상태: 미요청 / 요청중 / 회신완료 / 검토중 / 확정
  - 각 항목 수정 시 버전이력 저장, 이전 버전 비교
- **리마인드·체이싱**
  - 요청 기한 임박/지연 항목 자동 리마인드
  - 개인별 “나에게 온 요청” 인박스 화면 제공

### 3-2. ESG 데이터 (추천·생성·인포그래픽)

- **ESG 데이터 추천**
  - 업종·규모·프레임워크 기반으로 추천 KPI/지표 세트 제안
  - 예: 환경(E) KPI, 사회(S) KPI, 지배구조(G) KPI
- **데이터 생성·등록**
  - 각 KPI별 목표·실적 값 입력(연간/분기/월)
  - 기존 시스템(인사, 안전, 품질 등) 연동 데이터 조회·매핑
  - 원천 데이터 파일·근거 문서 첨부
- **인포그래픽 생성**
  - SR 보고서용 차트·그래프 템플릿 선택
  - 데이터 범위 선택 → 시각화 자동 생성
  - 내보내기(PNG/SVG) 및 보고서 레이아웃에 삽입

### 3-3. SR 보고서 생성 (다운로드·협업)

- **보고서 구조 설계**
  - 목차 템플릿: CEO 메시지, 비즈니스 개요, ESG 전략, E/S/G 챕터, 부록 등
  - 공시 프레임워크별 필요 섹션 자동 포함(IFRS, GRI, ESRS 등)
- **초안 작성·편집**
  - 섹션별 콘텐츠 블록 단위 편집 (텍스트, 표, 차트, 이미지)
  - GHG 모듈/ESG 데이터 모듈에서 가져온 값 자동 삽입
- **협업·검토**
  - 섹션 담당자 지정
  - 코멘트 달기, 변경요청, 수정 이력 확인
  - 상태: 초안 / 검토중 / 확정
- **최종물 생성·다운로드**
  - SR PDF, 웹용 HTML, KRX 제출용 파일(향후)
  - 버전 태깅(예: v1.0, v1.1(감사 의견 반영))

## 4. 대시보드

> ESG 팀장·임원·경영진이 한 화면에서 **공시/배출량/보고서 진행 현황** 을 모니터링하고, 필요한 액션(알림·승인·추진요청)을 바로 수행하는 공간.

### 4-1. 공시 진행률

- **프레임워크별 진행률 위젯**
  - IFRS S2 / KSSB / K-ETS / GRI / ESRS별 공시 항목 완료율
  - 상태별 개수: 미요청 / 요청중 / 회신완료 / 검토중 / 확정
- **액션**
  - 특정 프레임워크/항목 클릭 → 상세 화면 이동
  - 대량 리마인드 발송 (지연 항목 일괄 선택 후 `리마인드 보내기`)

### 4-2. GHG 진행률

- **GHG 산정 단계별 진행률**
  - 데이터 취합 완료율 (I/F 연동 vs 수기 업로드)
  - 이상치 검토 완료율
  - 배출계수 매핑 완료율
  - 산정 결과 승인 상태
- **경고·알림**
  - 검증 미완료 데이터, 누락 사업장, 전년 대비 급증 사업장 표시

### 4-3. SR 진행률

- **SR 보고서 섹션별 완료 상태**
  - 각 챕터(전략, 거버넌스, 리스크, E/S/G 등)별 초안/검토/확정율
  - SR 보고서 마일스톤 일정(초안 제출, 이사회 보고, 공시일 등) 캘린더 표현
- **협업 알림**
  - 나에게 배정된 섹션의 작업 요청/리마인드
  - 승인 대기 섹션 리스트

### 4-4. STATUS (알림·요청 허브)

- **내 알림함**
  - 계정 승인 결과, 권한 변경 결과
  - GHG 데이터 검증 요청·회신
  - 공시 항목 데이터 요청·리마인드
  - SR 섹션 리뷰 요청·결과
- **할 일(To-do) 리스트**
  - 오늘 해야 할 작업, 마감 임박 작업 정리
  - 각 항목에서 바로 상세 화면으로 이동 가능
- **경영진 뷰**
  - “이번 분기/연도 공시 준비 상태” 요약 카드
