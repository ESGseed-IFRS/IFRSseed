🚀 웹사이트 구조 리팩토링 및 내비게이션 연결 작업 지시서

1. 개요
파일 구조 리팩토링에 따라 끊어진 내비게이션을 복구하고, 'SR 보고서 작성' 페이지를 세부 탭 구조로 통합합니다. 특히 내비게이션 메뉴를 정리하여 사용성을 높입니다.

2. 주요 작업 요구사항
A. 내비게이션 바(GNB) 정리 및 연결
메뉴 정리: 상단 내비게이션에서 기존에 따로 존재하던 'IFRS/KSSB 작성' 버튼을 삭제하고, [SR 작성] 버튼 하나만 남깁니다.

링크 복구: 나머지 버튼(회사정보, GHG 산정, 도표 및 그림 생성, 문단생성, CDP 응답 등)을 리팩토링된 신규 파일 경로에 정확히 연결하세요.

디자인 유지: 메뉴 삭제 및 경로 변경 시에도 기존 UI 스타일(글자 크기, 간격, 초록색 포인트 컬러 등)이 변형되거나 글자가 깨지지 않도록 주의하세요.

B. 'SR 보고서 작성' 페이지 내부 탭 시스템 구현
위치: [SR 작성] 메뉴를 클릭해 들어오는 메인 페이지

기능: 페이지 내에서 **[SR 보고서 작성]**과 **[IFRS 보고서 작성]**을 선택할 수 있는 세부 탭(Tab) 기능을 구현하세요.

동작: - 탭 클릭 시 페이지 전체가 리로드되지 않고 내부 콘텐츠만 해당 보고서 양식으로 전환되어야 합니다.

리팩토링으로 분리되었던 개별 페이지 컴포넌트들을 이 탭 구조 안으로 통합하여 반영하세요.

C. UI/UX 및 데이터 안정성
데이터 유지: 사용자가 탭을 전환하더라도 작성 중인 내용이 초기화되지 않도록 상태(State)를 유지해 주세요.

시각적 일관성: 현재 UI 시안의 룩앤필을 그대로 유지하며, 레이아웃이 어긋나지 않도록 정밀하게 조정하세요.

3. 커서(Cursor)를 위한 실행 가이드
상단 GNB 컴포넌트 파일에서 'IFRS/KSSB 작성' 항목을 제거하고 경로를 수정해.

'SR 작성' 페이지 파일에서 useState 등을 활용해 탭 전환 로직을 만들고, 분리된 두 보고서 컴포넌트를 임포트해서 넣어줘.

CSS 작업 시 기존 스타일 시트를 참고해서 글자 깨짐이나 레이아웃 변형이 없게 해줘.
