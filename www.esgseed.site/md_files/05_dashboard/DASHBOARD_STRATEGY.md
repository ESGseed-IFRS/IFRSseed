# 대시보드 페이지 전환 전략

> **목적**: 기존 최종보고서 페이지를 **대시보드**로 전환하고, 전체 플랫폼의 진행 상태·확인 사항·보고서 출력을 한 화면에서 통합 관리할 수 있도록 전략 수립  
> **참조 문서**: `JOURNEYMAP_TOTAL.md`, `JOURNEYMAP_FINAL.md`, `FINAL_REPORT_DATA_STRATEGY.md`, 각 페이지 저니맵

---

## 1. 전환 개요

### 1.1 변경 방향

| 기존 (최종보고서) | 전환 후 (대시보드) |
|------------------|-------------------|
| 보고서 완성도·섹션별 진행 상황 중심 | **전체 플랫폼 진행 상태** + 보고서 완성도 통합 |
| 주로 SR·회사정보·차트 기반 진행률 | **모든 페이지**(회사정보, GHG, SR, 도표, 스마트 문단) 진행 상태 요약 |
| 최종보고서 관련 기능만 제공 | **확인 필요 사항**·**다음 액션**·**빠른 이동** 기능 추가 |
| PDF/Word 다운로드 | 유지 (대시보드 내 보고서 출력 영역) |

### 1.2 대시보드의 핵심 역할

1. **진행 상태 한눈에 보기**: 각 페이지별 완료·진행중·대기중 상태 요약
2. **확인 필요 사항 알림**: 미완료·미제출·검증 필요 항목 안내
3. **빠른 이동(Shortcut)**: 해당 페이지로 바로가기 링크
4. **보고서 출력**: 기존 최종보고서 기능(미리보기·PDF/Word 다운로드) 유지

## 2. 페이지별 대시보드 연동 전략

### 2.1 전체 페이지 매핑

| 순서 | 페이지 | 라우트 | 대시보드에 표시할 진행 상태 | 확인 필요 시 표시할 내용 |
|------|--------|--------|---------------------------|--------------------------|
| 1 | **회사정보** | `/company-info` | 기업 기본정보·연락처·ESG 목표·이해관계자 입력·제출 여부 | N개 필드 미입력, 최종 보고서 미제출 |
| 2 | **GHG 산정** | `/ghg-calculation` | 산정 결과 저장 여부, Scope 1·2·3 배출량 존재 여부, 리포트 생성 완료 여부 | 활동자료 미입력, 결과 미저장, Evidence Report 미생성 |
| 3 | **SR 작성** | `/report` | 목차별 작성·저장 여부, 전체 준수율, 데이터 최종 제출 여부 | 미준수 공시 기준, 미제출 지표, 준수율 90% 미만 |
| 4 | **스마트 문단생성** | `/content` | IFRS S2 체크리스트 진행률, 최종보고서 저장 문단 수 | 필수 정보 미체크, 저장된 문단 0건 |
| 5 | **도표 및 그림 생성** | `/charts` | Environmental/Social/Governance 차트·도표 저장 수 | 저장된 차트 0건, 주요 섹션별 추천 차트 미생성 |
| 6 | **보고서 출력** | (대시보드 내) | 보고서 완성도, PDF/Word 다운로드 가능 여부 | 회사정보 미제출, 필수 섹션 누락 |

### 3.2 블록별 상세

#### 블록 A: 전체 진행률 (헤더)

| 항목 | 설명 |
|------|------|
| **전체 진행률** | 페이지별 완료 여부 기반 퍼센트 (예: 6개 페이지 중 5개 필수 완료 = 83%) |
| **완료/대기/진행중** | 카운트 배지: 완료(모든 필수 항목 충족), 진행중(일부 완료), 대기(미시작) |
| **데이터 소스** | 각 페이지 store·API에서 상태 집계 |

#### 블록 B: 페이지별 진행 상태 카드

각 카드에 표시할 내용:

| 페이지 | 진행 상태 표시 | 퍼센트/배지 | 바로가기 버튼 |
|--------|---------------|-------------|---------------|
| **회사정보** | 완료 = 최종 제출됨 / 진행중 = 일부 입력 / 대기 = 미입력 | ✓ / ◐ / ○ | [회사정보로 이동] |
| **GHG 산정** | 완료 = 결과 저장 + 리포트 생성 / 진행중 = 활동자료 입력 중 / 대기 = 미시작 | ✓ / ◐ / ○ | [GHG 산정으로 이동] |
| **SR 작성** | 완료 = 데이터 최종 제출 + 준수율 90%↑ / 진행중 = 일부 작성 / 대기 = 미작성 | ✓ / ◐ / ○ | [SR 작성으로 이동] |
| **스마트 문단** | 완료 = 저장 문단 N개 + IFRS 체크 완료 / 진행중 = 작성 중 / 대기 = 미이용 | ✓ / ◐ / ○ | [스마트 문단으로 이동] |
| **도표 생성** | 완료 = 차트·도표 N개 저장 / 진행중 = 생성 중 / 대기 = 미생성 | ✓ / ◐ / ○ | [도표 생성으로 이동] |

#### 블록 C: 확인 필요 사항 · 다음 액션

| 조건 | 표시 메시지 | 이동 링크 |
|------|-------------|-----------|
| 회사정보 미제출 | "회사정보가 최종 보고서에 제출되지 않았습니다. PDF 생성을 위해 먼저 제출해 주세요." | `/company-info` |
| GHG 결과 미저장 | "GHG 산정 결과를 저장해 주세요. SR 작성의 정량 데이터에 반영됩니다." | `/ghg-calculation` |
| SR 준수율 90% 미만 | "SR 작성: N개 공시 기준 미준수 (현재 O%). 90% 이상 달성을 권장합니다." | `/report` |
| SR 데이터 미제출 | "데이터 최종 제출이 필요합니다. 인덱스별 데이터 입력·조회 후 제출해 주세요." | `/report` |
| 차트 0건 | "저장된 차트가 없습니다. 환경 성과 등 시각화를 추가하면 보고서 품질이 향상됩니다." | `/charts` |
| 필수 섹션 누락 | "보고서 필수 섹션: 회사정보, 환경 성과, 지속가능경영 전략 등이 비어 있습니다." | 해당 페이지 |

- **우선순위**: 회사정보 제출 > GHG 저장 > SR 준수/제출 > 차트·문단
- **표시 개수**: 최대 5~7개 (스크롤 시 전체)

#### 블록 D: 보고서 출력 (기존 최종보고서)

- **섹션별 진행 상황**: 회사정보, 지속가능경영 전략, 환경 성과, 사회적 책임, 지배구조, 성과 지표, 향후 계획
- **보고서 미리보기 · 목차 · 품질 체크리스트**: `JOURNEYMAP_FINAL.md`, `FINAL_REPORT_DATA_STRATEGY.md` 유지
- **PDF/Word 다운로드**: 기존 동작 그대로

## 4. 페이지별 진행 상태 연동 상세

### 4.1 회사정보 (`/company-info`)

| 상태 | 판단 기준 | 대시보드 표시 |
|------|----------|---------------|
| **완료** | `finalCompanyInfo` 존재, 최종 보고서 제출됨 | ✓ 회사정보 제출 완료 |
| **진행중** | 일부 필드 입력, 검증 통과 또는 N개 필드 미입력 | ◐ N개 필드 보완 필요 |
| **대기** | 입력 없음 또는 검증 미실행 | ○ 회사정보 입력 필요 |

**확인 필요 시**: "N개 필드가 비어 있습니다", "최종 보고서에 제출되지 않았습니다"

### 4.2 GHG 산정 (`/ghg-calculation`)

| 상태 | 판단 기준 | 대시보드 표시 |
|------|----------|---------------|
| **완료** | Scope 1·2·3 결과 저장됨, Evidence Report 또는 리포트 생성 탭 사용 이력 | ✓ GHG 산정 완료 |
| **진행중** | 활동자료 일부 입력, 결과 미저장 | ◐ 결과 저장 필요 |
| **대기** | 활동자료 없음 | ○ GHG 데이터 입력 필요 |

**확인 필요 시**: "활동자료를 입력한 후 결과 저장을 클릭해 주세요", "리포트 생성 탭에서 Evidence Report를 다운로드해 주세요"

### 4.3 SR 작성 (`/report`)

| 상태 | 판단 기준 | 대시보드 표시 |
|------|----------|---------------|
| **완료** | 데이터 최종 제출됨, 전체 준수율 90% 이상 | ✓ SR 작성 완료 |
| **진행중** | 목차별 일부 작성, 준수율 60~89% 또는 데이터 미제출 | ◐ O% 준수, 제출 필요 |
| **대기** | 작성 없음 | ○ SR 작성 시작 필요 |

**확인 필요 시**: "N개 공시 기준 미준수", "데이터 최종 제출이 필요합니다"

### 4.4 스마트 문단생성 (`/content`)

| 상태 | 판단 기준 | 대시보드 표시 |
|------|----------|---------------|
| **완료** | `contentSections` 저장 N건, IFRS S2 체크리스트 필수 항목 충족 | ✓ 문단 N개 저장됨 |
| **진행중** | 문단 생성·수정 중, 일부만 저장 | ◐ 저장 완료 필요 |
| **대기** | 미사용 | ○ (선택) 문단 생성 권장 |

**확인 필요 시**: "IFRS S2 필수 정보 N개 미체크"

### 4.5 도표 및 그림 생성 (`/charts`)

| 상태 | 판단 기준 | 대시보드 표시 |
|------|----------|---------------|
| **완료** | 차트 N개 + 도표 N개 저장됨 | ✓ 차트 N개, 도표 N개 |
| **진행중** | 일부만 저장 | ◐ 시각화 추가 권장 |
| **대기** | 저장 0건 | ○ 차트·도표 생성 권장 |

**확인 필요 시**: "환경 성과 섹션에 온실가스 차트 추가를 권장합니다"

## 5. 데이터 흐름 및 store 연동

### 5.1 대시보드가 참조할 데이터 소스

| 데이터 | 소스 | 용도 |
|--------|------|------|
| 회사정보 제출 여부 | `finalCompanyInfo` | 블록 B·C |
| GHG 결과·저장 여부 | GHG store (scope1/2/3, resultSaved) | 블록 B·C |
| SR 준수율·제출 여부 | report store (complianceRate, dataSubmitted) | 블록 B·C |
| 스마트 문단 저장 수 | `contentSections` | 블록 B |
| 차트·도표 저장 수 | `charts`, `esgTables`, `renewableTable` | 블록 B |
| 섹션별 진행 상황 | 기존 `FINAL_REPORT_DATA_STRATEGY` §3 | 블록 D |

### 5.2 추천 API/훅

- `useDashboardStatus()`: 위 데이터를 종합해 `{ overallProgress, pageStatuses, actionItems }` 반환
- `pageStatuses`: `{ companyInfo, ghg, sr, content, charts }` 각각 `{ status, percent?, message?, link }`
- `actionItems`: `{ id, priority, message, link, page }[]`
