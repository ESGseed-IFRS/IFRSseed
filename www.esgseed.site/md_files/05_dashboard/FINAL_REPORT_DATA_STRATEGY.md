# 최종보고서 탭 — 플랫폼에서 가져갈 내용 전략

본 문서는 **JOURNEYMAP_TOTAL**, **JOURNEYMAP_SR_REPORT**, **JOURNEYMAP_FINAL**을 참고하여, 우리 플랫폼의 **최종보고서** 탭에서 **어떤 데이터를 어디서 가져와** 통합·다운로드할지 전략을 정리합니다.

---

## 1. 전제: 최종보고서의 역할

- **목적**: 로그인 → 회사정보 → GHG 산정 → SR 작성 → 도표·그림 생성까지의 **산출물을 하나의 지속가능경영 보고서**로 통합해 검토·다운로드(PDF/Word)할 수 있게 함.
- **사용자 관점**: "보고서 완성도·섹션별 진행 상황 확인 → 미리보기·품질 체크 → PDF/Word 다운로드" (JOURNEYMAP_FINAL §2–3).
- **데이터 관점**: 각 메뉴에서 **"최종 보고서에 제출"** 또는 **"저장"**한 데이터만 최종보고서에 반영되며, PDF 생성 시 해당 데이터를 **한 번에 가져와** 문서에 넣는 구조가 적합합니다.

---

## 2. 최종보고서가 가져갈 데이터 소스 (권장)

아래는 **저니맵·현재 구현**을 기준으로, 최종보고서 탭이 **가져가는 것이 좋은 내용**을 출처별로 정리한 것입니다.

| 순서 | 데이터 종류 | 출처(메뉴/페이지) | 사용자 행동(반영 조건) | 최종보고서에 포함할 내용 | 비고 |
|------|-------------|-------------------|------------------------|---------------------------|------|
| 1 | **회사정보** | 회사정보 (`/company-info`) | **최종 보고서에 제출** 클릭 | 기업 기본정보(회사명, 사업자등록번호, 대표자명, 업종), 연락처(주소, 전화, 이메일, 웹사이트), ESG 목표·비전, 이해관계자 정보 | 필수. 미제출 시 PDF 다운로드 비활성 또는 경고. 현재 `finalCompanyInfo`로 반영됨 |
| 2 | **문단/본문** | SR 작성 (`/report`) | 목차별 **내용** 입력 후 **저장** · (선택) **최종보고서에 저장** | 목차(Table of Contents) 구조에 맞는 문단 텍스트, 정량 데이터(GHG 배출량 tCO₂e, 전년 배출량 등) | SR 본문이 최종보고서 저장소와 연동되도록 설계. 현재는 Content의 `contentSections` 등과 별도일 수 있음 → **통합 전략** 필요 |
| 3 | **문단/본문** | 스마트 문단생성·Content (`/content`) | **최종보고서에 저장** 클릭 | IFRS S2 등 공시 기준에 맞게 생성·보완한 문단(제목, 본문, ifrsCode). 실시간 보고서 문단 프리뷰에 표시된 내용 | 현재 `contentSections`(PreviewSection)로 저장·PDF에 포함. SR 작성 문단과 동일 저장소 또는 매핑 규칙 필요 |
| 4 | **차트·도표** | 도표 및 그림 생성 (`/charts`) | **최종보고서에 저장** (또는 차트/도표 갤러리에서 저장) | Environmental·Social·Governance 탭에서 생성한 차트(막대·원형·선 등) 이미지·데이터, **도표 갤러리** 테이블 | 현재 `charts`, `renewableTable`, `esgTables` 등. PDF에 차트 이미지·재생에너지 표·ESG 테이블 포함 |
| 5 | **GHG 산정 결과** | GHG 산정 (`/ghg-calculation`) | 산정 결과·리포트 생성 탭에서 **결과 저장** 또는 **Evidence Report/검증의견서 부속 서류** 다운로드 후, SR/최종보고서에서 **참조** | Scope 1·2·3 총 배출량(tCO₂e), 전체 사업장 합계, 적용 배출계수·산식 요약, (선택) 검증용 요약 | 저니맵: "SR 문단·차트·최종보고서에서 GHG 결과 참조 가능". SR 작성의 **정량 데이터**에 수동 입력 또는 GHG 스토어 연동으로 자동 반영 권장. 최종보고서 PDF에 "온실가스 배출량 요약" 등으로 포함 |
| 6 | **메타데이터** | 최종보고서 페이지 또는 공통 설정 | 보고 연도·보고 기간·언어·최종 수정일 | 표지·헤더: 보고 연도, 회사명, IFRS 기준 준수, 발행일, 보고 기간, 언어 | 현재 `reportMetadata` |

## 3. 섹션별 진행 상황과 가져갈 내용 매핑

JOURNEYMAP_FINAL·JOURNEYMAP_SR_REPORT에 나온 **섹션별 진행 상황**과, **어디서 데이터를 가져오는지**를 맞춰 두면, "어디가 비었는지" 안내와 품질 체크리스트 연동에 유리합니다.

| 섹션(진행 상황 카드) | 가져갈 데이터 소스 | 비고 |
|---------------------|---------------------|------|
| 회사정보 | `finalCompanyInfo` (회사정보 페이지 → 최종 보고서에 제출) | 완료 = 제출됨 |
| 지속가능경영 전략 | SR 작성 목차 해당 문단 + Content에서 저장한 문단 | 진행률 = 해당 목차 항목 작성·저장 여부 |
| 환경 성과 | SR 작성(온실가스 관리 등) + GHG 산정 결과 요약 + 환경 차트·도표 | GHG 결과 연동 시 자동 반영 권장 |
| 사회적 책임 | SR 작성 해당 문단 + Social 차트·도표 | |
| 지배구조 | SR 작성 해당 문단 + Governance 차트·도표 | |
| 성과 지표 | SR 작성·Content 문단 + ESG Data·도표 갤러리 테이블 | |
| 향후 계획 | SR 작성 해당 문단 | |

- **진행률 계산**: 위 소스별로 "데이터 존재 여부" 또는 "저장/제출 여부"를 합산해 완료/진행중/대기중을 구하면 됩니다.

---

## 4. 품질 체크리스트와 데이터 연동

JOURNEYMAP_FINAL §2의 **품질 체크리스트** 항목을, 실제 플랫폼 데이터와 연결하면 "최종 점검" 단계가 명확해집니다.

| 체크리스트 항목 | 연동할 데이터/조건 |
|-----------------|---------------------|
| IFRS 기준 준수 | SR 작성 **전체 준수율** 또는 Content IFRS S2 체크리스트 결과. (확장) 공시 기준별 준수 여부 |
| GRI 표준 적용 | SR 작성 목차·공시 기준 매핑에서 GRI 항목 작성·저장 여부 |
| 데이터 정확성 검증 | 회사정보 검증 완료, GHG 정량 데이터 입력 여부, 차트 데이터 소스 명시 |
| 이해관계자 검토 | 회사정보 이해관계자 섹션 입력 여부 (확장 시 검토 상태) |
| 법적 요구사항 충족 | 필수 섹션(회사정보, 환경 성과 등) 제출·작성 여부 |
| 외부 감사 완료 | (확장) GHG 검증의견서·리포트 생성 다운로드 여부 등 |

- 체크리스트는 **실제 저장·제출 데이터**와 연동되도록 하면, "미체크 시 어디서 채울지" 링크(회사정보/SR 작성/GHG 등)를 줄 수 있습니다.

---

## 5. 통합·연동 시 권장 사항

1. **단일 저장소 또는 명확한 매핑**  
   SR 작성 문단과 Content의 `contentSections`가 **동일한 최종보고서 본문**을 구성하도록, 하나의 저장소를 쓰거나 목차 ID·ifrsCode 기준으로 매핑 규칙을 두는 것이 좋습니다.

2. **GHG 결과 자동 반영**  
   GHG 산정 결과(Scope 1·2·3 tCO₂e, 전체 사업장 합계)를 SR 작성 **정량 데이터** 또는 최종보고서 전용 필드로 자동 채우면, 사용자 재입력 없이 "환경 성과" 등에 반영할 수 있습니다.

3. **필수 제출 순서 안내**  
   저니맵 순서대로 "① 회사정보 입력 후 최종 보고서에 제출 → ② SR 작성(목차별 내용)·(선택) 스마트 문단생성·도표 → ③ 최종보고서에서 PDF 다운로드"를 최종보고서 페이지에 짧게 안내하면, "어떤 내용을 가져가는지" 사용자가 이해하기 쉽습니다.

4. **미리보기·목차와 데이터 일치**  
   "보고서 미리보기"·"목차"에 표시되는 항목이 실제로 가져가는 데이터(회사정보, 문단 개수, 차트 개수, GHG 요약 포함 여부)와 일치하도록 하면, 다운로드 전 기대치를 맞출 수 있습니다.

---

## 6. 요약

- 최종보고서 탭이 **가져가는 내용**은 (1) 회사정보, (2) SR 작성·Content 문단, (3) 차트·도표·테이블, (4) GHG 산정 결과 요약, (5) 메타데이터로 정리할 수 있습니다.
- 각 항목은 **해당 메뉴에서 "최종 보고서에 제출" 또는 "저장"**이 된 데이터만 가져오며, **섹션별 진행 상황**·**품질 체크리스트**는 이 데이터와 연동해 완성도와 품질을 표시하는 전략이 적합합니다.
- 구현 시에는 **reportStore**(또는 동일한 역할의 저장소)에 회사정보·문단·차트·GHG 요약이 일관되게 쌓이도록 하고, 최종보고서 페이지는 해당 데이터만 읽어 PDF/Word를 생성하는 구조를 유지하면 됩니다.
