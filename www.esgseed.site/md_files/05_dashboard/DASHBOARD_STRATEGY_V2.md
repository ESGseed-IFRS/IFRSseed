# 대시보드 기능 전략

> **목적**: ESG 보고서 작성 전체 현황을 한눈에 파악하고, 필요한 작업을 빠르게 진행할 수 있는 대시보드 기능 정의  
> **환경**: On-premise  
> **사용자**: ESG 팀장 / ESG 팀원

---

## 1. 사용자 유형

| 구분 | ESG 팀장 | ESG 팀원 |
|------|----------|----------|
| 역할 | 최종 결재·승인 | 섹션별 작성 담당 |
| 추가 권한 | 팀원 승인, 최종 보고서 승인 | 담당 섹션 편집, 검토 요청 제출 |
| 다운로드 | ✅ 최종 승인 후 | ✅ 팀장 승인 완료 후 |

---

## 2. 페이지 카드 구성 (4개)

| 카드 | 라우트 | 완료 조건 | 진행중 조건 | 대기 조건 |
|------|--------|-----------|-------------|-----------|
| **회사정보** | `/company-info` | 최종 제출 완료 | 일부 입력, 미제출 | 미입력 |
| **GHG 산정** | `/ghg-calculation` | Scope 1·2·3 저장 + 리포트 생성 | 활동자료 입력 중, 결과 미저장 | 미시작 |
| **SR 작성** | `/report` | 준수율 90%↑ + 데이터 최종 제출 | 일부 작성, 준수율 90% 미만 | 미작성 |
| **도표 생성** | `/charts` | 차트·도표 1건↑ 저장 | 일부 저장 | 저장 0건 |

### SR 작성 카드 진행률

SR 작성 카드의 진행률은 **목차별 공시 기준 작성 완료 비율(준수율)** 을 그대로 사용한다.

- 준수율 = 작성 완료된 공시 기준 수 ÷ 전체 공시 기준 수 × 100
- 완료 기준: 준수율 90% 이상 + 데이터 최종 제출

## 3. 대시보드 블록 구성

### 블록 1: 전체 진행률

- 4개 카드 완료 여부 기반 퍼센트 표시
- 완료 / 진행중 / 대기 카운트 배지
- 마감일 설정 및 D-Day 카운트다운 (팀장 설정, 전체 표시)

### 블록 2: 페이지별 진행 상태 카드 (4개)

각 카드 구성:
- 완료 ✓ / 진행중 ◐ / 대기 ○ 배지
- 간단한 상태 메시지 및 진행률 퍼센트
- [이동 →] 바로가기 링크
- 담당 팀원 이름 표시 (팀장 뷰에서만)

### 블록 3: 확인 필요 사항 · 다음 액션

| 우선순위 | 조건 | 메시지 |
|----------|------|--------|
| 1 | 팀원 가입 승인 대기 (팀장만) | "N명의 팀원 가입 승인이 필요합니다." |
| 2 | 회사정보 미제출 | "회사정보를 최종 보고서에 제출해 주세요." |
| 3 | GHG 결과 미저장 | "GHG 산정 결과를 저장해 주세요." |
| 4 | SR 준수율 90% 미만 | "SR 작성: N개 기준 미준수 (현재 O%)." |
| 5 | SR 데이터 미제출 | "데이터 최종 제출이 필요합니다." |
| 6 | 차트 0건 | "저장된 차트가 없습니다." |
| 7 | 팀장 수정 요청 수신 (팀원만) | "팀장이 N개 섹션 수정을 요청했습니다." |

각 항목에 [해당 페이지로 이동] 링크 포함.

### 블록 4: 팀원 현황 (팀장 전용)

- 팀원 목록: 이름 / 소속 / 담당 카드 / 마지막 활동 시각 / 상태 배지
- [팀원 초대] 버튼
- 가입 승인 대기 목록: [승인] / [반려] 버튼

### 블록 5: 피드백 수신함 (팀원 전용)

- 팀장이 남긴 수정 요청 / 승인 결과 알림
- [확인 후 재요청] 버튼

### 블록 6: 보고서 출력

#### 보고서 완성도 스코어카드

4개 카드 상태를 한 줄 요약으로 표시. 모든 항목 충족 시 팀장의 [최종 승인] 버튼 활성화.

| 항목 | 충족 | 미충족 |
|------|------|--------|
| 회사정보 제출 | ✓ | ✗ |
| GHG 산정 결과 저장 | ✓ | ✗ |
| SR 준수율 90% 이상 | ✓ (현재 O%) | ⚠ (현재 O%) |
| 차트·도표 저장 | ✓ (N건) | ✗ (0건) |

#### 섹션별 진행 상황

회사정보 / 지속가능경영 전략 / 환경 성과 / 사회적 책임 / 지배구조 / 성과 지표 / 향후 계획

#### 다운로드

| 형식 | 용도 | 활성화 조건 |
|------|------|-------------|
| **Excel (.xlsx)** | GHG 수치·ESG 성과 지표 데이터 중심 | 팀장 최종 승인 후 팀장·팀원 모두 |
| **PowerPoint (.pptx)** | 경영진 보고용 슬라이드 형식 | 팀장 최종 승인 후 팀장·팀원 모두 |

- 보고서 미리보기는 승인 전후 모두 열람 가능 (팀장·팀원 공통)
- [최종 승인] 버튼은 팀장 전용, 승인 완료 시 전체 팀원에게 인앱 알림 발송

### 블록 7: 최근 활동 로그

- **팀장**: 팀 전체 활동 타임라인 표시
- **팀원**: 본인 활동 로그만 표시
- 표시 형식: `[이름] · [작업 내용] · [N분/시간/일 전]`
- 예시:
  - 홍길동 · GHG 산정 결과 저장 · 2시간 전
  - 이영희 · SR 작성 데이터 제출 · 어제
  - 김철수 · 도표 생성 차트 3건 저장 · 3일 전
- 최대 10건 표시, [전체 로그 보기] 링크 제공

## 6. 화면 구현 방식

### 단일 라우트 + 조건부 렌더링

라우트는 `/dashboard` 하나로 통일하고, 로그인한 사용자의 `role`에 따라 블록을 조건부 렌더링한다. 역할별 별도 페이지(`/dashboard/manager`, `/dashboard/member`)는 만들지 않는다.

```tsx
// 공통 블록은 항상 렌더링
<OverallProgress />       // 블록 1
<PageStatusCards />       // 블록 2
<ActionItems />           // 블록 3
<ReportExport />          // 블록 6
<ActivityLog />           // 블록 7

// 팀장 전용
{role === 'MANAGER' && <TeamMemberPanel />}   // 블록 4
{role === 'MANAGER' && <ApprovalQueue />}     // 블록 4

// 팀원 전용
{role === 'MEMBER' && <FeedbackInbox />}      // 블록 5
```

| 블록 | 팀장 | 팀원 |
|------|------|------|
| 블록 1 전체 진행률 | ✅ | ✅ |
| 블록 2 페이지별 카드 | ✅ (담당자 이름 포함) | ✅ |
| 블록 3 확인 필요 사항 | ✅ 전체 항목 | ✅ 본인 담당 항목만 |
| 블록 4 팀원 현황 + 가입 승인 | ✅ | ❌ 미표시 |
| 블록 5 피드백 수신함 | ❌ 미표시 | ✅ |
| 블록 6 보고서 출력 | ✅ (최종 승인 버튼 포함) | ✅ (승인 후 다운로드만) |
| 블록 7 활동 로그 | ✅ 전체 팀원 로그 | ✅ 본인 로그만 |
