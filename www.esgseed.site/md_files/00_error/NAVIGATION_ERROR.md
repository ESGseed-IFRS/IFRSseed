### 🚨 [긴급] 무한 루프 에러 및 성능 저하 해결 요청

**1. 에러 로그 분석 결과:**
`useReportLogic.ts:76`에서 `Maximum update depth exceeded` 에러가 발생하고 있습니다. 
`SRReportPage`가 렌더링될 때 `useReportLogic` 내부의 `useEffect`가 무한 루프를 유발하여 페이지가 프리징(Freezing)되는 현상입니다.

**2. 요청 사항:**
- **useReportLogic.ts (76라인 근처) 수정:** - `useEffect` 내에서 `setState`를 호출할 때 조건문을 추가하여 값이 실제로 변했을 때만 업데이트하도록 수정하세요.
  - 의존성 배열(`dependency array`)에 매번 새로 생성되는 객체나 함수가 들어있는지 확인하고 `useMemo`나 `useCallback`으로 최적화하세요.
- **SR 작성/IFRS 작성 탭 전환 로직 최적화:**
  - 탭을 바꿀 때마다 초기화 로직이 중복 실행되어 무한 루프를 트리거하는지 확인하세요.
- **내비게이션 연결 복구:**
  - 위 무한 루프가 해결되면 내비게이션 클릭 시의 버퍼링도 사라질 것입니다. 모든 GNB 버튼이 리팩토링된 경로로 정상 작동하는지 최종 확인하세요.

**3. 주의사항:**
- UI 레이아웃과 한글 텍스트가 깨지지 않도록 기존 CSS를 유지하며 로직만 수정하세요.
