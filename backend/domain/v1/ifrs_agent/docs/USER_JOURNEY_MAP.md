# IFRSseed AI 플랫폼 사용자 저니맵 (User Journey Map)

## 개요

이 문서는 IFRSseed AI 플랫폼의 각 탭별 사용자 여정을 단계별로 정리한 저니맵입니다. 사용자가 플랫폼을 처음 접속하여 최종 보고서를 생성하기까지의 전체 프로세스를 시각화합니다.

---

## 저니맵 구조

- **가로축 (Columns)**: 각 탭/페이지
- **세로축 (Rows)**: 사용자 입장에서의 단계별 사용 순서

---

## 전체 저니맵 테이블

| 단계                                | 로그인/회원가입                                                       | 회사정보                                                                                                                                                   | 문단생성                                                     | 도표 및 그림 생성                                        | 최종보고서                                                                         |
| ----------------------------------- | --------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------ | -------------------------------------------------------- | ---------------------------------------------------------------------------------- |
| **1단계: 진입/인식**          | 플랫폼 접속 → "로그인해야 이용 가능"하다는 것 인식                   | 로그인 완료 후 회사정보 탭 클릭                                                                                                                            | 회사정보 입력 완료 후 문단생성 탭 클릭                       | 문단 생성 완료 후 도표 생성 탭 클릭                      | 모든 데이터 입력 완료 후 최종보고서 탭 클릭                                        |
| **2단계: 가입/등록 선택**     | **회사 미등록**: 회사 등록 / **임원**: 최종 등록(사전 등록된 사람만) / **기존 회원**: 로그인 | 회사정보 입력 폼 구조 파악 (4개 섹션)                                                                                                                      | 문단생성 페이지 레이아웃 확인 (입력창, 체크리스트, 미리보기) | 도표 생성 페이지 구조 확인 (차트 타입 선택, 데이터 입력) | 최종보고서 페이지 레이아웃 확인 (진행률, 미리보기, 다운로드)                       |
| **3단계: 기본정보 입력**      | 임원 최종 등록: **이름·이메일**(사전 등록 매칭) → 인증 후 비밀번호; 기존 회원: 이메일·비밀번호 로그인 | 기업 기본정보 섹션부터 입력 시작                                                                                                                           | ESG/TCFD 텍스트 입력 또는 파일 업로드                        | 차트 타입 선택 (막대/선/파이/표)                         | 보고서 완성도 확인 및 데이터 검증                                                  |
| **4단계: 회사정보 확인/입력** | DART API로 불러온 회사정보 확인 (조회 전용)                           | **수정 권한**: 4개 섹션 순차 입력/수정 `<br>`**조회 권한**: 4개 섹션 정보 조회만 가능 (읽기 전용)                                            | 텍스트 입력 후 분석 버튼 클릭                                | 데이터 포인트 입력 (연도별 값)                           | 포함된 데이터 확인:`<br>`- 회사정보 `<br>`- AI 생성 문단 `<br>`- 저장된 차트 |
| **5단계: DART API 연동**      | DART API로 회사 공시정보 자동 불러오기 및 확인                        | 실시간 미리보기에서 입력 데이터 확인                                                                                                                       | AI 분석 결과 확인 (체크리스트, 검증 이슈)                    | 차트 미리보기 확인 및 조정                               | 섹션별 진행 상황 확인                                                              |
| **6단계: 인증 및 권한 부여**  | 임원: 이메일 인증 완료 + 최초 관리자가 사전 등록한 권한 확인(읽기 전용) | **수정 권한**: "정보 확인 및 검증" 버튼 클릭 → 검증 완료 후 "최종 보고서에 제출" 클릭 `<br>`**조회 권한**: 회사정보 확인 완료 체크박스 클릭 | 생성된 문단을 "보고서에 추가" 버튼으로 저장                  | "차트 저장" 버튼 클릭하여 최종보고서에 포함              | "PDF 다운로드" 버튼 클릭                                                           |
| **7단계: 서비스 이용 시작**   | 회원가입 완료 → 권한 레벨에 따른 기능 접근 가능, DART 정보 자동 반영 | **수정 권한**: 제출 완료 메시지 확인 및 최종보고서 탭으로 이동 가능 `<br>`**조회 권한**: 회사정보 확인 완료 메시지 표시                      | 저장된 문단이 최종보고서에 포함됨 확인                       | 저장된 차트가 최종보고서에 포함됨 확인                   | PDF 다운로드 완료 및 보고서 생성 완료                                              |

## 상세 저니맵: 담당 탭별 상세 분석

### 1. 로그인 탭 (LoginPage)

#### 사용자 여정 단계별 상세

| 단계                                          | 사용자 행동                                                                                                                                                                                                                                                                                                                                | 화면 상태                                                                   | 감정/생각                                           | 개선 포인트                                          |
| --------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | --------------------------------------------------------------------------- | --------------------------------------------------- | ---------------------------------------------------- |
| **1. 진입 및 인식**                     | 홈페이지 접속 후 플랫폼 첫 화면 확인                                                                                                                                                                                                                                                                                                       | 로그인/회원가입 화면 표시                                                   | "이 플랫폼을 이용하려면 먼저 로그인해야 하는구나"   | 플랫폼 가치 제안 명확히 표시                         |
| **2. 회사 등록 또는 가입 경로 선택**    | "회사 등록", "임원 최종 등록(회원가입)", "로그인" 중 선택                                                                                                                                                                                                                                                                                     | 회사 등록 / 임원 최종 등록 / 로그인 화면으로 이동                              | "회사가 없으면 등록하고, 임원은 초대·사전등록 후 가입하는구나" | 회사 등록·임원 최종 등록·로그인 구분 및 안내 문구        |
| **3. 회사 등록 - 회사정보 입력**        | 회사명, 사업자등록번호 입력                                                                                                                                                                                                                                                                                                                | DART API 연동 안내 메시지 표시                                              | "회사 정보를 입력하면 자동으로 불러올 수 있나?"     | 사업자등록번호 형식 검증 (000-00-00000)              |
| **4. 회사 등록 - DART API 연동**        | "회사정보 자동 불러오기" 버튼 클릭                                                                                                                                                                                                                                                                                                         | DART API 호출, 회사 공시정보 자동 조회                                      | "자동으로 회사 정보가 채워지니까 편하다"            | 로딩 상태 표시 및 API 실패 시 수동 입력 옵션 제공    |
| **5. 회사 등록 - 회사정보 확인**        | DART API로 불러온 회사 기본정보 확인 (조회 전용, 수정 불가):`<br>`• 회사명 확인 `<br>`• 대표자명 확인 `<br>`• 본사 주소 확인 `<br>`• 업종 확인 `<br>`• 대표이사 이메일 확인 (초대 링크 발송용)`<br><br>`**목적**: 회사 등록 시 본인이 속한 회사가 맞는지만 확인                                                                                                         | 자동 입력된 회사정보가 읽기 전용으로 표시, 수정 불가                        | "우리 회사 정보가 맞는지 확인하자"                  | 자동 입력된 필드 강조 표시, "확인 완료" 체크박스     |
| **6. 회사 등록 완료 및 초대 링크 발송** | 회사 등록 완료:`<br>`• 회사 ID 생성`<br>`• 대표이사 이메일로 초대 링크 자동 발송`<br>`• "초대 링크 발송 완료" 메시지 표시`<br>`• 초대 링크 유효기간 안내 (예: 7일)`<br><br>`**참고**: 회사 등록은 HRIS 불필요, 최초 관리자 생성은 초대 링크 방식 | 회사 등록 완료 메시지, 초대 링크 발송 완료 안내, 대표이사 이메일 표시 (마스킹) | "회사 등록이 완료되었고, 대표이사에게 초대 링크가 발송되었구나" | 초대 링크 재발송 옵션, 유효기간 표시, 초대 링크 상태 확인 |
| **7. 최초 관리자 - 초대 링크 수신**     | 대표이사 이메일로 초대 링크 수신:`<br>`• 이메일 제목: "[IFRSseed AI] 회사 등록 완료 - 관리자 초대"`<br>`• 초대 링크 클릭`<br>`• 초대 링크 페이지로 이동 | 초대 링크 페이지 표시, 회사 정보 확인 (회사명, 대표자명), "관리자로 등록하기" 버튼 표시 | "회사가 등록되었고, 내가 관리자로 등록해야 하는구나" | 초대 링크 유효기간 표시, 회사 정보 확인               |
| **8. 최초 관리자 - 비밀번호 설정**      | 비밀번호 설정:`<br>`• 비밀번호 설정 (특수문자, 영문, 숫자 포함, 8~14자)`<br>`• 비밀번호 확인 입력`<br>`• 비밀번호 강도 표시`<br>`• "비밀번호 생성 완료" 버튼 클릭 | 비밀번호 입력 필드, 비밀번호 확인 입력 필드, 비밀번호 강도 표시, 보안 규칙 안내 | "안전한 비밀번호를 설정하자"                       | 비밀번호 보안 강도 체크, 실시간 강도 표시, 보안 규칙 안내 |
| **9. 최초 관리자 - 계정 생성 완료**     | 최초 관리자 계정 생성:`<br>`• 이메일, 비밀번호 저장 (암호화)`<br>`• 자동으로 "최종 승인권자" 역할 부여`<br>`• 최초 관리자 플래그 설정`<br>`• 관리자 대시보드로 이동 | 최초 관리자 등록 완료 메시지, "최종 승인권자" 역할 안내, 관리자 대시보드 표시 | "이제 관리자로 로그인해서 임원을 등록하고 관리할 수 있겠다" | 관리자 기능 안내, 임원 사전 등록 가이드 제공            |
| **10. 최초 관리자 - 임원 사전 등록**     | 관리자 대시보드(또는 사용자 관리)에서 회사 계정을 쓸 임원을 사전 등록:`<br>`• 임원 **이름** 입력`<br>`• 임원 **이메일** 입력`<br>`• **권한(역할)** 지정 (예: 최종 승인권자, ESG팀, 현업팀, 일반 사용자 등 4단계 역할 체계)`<br>`• 저장 시 해당 이메일로 **최종 등록 안내**(선택: 초대 링크 또는 안내 메일) | 임원 목록 테이블, 추가/수정/삭제, 권한 드롭다운, 중복 이메일 방지, 발송 상태 표시 | "우리 회사 임원만 이름·이메일·권한을 미리 박아두면 된다" | 일괄 등록(CSV)·초대 메일 재발송, 사전 등록 대비 미완료 임원 알림 |
| **11. 사전등록 임원 - 본인 정보 입력 및 매칭** | 플랫폼 **회원가입(최종 등록)** 화면에서 본인 **이름**과 **이메일** 입력:`<br>`• 최초 관리자가 등록한 사전 등록 레코드와 **이름·이메일 일치** 검증`<br>`• 미사전 등록 이메일이면 진행 불가 안내 및 담당자 문의 안내 | 이름/이메일 입력 필드, 매칭 성공·실패 메시지, 회사명(읽기 전용) 표시 | "내가 관리자에게 등록된 사람이 맞는지 확인하는구나" | 오타 힌트, 담당자 연락처, 사전 등록 목록과의 diff 안내(보안상 마스킹) |
| **12. 사전등록 임원 - 이메일 인증** | **이메일 인증**으로 본인 소유 확인:`<br>`• "이메일 인증" 버튼 클릭 → 인증 메일 발송`<br>`• 인증코드 입력 또는 링크 클릭`<br>`• "인증 완료" | 인증코드 입력, 유효시간(예: 10분), 재발송 | "이메일만 내 것이면 다음으로 넘어간다" | 재발송, 남은 시간 표시 |
| **13. 사전등록 임원 - 비밀번호 설정**     | 비밀번호 설정:`<br>`• 비밀번호 설정 (특수문자, 영문, 숫자 포함, 8~14자)`<br>`• 비밀번호 확인 입력`<br>`• 비밀번호 강도 표시`<br>`• "비밀번호 생성 완료" 버튼 클릭 | 비밀번호 입력 필드, 비밀번호 확인 입력 필드, 비밀번호 강도 표시, 보안 규칙 안내 | "안전한 비밀번호를 설정하자"                       | 비밀번호 보안 강도 체크, 실시간 강도 표시, 보안 규칙 안내 |
| **14. 사전등록 임원 - 권한 확인 및 최종 등록 완료** | 사전 등록 시 부여된 **권한(역할)** 을 읽기 전용으로 확인:`<br>`• 역할별 권한·책임 안내`<br>`• "확인" 후 **계정 활성화**(비밀번호 암호화 저장, 회사 연동)`<br>`• 자동 로그인 또는 로그인 페이지로 이동 | 환영 화면, 부여 권한 배지, Workflow 안내(역할별), 온보딩(선택) | "관리자가 정해 준 권한으로 쓰면 되겠다" | 역할 변경은 관리자에게 요청 안내, 온보딩 가이드 |
| **15. 로그인 (기존 회원) 및 권한 확인** | 회사 이메일과 비밀번호 입력 후 로그인                                                                                                                                                                                                                                                                                                      | 로그인 처리 중, 세션 생성, 사용자 권한 레벨 확인 및 적용                    | "로그인하면 내 권한에 맞는 기능을 사용할 수 있겠다" | 자동 로그인 옵션, 비밀번호 찾기 링크, 권한 레벨 표시 |
| **16. 서비스 이용 시작**                | 네비게이션 바에 사용자, 회사 정보 및 권한 레벨 표시                                                                                                                                                                                                                                                                                        | 로그인 탭 숨김, 권한에 따른 기능 탭 접근 가능                               | "내 권한에 맞게 회사정보를 조회/수정할 수 있겠다"   | 다음 단계 안내 (권한별 맞춤 안내)                    |

#### 주요 기능

- **회사 등록 (2단계 구조 - Step 1)** ⭐ 신규
  - 회사명, 사업자등록번호 입력
  - DART API 연동을 통한 회사 정보 자동 조회
  - 회사정보 확인 (조회 전용, 수정 불가)
  - 회사 ID 생성
  - 대표이사 이메일로 초대 링크 자동 발송
  - 초대 링크 유효기간 관리 (7일)
  - 초대 링크 재발송 옵션
  - **HRIS 불필요**: 회사 등록은 HRIS 연동 없이 진행
- **최초 관리자 생성 (2단계 구조 - Step 2)** ⭐ 신규
  - 대표이사 이메일로 초대 링크 발송
  - 초대 링크 클릭 → 비밀번호 설정 → 계정 생성(암호화)
  - 자동으로 "최종 승인권자" 역할 부여
  - 최초 관리자 플래그 설정 (is_first_admin=True)
  - 관리자 대시보드 접근 권한 → 이후 **임원 사전 등록(이름·이메일·권한)** 수행
  - **HRIS 불필요**: 최초 관리자는 HRIS 연동 없이 생성 가능
- **임원 최종 등록 (사전 등록 기반 - Step 3)**
  - 최초 관리자가 **이름·이메일·권한**으로 사전 등록한 임원만 가입 가능
  - 본인 **이름·이메일** 입력 → 사전 등록과 **매칭** 후 이메일 인증·비밀번호 설정
  - 권한은 **관리자가 사전에 지정한 역할**을 읽기 전용으로 확인 후 확정 (가입 시 역할 임의 선택 없음)
- **DART API 연동**
  - 사업자등록번호 기반 회사 공시정보 자동 조회
  - 회사명, 대표자명, 본사 주소, 업종, 대표이사 이메일, 회사 ID 등 자동 입력
  - API 실패 시 수동 입력 옵션
- **회사정보 확인 (조회 전용)**
  - DART API로 불러온 회사정보를 조회만 가능
  - 회사 등록 또는 회원가입 시 본인이 속한 회사가 맞는지만 확인
  - 수정 불가 (회사정보 탭에서 권한에 따라 수정 가능)
- **최초 관리자 - 임원 사전 등록** ⭐ 신규(핵심)
  - 임원 **이름**, **이메일**, **권한(역할)** 등록
  - (선택) 해당 이메일로 최종 등록 안내 또는 초대 링크 발송
  - 미완료 초대·중복 이메일 관리
- **사용자 이메일 인증** (임원 최종 등록 단계)
  - 사전 등록 매칭 완료 후 동일 이메일로 소유권 인증
  - 인증 코드 발송·검증, 유효시간(예: 10분), 재발송
- **비밀번호 설정**
  - 비밀번호 보안 강도 체크 (특수문자, 영문, 숫자 포함, 8~14자)
  - 비밀번호 확인 일치 검증
  - 실시간 강도 표시
  - 보안 규칙 안내
- **권한(역할) 부여 방식: 관리자 사전 등록 + 가입 시 확인 (4단계 구조)** ⭐ 핵심 기능
  - 임원의 **권한은 최초 관리자가 사전 등록 시 지정**; 임원은 최종 등록 화면에서 **읽기 전용으로 확인**만 함
  - **역할 1: 최종 승인권자** — 전체 데이터 관리·최종 승인·**사용자(임원) 관리**·데이터 소스·최종 보고서 제출
  - **역할 2: ESG팀** — ESG 데이터 통합, 현업 검토/승인, 데이터 소스 설정, 보고서 생성·검토
  - **역할 3: 현업팀** — 담당 섹션 입력, ESG팀 검토 요청, 템플릿·파일 업로드
  - **역할 4: 일반 사용자** — 조회 위주, 수정 제한(정책에 따름)
  - 권한별 UI/기능 차별화, Workflow 기반 승인(현업팀 → ESG팀 → 최종 승인권자)
  - *(선택 확장)* HRIS 연동 시 직급·부서를 **참고 정보**로 표시하거나, 사전 등록과 대조하는 보조 검증에 활용 가능
- **회사 및 개인 인증**
  - 최초 관리자: 회사 등록 후 **이메일 초대 링크**로 본인 확인
  - 임원: **이름·이메일 사전 등록 매칭** + **이메일 인증**으로 소속·소유권 확인
  - 사업자등록번호 검증(회사 등록 단계)
- **로그인 (기존 회원)**
  - 회사 이메일/비밀번호 로그인
  - 자동 로그인 옵션
  - 비밀번호 찾기/재설정
  - 로그인 시 권한 레벨 확인 및 적용
- **회원가입 완료 후 자동 데이터 연동**
  - 회사정보 탭에 DART API로 불러온 정보 자동 반영(회사 등록 시 수집된 데이터 기준)
  - 권한에 따라 수정 또는 조회만 가능

#### 주요 기능

##### 1. 데이터 소스 관리 시스템 (신규)

###### 1.1 입력 방식 선택

사용자는 각 ESG 데이터 섹션별로 최적의 입력 방식을 선택할 수 있습니다:

**A. 수동 입력 (Manual Entry)**

- 웹 폼에 직접 데이터 입력
- Excel 템플릿 다운로드 → 작성 → 업로드
- 적합한 경우: 소규모 기업, 초기 설정, 시스템 없는 경우
- 장점: 즉시 시작 가능, 별도 설정 불필요
- 단점: 매번 수동 입력 필요, 시간 소요

**B. 시스템 API 연동 (System Integration)**

- 기업 내부 시스템과 실시간/주기적 자동 동기화
- 지원 시스템:
  - **ERP** (SAP, Oracle ERP, 더존 ERP): 인력, 지배구조 데이터
  - **EMS** (환경관리 시스템): 온실가스, 에너지, 물, 폐기물
  - **HRIS** (인사정보 시스템): 인력 구성, 다양성, 교육
  - **BI/DW** (데이터웨어하우스): 모든 KPI 데이터
  - **Custom API**: 자체 개발 시스템
- 설정 절차:
  1. 시스템 종류 선택
  2. API 엔드포인트 및 인증 정보 입력
  3. 필드 매핑 (시스템 필드명 → 플랫폼 필드명)
  4. 동기화 주기 설정 (실시간/일일/주간/월간)
  5. 테스트 연결 및 첫 동기화
- 적합한 경우: 중대형 기업, ERP/EMS 시스템 보유
- 장점: 완전 자동화, 실시간 업데이트, 사람 개입 최소화
- 단점: 초기 설정 필요, IT 부서 협조 필요

**C. 파일 자동 업로드 (Scheduled File Upload)**

- 정해진 위치의 파일을 주기적으로 자동 수집
- 지원 방식:
  - **FTP/SFTP**: 서버에 파일 업로드 → 자동 수집
  - **클라우드 드라이브**: Google Drive/OneDrive 특정 폴더 연동
  - **이메일**: 정해진 형식으로 이메일 발송 → 자동 파싱
- 지원 파일 형식: Excel (.xlsx, .xls), CSV, JSON, XML
- 스케줄 설정: Cron 표현식 또는 UI에서 "매월 1일 09:00" 등 설정
- 적합한 경우: 중소기업, 레거시 시스템, 주기적 보고 체계
- 장점: 기존 보고 프로세스 활용, 반자동화
- 단점: 파일 작성은 여전히 필요, 지연 가능성

**D. 하이브리드 모드 (Hybrid)**

- 일부 필드는 자동 연동, 일부는 수동 입력
- 예시:
  - 환경 데이터: EMS 자동 연동 (온실가스, 에너지)
  - 사회 데이터: HRIS 자동 연동 (인력 구성) + 수동 입력 (지역사회 공헌)
  - 지배구조: 수동 입력 (정책 수립 여부)
- 적합한 경우: 데이터 소스가 분산된 경우
- 장점: 최대 유연성, 점진적 자동화 가능
- 단점: 관리 복잡도 증가

###### 1.3 데이터 동기화 관리

- **동기화 이력**: 모든 동기화 기록 추적 (성공/실패, 시간, 레코드 수)
- **실패 처리**:
  - API 연결 실패 시 이메일/SMS 알림
  - 자동 재시도 (최대 3회)
  - 실패 시 이전 데이터 유지 또는 수동 입력으로 대체
- **데이터 검증**:
  - 동기화된 데이터 자동 검증 (형식, 범위, 이상치)
  - 검증 실패 시 경고 표시, 수동 확인 요청
- **버전 관리**:
  - 동기화할 때마다 버전 생성
  - 이전 버전으로 롤백 가능
- **변경 알림**:
  - 데이터가 자동 업데이트되면 담당자에게 알림
  - 변경 사항 요약 (전월 대비 증감)

- **수동 수정 가능**: 자동 연동 필드도 필요시 수정 가능 (다음 동기화 시 덮어씌워짐 경고)

###### 1.5 ERP 데이터 검증 및 수정 요청 파이프라인

ERP 시스템에서 ESG 플랫폼으로 데이터가 들어올 때, 자동으로 전처리, 검증, 이상 탐지 및 수정 요청이 수행됩니다.

**파이프라인 흐름:**
1. **데이터 수집**: ERP API 호출로 원시 데이터 수집
2. **전처리** (Python 로직):
   - 데이터 타입 변환 (문자열 → 숫자)
   - 단위 통일 (kWh → MWh, kg → t)
   - 필드명 매핑 (ERP 필드 → ESG 필드)
   - 누락값 처리, 중복 제거
3. **검증** (Python 로직):
   - 필수 필드 검증
   - 범위 검증 (0-100%, 최소/최대값)
   - 일관성 검증 (합계, 비율)
   - 전년도 비교 (이상 수치 탐지)
   - 업계 평균 비교
4. **이상 탐지 에이전트** (LLM 기반):
   - 전년도 대비 급격한 변화 분석
   - 맥락 기반 정상/오류 구분
   - 원인 추론 및 확인/수정 필요 여부 판단
5. **수정 요청 에이전트** (LLM 기반):
   - 구체적이고 실행 가능한 수정 요청 생성
   - 심각도 분류 및 기한 설정
6. **사용자 알림**: 이메일/앱 알림으로 확인/수정 요청 전달

**실제 기업 방식과의 일치:**
- SAP Green Ledger, Oracle Fusion Cloud Sustainability와 동일한 검증 프로세스
- LLM 기반 이상 탐지 및 수정 요청 생성으로 고도화

**사용자 경험:**
- ERP 데이터 동기화 시 자동으로 검증 수행

###### 1.8 데이터 매핑 자동화

**자동 매핑 엔진 활용**

현재 개발 중인 `auto_suggest_mappings_improved.py` 스크립트를 활용하여 ERP 필드와 IFRS S2/GRI Data Point 간 자동 매핑:

**매핑 프로세스**
1. **ERP 필드 추출**: 연동된 ERP 시스템에서 필드 목록 자동 조회
2. **벡터 검색**: ERP 필드명을 임베딩하여 유사한 IFRS/GRI Data Point 검색
3. **구조적 매핑**: 필드 타입, 단위, 범위 등을 고려한 구조적 유사도 계산
4. **하이브리드 점수**: 벡터 유사도 + 구조적 유사도 결합
5. **자동 매핑 제안**: 임계값 이상의 매핑 자동 확정, 미만은 검토 필요로 표시

**매핑 예시**
```python
# ERP 필드 → IFRS S2 Data Point 자동 매핑
erp_field = "SAP_ENERGY_CONSUMPTION_MWH"
ifrs_dp = mapping_service.suggest_mapping(
    source=erp_field,
    target_standard="IFRS_S2",
    vector_threshold=0.70,
    structural_threshold=0.50
)
# 결과: "IFRS_S2-Energy-Usage" 자동 매핑 (final: 0.85)
```

**수동 매핑 지원**
- 자동 매핑이 불확실한 경우 사용자에게 매핑 제안
- 드롭다운에서 ERP 필드 선택 → IFRS/GRI Data Point 선택
- 매핑 이력 저장 및 재사용

###### 1.9 실시간 데이터 동기화

**동기화 방식**

**1. 배치 동기화 (권장)**
- **주기**: 일일, 주간, 월간 선택 가능
- **시간**: 사용자 지정 (예: 매일 09:00, 매월 1일 09:00)
- **범위**: 전체 데이터 또는 최근 N일 데이터만
- **장점**: ERP 시스템 부하 최소화, 안정적

**2. 실시간 동기화 (선택)**
- **트리거**: ERP 데이터 변경 시 즉시 동기화
- **방식**: Webhook 또는 Event-driven Architecture
- **장점**: 최신 데이터 보장
- **단점**: ERP 시스템 부하 증가 가능

**3. 수동 동기화**
- 사용자가 "지금 동기화" 버튼 클릭
- 즉시 데이터 업데이트
- 테스트 및 검증 시 활용

**동기화 프로세스**
```python
@schedule(cron="0 9 1 * *")  # 매월 1일 09:00
def sync_erp_data():
    """ERP에서 ESG 데이터 자동 동기화"""
    # 1. ERP API 호출
    erp_data = erp_connector.fetch_all()
    
    # 2. 데이터 변환 (ERP 필드 → ESG 필드)
    esg_data = transform_to_esg_format(erp_data)
    
    # 3. 데이터 검증
    validated_data = validate_esg_data(esg_data)
    
    # 4. 플랫폼에 저장
    platform.save(validated_data)
    
    # 5. 동기화 이력 기록
    log_sync_history(erp_data, esg_data, status="success")
```

**동기화 이력 관리**
- 모든 동기화 기록 추적 (성공/실패, 시간, 레코드 수)
- 실패 시 자동 재시도 (최대 3회)
- 실패 알림 (이메일/SMS)
- 동기화 전/후 데이터 비교

##### 3. 8개 섹션 데이터 수집

**기존 4개 섹션 (DART API 보완)**

1. **기업 기본정보**: 회사명, 사업자등록번호, 대표자명, 업종
2. **연락처 정보**: 본사 주소, 대표 전화번호, 대표 이메일, 웹사이트
3. **ESG 목표 및 비전**: 미션, 비전, ESG 핵심 목표, 탄소중립 목표 연도
4. **이해관계자 정보**: 임직원 수, 주요 주주, 기타 이해관계자, 소통 채널

**신규 3개 섹션 (ESG 실적 데이터)**
5. **환경 데이터 (Environment)**

- 온실가스 배출량: Scope 1, 2, 3 (단위: tCO2eq, 연도별)
- 에너지: 총 사용량 (MWh), 재생에너지 비율 (%), 재생에너지 종류
- 물 관리: 총 사용량 (톤), 재활용률 (%)
- 폐기물: 총 발생량 (톤), 재활용률 (%), 유해폐기물 (톤)
- 환경 인증: ISO 14001, 탄소중립 인증, 발급일

6. **사회 데이터 (Social)**

   - 인력 구성: 총 임직원 수, 성비 (%), 장애인 고용률 (%), 평균 연령, 이직률 (%)
   - 산업안전보건: 재해율 (%), 안전교육 시간 (시간/인)
   - 인권: 정책 수립 여부, 실사 실시 여부, 고충처리 건수
   - 협력업체: 총 협력업체 수, ESG 평가 업체 수, 평가 비율 (%)
   - 지역사회: 공헌 투자액 (원), 봉사 시간 (시간), 수혜자 수
7. **지배구조 데이터 (Governance)**

   - 이사회: 총 이사 수, 독립이사 수, 여성 이사 수, ESG 위원회 설치 여부, 연간 회의 수
   - 내부통제: 감사 실시 여부, 내부감사팀 규모, 독립성 여부
   - 윤리경영: 행동강령 제정, 윤리교육 시간, 위반 건수, 내부고발 시스템
   - 리스크 관리: 기후리스크 평가 여부, 정보보안 투자액 (원), 데이터 유출 사고
8. **데이터 소스 관리 (Data Source Management)** ⭐ 신규

   - **입력 방식 설정**: 각 섹션별 수동/자동 선택
   - **시스템 연동**: ERP/EMS/HRIS/BI API 설정
   - **파일 자동 업로드**: FTP/클라우드/이메일 연동
   - **동기화 관리**: 스케줄, 이력, 실패 처리
   - **필드 매핑**: 외부 시스템 필드 → 플랫폼 필드
   - **데이터 검증**: 자동 연동 데이터 검증 및 이상치 탐지

##### 4. 데이터 입력 편의 기능

- **자동 연동 우선** ⭐ 신규:
  - 시스템 API 연동 시 자동 데이터 수집
  - "🔄 자동 연동됨" 배지 표시
  - 마지막 동기화 시간 표시
  - 수동 동기화 버튼 제공
- **DART API 자동 입력**: 기본정보 자동 채움, "자동 입력됨" 배지 표시
- **템플릿 제공**:
  - 업종별 기본 템플릿 (제조업, IT, 금융, 서비스업 등)
  - 이전 연도 데이터 불러오기
  - 업계 평균 참고 데이터 표시
  - **시스템별 Excel 템플릿** (ERP/EMS/HRIS 출력 형식에 맞춤)
- **파일 업로드**:
  - Excel 템플릿 다운로드 → 작성 → 업로드
  - 기존 SR 보고서 PDF 업로드 → AI 자동 파싱
  - ERP/EMS 시스템 CSV 임포트
  - **자동 스케줄 업로드** (FTP/클라우드/이메일) ⭐ 신규
- **자동 계산**:
  - Scope 1+2+3 자동 합산
  - 재활용률 자동 계산 (재활용량/총량 × 100)
  - 전년 대비 증감률 자동 표시
  - 업계 평균 대비 비교

##### 5. 데이터 검증 및 품질 관리

- **필수 항목 검증**: IFRS S1·S2 필수 공시 항목 체크
- **데이터 형식 검증**: 숫자, 날짜, 백분율, 단위 등 형식 자동 검증
- **이상치 경고**: 업계 평균 대비 ±50% 이상 차이 시 경고
- **완성도 표시**:
  - 섹션별 완성도 (0-100%)
  - 전체 진행률 표시
  - 필수 항목 vs 권장 항목 구분
- **연도별 데이터 일관성**: 전년 대비 급격한 변화 감지 및 확인 요청

##### 5. 실시간 미리보기 및 인사이트

- **실시간 미리보기**: 오른쪽 카드에서 입력 정보 실시간 확인
- **데이터 인사이트**:
  - 전년 대비 주요 지표 변화율
  - 업계 평균과의 비교
  - ESG 점수 예상 (환경/사회/지배구조별)
- **시각화**:
  - 온실가스 배출량 트렌드 그래프
  - 재생에너지 비율 파이 차트
  - 성비/다양성 막대 그래프

##### 6. 자동 저장 및 버전 관리

- **자동 저장**: 5초마다 입력 내용 자동 저장
- **임시 저장**: 작업 중단 시 임시 저장 후 나중에 이어서 작업
- **버전 관리**: 수정 이력 추적, 이전 버전 복원 가능
- **변경 알림**: 다른 사용자가 동시 수정 시 알림

##### 7. 멀티모달 에이전트 연동

- **Data Point 매핑**: 입력된 데이터를 IFRS Data Point로 자동 매핑
- **RAG 검색 지원**: 입력된 데이터 기반으로 관련 기준서 요구사항 검색
- **문단 생성 지원**: 실제 기업 데이터를 반영한 맞춤형 문단 생성
- **검증 강화**: 입력된 데이터의 완전성 및 정확성 검증

##### 8. 승인 워크플로우 (부서별 입력 권한)

- **입력 단계**: 부서별 담당자가 해당 섹션 데이터 입력
- **검증 요청**: "검증 요청" 버튼 클릭 → 담당 임원에게 알림
- **승인 단계**: 임원이 검토 후 승인/반려
- **반려 시**: 수정 요청 사항과 함께 담당자에게 반송
- **승인 완료**: 최종보고서에 데이터 반영

#### 주요 기능

##### 1. 보고서 완성도 관리

- **전체 진행률 표시**: 실시간 퍼센트(%) 및 진행 바
- **섹션별 상태 표시**:
  - ✅ 완료 (Complete)
  - 🔄 진행중 (In Progress)
  - ⏳ 대기중 (Pending)
  - ⚠️ 누락 (Missing)
- **필수 항목 체크리스트**: IFRS S1·S2 기준 필수 항목 자동 체크
- **완성도별 권장 사항**:
  - 90% 이상: "다운로드 준비 완료"
  - 70-90%: "몇 가지 항목 추가 권장"
  - 70% 미만: "필수 항목 입력 필요"

##### 2. 실시간 미리보기

- **PDF 미리보기**: 오른쪽 패널에 실시간 PDF 레이아웃 표시

- **페이지 번호 표시**: 각 섹션의 페이지 번호 표시

##### 3. 직접 수정 기능

- **WYSIWYG 편집기**:

## 사용자 페인 포인트 및 개선 사항

### 로그인/회원가입 탭

- **페인 포인트**:
  - 회사 등록 → 최초 관리자 → 임원 사전 등록 → 임원 최종 등록으로 단계가 길어 이탈 가능
  - DART API 연동 실패 시 수동 입력 필요
  - 이메일 인증 대기 시간
  - 사전 등록 없이 가입 시도 시 **매칭 실패**로 막혀 담당자 문의가 필요할 수 있음
  - 관리자가 임원 사전 등록을 늦게 하면 임원이 대기하는 구조
- **개선 사항**:
  - **4단계 역할 구조** (KPMG 기반): 역할은 **최초 관리자가 사전 등록 시 지정**, 임원은 최종 등록에서 **읽기 전용 확인**
  - **Workflow 승인 프로세스**: 현업팀 → ESG팀 → 최종 승인권자
  - 역할별 권한·책임 안내 (조직도, Workflow 다이어그램), 임원별 **초대 재발송**·미가입 알림
  - DART API 실패 시 명확한 안내 및 수동 입력 옵션
  - 인증번호 재발송 옵션 및 유효시간 표시
  - 회원가입·최종 등록 단계 **진행률** 표시 (N/M 단계)
  - 가입 완료 후 온보딩 가이드 제공 (역할별 맞춤 안내)

### 회사정보 탭

- **페인 포인트**:
  - DART API로 불러온 정보만으로는 SR 보고서 작성 불가 (ESG 실적 데이터 누락)
  - 7개 섹션의 많은 필드를 매번 수동 입력하는 피로도 ⭐
  - ESG 데이터를 어디서 가져와야 할지 모름 (ERP, EMS 시스템 등)
  - **기업은 이미 ERP/EMS에 데이터가 있는데 왜 다시 입력해야 하나?** ⭐ 핵심 불만
  - 데이터가 업데이트될 때마다 수동으로 다시 입력해야 함 (매월/분기)
  - 부서별로 담당 데이터가 다른데 권한 분리가 필요
  - 데이터 입력 실수 (단위 오류, 이상치)
  - 조회 권한 사용자가 수정이 필요한 경우 관리자에게 요청하는 프로세스 불명확
- **개선 사항**:
  - **✨ 데이터 소스 관리 시스템 (핵심 개선)**:
    - 🔌 **시스템 API 연동**: ERP/EMS/HRIS/BI 시스템과 자동 동기화
    - 📁 **파일 자동 업로드**: FTP/클라우드/이메일로 정기 자동 수집
    - 🔄 **스케줄링**: 매월/분기 자동 업데이트, 수동 개입 불필요
    - 🎯 **필드 매핑**: 외부 시스템 필드를 플랫폼 필드로 자동 매핑
    - 📊 **동기화 이력**: 모든 업데이트 추적, 실패 시 알림
  - **ESG 데이터 섹션 3개 추가**: 환경/사회/지배구조 실적 데이터 수집
  - **템플릿 및 파일 업로드**: Excel 템플릿, PDF 자동 파싱, CSV 임포트
  - **자동 계산 및 검증**: 합산, 비율 계산, 이상치 경고, 업계 평균 비교
  - **역할 기반 권한 시스템 강화** ⭐ 4단계 구조 적용:
    - **역할 1: 최종 승인권자**: 전체 관리 + 최종 승인 + 사용자 관리 + 데이터 소스 관리
    - **역할 2: ESG팀**: 전체 관리 + 현업팀 데이터 검토/승인 + 데이터 소스 설정
    - **역할 3: 현업팀**: 환경팀/인사팀/재무팀/경영지원팀 각자 담당 섹션만 입력
    - **역할 4: 일반 사용자**: 조회만 가능
  - **승인 워크플로우 명확화**: 현업팀 입력 → ESG팀 검토 → 최종 승인권자 승인
  - **섹션별 완성도 표시**: 실시간 진행률 (0-100%)
  - **DART API 자동 입력**: "자동 입력됨" 배지 표시
  - **임시 저장 강화**: 5초마다 자동 저장, 작업 중단 후 이어서 가능
  - **데이터 인사이트**: 전년 대비 변화율, 업계 평균 비교, ESG 점수 예상

**작성일**: 2024년  
**프로젝트명**: IFRSseed AI  
**버전**: 7.1 (임원 사전 등록 → 본인 이름·이메일 최종 등록 흐름 반영) ⭐  
**담당자**: 로그인/회원가입, 회사정보, 최종보고서 탭 담당

**주요 업데이트**:

- **v7.1 — 임원 온보딩**: 최초 관리자가 **이메일 초대 → 비밀번호 → 계정 생성**까지 완료한 뒤, 관리자 대시보드에서 임원 **이름·이메일·권한**을 **사전 등록**하고, 사전 등록된 임원이 플랫폼에서 **본인 이름·이메일**을 입력해 매칭 후 **이메일 인증·비밀번호·권한 확인**으로 **최종 등록**을 마침. 임원 가입 단계의 **HRIS 자동 역할 할당**은 기본 흐름에서 제외하고, **선택 확장(보조 검증)** 으로 정리함.

- **⭐⭐ 4단계 역할 구조 도입 (KPMG 기반 - 핵심 개선)**:
  - **역할 1: 최종 승인권자** (CEO, 대표이사, 사장)
    - 전체 데이터 관리 및 최종 승인
    - 사용자 관리 권한
    - 데이터 소스 관리 (API 연동)
    - 최종 보고서 제출
  - **역할 2: ESG팀** (ESG 담당자, ESG 협업팀)
    - ESG 데이터 통합 관리
    - 현업팀 데이터 검토 및 승인
    - 데이터 소스 설정 가능
    - 보고서 생성 및 검토
  - **역할 3: 현업팀** (부서 담당자)
    - 부서별 데이터 입력 (환경안전팀, 인사팀, 재무팀, 경영지원팀)
    - 담당 섹션만 입력/수정 가능
    - ESG팀에 검토 요청
  - **역할 4: 일반 사용자**
    - 데이터 조회만 가능

- **⭐ Workflow 승인 프로세스 자동화**:
  - **현업팀** 데이터 입력 → "검토 요청"
  - **ESG팀** 데이터 검토/수정 → "승인 요청"
  - **최종 승인권자** 최종 승인 → "보고서 제출"
  - 실시간 상태 추적 및 알림
  - 피드백 및 반려 프로세스

- **역할별 사용자 여정 상세화**:
  - 최종 승인권자 여정 (경영진 대시보드, 최종 승인)
  - ESG팀 여정 (데이터 통합 관리, 검토 및 승인)
  - 현업팀 여정 (부서별 데이터 입력, 검토 요청)
  - 일반 사용자 여정 (조회만)

- 회사 이메일 기반 회원가입 프로세스 상세화
- DART API 연동을 통한 회사정보 자동 불러오기 기능 추가
- 이메일 인증을 통한 회사 및 개인 인증 프로세스 추가
- 역할 부여: 초기안은 회원가입 시 선택 → **v7.1부터 임원은 관리자 사전 등록 시 권한 지정, 가입 시 읽기 전용 확인**
- 회사정보 탭 권한별 접근 제어 강화
- 회원가입 시 회사정보 확인 단계를 조회 전용으로 변경

  - **신규 3개 ESG 섹션**:
    - **환경 데이터**: 온실가스(Scope 1/2/3), 에너지, 물, 폐기물, 환경 인증

    - **지배구조 데이터**: 이사회 구성, 내부통제, 윤리경영, 리스크 관리
  - **⭐ 신규 데이터 소스 관리 섹션**:
    - **시스템 API 연동**: ERP/EMS/HRIS/BI 자동 동기화
    - **파일 자동 업로드**: FTP/클라우드/이메일 정기 수집
    - **스케줄링**: 매월/분기 자동 업데이트
    - **필드 매핑**: 외부 시스템 → 플랫폼 자동 매핑
    - **동기화 관리**: 이력 추적, 실패 처리, 알림
    - **하이브리드 모드**: 일부 자동, 일부 수동
  - 템플릿 제공 및 파일 업로드 (Excel, PDF, CSV)
  - 자동 계산 및 검증 (합산, 비율, 이상치 경고)
  - 역할 기반 승인 워크플로우 (현업팀 → ESG팀 → 최종 승인권자)
  - 데이터 인사이트 (전년 대비, 업계 평균, ESG 점수)
