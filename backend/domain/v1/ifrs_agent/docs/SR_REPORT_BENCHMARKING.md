## 1. 개요

### 1.1 목적

외부 크롤링 데이터(뉴스, 공시)와 경쟁사 SR 보고서 평가를 활용하여 현재 기업의 SR 보고서를 분석하고, 개선점을 도출하여 자동으로 조치를 적용하는 시스템입니다.

### 1.2 핵심 기능

1. **외부 데이터 수집**: 뉴스, 공시 데이터 주기적 크롤링
2. **현재 보고서 평가**: 외부 피드백 기반 종합 평가 및 점수 산출
3. **경쟁사 벤치마킹**: 동일 업종 경쟁사 SR 보고서 수집 및 평가
4. **개선점 도출**: 평가 결과 기반 구체적 개선 사항 식별
5. **자동 조치 적용**: 우수한 경쟁사 보고서 참조하여 자동 개선

### 1.3 전체 플로우

```
┌─────────────────────────────────────────────────────────────────┐
│ Phase 1: 외부 데이터 수집 (주기적 또는 실시간)                     │
└─────────────────────────────────────────────────────────────────┘

[1-1] 뉴스 크롤링
    ├─→ 기업 ESG 관련 뉴스 수집
    ├─→ 감정 분석 (긍정/부정/중립)
    └─→ 핵심 이슈 추출
    ↓
[1-2] 데이터 저장
    └─→ news_articles 테이블

┌─────────────────────────────────────────────────────────────────┐
│ Phase 2: 현재 보고서 평가 (주 1회 또는 수동 실행)                  │
└─────────────────────────────────────────────────────────────────┘

[2-1] 현재 SR 보고서 로드
    ├─→ sr_report_content 테이블에서 조회
    └─→ 생성된 섹션들 수집
    ↓
[2-2] 외부 피드백 수집 및 분석
    ├─→ 뉴스 기사 분석
    │   ├─→ ESG 관련 뉴스 수집 (최근 3개월)
    │   ├─→ 감정 분석 (긍정/부정/중립)
    │   ├─→ 감정 점수 계산 (-1~1)
    │   ├─→ 핵심 이슈 추출
    │   │   - 언급된 ESG 주제 (환경/사회/지배구조)
    │   │   - 관련 Data Point 식별
    │   │   - 중요도 점수 산출
    │   └─→ 뉴스 기사별 평가 점수 계산
    │
    ├─→ 공시 데이터 분석
    │       ├─→ 소셜 미디어/온라인 커뮤니티 분석 (선택적)
    │   ├─→ 기업 ESG 관련 온라인 논의 수집
    │   ├─→ 이해관계자 의견 분석
    │   └─→ 주요 관심사 추출
    │
    └─→ 외부 평가 기관 리포트 분석 (선택적)
        ├─→ ESG 평가 기관 리포트 수집
        │   - MSCI ESG Rating
        │   - Sustainalytics
        │   - CDP (Carbon Disclosure Project)
        ├─→ 평가 점수 및 피드백 추출
        └─→ 개선 권고사항 식별
    ↓
[2-3] 외부 피드백 기반 보고서 평가
    ├─→ 뉴스 감정 점수 반영
    │   ├─→ 긍정 뉴스 비율 계산
    │   ├─→ 부정 뉴스 비율 계산
    │   ├─→ 평균 감정 점수 산출
    │   └─→ 보고서 신뢰도 점수에 반영
    │       - 긍정 뉴스 많음 → 신뢰도 +5점
    │       - 부정 뉴스 많음 → 신뢰도 -10점
    │
    ├─→ 핵심 이슈 반영도 평가
    │   ├─→ 뉴스에서 언급된 핵심 이슈 추출
    │   │   예: "탄소중립 목표", "재생에너지 전환", "공급망 관리"
    │   ├─→ SR 보고서에서 해당 이슈 다루는지 확인
    │   │   - 섹션별로 핵심 이슈 언급 여부 검사
    │   │   - 언급 깊이 평가 (단순 언급 vs 상세 설명)
    │   └─→ 이슈 반영도 점수 계산
    │       - 모든 핵심 이슈 상세 설명 → 100점
    │       - 대부분 이슈 언급 → 80점
    │       - 일부 이슈만 언급 → 60점
    │       - 핵심 이슈 누락 → 40점 이하
    │
    ├─→ 외부 평가 기관 피드백 반영 (있는 경우)
    │   ├─→ MSCI ESG Rating 피드백
    │   │   - 강점 영역 식별
    │   │   - 개선 필요 영역 식별
    │   ├─→ CDP 평가 결과
    │   │   - 기후변화 대응 점수
    │   │   - 물 관리 점수
    │   │   - 산림 보호 점수
    │   └─→ 평가 기관 권고사항 반영
    │
    └─→ 종합 평가 점수 산출
        ├─→ 가중 평균 계산
        │   - 뉴스 감정 점수: 20% 가중치
        │   - 이슈 반영도: 50% 가중치
        │   - 외부 평가 기관 피드백: 30% 가중치 (있는 경우)
        │
        ├─→ 기본 평가 항목 (간소화)
        │   - IFRS 준수도: 기본 점수 80점 (외부 피드백으로 조정)
        │   - 완성도: 이슈 반영도 등 기반 계산
        │   - 신뢰도: 뉴스 감정 점수 기반 계산
        │
        └─→ 최종 종합 점수 (0~100)
    ↓
[2-4] 평가 결과 저장
    └─→ sr_report_evaluations 테이블
        - 종합 점수
        - 뉴스 감정 점수
        - 이슈 반영도 점수
        - 개선 필요 영역

┌─────────────────────────────────────────────────────────────────┐
│ Phase 3: 경쟁사 보고서 수집 및 평가 (분기별 1회)                   │
└─────────────────────────────────────────────────────────────────┘

[3-1] 경쟁사 식별
    ├─→ 동일 업종 기업 조회
    ├─→ 시가총액/매출 기준 상위 기업 선정
    └─→ 최근 SR 보고서 발행 기업 우선
    ↓
[3-2] 경쟁사 SR 보고서 수집
    ├─→ DART API로 보고서 다운로드
    ├─→ PDF 파싱 (Docling)
    └─→ Index/본문 분리 저장
    ↓
[3-3] 경쟁사 보고서 평가
    ├─→ 동일한 평가 기준 적용
    ├─→ 섹션별 평가
    └─→ 우수 사례 추출
    ↓
[3-4] 평가 결과 저장
    └─→ competitor_report_evaluations 테이블

┌─────────────────────────────────────────────────────────────────┐
│ Phase 4: 벤치마킹 및 개선점 도출                                   │
└─────────────────────────────────────────────────────────────────┘

[4-1] 현재 vs 경쟁사 비교
    ├─→ 평가 점수 비교
    ├─→ 섹션별 비교
    └─→ 차이점 분석
    ↓
[4-2] 개선점 도출
    ├─→ 부족한 섹션 식별
    ├─→ 우수 사례 섹션 식별
    └─→ 구체적 개선 방안 제시
    ↓
[4-3] 개선 우선순위 결정
    ├─→ 영향도 분석
    ├─→ 구현 난이도 분석
    └─→ 우선순위 점수 산출

┌─────────────────────────────────────────────────────────────────┐
│ Phase 5: 자동 조치 적용                                            │
└─────────────────────────────────────────────────────────────────┘

[5-1] 개선 제안 생성
    ├─→ 개선이 필요한 섹션 식별
    ├─→ 참조할 경쟁사 섹션 선택
    └─→ 개선 제안서 생성
    ↓
[5-2] 사용자 승인 또는 자동 적용
    ├─→ 사용자에게 개선 제안 표시
    ├─→ 승인 시 자동 개선 실행
    └─→ 또는 자동 적용 모드
    ↓
[5-3] Gen Node로 문단 재생성
    ├─→ 경쟁사 우수 사례 참조
    ├─→ 현재 데이터 반영
    └─→ 개선된 문단 생성
    ↓
[5-4] 개선 결과 검증
    ├─→ 개선 전후 비교
    └─→ 평가 점수 재계산
```

## 2. 데이터베이스 스키마

### 2.1 뉴스 기사 테이블

```sql
CREATE TABLE news_articles (
    id UUID PRIMARY KEY,
    company_id UUID NOT NULL,
    
    -- ===== 기사 정보 =====
    title TEXT NOT NULL,
    content TEXT NOT NULL,
    source TEXT NOT NULL,  -- 'naver', 'daum', 'google' 등
    url TEXT,
    published_at TIMESTAMPTZ NOT NULL,
    
    -- ===== ESG 관련성 =====
    esg_relevance_score DECIMAL(5, 2),  -- ESG 관련성 점수 (0~100)
    esg_topics TEXT[],  -- ['환경', '사회', '지배구조']
    related_dp_ids TEXT[],  -- 관련 DP 목록
    
    -- ===== 감정 분석 =====
    sentiment TEXT,  -- 'positive' | 'negative' | 'neutral'
    sentiment_score DECIMAL(5, 2),  -- 감정 점수 (-1~1)
    
    -- ===== 핵심 이슈 =====
    key_issues JSONB,  -- 추출된 핵심 이슈
    -- 예: [
    --   {"issue": "온실가스 감축 목표", "importance": 0.9},
    --   {"issue": "재생에너지 전환", "importance": 0.8}
    -- ]
    
    -- ===== 메타데이터 =====
    crawled_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    processed_at TIMESTAMPTZ,
    
    INDEX idx_news_company_date (company_id, published_at),
    INDEX idx_news_relevance (esg_relevance_score),
    INDEX idx_news_sentiment (sentiment)
);
```

### 2.2 SR 보고서 평가 테이블

```sql
CREATE TABLE sr_report_evaluations (
    id UUID PRIMARY KEY,
    company_id UUID NOT NULL,
    report_year INTEGER NOT NULL,
    evaluation_date TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    
    -- ===== 종합 평가 =====
    overall_score DECIMAL(5, 2) NOT NULL,  -- 종합 점수 (0~100)
    
    -- ===== 외부 피드백 기반 평가 =====
    news_sentiment_score DECIMAL(5, 2),  -- 뉴스 감정 점수 평균 (-1~1)
    news_sentiment_impact DECIMAL(5, 2),  -- 뉴스 감정이 종합 점수에 미친 영향 (+/-)
    disclosure_alignment_score DECIMAL(5, 2),  -- (선택) 공시·요건 충족 기반 일치도 (0~100, ghg_disclosure_requirements 등)
    issue_coverage_score DECIMAL(5, 2),  -- 핵심 이슈 반영도 (0~100)
    external_rating_score DECIMAL(5, 2),  -- 외부 평가 기관 점수 (있는 경우)
    
    -- ===== 기본 평가 항목 (간소화) =====
    ifrs_compliance_score DECIMAL(5, 2),  -- IFRS 준수도 (기본 80점, 외부 피드백으로 조정)
    completeness_score DECIMAL(5, 2),  -- 완성도 (이슈 반영도 등 기반)
    credibility_score DECIMAL(5, 2),  -- 신뢰도 (뉴스 감정 점수 기반)
    
    -- ===== 외부 피드백 상세 =====
    news_feedback_summary JSONB,  -- {
    --   "total_articles": 45,
    --   "positive_count": 30,
    --   "negative_count": 5,
    --   "neutral_count": 10,
    --   "average_sentiment": 0.6,
    --   "key_issues": [
    --     {"issue": "탄소중립 목표", "mentioned_in_report": true, "coverage_depth": "detailed"},
    --     {"issue": "재생에너지 전환", "mentioned_in_report": true, "coverage_depth": "brief"}
    --   ]
    -- }
    
    disclosure_feedback_summary JSONB,  -- {
    --   "total_disclosures": 12,
    --   "aligned_count": 10,
    --   "discrepancy_count": 2,
    --   "discrepancies": [
    --     {
    --       "data_point": "GHG 배출량",
    --       "difference_percent": 4.2,
    --       "severity": "medium"
    --     }
    --   ]
    -- }
    
    external_rating_feedback JSONB,  -- {
    --   "msci_rating": "AA",
    --   "msci_feedback": {
    --     "strengths": ["기후변화 대응", "공급망 관리"],
    --     "weaknesses": ["물 관리", "인권"]
    --   },
    --   "cdp_score": {
    --     "climate": "B",
    --     "water": "C"
    --   }
    -- }
    
    -- ===== 개선점 =====
    improvement_areas JSONB,  -- [
    --   {
    --     "area": "공시 데이터 일치도",
    --     "issue": "GHG 배출량 불일치 (4.2% 차이)",
    --     "priority": "high",
    --     "suggestion": "공시 데이터와 SR 보고서 데이터를 재검토하여 일치시키기"
    --   },
    --   {
    --     "area": "핵심 이슈 반영",
    --     "issue": "재생에너지 전환 이슈가 단순 언급 수준",
    --     "priority": "medium",
    --     "suggestion": "재생에너지 전환 계획과 실적을 상세히 설명"
    --   }
    -- ]
    
    -- ===== 평가 근거 =====
    evaluation_rationale TEXT,  -- 평가 근거 설명
    
    -- ===== 메타데이터 =====
    evaluated_by TEXT DEFAULT 'system',  -- 'system' | 'user_id'
    
    INDEX idx_eval_company_year (company_id, report_year),
    INDEX idx_eval_score (overall_score),
    INDEX idx_eval_date (evaluation_date)
);
```

## 3. 외부 데이터 수집 및 분석 (Phase 1)

### 3.1 뉴스 크롤링 및 분석

**수집 주기**: 일 1회 (자동) 또는 실시간 (웹훅)

**수집 소스**:
- 네이버 뉴스
- 다음 뉴스
- 구글 뉴스
- 전문 ESG 뉴스 사이트 (선택적)

**분석 프로세스**:

1. **ESG 관련성 분석**
   - LLM이 기사 제목과 내용을 분석하여 ESG 관련 여부 판단
   - 관련성 점수 산출 (0~100)
   - 관련 ESG 주제 분류 (환경/사회/지배구조)
   - 관련 Data Point 식별

2. **감정 분석**
   - 긍정/부정/중립 분류
   - 감정 점수 계산 (-1~1)
   - 긍정: 기업의 ESG 성과, 목표 달성 등
   - 부정: ESG 관련 비판, 문제점 지적 등

3. **핵심 이슈 추출**
   - 언급된 주요 ESG 이슈 식별
   - 이슈별 중요도 점수 산출
   - 예: "탄소중립 목표" (중요도 0.9), "재생에너지 전환" (중요도 0.8)

**저장**: `news_articles` 테이블에 저장

### 3.2 소셜 미디어/온라인 커뮤니티 분석 (선택적)

**수집 대상**:
- 기업 ESG 관련 온라인 논의
- 이해관계자 의견
- 전문가 분석

**분석 항목**:
- 주요 관심사 추출
- 이해관계자 우려사항 식별
- 전문가 평가 및 권고사항

### 3.4 외부 평가 기관 리포트 분석 (선택적)

**수집 대상**:
- MSCI ESG Rating 리포트
- Sustainalytics 평가 리포트
- CDP (Carbon Disclosure Project) 평가 결과
- 기타 ESG 평가 기관 리포트

**분석 항목**:
- 평가 점수 및 등급 추출
- 강점 영역 식별
- 개선 필요 영역 식별
- 구체적 권고사항 추출

## 4. 현재 보고서 평가 (Phase 2) - 외부 피드백 중심

### 3.1 뉴스 크롤링 및 분석

**수집 주기**: 일 1회 (자동) 또는 실시간 (웹훅)

**수집 소스**:
- 네이버 뉴스
- 다음 뉴스
- 구글 뉴스
- 전문 ESG 뉴스 사이트 (선택적)

**분석 프로세스**:

1. **ESG 관련성 분석**
   - LLM이 기사 제목과 내용을 분석하여 ESG 관련 여부 판단
   - 관련성 점수 산출 (0~100)
   - 관련 ESG 주제 분류 (환경/사회/지배구조)
   - 관련 Data Point 식별

2. **감정 분석**
   - 긍정/부정/중립 분류
   - 감정 점수 계산 (-1~1)
   - 긍정: 기업의 ESG 성과, 목표 달성 등
   - 부정: ESG 관련 비판, 문제점 지적 등

3. **핵심 이슈 추출**
   - 언급된 주요 ESG 이슈 식별
   - 이슈별 중요도 점수 산출
   - 예: "탄소중립 목표" (중요도 0.9), "재생에너지 전환" (중요도 0.8)

**저장**: `news_articles` 테이블에 저장

## 4. 현재 보고서 평가 (Phase 2) - 외부 피드백 중심

### 4.1 외부 피드백 수집 및 분석

#### 4.1.1 뉴스 기사 분석

**수집 범위**: 최근 3개월간의 ESG 관련 뉴스

**분석 항목**:

1. **감정 분석**
   - 각 기사의 감정 점수 계산 (-1~1)
   - 전체 기사의 평균 감정 점수 산출
   - 긍정/부정/중립 기사 비율 계산
   - 예: 긍정 30개, 부정 5개, 중립 10개 → 평균 감정 점수 0.6

2. **핵심 이슈 추출**
   - 뉴스에서 언급된 주요 ESG 이슈 식별
   - 이슈별 언급 빈도 및 중요도 계산
   - 예:
     - "탄소중립 목표": 15회 언급, 중요도 0.9
     - "재생에너지 전환": 12회 언급, 중요도 0.8
     - "공급망 관리": 8회 언급, 중요도 0.7

3. **SR 보고서 반영도 확인**
   - 각 핵심 이슈가 SR 보고서에서 다뤄지는지 확인
   - 언급 깊이 평가:
     - **상세 설명**: 해당 이슈에 대해 구체적인 계획, 실적, 목표 등을 상세히 설명
     - **간단 언급**: 해당 이슈를 언급만 하고 구체적 설명 없음
     - **누락**: 해당 이슈를 전혀 다루지 않음

4. **뉴스 기사별 평가 점수**
   - 각 기사가 SR 보고서에 미치는 영향 평가
   - 긍정 뉴스: 신뢰도 향상 (+)
   - 부정 뉴스: 신뢰도 하락 (-)
   - 중요 이슈 언급: 반영도 평가

#### 4.1.2 소셜 미디어/온라인 커뮤니티 분석 (선택적)

**수집 대상**:
- 기업 ESG 관련 온라인 논의
- 이해관계자 의견
- 전문가 분석

**분석 항목**:
- 주요 관심사 추출
- 이해관계자 우려사항 식별
- 전문가 평가 및 권고사항

#### 4.1.3 외부 평가 기관 리포트 분석 (선택적)

**수집 대상**:
- MSCI ESG Rating 리포트
- Sustainalytics 평가 리포트
- CDP (Carbon Disclosure Project) 평가 결과
- 기타 ESG 평가 기관 리포트

**분석 항목**:
- 평가 점수 및 등급 추출
- 강점 영역 식별
- 개선 필요 영역 식별
- 구체적 권고사항 추출

### 4.2 외부 피드백 기반 보고서 평가

#### 4.2.1 뉴스 감정 점수 반영

**평가 프로세스**:

1. **평균 감정 점수 계산**
   - 최근 3개월간 ESG 관련 뉴스의 평균 감정 점수 산출
   - 예: 평균 감정 점수 0.6 (긍정적)

2. **신뢰도 점수에 반영**
   - 긍정 뉴스 비율이 높을수록 신뢰도 점수 상승
   - 부정 뉴스 비율이 높을수록 신뢰도 점수 하락
   - 반영 규칙:
     - 평균 감정 점수 > 0.5: 신뢰도 +5점
     - 평균 감정 점수 0.0~0.5: 신뢰도 +2점
     - 평균 감정 점수 -0.5~0.0: 신뢰도 -5점
     - 평균 감정 점수 < -0.5: 신뢰도 -10점

3. **핵심 이슈 반영도 평가**
   - 뉴스에서 언급된 핵심 이슈 목록 생성
   - SR 보고서에서 각 이슈를 다루는지 확인
   - 언급 깊이 평가:
     - **상세 설명**: 해당 이슈 섹션에서 구체적인 계획, 실적, 목표 등을 상세히 설명
     - **간단 언급**: 해당 이슈를 언급만 하고 구체적 설명 없음
     - **누락**: 해당 이슈를 전혀 다루지 않음

4. **이슈 반영도 점수 계산**
   - 모든 핵심 이슈를 상세히 설명: 100점
   - 대부분 이슈를 상세히 설명: 90점
   - 대부분 이슈를 언급 (일부 상세): 80점
   - 일부 이슈만 언급: 60점
   - 핵심 이슈 대부분 누락: 40점 이하

#### 4.2.2 외부 평가 기관 피드백 반영 (있는 경우)

**평가 프로세스**:

1. **MSCI ESG Rating 피드백**
   - 등급 추출 (AAA, AA, A, BBB, BB, B, CCC)
   - 강점 영역 식별
   - 개선 필요 영역 식별
   - 구체적 권고사항 추출

2. **CDP 평가 결과**
   - 기후변화 대응 점수 (A, A-, B, B-, C 등)
   - 물 관리 점수
   - 산림 보호 점수
   - 평가 기관의 피드백 및 권고사항

3. **평가 기관 권고사항 반영**
   - SR 보고서에서 권고사항을 다루는지 확인
   - 권고사항 반영도 점수 계산
   - 종합 점수에 반영 (20% 가중치)

### 4.3 종합 평가 점수 산출

**가중 평균 계산**:

1. **가중치 배분**
   - 뉴스 감정 점수: 20% 가중치
   - 이슈 반영도: 50% 가중치
   - 외부 평가 기관 피드백: 30% 가중치 (있는 경우)

2. **기본 평가 항목 (간소화)**
   - **IFRS 준수도**: 기본 점수 80점 (외부 피드백으로 ±10점 조정)
   - **완성도**: 이슈 반영도 등 기반 계산
   - **신뢰도**: 뉴스 감정 점수 기반 계산 (기본 80점 + 감정 점수 반영)

3. **최종 종합 점수 계산**
   ```
   종합 점수 = 
     (뉴스 감정 기반 신뢰도 점수 * 0.2) +
     (이슈 반영도 점수 * 0.5) +
     (외부 평가 기관 점수 * 0.3) [있는 경우]
   ```

4. **개선 필요 영역 식별**
   - 뉴스에서 부정적으로 언급된 이슈
   - 핵심 이슈 중 SR 보고서에서 누락되거나 간단히만 언급된 항목
   - 외부 평가 기관의 개선 권고사항

### 4.4 평가 결과 저장

**저장 항목**:
- 종합 점수
- 뉴스 감정 점수 및 상세 분석
- 이슈 반영도 점수 및 누락 이슈
- 외부 평가 기관 피드백 (있는 경우)
- 개선 필요 영역 및 우선순위

**저장 위치**: `sr_report_evaluations` 테이블

## 5. 경쟁사 보고서 수집 및 평가 (Phase 3)

### 5.1 경쟁사 식별

**식별 기준**:
1. 동일 업종 기업

3. 최근 SR 보고서 발행 기업 우선

### 5.2 경쟁사 SR 보고서 수집

1. DART API로 경쟁사 SR 보고서 다운로드
2. PDF 파싱 (Docling 사용)
3. Index/본문 분리 저장 (HISTORICAL_REPORT_PARSING.md 참고)
4. `historical_sr_reports` 테이블에 저장

### 5.3 경쟁사 보고서 평가

**평가 기준**: 현재 보고서와 동일한 평가 기준 적용
- 외부 피드백 기반 평가 (뉴스, 이슈 반영도)
- 종합 점수 산출

**우수 사례 추출**:
- 평가 점수 85점 이상인 섹션 식별
- 해당 섹션의 문단 추출
- 강점 분석 (재무 연결성 명확, 시나리오 분석 상세 등)

## 6. 벤치마킹 및 개선점 도출 (Phase 4)

### 6.1 현재 vs 경쟁사 비교

**비교 항목**:
- 종합 점수 비교
- 섹션별 점수 비교
- 외부 피드백 반영도 비교

### 6.2 개선점 도출

**도출 기준**:
- 경쟁사 대비 5점 이상 낮은 섹션
- 80점 미만 섹션
- 외부 피드백에서 지적된 문제점

### 6.3 개선 우선순위 결정

**우선순위 계산**:
- 영향도: 점수 차이 (gap)
- 구현 난이도: 낮을수록 우선
- 우선순위 점수 = 영향도 / 난이도
