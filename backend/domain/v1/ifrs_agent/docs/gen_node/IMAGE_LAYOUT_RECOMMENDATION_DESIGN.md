# gen_node 확장: 전년·전전년 이미지 기반 배치·표 추천 설계

전년·전전년 SR 본문(`body_text`)과 함께 수집된 **이미지 메타데이터**(캡션, 타입, 크기, 페이지 내 좌표 등)를 활용해, 신규 보고서 문단 작성 시 **“이미지·표·본문의 상대적 배치”를 추천**하고 사용자 편의를 높이기 위한 설계 문서다. 구현 시 본 문서의 **출력 스키마**와 **역할 분리(백엔드 vs UI)** 를 기준으로 맞춘다.

---

## 1. 배경과 목표

### 1.1 현재 상태

- `gen_input`의 `ref_2024` / `ref_2023`에는 `body_text`, `page_number`, `images[]`가 포함될 수 있다.
- `prompts.py`의 참조 섹션에서는 이미지를 **최대 5개**, `image_type` + `caption` 수준으로만 프롬프트에 넣는다 (`_build_reference_section`).

### 1.2 목표

1. **추천**: 전년·전전년 레이아웃과 이미지 설명을 참고해, 신규 연도 본문에서 **표 사용 여부**, **이미지 삽입 위치**, **어떤 이미지(또는 유사 역할)를 쓸지**를 자연어·구조화 형태로 제안한다.
2. **출력**: 사용자가 편집기에 붙여 넣거나, **전용 뷰어**가 그릴 수 있도록, (최소) **마크다운** 또는 (권장) **구조화 JSON**을 제공한다.
3. **범위**: PDF 픽셀 단위로 과거와 동일한 위치에 이미지를 “복제”하는 것이 아니라, **의미적 배치**(절·표·단락 옆/아래)와 **콘텐츠 추천**에 초점을 둔다.

---

## 2. 활용 가능한 데이터

| 소스 | 필드 예시 | 용도 |
|------|-----------|------|
| `ref_YYYY.body_text` | 마크다운/텍스트 | 문체·절 구조·소제목 패턴 |
| `ref_YYYY.images[]` | `image_url` / `sr-image:…` id, `caption`, `image_type` | **추천 대상 식별**, 설명 문구 |
| API/DB 확장 시 | `image_width`, `image_height`, `placement_bboxes` | **대략적** 위치(전년 페이지 기준), “이미지가 본문 중 어디쯤에 있었는지” 힌트용 |

**주의**: `placement_bboxes`는 **해당 연도 PDF 페이지 좌표**다. 생성 문단의 **문장 인덱스**와 자동으로 1:1 정렬되지 않는다. 추천은 **“이 절/이 표 옆”** 수준의 서술이 현실적이다.

---

## 3. 역할 분리 (백엔드 vs 클라이언트)

| 구분 | 책임 |
|------|------|
| **gen_node(또는 확장 모듈)** | 추천 근거가 있는 **텍스트 + 구조화 힌트**(이미지 id, 표 마크다운, 배치 설명) 생성 |
| **API/오케스트레이터** | 응답에 `generated_text` 외 **`layout` / `image_recommendations`** 필드 허용 여부 결정 |
| **프론트(또는 PDF 렌더러)** | 2열 레이아웃·표 안 이미지 등 **시각적 배치**는 구조 JSON 또는 HTML을 받아 렌더 |

사용자가 그린 것처럼 **박스로 둘러싼 2열 레이아웃**을 “문자열만”으로 완벽히 재현하려면, 최종은 **UI 컴포넌트**가 구조 데이터를 받는 편이 안정적이다.

---

## 4. 출력 형식 옵션

### 4.1 옵션 A — 단일 마크다운 (`generated_text` 확장)

- 본문에 **표**(`| --- |`)와 **이미지** `![캡션](sr-image:uuid)` 를 삽입.
- “추천” 문구는 **주석 블록** 또는 **굵은 안내 문단**으로 구분.

**장점**: 기존 API 필드 하나로 전달 가능.  
**한계**: 복잡한 2열·중첩 표는 마크다운 표현 한계가 있다.

### 4.2 옵션 B — 구조화 JSON (권장, 확장 시)

`generated_text`는 “순수 서술”만 두고, 배치는 별도 필드로 분리한다.

```json
{
  "generated_text": "…",
  "layout": {
    "version": 1,
    "blocks": [
      {
        "kind": "paragraph",
        "text": "…"
      },
      {
        "kind": "table",
        "markdown": "| … |\n|---|",
        "note": "전년도 대비 동일한 정보 밀도를 위해 표 형식 권장"
      },
      {
        "kind": "image_recommendation",
        "image_ref": "sr-image:d0db1d3b-f648-45dd-a8a9-23367b39206e",
        "role": "알고리즘 특강 홍보 배너",
        "placement_hint": "해당 절의 서술 직후, 본문 오른쪽 열 또는 표 하단",
        "rationale_ko": "전년 참조본에서 동일 주제 옆에 배치된 이미지(캡션: …)와 역할이 유사함."
      }
    ]
  }
}
```

**장점**: UI가 블록 단위로 렌더·드래그 수정 가능.  
**단점**: 오케스트레이터·API 스키마·클라이언트 수정 필요.

### 4.3 옵션 C — 2단계 파이프라인

1. **레이아웃/이미지 추천** 전용 호출(또는 동일 모델의 첫 턴): JSON만 출력, 스키마 고정.  
2. **본문 생성** 두 번째 호출에 1번 결과를 넣어 최종 문단 생성.

**장점**: 디버깅·재시도 쉬움.  
**단점**: 지연·비용 증가.

## 5. 추천 로직 설계 (프롬프트 관점)

### 5.1 입력에 추가할 정보

- 연도별 `images[]` 전체(또는 상위 N개)에 대해: `image_id`, `caption`, `image_type`, `width`×`height`, (선택) `placement_bboxes` 요약.
- `body_text` 내 `<!-- image -->` 등 **플레이스홀더**가 있으면 “이미지가 끼어든 위치” 힌트로 활용.

### 5.2 모델 지시 예시 (의도)

- 전년·전전년에서 **같은 주제**(예: 대학생 알고리즘 특강)에 쓰인 이미지를 우선 매칭.
- **표**는 수치·절차 비교가 필요할 때만 제안하고, 불필요한 표 남발은 금지.
- 각 추천에 **한국어 근거**(`rationale_ko`) 한 줄 이상.

### 5.3 토큰 관리

- 이미지 개수가 많으면 **캡션 길이 제한**, **연도당 최대 N개**, **본문 참조는 앞부분만** 등으로 상한을 둔다.

## 6. 사용자 경험 목표 (레이아웃 예시)

아래는 **최종 UI가 그릴 수 있는 목표**를 ASCII로 표현한 것이다. API는 이와 동일한 ASCII를 출력할 필요는 없고, **구조 JSON + 마크다운**으로 대체 가능하다.

```
|——————————————————————————————————————————|
|  (본문 텍스트 열)              |  (이미지/표 열)        |
|  신입사원은 상/하반기…        |  [추천] 알고리즘 특강  |
|  경력사원은 수시…            |  홍보 배너 이미지      |
|  2023년부터 인턴…            |  sr-image:…            |
|——————————————————————————————————————————|
```

또는 본문 흐름 중간에 표를 삽입한 뒤, 그 옆/아래에 이미지를 두는 형태:

```
… 인턴 모집을 재개하여 …  |————————————————————————|
… 부서 실습 기회를 …      |  (표: 채용 채널 요약)   |
…                        |————————————————————————|
```

## 7. 구현 단계 제안

| 단계 | 내용 |
|------|------|
| **0** | API 응답에 `layout` 또는 `image_recommendations` 필드 추가 여부 결정 |
| **1** | `prompts.py` 참조 섹션: 이미지 메타 확장( id, 크기, bbox 요약, 상한 N ) |
| **2** | 단일 생성 호출에서 **JSON 모드** 또는 **마크다운 + 구분선**으로 “추천” 문단 출력 |
| **3** | (선택) 2단계 파이프라인으로 분리 |
| **4** | 프론트: `blocks` 렌더러 또는 마크다운 확장 |

---

## 8. 검증·리스크

- **환각**: 존재하지 않는 `image_ref`를 쓰지 않도록, **허용 id 목록**을 프롬프트에 명시하고, 후처리로 필터링.
- **validator_node**: 새 출력 형식이 생기면 “추천 블록은 검증 제외” 등 정책 정합.
- **접근성**: 이미지 추천 시 `caption`을 대체 텍스트로 사용하도록 유도.

## 10. 요약

- 전년·전전년 **이미지 메타 + 본문**은 이미 파이프라인에 존재하므로, **추천·배치 힌트**는 프롬프트·출력 스키마 확장으로 구체화할 수 있다.
- **픽셀 단위 복제**는 기대하지 않고, **의미적 추천 + 구조화 JSON(권장)** + **UI 렌더** 조합이 유지보수·확장에 유리하다.
- 단일 `generated_text`만으로도 1차 PoC는 가능하나, **2열·복잡 표**는 **옵션 B/C**를 검토하는 것이 좋다.
