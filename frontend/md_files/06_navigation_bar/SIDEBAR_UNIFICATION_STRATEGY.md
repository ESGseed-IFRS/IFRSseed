# 2차 사이드바 통일 전략 (대시보드 기준 · SR·GHG 정렬)

**목적:** 워크스페이스 내 **대시보드 / SR 보고서 / GHG 산정**의 2차 사이드바가 서로 다른 배경색·폭·타이포·밀도를 쓰고 있어 시각적 통일성이 깨진다. **대시보드 사이드바(`DashboardNewSidebar`)를 단일 기준(레퍼런스)**으로 삼고, SR·GHG를 같은 토큰·비율로 맞춘다.  
**범위:** 2차 사이드바만. 1차 모듈 레일(`AppModuleRail`)은 별도 유지.

---

## 1. 현황 요약

| 영역 | 구현 파일 | 배경색(대략) | 폭 | 타이포 특징 |
|------|-----------|--------------|-----|-------------|
| 대시보드 | `DashboardNewSidebar.tsx` | `C.navy` `#0C2340` | **236px** | 섹션 10px·700~800, 본문 12px·400~600 혼합 — **상대적으로 가장 정돈** |
| SR 보고서 | `SrReportSidebar.tsx` | `#0C2340` (동일 계열) | **270px** | `fontWeight` **800~900** 다수, 헤더·메타·칩·카드가 **굵고 큼** |
| GHG 산정 | `ghg_calc/.../Sidebar.tsx` | `#0d1b36` | **220px** | Tailwind + 인라인 700, 상단 **에메랄드** 브랜딩으로 **톤이 다름** |

**문제:** 색·폭·굵기·카드 패딩이 제각각이라 “한 제품의 작업 패널”로 보이기 어렵다. SR은 사용자 피드백대로 **특히 굵고 두꺼운 인상**이 강함.

---

## 2. 설계 원칙 (대시보드 = 소스 오브 트루스)

1. **색상:** 배경은 **`C.navy` (`#0C2340`)** 단일화. 보조면 `C.navyM` 등은 대시보드와 동일 규칙으로만 사용. GHG의 `#0d1b36`·에메랄드 헤더 아이콘 배경은 **제거 또는 `C.blue` 계열로 치환**해 브랜드 축을 맞춘다.
2. **폭:** 2차 사이드바 **고정 폭 236px**로 통일 (대시보드 `DashboardNewSidebar`와 동일).
3. **타이포 스케일:** 대시보드에 맞춘 **단계만 허용** (아래 §4). SR에 흔한 **800/900**은 **600 이하**로 내린다.
4. **밀도:** 세로 패딩·카드 라운드·프로그레스 바 두께를 **한 단계씩 줄여** “얄쌍한” 비율로 수렴 (SR 우선 적용).
5. **폰트 패밀리:** 작업 영역 전체가 이미 `Noto Sans KR` / `Pretendard`를 쓰는 경우가 많으므로, SR 사이드바의 **순수 인라인 스택만** 대시보드와 동일 `fontFamily`로 통일한다.

---

## 3. 레퍼런스 추출 (대시보드에서 가져올 값)

구현 시 `src/app/(main)/dashboard/lib/constants.ts`의 **`C`** 객체를 **공유 import**하는 것을 권장한다 (중복 hex 금지).

- **배경:** `C.navy`
- **강조 보더/포인트:** `C.blue`, `C.teal` (모듈 성격에 따라 기존 대시보드 규칙 유지)
- **폭:** `236` (px)
- **구분선:** `rgba(255,255,255,.08)` ~ `.1` (대시보드와 동일 톤)

대시보드 사이드바 내부의 **헤더 블록·SR 완료율·접힘 섹션** 레이아웃을 “SR/GHG 상단 브랜딩 영역”의 **레퍼런스 레이아웃**으로 삼는다 (로고 크기 32px 전후, 타이틀 12px·700 등).

---

## 4. 타이포그래피 토큰 (통일안)

SR에서 **굵기 완화**를 위해 아래 상한을 넘지 않도록 한다.

| 용도 | fontSize | fontWeight | 비고 |
|------|----------|------------|------|
| 섹션 라벨(대문자/캡션) | 10px | **600** | 기존 800→600 |
| 2차 메뉴 텍스트 | 12px | 활성 **600**, 비활성 **400** | 700 남용 금지 |
| 헤더 타이틀(회사명 등) | 12px | **600** | 기존 900 금지 |
| 헤더 부제 | 10~11px | **400** | opacity로 위계 |
| 메타 라벨(작성 주체 등) | 10px | **500~600** | 값 텍스트는 **500** |
| 배지/칩 텍스트 | 9~10px | **600** | 배경색으로 구분 |

**금지:** 본문/라벨에 `fontWeight: 800`·`900` 사용 (숫자 강조·퍼센트만 예외적으로 **700** 허용 가능).

## 5. SR 보고서 사이드바(`SrReportSidebar`) 조정 항목

파일: `src/app/(main)/sr-report/components/SrReportSidebar.tsx`

### 5.1 폭·색

- `width: 270` → **236**
- `bg: '#0C2340'` → **`C.navy`** (import 통일)

### 5.2 헤더·메타

- 로고 박스: `40×40` → **32×32** (대시보드 썸네일과 동일 계열)
- “Samsung SDS” 등: **14px/900 → 12px/600**
- 부제·라벨: **800 → 500~600**, 값: **900 → 500~600**

### 5.3 전체 준수율·프로그레스 카드

- 외곽 패딩: `padding: 12` → **8~10**
- `borderRadius: 14` → **10~12**
- 메인 프로그레스 바 높이: `8px` → **4~5px** (대시보드 얇은 바와 유사)

### 5.4 직접 공시기준 블록

- 블록 제목: **800/12px → 600/11px**
- 각 기준 행 간 `gap`·세로 패딩 **20~30% 축소**
- 기준별 미니 바 높이 **얇게** (대시보드 `Pbar` 또는 높이 3~4px 패턴 참고)

### 5.5 DP 목록 카드

- 카드 `padding` **축소** (예: 12→8, 10→7)
- 카드 간 `gap` 축소
- 제목: **굵은 대형** → **12~13px / 600**, 부제는 **10px / 400** 유지
- 상태 칩: 패딩·`fontSize` 한 단계 축소, **칩 텍스트 600** 유지 가능
- 선택 테두리는 `C.blue` 유지하되 **두께 1~2px**로 정리

### 5.6 지주사(holding) 모드

- 동일 토큰 적용. 진행률 카드만 유지하고 스타일은 subsidiary와 **동일 밀도**.

## 6. GHG 산정 사이드바(`Sidebar.tsx`) 조정 항목

파일: `src/app/(main)/ghg_calc/components/layout/Sidebar.tsx`

- `w-[220px]` → **`w-[236px]`** (또는 `style={{ width: 236 }}`로 대시보드와 동일)
- `bg-[#0d1b36]` → **`C.navy`** (Tailwind arbitrary `bg-[#0C2340]` 또는 공통 클래스)
- 상단 아이콘: **에메랄드 원형** → **`C.blue` 기반** 사각/라운드 (대시보드 블루 포인트와 통일)
- 제목 “ESG Platform” 등: **700 → 600**, 캡션 **10px / 400~500**
- 메뉴 행 높이·`padding`을 대시보드 서브메뉴 버튼(`padding: 6px 16px` 전후)에 **맞춰 소폭 조정**

아이콘(lucide) 크기는 **13px 유지** 가능하나, 행 간격만 대시보드와 시각적으로 맞출 것.

---

## 7. 공통화 옵션 (권장 순서)

1. **1단계 (저비용):** `C`만 SR·GHG에서 import하고, 폭·hex·fontWeight를 위 표대로 **파일별 수정**.
2. **2단계 (유지보수):** `src/components/workspace/sidebarTokens.ts` (또는 `dashboard/lib/sidebarTheme.ts`)에  
   `SIDEBAR_WIDTH`, `sidebarSectionLabelStyle`, `sidebarMetaRowStyle` 등을 두고 세 컴포넌트가 참조.
3. **3단계 (선택):** 헤더·메타·프로그레스 카드를 `SidebarBrandHeader`, `SidebarProgressSummary` 같은 **작은 프레젠테이션 컴포넌트**로 분리해 SR·대시보드가 공유 (대시보드 리팩터 범위가 커지면 2단계 후 검토).

## 8. 검수 체크리스트

- [ ] SR·GHG·대시보드 2차 사이드바 **배경색이 동일**하게 보이는가 (스크린 병치).
- [ ] **폭 236px**로 스크롤바·메인 영역 레이아웃이 깨지지 않는가.
- [ ] SR에서 **800/900이 제거**되었는가 (검색으로 확인).
- [ ] DP 카드·프로그레스 영역이 이전 대비 **세로로 압축**되었는가.
- [ ] 키보드 포커스·대비(WCAG)가 **얇아진 후에도** 읽기 가능한가.
