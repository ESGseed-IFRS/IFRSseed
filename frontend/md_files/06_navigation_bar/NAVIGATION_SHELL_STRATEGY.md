# 네비게이션 셸 전략 (홈 전용 상단바 · 앱 이중 사이드바)

**문서 목적:** 현재 모든 주요 화면에 동일한 상단 `Navigation`이 노출되는 구조를 바꾸고, **홈(`/`)만** 참고 이미지 4·5번에 가까운 **텍스트 중심 상단바**를 두며, **대시보드·SR 보고서·GHG 산정** 구간은 참고 이미지 1·2번에 가까운 **1차(모듈) 레일 + 2차(기존 페이지) 사이드바** 구조로 정리한다.  
**범위:** 레이아웃·네비게이션·헤더 영역. 배경 이미지·풍경 히어로 등은 **의도적으로 따르지 않는다.**

---

## 1. 현황과 문제

| 구분 | 현재 (`AppLayoutShell` + `Navigation`) |
|------|--------------------------------------|
| 노출 범위 | `/`를 제외한 대부분 경로에서 고정 상단바 + `pt-16` 메인 (로그인/등록만 예외) |
| 대시보드 등 | `DashboardNewShell` 등 **자체 사이드바**가 있음에도, 그 위에 **동일한 글로벌 상단 메뉴**가 겹쳐 보임 |
| 홈 상단바 | 로고와 메뉴가 한 줄 `justify-between`에 가깝고, 각 탭에 **아이콘(lucide)** 이 붙어 있음 |

**사용자 기대**

- **홈:** 상단바만 사용. 로고는 **왼쪽 끝**, 메인 링크(홈·대시보드·GHG·SR 등)는 **가로 중앙 근처**에 모여 있고, **아이콘 없이 텍스트만** (참고 5번의 “에어리한 텍스트 내비” 느낌). 배경은 흰/단색 유지.
- **대시보드 / `sr-report` / `ghg_calc`:** 글로벌 상단 메뉴 **비표시**. 대신 **맨 왼쪽 좁은 레일**에서 세 모듈 전환, **그 옆**에 현재 각 페이지의 **기존 사이드바** 유지.
- **앱 상단(2차 영역 위):** 참고 2번처럼 **IFRSseed 브랜딩**과 **계열사 / 지주사** 전환(또는 동일한 “관점 전환” UI)이 **한눈에** 보이도록 배치.

---

## 2. 목표 아키텍처

### 2.1 라우트별 셸(Shell) 분류

| 셸 유형 | 경로(예) | 상단 글로벌 `Navigation` | 좌측 1차 레일 | 2차 사이드바 |
|---------|----------|-------------------------|---------------|--------------|
| **마케팅/랜딩** | `/` | ✅ (홈 전용 스타일) | ❌ | ❌ |
| **앱 작업** | `/dashboard`, `/sr-report`, `/ghg_calc` (및 하위) | ❌ | ✅ 모듈 3종 | ✅ 기존 컴포넌트 유지 |
| **인증** | `/login`, `/register` | ❌ (현행 유지) | ❌ | ❌ |
| **기타** | 정책 페이지 등 추후 | 문서화 시 결정 | — | — |

### 2.2 이중 사이드바 개념 (참고 1·2번)

1. **1차 사이드바 (모듈 레일)**  
   - 폭: **좁게** (아이콘 + 툴팁 또는 짧은 라벨).  
   - 항목: **대시보드**, **SR 보고서**, **GHG 산정** (3개 고정).  
   - 역할: 앱 내 **최상위 모듈 전환**만 담당.  
   - 시각: 참고 1의 틸/다크 스트립에 가깝게 할지, 현재 대시보드 네이비 톤에 맞출지는 **디자인 토큰**으로 결정 (일관성 우선).

2. **2차 사이드바 (컨텍스트 내비)**  
   - **현재 구현 유지:**  
     - 대시보드: `DashboardNewSidebar`  
     - SR / GHG: 각 라우트에 이미 있는 좌측 패널(있으면 동일 패턴으로 래핑)  
   - 1차 레일 **오른쪽**에 붙이고, **메인 콘텐츠**는 그 오른쪽 전체 폭 사용.

3. **앱 헤더 (모듈 레일·2차 사이드바 위가 아니라, 작업 영역 상단)**  
   - 참고 2번: **로고/서비스명(IFRSseed)** + **계열사 | 지주사** 토글(또는 세그먼트).  
   - 대시보드는 이미 `NewDashboardTopBar`에 `mode === 'holding' ? '지주사' : '계열사'` 표기가 있으므로, **토글 UI를 이 헤더 쪽으로 승격·통합**하는 방향이 자연스럽다.  
   - SR·GHG에서도 **동일한 관점 상태**가 필요하면 URL 쿼리(`?mode=holding`) 또는 **상위 React context**로 공유하고, 각 셸이 구독.

## 3. 홈 전용 상단바 (참고 4·5번, 배경 제외)

### 3.1 레이아웃 규칙

- **컨테이너:** `position: fixed`, 전체 너비, 높이 고정(예: 64px), 배경 **흰색**, 하단 **얇은 보더**.
- **3분할 그리드 권장:**  
  - **왼쪽:** 로고(스프라이트 아이콘은 유지 가능하되, **메뉴 항목에는 아이콘 없음**).  
  - **가운데:** `absolute` + `left-1/2 -translate-x-1/2` 또는 `grid`의 중앙 컬럼에 **내비 링크만** 배치해 **시각적 중앙** 정렬.  
  - **오른쪽:** 로그인/로그아웃, 사용자명, (선택) 유틸 아이콘.

### 3.2 타이포·스타일 (참고 5번 느낌만)

- 링크: **아이콘 제거**, `font-medium`~`semibold`, 활성 시 밑줄 또는 **미세한 대비** (과한 pill은 홈에서만 최소화).  
- 참고 5의 **검색·다운로드·글로브** 아이콘은 **필수 아님** — 필요 시 우측 유틸로만 추가.

### 3.3 홈에서 보여줄 링크 범위

- 최소: **홈, 대시보드, GHG 산정, SR 보고서** + 비로그인 시 **로그인**.  
- 로그인 후: **로그아웃**·사용자 표시는 우측 유지.

## 4. 구현 시 기술적 접근 (Next.js App Router)

### 4.1 `AppLayoutShell` 분기 확장

현재: `hideChrome` = `/login`, `/register`.  

**제안:**  
- `usePathname()`으로 **앱 작업 구역** 여부 판별 (예: `pathname.startsWith('/dashboard')` 등).  
- 앱 작업 구역이면 **`<Navigation />` 미렌더**, `main`의 `pt-16`도 **앱 셸이 자체 상단을 가지므로** 조정 (또는 앱 레이아웃이 `min-h-screen`을 전부 채우도록 루트에서 패딩 제거).

### 4.2 신규 컴포넌트 (이름 예시)

| 컴포넌트 | 역할 |
|----------|------|
| `HomeNavigation` | 홈 전용 3분할 상단바 (텍스트만 중앙) |
| `AppModuleRail` | 3모듈 1차 사이드바 + `Link`/`usePathname` 활성 상태 |
| `AppWorkspaceLayout` | `AppModuleRail` + `children`(각 페이지가 2차 사이드+콘텐츠 삽입) |
| `AppChromeHeader` | IFRSseed + 계열사/지주사 토글 + (선택) 검색 |

### 4.3 라우트 그룹 (권장)

- `src/app/(marketing)/page.tsx` → 홈만 `HomeNavigation` 사용.  
- `src/app/(workspace)/dashboard/...`, `(workspace)/sr-report/...`, `(workspace)/ghg_calc/...` → 공통 `layout.tsx`에서 `AppWorkspaceLayout` 래핑.

**이동 비용이 크면** 1단계로 `AppLayoutShell`만 분기하고, 페이지별로 `AppWorkspaceLayout`을 삽입하는 방식도 가능.

### 4.4 상태: 계열사 / 지주사

- **옵션 A:** URL 쿼리 `?perspective=subsidiary|holding` — 새로고침·공유에 유리.  
- **옵션 B:** React Context + `sessionStorage` — 대시보드와 동일 패턴 확장.  
- SR·GHG가 아직 모드를 쓰지 않으면, 헤더 토글은 **대시보드에만 반영**하고 SR/GHG는 “UI만 선반영” 또는 추후 연동으로 단계화.

## 5. 단계별 로드맵

| 단계 | 내용 | 완료 기준 |
|------|------|-----------|
| **1** | `AppLayoutShell`에서 앱 경로 시 `Navigation` 숨김 | 대시보드 진입 시 글로벌 상단바 미노출 |
| **2** | `AppModuleRail` 추가 + 3모듈 링크 | 좌측 좁은 레일에서 모듈 전환 가능 |
| **3** | 대시보드 `DashboardNewShell`을 레일 오른쪽에 맞게 래핑 | 기존 `DashboardNewSidebar` 위치·동작 유지 |
| **4** | `AppChromeHeader`에 IFRSseed + 계열사/지주사 토글 | 참고 2번과 유사한 정보 계층 |
| **5** | SR·GHG 페이지에 동일 레이아웃 적용 | 세 모듈 모두 동일 셸 |
| **6** | 홈 `Navigation` → `HomeNavigation` 리디자인 | 로고 좌측·링크 중앙·아이콘 제거 |

---

## 6. 비기능·일관성

- **접근성:** 모듈 레일 아이콘만 있을 경우 `aria-label`·툴팁 필수.  
- **반응형:** 모바일에서는 1차 레일을 **하단 탭** 또는 **햄버거**로 축약하는 후속 과제로 문서화.  
- **성능:** 레이아웃은 Client Component 최소화; 경로 판별만 클라이언트여도 무방.

---

## 7. 열린 결정 사항

1. 모듈 레일 **색상**: 틸(참고 1) vs 기존 네이비(현 대시보드) 통일 여부.  
2. **지주/계열** 토글을 SR·GHG 데이터에 연결할 시점과 API 계약.  
3. 홈 우측 유틸(검색 등) 도입 여부.
