# SR보고서 페이지 — 계열사/지주사 통합 전환 전략

> 대상: `src/app/(main)/sr-report/`  
> 목적: SR보고서 페이지를 **계열사/지주사 토글형 단일 화면**으로 구성하되,  
> - 계열사: 현재 구현 화면 유지  
> - 지주사: SR 플랫폼의 지주사 화면을 가져오되 일부 기능 제거/유지 기준 적용  
> - 사이드바: SR보고서 페이지의 현재 내비 스타일로 통일

---

## 1. 요구사항 정리

### 1-1. 계열사
- 현재 SR보고서 페이지 구현 상태를 기본값으로 유지
- 첫번째 이미지의 `계열사` 탭과 동일 동작

### 1-2. 지주사
- `지주사` 탭 선택 시 SR 플랫폼의 지주사 화면 구조를 가져옴
- 단, 아래는 제외:
  - `공시데이터 취합`
  - `공시데이터 검토`
- 아래는 유지:
  - `공시데이터 작성`(현재 작성 형식 유지)
  - `페이지별 작성`
  - `보고서 생성`

### 1-3. 내비게이션
- 지주사 화면으로 전환되어도 좌측 내비는 SR보고서 페이지 사이드바 스타일 사용

---

## 2. 화면 구조(단일 페이지 + 모드 전환)

```
SR보고서 page
 ├─ 상단 모드 토글: [계열사] [지주사]
 ├─ 좌측: SR보고서 공통 사이드바(항상 동일)
 └─ 우측 메인:
    ├─ mode=subsidiary -> 기존 계열사 작성 화면
    └─ mode=holding    -> 지주사 작성/페이지별 작성/보고서 생성
```

핵심 원칙:
- 라우트는 단일 유지 (`/sr-report`)
- 모드는 URL query와 내부 state 동기화
  - 예: `/sr-report?mode=subsidiary`, `/sr-report?mode=holding`

---

## 3. 정보구조(IA) 재정의

### 3-1. 지주사 메뉴 재구성

기존 SR 플랫폼(지주사) 메뉴에서 다음만 노출:

1. 공시데이터 작성
2. 페이지별 작성
3. 보고서 생성

비노출:
- 공시데이터 취합
- 공시데이터 검토

### 3-2. 사이드바 구성

사이드바는 SR보고서 기준으로 고정:
- 상단 헤더/브랜드
- 진행 현황
- DP/단계 네비
- 상태 배지(chip)

지주사 탭에서도 동일 UI를 사용하고, 메뉴 데이터만 지주사용으로 교체

---

## 4. 컴포넌트 설계안

### 4-1. 컨테이너
- `SrReportPageContainer` (신규 or 기존 `page.tsx` 확장)
  - `mode: 'subsidiary' | 'holding'`
  - URL query sync

### 4-2. 모드 토글
- `SrReportModeSwitch` (신규)
  - 상단 `계열사/지주사` pill UI
  - 클릭 시 `setMode` + `router.replace` query 반영

### 4-3. 메인 뷰
- `SubsidiarySrWorkspace` (기존 유지)
- `HoldingSrWorkspace` (신규)
  - `공시데이터 작성` 탭: 기존 작성 UI 재사용
  - `페이지별 작성` 탭: SR 플랫폼 지주사 편집 파트 이관
  - `보고서 생성` 탭: SR 플랫폼 보고서 생성 파트 이관
  - `취합/검토` 탭/컴포넌트는 이관 금지

### 4-4. 사이드바
- `SrReportSidebar` (공통)
  - mode별 메뉴 소스만 다르게 주입
  - UI/스타일 컴포넌트는 단일 유지

## 5. 데이터/상태 설계

### 5-1. 공통 상태
- `mode`
- `activeMenu` (작성/페이지별/보고서생성)
- `activeDpId`
- `statusByDpId`

### 5-2. 분리 상태
- 계열사 전용 draft/editor 상태
- 지주사 전용 page-builder/report-generator 상태

### 5-3. 상태 보존 정책
- mode 전환 시 서로의 편집 상태를 보존 (unmount 시에도 store 유지)
- 초기에는 local state, 이후 필요 시 zustand/store 분리

## 6. 이관 기준 (SR 플랫폼 -> SR보고서)

### 가져올 것
- 지주사 `공시데이터 작성`
- 지주사 `페이지별 작성`
- 지주사 `보고서 생성`

### 가져오지 않을 것
- 지주사 `공시데이터 취합`
- 지주사 `공시데이터 검토`

### 이관 방식
1. 로직 우선 이관: 상태/이벤트 핸들러
2. UI는 SR보고서 디자인 토큰으로 재스킨
3. 사이드바/헤더는 SR보고서 컴포넌트 사용

---

## 7. 사용자 흐름

### 계열사 흐름
1. `/sr-report` 진입 (기본 `mode=subsidiary`)
2. 기존 작성/제출 흐름 유지

### 지주사 흐름
1. 상단 토글에서 `지주사` 선택
2. 기본 탭 `공시데이터 작성` 진입
3. 필요 시 `페이지별 작성` -> `보고서 생성` 순 이동
4. 취합/검토는 사이드바/탭 어디에서도 노출되지 않음
