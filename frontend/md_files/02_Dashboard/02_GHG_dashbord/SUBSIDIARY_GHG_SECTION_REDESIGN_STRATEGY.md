# 계열사 대시보드(신규) — GHG 섹션 개편 전략

> 대상: `src/app/(main)/dashboard/components/DashboardNewContent.tsx`의 `renderGhg()` (계열사 모드)  
> 질문: `Raw 충실도 / 이상치 미조치 / 산정 적합성` KPI를 유지할지, 무엇을 대시보드에 두고 무엇을 `GHG 산정(신규)`로 넘길지

---

## 1. 결론 요약

- **현재 3개 KPI는 모두 완전 제거 대상은 아님**
- 다만 계열사 관점에서는 의미를 다음처럼 재정의해야 함:
  - `Raw 충실도` -> `필수 데이터 입력률`
  - `이상치 미조치` -> `미조치 이슈`
  - `산정 적합성` -> `검증 통과율` (혹은 `검증 상태`)
- 상세 산정 논리(계수, 경계, 원천 라인)는 대시보드가 아닌 `GHG 산정(신규)`로 이동

---

## 2. 현재 문제

현재 `renderGhg()` 계열사 분기는 다음 구조입니다.

1. KPI 3개 (`Raw 충실도`, `이상치 미조치`, `산정 적합성`)
2. Scope 1/2/3 리스트
3. Raw data 항목 리스트
4. 이상치 리스트

문제:

- KPI 명칭이 "데이터 엔지니어링/검증팀" 관점에 가깝고 실무자 업무 언어와 거리 있음
- 항목 수가 많아져서 대시보드가 `준 상세화면`처럼 보임
- 결국 상세 확인은 `ghg_calc`로 가야 하므로 중복정보가 많음

---

## 3. 권장 IA (정보 구조)

### A. 요약(대시보드용) — 남김
- KPI 3개는 유지하되 명칭 변경
- "오늘 처리해야 할 항목" 3~5개만 노출

### B. 상세(작업용) — 이관
- Scope별 산정값 상세
- Raw data 전체 라인아이템
- 이상치 전체 리스트와 이력

이 영역은 `GHG 산정(신규)` 페이지가 담당

---

## 4. KPI 재정의안

| 현재 | 개편안 | 계산 기준(예시) | 사용자 행동 |
|------|--------|------------------|-------------|
| Raw 충실도 | 필수 데이터 입력률 | `done 필수 raw / 필수 raw` | 누락 항목 입력 |
| 이상치 미조치 | 미조치 이슈 | `status=unresolved` 건수 | 이상치 조치 |
| 산정 적합성 | 검증 통과율 | `(검증 통과 항목 / 검증 대상)` | 검증 실패 원인 확인 |

표시 방식:

- KPI 카드 하단에 항상 `GHG 산정(신규)에서 상세 확인` 링크 노출

---

## 5. 화면 구성안 (계열사)

```
[GHG 요약 배너]
  - 필수 데이터 입력률 / 미조치 이슈 / 검증 통과율 (3 KPI)
  - 우측: GHG 산정(신규) 이동

[내 조치 필요 TOP 5]
  - 미조치 이슈 + 누락 입력 항목 혼합 리스트
  - 각 행: 항목명 / 상태 / 기한 / "상세 처리" 버튼

[최근 변경 이력 3건]
  - 최근 조치/검증 로그 요약
```

삭제/이관 권장:

- Scope 1/2/3 전체 상세 블록
- Raw data 전체 항목 목록
- 이상치 전체 테이블

---

## 6. 버튼/내비게이션 정책

모든 상세성 버튼은 `ghg_calc`로 단일 수렴:

- `상세 처리` -> `/ghg_calc?focus=anomaly&id=<anomalyId>`
- `누락 입력` -> `/ghg_calc?focus=raw&id=<rawId>`
- `검증 실패` -> `/ghg_calc?focus=validation`

대시보드는 "요약 + 우선순위 제시" 역할만 수행

---

## 7. 구현 단계

### Phase 1 (UI 간소화)
1. `renderGhg()`에서 Scope/Raw/이상치 전체 블록 제거

3. 이동 버튼을 `ghg_calc` query 딥링크로 통일

### Phase 2 (KPI 재정의)
1. 카드 라벨 변경 (`필수 데이터 입력률`, `미조치 이슈`, `검증 통과율`)
2. 계산식 명확화 및 tooltip 추가

### Phase 3 (데이터 정합)
1. `mockData.ts`에 `required`, `owner`, `dueDate` 메타 보강
2. 계열사 사용자 기준 필터(`isMine`) 적용
